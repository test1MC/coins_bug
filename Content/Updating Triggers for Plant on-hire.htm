Updating Triggers for Plant On Hire

Triggers allow you to keep track of when events related to plant or vehicle assets are due. You can automatically update hourly and daily triggers for plant which is on hire.

For example, suppose you have a plant item which you want to inspect when it has been on hire for 60 days or more. Using Fixed Assets > Setup > Maintenance > Asset Trigger Definitions, you would set up an integer trigger, with a TT (trigger type) of GE (greater than or equal to) and a reset Value of 60.

Then you would enter that trigger against the asset using Asset Maintenance.

You should update the daily (and hourly) triggers periodically, and before reporting on trigger expiry dates, for example. This will update the value of the trigger; when this value reaches (or exceeds) the Expires value (60 in this example), the trigger will fire, and will appear on the Trigger Enquiry and the Fixed Assets > Reports > Operational Reports > Asset Triggers Report.

To update triggers:

  1. Go to Plant Control > Processing > Update Plant On Hire Triggers.
  2. Fill in the fields as follows:
  • Type Select whether to update triggers based on Daily or Hourly hire.
  • End Date The date to use to update the triggers.
  1. To update triggers for selected assets only, use the Asset Selection tab to specify the assets.

If you are using captions on timesheets to update a trigger, do not also use Plant Control > Processing > Update Plant On Hire Triggers to update the same trigger. You could have different triggers that use the different methods; so you could use TRIGHOUR to drive hourly-based servicing, and have a trigger for off hire inspection status updated by a caption.
