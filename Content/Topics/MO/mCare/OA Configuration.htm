Adding inspections

Once the user has logged into OA, they will need to select the correct contract and module.

For the purpose of the app the user will need to select KCO and then ‘Inspections’ Module.

The user taps Inspection Workbench

  • At the bottom of the page the user selects .

The user will then need to add the following details:

  • Development
  • Plot
  • Type
  • Inspector
  • Status
  • Date
  • Time
  • Notes

Once the user goes back to the Inspection workbench, they will be able to view their created inspection which would be at the top of the list (most recently created).

Multiday view

In order for the user to be able to view multiple days on the device, the user mobile configuration (CCVISITLISTTYPE and CCVISITLISTNUMBEROFDAYS) must be set up.

This can be done under the ‘System’ module on OA.

The user taps ‘System Setup’.

Mobile setup

User Mobile Configuration

The user opens CCVISITLISTTYPE and sets this to MULTIDAY and the tap save.

Then the user can open CCVISITLISTNUMBEROFDAYS and select how many days in the week that the can view.

Once this has been saved it will be sent to the ESB Outbound Messages.

 

  • Once successfully published then the update will be sent to the device.

 

From the User Mobile Configuration settings, the user can also set the CCADDNEWISSUE function for repair tasks.

This parameter will allow the user to add a new issue and display the ‘Add issue’ button on a repairs task.

To enable this function, the user must open this parameter, logical must be ticked, and parameter must then be saved. The value for this parameter will then be set to ‘Y’.

Similarly this can also be done for CCCOMPLETENEWISSUE. This parameter will provide the ability for the to complete new issues identified on the current visit.
