Tick Groups

Within a Plot, the user will be able to perform valuations on Tick Groups and/or payment stages.

The Tick Group details include the group number, description, budget and amount not valued.

A selected Tick Group is highlighted with orange border.

Tapping on the ‘white arrow’ allows to show or hide comments and photo options.

The Show Fully Valued button, allows the user to see the ‘Fully Valued’ Tick Groups highlighted in green.

Users can change the ‘Value’ in price or update the ‘Progress’ in percentage.

Value and Progress are co-dependent, when one is changed, the other is automatically updated based on the budget.

The user will be prevented from entering a ‘Value’ that is greater than the budget.

Users can add ‘comments’ to each Tick Group item which also get updated onto .

When the ‘Value’ and ‘Progress’ amounts have been changed, the app can enforce the user to take photos.

User can tap on ‘Take Photo’ on valuation or within ‘Review Photos’ option.

Photo mandatory, max and minimum amount are OA configurable. Including the maximum number of photos per Tick Group or Payment Stage.

Clicking the image will open a preview application and the "X" will deleted it.

Once photos have been captured, they will be displayed in the photos screen where notes can be added against each and the user can take more photos.

Tapping on the ‘Notes’ field will bring up the on screen keyboard.

Photos will be saved and displayed in the order in which they were taken.

To return to previous screen user has to click back button or the mTick home button in the top left corner.

After photos have been saved and the user taps back to the plots screen, the user can review the captured photos and notes.

Once the user submits data the photos will no longer be visible to review.
