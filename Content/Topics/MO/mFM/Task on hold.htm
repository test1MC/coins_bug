Task on hold

The parameters for this option are located in Mobile Configuration Settings (Company). The following definitions are available:

FMTASKONHOLD

FMTASKONHOLDSTATUS

FMTASKRESUMESTATUS

On Hold Button

When the user is on a task they can place a job on hold. This will allow the user to complete other tasks and resume the on-hold task at a later time.

On hold button will appear on an active task on the right side of the title bar in all screens except Sign screen as long as customer has not signed yet. Once a customer signs the on hold button will not be visible.

Placing task on hold

When user taps the on hold button he will be prompted to confirm the action.

If confirmed the task will be placed on hold and the Task List screen will be displayed.

Tasks on hold will show with orange in the Task List screen.

Resuming task

Selecting a task on hold from the Task List will display the ‘Tenant’, ‘Task’, ‘History’ (if enabled) and ‘Actions’ tabs. If there is no other active task, Actions tab will display as well.

The Actions screen will show the play icon (top right corner), if the button is pressed the user will be prompted to confirm resuming task.

Selecting ok will activate the task and display the Status screen with no cancel option.

Once the user has resumed the task they will be taken back to the beginning of the task where they will need to select On Route or Navigate.

Logging off

When user logs off the app will persist all the data such as text inputs photos, SORs and signatures.
If a task is active when the user logs off he will be warned that this action will clear the task data. The user should place the task on hold before logging off if the data needs to be persisted.

If all tasks are no tasks left incomplete/unfinished then the application will display a standard logout message.
