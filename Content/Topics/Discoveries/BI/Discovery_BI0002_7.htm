Discovery BI0002.7 - Writing an Inline Frame

 

A Drill Down Browse Screen

The exercise is to create a simple Cost enquiry screen. The screen will work with a hard coded contract and return basic information from the jc_costcode record.

Create the Function

Create a new function. (Currently to be run as standalone page).


Function Code Initials + Exercise No    Eg    NLM3Exercise2
Function Name DescriptionEg.   NLM3Exercise2
Function Type – Function Function Module
Contract Status Function Category
Browse Function Context – Leave as defaulted from the Function Description.
Function Program wou005

 

 

Create the Page

Create the page section (with the same name as the function).  Enter the information into the Page Summary as follows:

Header Table

jc_config,co_config

Body Table

jc_costcode

 

In this exercise we will use a hard-coded contract.  The exercise uses a Contract 1000, but you should substitute a contract number that exists on your system if testing on your own data.

BodyRows

12

Body Query

FOR EACH jc_costcode WHERE jc_costcode.kco = {kco}
AND jc_costcode.job_num = '1000'

 

Click complete the Page Summary.

Create the Forms

Select the Form tab and add a Body and a Totals Form

Click to create the forms.

Add the Fields

In the Fields Tab go to the FORM selector at the bottom of the page, select Body and click Apply Filter.

NOTE: You should always do this to ensure you are adding fields to the correct form – adding fields to an incorrect form will mean that they do not display.

Adding a Sort to a Column

A column header can be used to sort the data. To add a sort enter the field name followed by either a + for ascending or – for descending.

For this example, add a sort to the jc_jcc field. (kco+, job_num+, jph_phase+, jcc_cc+)

This will also provide a simple search filter to the Enquiry screen.
