Overview

OA online documentation gives you information that is standard for all users, such as logging into the system, module-specific setup and configuration, etc. It is found on the OA Online Documentation Menu, which can be reached by clicking  at the bottom of each OA screen.

This documentation is organed through the use of menu books and topics . Customers can modify the standard documentation to reflect their own business processes and their specific use of . The purpose of this document is to show you how to perform these modifications.

EXAMPLE:

Let's say your (ABC) and another (DEF) use OA on different systems (or environments); for simplicity, we'll say that ABC uses an ABC system and DEF uses a DEF system. Suppose that ABC users need to know how to perform a task specific only to the ABC environment. The ABC can therefore create new - or modify existing - menu books and topics for that task on the OA Online Documentation Menu within its ABC environment, which means that only your users in ABC can see these menus/topics, and DEF users cannot.
