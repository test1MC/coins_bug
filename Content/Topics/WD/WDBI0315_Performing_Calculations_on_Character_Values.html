Performing Calculations on Character Values

INLIST

As IF statements can perform comparisons on numerical values, then the INLIST function can be used to perform such on character values. INLIST should be used in the format :

     INLIST('string1','string1,string2')

and will return 1 for a true result and 0 for a false result.

Therefore

INLIST('a','a,b,c')        would return 1

INLIST('a','b,c')          would return 0

Using INLIST to assign text values to a column :

The example below checks the flag on a Purchase Order Header and instead of returning a TRUE or FALSE value will return either E-Commerce or Standard.

$IF$((INLIST(\,'yes')),EQ,1,'E-Commerce','Standard');


CAN-DO

The CAN-DO function tests a value against a comma separated list of criteria in a similar way to INLIST but will allow the use of wildcards and exclusions. It should be used in the format :

     CAN-DO('strin*','string1')

For example :

     CAN-DO('A*','A1000') would return 1

MASK

The MASK function will return a 1 or 0 for a true or false result when comparing a value against a combination of a RANGE of values and a CAN-DO list and should be used in the format :

     MASK(a,b,c,d)

Where a is the range from, b the range to, c is a can-do list and d the value to be masked. In effect this function behaves as a From, To, List selection in .

For example to check a GL Account ? element 2 within a range from 2000 to 2180 but excluding 2170 :

MASK('2000','2180','!2170,*','\');
