Posting a Return

After the Return has been submitted, it must be posted.

To post the Return:

  1. Go to Returns Workbench.
  2. On the Returns tab, select the Return.
  3. Choose the Post Return to GL action.

Posting a Return will transfer the returned balances from the  control accounts to a separate  Debtor/Creditor Account. The balance on this account will then be cleared by posting of the cash payment/receipt to/from HMRC. This process avoids the need to allow manual postings to the  control accounts for this monthly/quarterly cash posting.

Only unposted returns can be posted. This will create and post a GL batch with one line per unique GL control account for codes (excluding non-recoverable codes) included in the return with a value of the total for Codes linked to that GL Control Account (Positive creates a Debit entry and Positive creates a Credit Entry). The total of these lines is then reversed and posted to the Creditor/Debtor Account for the Registration. The Posting Status for the Return is changed to "Posted".

A posted return can be unposted.

  1. Go to Returns Workbench.
  2. On the Returns tab, select the Return.
  3. Choose the Unpost Return to GL action.

This makes the reverse entries to those that were made when the Return was posted.
