Configurable PL and Remittances

allows you to design and create various layouts for printing and remittances onto your own company pre-printed stationery.

Define the layout of and remittances using Document Designer in the System Module. See Configurable Documents for information on how to define configurable documents.

In , you can define three types of documents to print: only, remittances only, and combined and remittances. You can create different documents for each type, which will enable you to print various layouts for a particular type of form. When you print the or remittance, lets you specify which document layout you want to use.

To configure forms for or remittances:

  1. Use the document type /Remittances (PL).
  2. In the Layout Type field, select:
    • to define the layout for ;
    • Combined to define the layout for combined and remittances;
    • Remittance to define the layout for remittances.
  3. You can set up the following forms:
    • header/footer - In the header of the form, put information to appear on the top of the remittance advice, such as the 's name and address. In the footer, put any additional information that is needed, and for prints, the details that will be printed on the . Header and footer use tables ap_check, ap_vendor, sc_mast.
    • BODY - In the body of the form, put a single line containing details of the invoices paid. Body forms use tables ap_pay, ap_invoice, jc_job.

      NOTE

      Each document needs to have a BODY form (even if it doesn't print anything).

    • DISC - To print discount on a separate line of the remittance (for example, underneath the invoice amount instead of alongside it), create a row with form type DISC, containing the field apy_discamt. This will only print if the discount amount is not zero.
    • TAX - To print tax deducted on the element of an invoice as a separate line, create a row with form type TAX, containing the field ain_paye. This will only print if tax was deducted.
    • JOB - To print remittance comments, Create a row with form type JOB, containing the field job_remcode. Remittance Comments are lines of standard text that can be associated with a . A typical remittance comment might be: Paid on behalf of ABC Ltd. (See Remittance Comments Maintenance). If you use this section type, the Break on Job field; sorts the invoices being paid into order and prints a separate per .
    • PAY - To print manual payments issued since the previous remittance and the invoices they were allocated to, use a form type PAY. This form can be configured to print the number, if applicable. DISC, TAX and JOB are repeated for these payments also.
  4. To configure a combined /remittance document, configure the body form to print the remittance lines. The can be either at the top or at the bottom of the page. If the remittance lines take up more than one page, you will need to print a "VOID" form on the portion of the stationery on all pages after the first one. Configure a form called VOID, then, depending on whether the is at the top or bottom of the stationery, do one of the following:
    • If the is at the top: in the Location field, select Top of Page. In the Continuation Header Form field, enter VOID. Configure the Header Form to print the details.
    • If the is at the bottom: in the Location field, select Bottom of Page. Configure a form to print the details, and enter that form name in both the Footer Form and Continuation Footer Form fields. then replaces the footer with the VOID form after the first page.
  5. To print one per page, leave the Multiple per Page?box blank, and configure the header or footer form to print the details. To print multiple per page, the Multiple per Page?box, and configure the body form to print the details.

Fields

RO_SpelledAmount - This is the amount spelled in words all on one line based on USA standards.

RO_usaAdd1, RO_usaAdd2, and RO_usaAdd3: Use these each on a separate line to print the address in USA formatting and without a blank line in the middle of the address if the name/address only uses three lines instead of four.
