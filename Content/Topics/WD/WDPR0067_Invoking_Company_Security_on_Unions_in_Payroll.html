Invoking Security on Unions in Payroll

In OA Payroll, Unions, Union Rates, and Union Calculations are global files. They are maintained and accessible across all under Global Setup. In order to restrict union access by , the process described below can be implemented.

How to Set Up Security on Unions:

By following the process below, you can define the Union Group codes that will be valid for a .

  1. The screenshot below depicts all of the unions on file in without restrictions. Note the types of Union Group codes listed for each union, such as g1, Dutchess, ff.

    For this example, we want to restrict the 's unions to those with Union Group codes containing a1, g1, and u.

  2. Go to the UNIONBR parameter in Parameters. We will use this parameter to define the valid Union Group codes for the .
  3. Select the UNIONBR parameter.
  4. Under Choose Action, select Specific and enter the value(s) to be unique for the . In our example, the entries will be *A1*,*g1* ,*u* (see the screenshot below), meaning that a valid match for the Union Group can be any value that contains A1, any value that contains g1, or any value that contains the letter U.

    The value(s) entered are compared to the value(s) entered in the Union Group field in . Matches can include wild cards (*), exceptions (!), and can-do or comma separated lists.

  5. If you return to , you can see that the list of Unions for all users in the has now been limited based on the values you entered in Parameter UNIONBR. The screenshot below gives you an example of this limited list.
