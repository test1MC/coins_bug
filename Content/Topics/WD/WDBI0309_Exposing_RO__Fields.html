Exposing RO_ Fields

In order to use the Read Only (RO_) fields in the OA Report Writer and coins OA Application Adapter, it is first necessary to expose these fields, that is, document the parameters which are to be passed to a field, within the database.

In the System module in OA, System Maintenance and Table Documentation select the table in which the field you wish to pass parameters to.

The Parametered Fields section then needs to be populated with the appropriate RO_ field and any parameters that are required.

A single field can be documented more than once with different parameters, for each a balance field can be documented with several dates, such as a year of period end dates.

Another example would be to pass various field extents to the same field such as different currency values of transactions.

Examples of an RO_ field and associated parameters are :

jc_job.RO_job_costs^TP|-3           Costs ? This Period with a period offset of -3

cs_certificate.RO_cst_ctd_cum^10 Certificate Item ? Cumulative to Date

                                                  Certificate Line 10

Once a RO_ field has been entered in the Table Documentation with appropriate parameters, it will then become available in OA Application Adapter and Report Writer for selection as per standard database fields. It will also appear in the Database with a field type of EX.
