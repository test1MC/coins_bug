Customing Using Terminology

Different varieties of English (for example, British and American English) sometimes use different words and phrases for the same concepts.  Also, various business areas and individual companies may refer to the same thing in different ways.

To allow for this diversity, you can use terminology to change a number of words and phrases to suit the way you work.  These changes can apply to fields, function names and messages, as well as side-frame help and online documentation.

How terminology works

includes a number of pre-defined tokens which represent specific words or phrases with display values that you can change. If you change the display value of a token, displays the new value wherever that token is used.  For example, the token "kco company" refers to internal companies in .  If you use companies to represent business units, you can change the display value of this "kco company" token to "business unit"; as a result, "company" will appear as "business unit" (wherever it refers to a internal company).

provides pre-defined terminology sets which contain standard terminology equivalents for American, Irish and Australian English.  You can select one of these terminology sets to use as a starting point (see below); for instance, a US business would most likely select the American set.  You can then override the display values in this terminology set for any of its individual terms.  Alternatively you can create your own terminology set (see below).

You can select terminology sets on a system-wide or -specific basis.

NOTE

You can only change words and phrases that have been specifically catered for in .  You cannot add a term to the list and expect it to change.  However, if you have a requirement to change a term  has not catered for, contact your representative to see if it may be added.

To select a terminology set:

  • In SY Parameters, set the parameter SY/TERMCODE to the code for the terminology set you want to use.

    This can be one of the standard terminology codes, or the code of a terminology set you create (see below).  Leave SY/TERMCODE blank to use the standard terminology set, which is based on British English.

 

To override the display of an individual term:

  1. Go to Terminology.
  2. Using the Terminology selector at the bottom, select the terminology set you are using.
  3. Click next to the token you want to change.  (Hint: you can filter on the Display Text field to find the token.)
  4. In the System Override field, enter the display text you want to use. (Standard terminology sets will be overwritten when is upgraded, but the System Override field will not be overwritten.)

    NOTES

    • terminology applies capitalation rules where appropriate when displaying text.  Enter the text in lower case unless you want it always to display in upper case.
    • If a noun has separate singular and plural tokens, remember to change the display text for both.
    • Sometimes the same word has different meanings; terms will only be changed where appropriate.  For example, there is a token "jc contract" which refers to contract records; if you change the display text for this, references to employment contracts or house purchase contracts within will not change.  Similarly, in the American terminology set, the token "" has the display text "check".  If you override this, the display for "check" (meaning a method of payment) will not change, because this uses a different token.

Configuration

To create a new terminology set:

  1. Go to Terminology Code.
  2. Add a new entry (Code and Description) for the terminology set you want to create.
  3. Click .
  4. Go to Terminology.
  5. Using the Terminology selector at the bottom, select the new terminology set created in step 2.
  6. Add terms to the list.  These terms must exist in the Standard terminology set.  Enter the terms exactly as they appear in the Standard set.
  7. In the Display Text field, enter the text you want to display.
