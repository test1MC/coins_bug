Memorandum Orders

"Memo Orders" serve as a simple "authoration to proceed". They carry no value and so cannot produce a committed cost nor any form of accrued cost. Line details may be entered if required but no quantities, prices, dates etc will be entered. The main reasons for entering line details would be to provide an indication what type of is to be and to provide some default cost coding for the 's invoice.

When entering a memo order, you specify the commodity code, override the description where required and enter any WBS code where relevant. Line input is simplified compared with full orders, and only , text and clause line types are available. It is perfectly feasible that no line items details are entered at all.

Normally, if a valid purchase order number has been entered, a new tab called "Matching" appears once the initial entry screen has been saved. If the order specified is a Memo Order this Matching tab will not appear; instead the tab called "Costing Distribution" will be shown where you can enter the costing details manually.

Committing Orders

Just like any other order, memo orders must be committed before invoices can be entered against them.

has the capability of sending order details electronically to a vendor (using the -etc Hub). Memo orders will NOT be sent electronically (even if the vendor is configured as an electronic recipient of orders). This is because memo orders do not carry information that is sufficiently precise to enable electronic trading.

Some systems have the PO parameter WARNUNPR set to Y. This issues a warning if a user attempts to commit a purchase order that contains an item with a zero price. This parameter will be ignored if the order being committed is a memo order.

Entry

can be entered against memo orders. The method of input is exactly the same as for the original order.

The only way to prevent further invoices from being entered against a memo order is to use "Force Complete Orders".

Setup

In Purchase Order Types, add a new type of order to represent Memo Order.  You can use any spare letter to identify this order type (however, it is essential that the code used does not occur in the Subcontract Order Types table), and the description can be whatever you like.

Enter the code as the value of the PO parameter POMEMTYP.

Note that Purchase Order Types apply to all purchase orders (apart from subcontract orders which have their own set of Subcontract Order Types).  Memo Orders cannot be used against any other major category of orders.  In other words we do not support Material Memo Orders, Overhead Memo Orders and so on.
