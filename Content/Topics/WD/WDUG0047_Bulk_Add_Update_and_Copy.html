Bulk Add, Update and Copy

Many transaction pages allow you to enter or update several records at once, by pasting from an Excel spreadsheet.  validates the records in order, in exactly the same way as if they had been entered by hand, and any errors will back out the whole transaction.

The bulk add feature is useful for loading large numbers of records from a spreadsheet, both for standard details (such as details) and also transactions (for example, for accountants loading substantial line entries for General Ledger month-end journals).

NOTE

Bulk maintenance is a powerful tool, but should be used with care.

More details...

The main consideration is the volume of data being accessed via bulk maintenance. You might experience time out or data issues if the volume of data is significant. If you need to maintain many records, consider the nature of the data you are loading and the number of records in your source spreadsheet.

Each browse will be different, for example a simple lookup table with just a code and description may successfully handle 1000s of records. In contrast, costing distribution records with many fields, most of which will require validation (, costing analysis, , cost effective dates, GL accounts), may only process a few hundred records in a single transaction. cannot be definitive on exact numbers for each browse as there are many internal and external factors that may affect each transaction.

For large numbers of more complex records, would recommend breaking down the bulk transaction into several, smaller blocks of data to ensure that the paste and save complete successfully.

If you use bulk maintenance is used on a regular basis for processing data, you should consider other options for data loading in the system. There are many other options (Data Import routines, Workspaces, WebServices) that may be much more appropriate to meet the business needs.

Bulk update features - configuration

The Bulk button is only available on a browse if there is a corresponding function ending with "B".  For example, %WSC2013BVPT is Subcontract Rates, and %WSC2013BVPTB is the function that enables the Bulk button on the Subcontract Rates page.

Bulk maintenance needs to be used carefully. If used incorrectly it could cause data problems.  You should consider restricting access to bulk maintenance.
