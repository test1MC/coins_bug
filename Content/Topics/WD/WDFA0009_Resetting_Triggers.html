Resetting Triggers

You can reset a trigger when the event to which the trigger refers (such as the service) has been completed.  This prepares it to track the next occurrence of the event. 

Resetting a trigger clears the Off box (so the trigger is not expired) and resets the Expires field or the current value (depending on the definition of the trigger).

To reset a trigger manually:

  1. Open the asset.
  2. On the Triggers tab, select the trigger(s) you want to reset.
  3. From the action list, choose Reset Trigger and click .

Resetting triggers using workshop

automatically resets triggers which are specified by elements when a transaction is posted to the cost code. You can use the elements in different ways to control which triggers resets.

For example, you might want to reset triggers according to the reason for the work, so that the triggers for cam-belt change and filter change are reset for a 30,000 mile service, and oil change is set for all services. Assuming you have three elements set up as above, you could set up the trigger list on the first element (parts) to include the trigger for that item; the replace activity for the second element to reset all triggers, and the 30,000 mile service reason for the 3rd element to reset the triggers for cam-belt, filter change and oil change. The trigger list for other services would include oil change.
