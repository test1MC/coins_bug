Account Masking

derives the General Ledger account for a cost from both the -specific cost code and the on which it is used. The cost code and can both specify an account mask; the masks identify, in full or in part, a General Ledger account. To take a simple example, if part of the account code denotes the region (01 - North, 02 - West, 03 - South) and part of the code denotes the type of cost (1200 - materials costs; 1300 - costs; 1400 - subcontract costs), would be set up with account masks according to their region:

and would be set up with account masks against the appropriate categories according to the type of cost:

  • Material would have _ _.1200 against the material category
  • would have _ _.1300 against the category
  • Subcontract would have _ _.1400 against the subcontract category

Then when a material cost is recorded on a northern , the masks would combine so that the cost is recorded against account 01.1200.

NOTE

The mask on the cost code defaults from the mask on the standard on which the cost code is based. However, if you subsequently change the mask on a , this will not affect the mask on any cost codes based on it.

You can set masks on the Group, Type, and Location records (typically for different parts of the account code). The Group, Type, and Location specified on a will then build up the account code masks on the GL Codes tab in {Jc Contract} Details. You can override individual accounts on the GL Codes tab (but note that if you subsequently change the Group, Type, or Location, the relevant part of the account mask will be re-set).
