Sector-Based Ledgers

Sector-based ledgers enable you to produce separate payment runs (for example, in and Subcontract Ledger) for each sector, which in turn produce a separate for each sector.  In effect it is as if you had a separate ledger for each sector, but with a single set of .  You might want to do this if, for example, your sectors correspond to separate legal entities, or if they have different branding.

When users process a payment run, they must first specify the sector, and will only be able to select transactions for that sector.

You may also want to consider using sector security with sector-based ledgers.

To enable sector-based ledgers:

  1. Sectors must be enabled (see Sectors).
  2. Set the SECTLED parameter in the parameters list for each ledger (Expenses, , and Subcontract Ledger) to Y. You can do this for some ledgers and not others.
  3. If you are using sector security, set the parameter BNKSECSC to Y.
