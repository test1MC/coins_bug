Entering Dates

To enter a date, you can either use the calendar lookup, or type the date, using the format dd/mm/yymm/dd/yy.

When you type dates, you can enter part of the date and fills in the missing parts based on today's date.

For example, if today's date is 3rd February 2012, you can enter the following:

/9 Return will give 09/02/1202/09/12 (that is, 9th of the current month).

9/4Return will give 09/04/1204/09/12 (that is, 9th April of the current year).

 

You can also enter date calculations:

$ represents today's date.  You can enter $ followed by + or - then a number and a unit (D = days, M = months, Y  = years)

For example (again, if today's date is 3rd February 2012):

$ Return will give 03/02/1202/03/12.

$+1M Return will give 03/03/12.

$-1MReturn will give 03/01/1201/03/12.

$+5DReturn will give 08/02/1202/08/12.

$+1Y Return will give 03/02/1302/03/13.

Dates before 1st January 2000 are not a problem: will interpret the date 10/01/981/10/98 as 10th January 1998, 15/3/053/15/05 as 15th March 2005, and so on.

Note that when you enter a period code, for example 0998, you do not need to enter the initial 0.

NOTE

This assumes that you are set up to use the MDY DMYdate format.  The date format can be set differently for individual users.
