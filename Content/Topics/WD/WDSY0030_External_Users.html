External Users

In addition to the password security, you can also provide different security according to where the users log in from.  This allows you, for example, to grant limited access to people logging in to from outside your organation.

You specify which IP address or addresses are counted as internal addresses; if anyone logs in from a different IP address they are deemed to be an external user. 

External users will only be allowed to log in to if the Extranet User field on the user record is ed.  This prevents someone being able to log in to from outside by guessing the user ID and password of an internal user.

In addition, you can specify, for each user ID, the IP addresses from which they are allowed to log in.  This means that, for example, you could specify the router address of the company from at which a user works.  This applies to internal users as well as external users.

For additional control over the access of external users, you can set up a dummy user which you set security against; the security settings for this user will be applied to all external users.  In order for an external user to be able to run a function, both the actual user ID and the dummy user ID must have access to that function.

External users - configuration

Create a dummy user record and give it the access permissions you want to give external users.  Enter the user ID of this record in the SY/EXTUSER parameter.
