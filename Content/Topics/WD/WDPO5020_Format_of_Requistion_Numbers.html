Format of Requisition Numbers

You can configure the format of purchase requisition numbers, so that they show information about the requisition.

Requisition numbers can be up to 15 characters long.

The simplest format for a requisition number would be a single numerical sequence for your entire . To do this, set PRNUM1 to comp, and PRFORM to 99999999 (or however many digits you want in the number, up to 15).

But you may want the number to show other information:

  • You can show any one of the following as the base part of the requisition number:
    • For material, or subcontract requisitions: , location, group, type, or any one of the four analysis sets. Use the PO parameter PRNUM1.
    • For overhead requisitions: department or one of the four General Ledger account elements. Use the PO parameter PRNUM2.
    • For workshop requisitions: location or . Use the PO parameter PRNUM3.
    • For overhead requisitions: asset code or department code. Use the PO parameter PRNUM4.

      • For example, to include the code in the requisition number, set PRNUM1 to . prefixes the number with the code. There will be a different sequence of numbers for each prefix; for example: 1/00001, 1/00002 and 2/00001, 2/00002.

      NOTE: To generate different sequences for each , set the above parameters to comp. The number is not shown as part of the requisition number.

  • To indicate the type of requisition (for example, material, or subcontract), enter the prefixes against the following PO parameters:
    • ASREQPFX (Asset)
    • OVREQPFX (Overhead)
    • PLREQPFX ()
    • PRREQPFX (Material)
    • SCREQPFX (Subcontract)
    • STREQPFX ()
    • WSREQPFX (Workshop)

    For example, MAT-, OHD-, PLT- would produce requisition numbers such as MAT-00001, MAT-00002, OHD-00001, PLT-00001, OHD-00002 and so on.

You can combine any of the above, and all are optional. generates separate sequences for each different prefix.

Subcontract requisitions always have different sequences from material/ requisitions, even if you are not using any prefixes. The parameter SRFORM governs the format of subcontract requisition sequence numbers, and SRNUM1 governs the base.

You can change the next number in any of the sequences. See Number Sequences.
