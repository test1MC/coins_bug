Rates

Rates allows you to store rate charges for billing purposes. It allows payroll transactions, imported through Load Transactions, to be re-charged to the customer.

This assumes is charged at an agreed rate per hour, with different rates applying for different classes of employee. From this table the ' Transaction' processing will determine which SOR item to create for each imported ' transaction' record.

For each employee position, earnings code, there is one entry in all the other fields (sell rate per hour, costing group, SOR item, invoice class), and which rate applies for a transaction is determined by a hierarchy of customer, minor category, major category, as per mark-ups. rates - configuration

Rates can be found under FM Setup - 'Invoicing Setup' tab group.

The SOR item must be set up as a standard one against the current SOR type. However, it is not possible to check whether the SOR item is a standard one at the configuration stage because we do not know which major category is going to apply.

As a compromise, an error will be given if the SOR item does not exist at all on any major categories for the current SOR type. A warning will be given if it is missing off some (the message displays which ones).

Standard Auditing is available on this table.

You can import rates from a CSV file using Load Rates. The format for the CSV file is as follows (all field values are text unless stated):

  • Major category (Optional)
  • Minor category (Optional)
  • Customer (Optional)
  • Position (Optional)
  • Position description (Optional)
  • Earnings code (Optional)
  • Description (Optional)
  • Cost Group (Mandatory)
  • SOR Item (Mandatory)
  • Invoice class (Optional)
  • Rate per Hour (Mandatory)
