FM Invoice Configurable Documents

FM Invoice documents (document type FMINV) are used to print invoice documents, whether created from 'normal' FM processing, or via the /certification process.

Define the layout of configurable documents using Document Designer in the System module. See Configurable Documents for information on how to define configurable documents.

Invoice Style

All FM and invoices have an 'invoice style'. This can be 'SINGLE', 'CONSOLIDATED', 'MAJOR CATEGORY', 'CROSS-'. This affects the of the documents:

  • SINGLE - the header is a FM and the individual lines shown are items (SOR items or cost transactions). You can subhead/subtotal by 'invoice class' ('invoice class' is a category you can apply to a SOR item or cost transaction that enables you to group these items in non-standard ways - and add mark-ups to them).
  • CONSOLIDATED - the header is customer/ and the individual lines shown are FM within this .
  • MAJOR CATEGORY - the header is customer/major category and the individual lines shown are FM within the major category.
  • CROSS- - the header is customer and the individual lines shown are FM for the customer.

 

To configure an invoice document:

  1. Go to Document Designer.
  2. Use the document type Invoice (FM) (FMINV).
    • For single invoice styles
      • For a summary invoice (one line per invoice class), Summary and configure the Invoice Class Subtotal form. For example, if you have one invoice class for '' and one for 'Materials' you can show:
        21.00
        Materials50.24
      • For a detailed invoice (individual invoice lines), unSummary and configure the standard Header, Body and Footer forms. If you also want subtotals by invoice class, configure the Invoice Class Subtotal form. For example, if you have one invoice class for '' and one for 'Materials' you can show:
          
         Task 110.00
         Task 211.00
         21.00
        Materials  
         Mat 130.23
         Mat 220.01
        Materials 50.24
    • For other invoice styles, Summary and configure the CONSOLIDATED Header, Body and Footer forms on the Extras tab.
  3. To configure a credit note document, the Credit Note box; to configure an invoice, leave it uned. The credit note flag only applies if the separate Credit Notes workbench is being used. In this case credit notes will only ever include negative items - and positive items will appear on an invoice. This type of credit note is always 'single' style, the idea being that giving credit would be an exception so would normally only apply to single . If the separate workbench is not used, an invoice would be a credit note if the total is negative and would appear on the normal Invoices workbench.
