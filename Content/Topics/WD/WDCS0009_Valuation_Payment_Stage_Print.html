Valuation Payment Stage Print

The Valuation Payment Stage Print document format allows you to print out the Sales Valuation Payment Stages details.

Valuation Payment Stage Print documents show a line for each payment stage.

Define the layout of configurable documents using Document Designer in the System Module. See Configurable Documents for information on how to define configurable documents.

To define the format for valuation payment stage prints:

  1. Use the document type Valuation Payment Stage Print (CS).

The following tables are available:

cs_valuation, jc_job, cs_valcert, cs_series, co_config, ar_cust

The BODY form shows: the stage in the valuation cycle, the date of the valuation, the date when payment for this valuation is due, stage details such as Gross Cumulative, Retention Cumulative, Discount Cumulative and Net Cumulative. Use the field RO_cvc_pvalue^ plus the number that identifies the item in the certificate definition (1 is GRC, 2 is RET, 3 is COS and so on), and /P to use the This Period values. For example: RO_cvc_pvalue^3/P.

There are other forms:

  • BODYHEAD - labels for body fields. To display labels for the items, use the field RO_cvl_plabel^ plus the corresponding number and value type. For example: RO_cvl_plabel^3/P.
  • CASH - cash received. This shows: reference, number, date, bank, gross amount, net amount, void flag. Tables are cs_pay, cs_check.
  • HEAD - header. This shows: , phase, series, valuation, customer.
  • TOTAL - totals. This shows: totals of applied for, gross certified, retention, discount, , net payment due, cash received, C/S balance, balance. Table is cs_valopen. Use the field RO_Total^ plus the ID of the item you want to total. For example: RO_Total^CSINV.
