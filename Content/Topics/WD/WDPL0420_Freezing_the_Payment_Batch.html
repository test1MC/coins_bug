Freezing the Payment Batch

When the invoices have been selected for payment, automatically puts them into a payment batch and allocates a batch number. The payment batch will need to be frozen (closed) so that no other payments can be added to it. This needs to be done before the payment run can be made. Any other invoices that are selected after a batch has been frozen will subsequently be placed in another payment batch.

To freeze the payment batch:

  1. Go to Payment Batches. 

    displays a list of payment batches.

    If you want, you can view the selected invoices by clicking the link in the Batch Number column.

    NOTE

    If your system is set to run multi-currencies and/or sector based ledgers, then these will determine whether users are able to create multiple (unposted) payment batches. allows for only one payment batch, per currency, per sector. 
  2. From the list, select the payment batch or batches that you want to freeze.
  3. Choose Freeze Payment Batches from the list.

You have now frozen the payment batch. 

 

You cannot add or delete invoices from a frozen batch. If you want to make any changes, you will need to unfreeze it

 

After freezing the batch, the next step is to print the and generate the payments (see Printing and Remittance Advices and Payments).
