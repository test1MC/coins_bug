It may also be necessary to amend the current PROGRESS Parameter on the database to ensure that it is set to the correct value to allow connection by the WebSpeed brokers and also the Report Generator for .

The value for the PROGRESS parameter (-n Maximum Number of Users) should be calculated using the following values :

  • The total number of system connections to the database (that is, Watchdogs, APIs, Brokers)
  • The total number of Licensed users
  • The total number of Report Generators to be run (value in Print Manager Parameter GENRPOCS)
  • The total number of WebSpeed agents running on the Environment

NOTE

If the required value for the PROGRESS Parameter exceeds the current PROGRESS Enterprise Database Count please contact your Account in the first instance.
