To set the style parameters:

  1. Using SY Parameters, set the filter to Style and click .
  2. Update the parameters for the features you want to change:
    • If the parameter takes a , you can enter either a name that your browser recognes, or a six-digit hexadecimal code (see About in HTML and FOP for more information).
    • If the parameter takes a cascading style sheet style declaration, this is of the form property:value, with multiple declarations separated by semi-colons. Some common properties include:
      • background-color (note the spelling).  This can take a name or hexadecimal code as its value.
      • color (note the spelling).  This is the foreground (for example, the text ).  This can take a name or hexadecimal code as its value.
      • font-family.  For example, "Times Roman" or "System".  You can enter a list of font families, separated by a comma; if the first one in the list is not available, the browser will use the next available font in the list.
      • font-weight.  For example, "normal" or "bold".

      So, to set the title banner to purple text on a white background, using "Georgia" as the font if it's available, and "Times New Roman" if not, set BANSTYLE to background-color:white; color:purple; font-family:Georgia, Times New Roman, serif; font-weight:normal

      NOTE

      In the standard implementation, the shaded background for the body, column headings and tabs are actually images, defined by the parameters BODYIMG, COLHDIMG and TABIMG; these override the plain background .  If you want to change these , you need either to set the image parameters to none and use the parameters BODYBG, COLHDBG and TABBG, or to set the image parameters to point to alternative images.
If you want to change the and styles of screens in ways that the style parameters will not allow, you can set up a user-defined cascading style sheet file to override the standard styles. 

NOTE

If you want to remove the title banner from a specific page, add NoBanner=Y as a parameter on the page function.
