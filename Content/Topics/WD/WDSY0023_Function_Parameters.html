Function Parameters

Many programs within can be set up to run in different ways. Function parameters (also called menu parameters) determine how the program runs. For example:

  • The same order entry program is used for entering material orders and subcontract orders, and for site orders and head office orders. So:

    po101.p with parameters (MATERIAL,C,A,B,M,N,H) = Enter head office material order,

    but

    po101.p with parameters (TRADE,F,C,C,,N,S) = Enter site subcontract order

You need to set up a different function for each different way you want the program to run; in fact most of the commonly required functions are already provided as standard.

You can allocate functions to users (see How Function Access Works) to control exactly which users can run which routines (for example, Browse can be made available to many users while full Maintenance is made available only to a limited number of trained users).
