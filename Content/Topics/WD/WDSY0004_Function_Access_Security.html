Function Access Security

You can control which users have access to which functions. For any user, you can control whether or not they have access to any given function. You can also give groups of users the same permissions. See How Function Access Works. It is important that you ensure that users do not have access to functions they should not be allowed to use. For example:

  • Users and User Function Access form part of security, and so only those users who have responsibility for security, such as the System Administrator, should be allowed access to these procedures.
  • Management reporting functions may not be relevant to data entry staff, so these staff members do not need access to these functions. 

Obviously, the access you allow or disallow depends on the requirements of your business and the responsibility of individuals within it.
