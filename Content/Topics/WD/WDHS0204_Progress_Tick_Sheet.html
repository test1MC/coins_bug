Record Build Progress – Progress Tick Sheet

The Progress Tick Sheet provides a quick method of updating the actual construction progress.  It will allow quick update of plots and stages which have been completed on the same date.

The layout is designed to be in a format which matches the House Sales > Plot Sales > Reports > Construction Progress Tick Sheet report which can be sent to site for update.

This screen excludes plots for which all applicable stages have been ticked, and plots which have not had any Build Release and which have not had any stages ticked.

Select the Progress Tick Sheet Tab.

Click

The plots are sorted as follows:

  • Primary Sort: Build Sequence
  • Within that sort by Plot Group
  • Within that sort by Plot Number

Plots for which all applicable stages have been ed are excluded.  Also, plots which have not had any Build Release and which have not had any stages ed are excluded.

The first three columns show the build sequence number for the plot, the plot group code and the plot number.  The remaining columns represent all construction stages which are applicable to the .

To update the sheet:

  1. Click to open the screen for editing.

    • Stage already ed (not updateable here)
    •  Stage available to (click to )
    •  Stage ed not yet saved (click to un)
    • Stage not build released.

      NOTE

      In this tab you cannot a plot/stage that has not been build released, unless the HS parameter CSVALREL is blank (in which case stages will be displayed as updateable regardless of build releases). If CSVALREL is set to "warning" then it will be possible to complete plots/stages that have not been build released elsewhere but not on this tab.  If CSVALREL is set to "error" then it will not be possible to complete plots/stages that have not been build released..
    • Blank - The stage is excluded from the construction for the plot; not updateable.

      NOTE

      If a stage does not apply to any plots, it will not be shown here.
  3. To un a stage which has already been ed on a plot, you need to delete the construction stage event either through the history tab on the Sales Workbench or through the history tab on the Plot level Construction Workbench.
  4. Having ed the required cells, click  to save the updates. 

    After saving, the screen will be redisplayed (any newly ed cells will now display as  indicating that they have been saved).  This creates construction stage events in the history on the sales workbench for each plot/stage ed.  For plots with a construction method included in HS parameter CSFILL then any earlier stages on the plot which have not yet been completed will also have been automatically completed.

    NOTE

    If you save an incorrect change to a stage, you cannot undo it here; you need to go to the plot history to delete the relevant event.
