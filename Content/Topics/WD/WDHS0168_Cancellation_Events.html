Cancellation Events

Cancellation Workbench Events allows you to set up the events you can run for a cancellation.  These might include recording a cash transaction, notifiying solicitors or closing the cancellation record.

To set up cancellation events:

  1. Go to Cancellation Workbench Events.
  2. Click to add a new event.
  3. Enter a code for the new event.
  4. If the event is one that is not run manually from the Cancellations Workbench but is run automatically by a process like sending a letter, System Generated.
  5. Select the system event. This determines the type of event you are setting up. There are four types of event:
    • CASH - allows you to record cash transactions (such as refund of deposit).
    • RETAIN - allows you to record that you have retained part of the deposit.
    • CLOSE - closes the cancellation file (you can filter out closed cancellations).
    • MEMO - allows you to record additional information about the cancellation.
  6. Enter a full description for the event. (The short description is only used in .)
  7. If you want to allow users to run the event more than once, Allow Duplicates.
  8. If you only want to allow the event to be run if other specified events have already been run, enter the codes for those events in the Previous Events field.
  9. If you want to prevent the event if other specified events have already been run, enter the codes for those events in the Stop Events field.
  10. To restrict how many days before or after today users can enter for the date of the event, the Restrict Backdating or Restrict Forward Dating field (or both), and enter the number of days. For example, to stop users entering a date more than two days before today, set Allowable Days Backdating to 2. If both are set to 0, users will be forced to enter today's date.
  11. Enter the default comments; this is the text that will appear in the Comments field when the event is run (the user can enter different text if they want).
  12. If you want to update captions when the event is run, select the caption groups to update.
  13. Click .

Cancellation Events - configuration

Set up which users can run the event, using Cancellation Event Security (see Event Security).

To copy the comments for the event to the Notes automatically, enter the event code as one of the values of the HS parameter COMNTVIS.
