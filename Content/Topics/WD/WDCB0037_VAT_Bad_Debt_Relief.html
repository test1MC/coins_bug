There are a number of rules governing the reclaim of on bad debts for Outputs (i.e.
sales). Those that are relevant to processing within are as follows:
_ You must have written off the debt in your day-to-day accounts and transferred
it to a separate bad debt account.
_ The debt must have remained unpaid for a period of six months after the later of the
time payment was due and payable and the date of the supply.
The rules for repaying already claimed on Inputs (i.e. purchases) are slightly
simpler:
_ You are required to repay input tax if you do not pay for the supplies within six
months of the relevant date (i.e. the later of the date of supply and the due date for
payment).

The basic process is the same for each of the four ledgers affected (,
, Subcontract Ledger, Ledger):

  1. Select the transactions which will be flagged as bad debt
  2. Create a “bad debt” batch from the selection
  3. Print a batch listing report (optional)
  4. Post the batch
  5. Print a batch posting report (optional)

Only one unposted bad debt batch can exist for each ledger. The selection can be run
multiple times, each time will clear out the contents from the previous run. Transactions
using codes set up to be “ at pay” cannot be selected, as the transactions
have not yet been created.
Posting the batch will do the following:
_ Flag the transaction as ?bad debt?. This will prevent it from being selected in future
bad debt selection.
_ Create reversing transactions for the outstanding balance
_ Transfer the outstanding amount from the suspense account to the bad
debt suspense account
_ Transfer the outstanding balance from the ledger?s control account to the bad debt
control account
If a payment is later made to a transaction flagged as ?bad debt? then the following will
happen:
_ a transaction will be created when the payment is posted (e.g. as if the
code was set up as ? at pay?)
_ the amount will be transferred from the bad debt suspense account (rather
than the standard suspense account)
_ the payment amount will be transferred from the ledger?s bad debt control account
(rather than the ledger?s control account)
