Configurable Statements

lets you design documents for printing statements. In this way you can print to your company's requirements and use pre-printed stationery.

Define the layout of configurable documents using Document Designer in the System Module. See Configurable Documents for information on how to define configurable documents.

To define the format for printing statements:

  1. Use the document type Statements (CS).
  2. In the Document field, enter a name for the statement document (for example: CSSTMT). You can define different documents to print statements with different layouts. When you print the statement, gives you a choice of which document to use.
  3. In the Type and Days fields, specify how you want the outstanding total amounts to be broken down over time.

statement documents use the following fixed form types:

  • CERT: Lines for certified amounts not fully paid. These use the tables cs_certificate, cs_series, jc_job.
  • INV: Lines for invoices not fully paid. These use the tables ar_invoice, jc_job.
  • CSCASH: Lines for cash received but not allocated to a certificate. These use the tables cs_check, jc_job.
  • SLCASH: Lines for cash received but not allocated to an invoice. These use the tables ar_check, jc_job.

In addition, each of the 4 body forms (CERT, INV, CSCASH, SLCASH) has the following calculated fields available: RO_txn_aged[6], RO_txn_amt, RO_txn_bal, RO_txn_unalloc.

The following running totals are also available: RO_tot_aged[6], RO_tot_amt, RO_tot_bal, RO_tot_unalloc, RO_tot_tpByID^CSCERTID, RO_tot_cumByID^CSCERTID.
