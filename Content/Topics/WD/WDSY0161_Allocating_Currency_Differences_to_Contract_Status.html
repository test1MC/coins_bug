Allocating Foreign Currency Differences to Status Instead of General Ledger

Some users may wish to allocate their foreign exchange variances, both realed and unrealed, as a Status cost transaction instead of directly to a General Ledger account. By appropriate configuration of the GL account relationship in the normal manner, these entries will pass through to associated GL accounts as well, like any other cost posting.

Identify the following:

  • If WBS is in use on , a default WBS to be used for the exchange variance postings.
  • A and cost category to be used.

Set up the following

  • JC parameters FXCHCAT and FXCHCATU with the and cost category, separated by a in the normal cost code way.
  • The default WBS code for Foreign Exchange Variance, if WBS codes are in use (use Default WBS Codes Maintenance in ).

It is the presence of a setting in the parameters FXCHCAT and FXCHCATU that turns this alternative feature on. If only one of the two items is desired to be processed to costing, populate only that parameter.

When processing, the is taken from the default on the invoice header (PL and SL) or the relevant (CS and SC). This allows postings to from purchase invoices which are only registered and not yet fully costed.

Note that the option for foreign exchange postings is only for P&L accounts, and not for Balance Sheet accounts.
