Changing Users' Menus

comes with a set of standard menus. Standard functions have codes beginning with a percent sign '%'. These functions and the standard menus are re-issued at each release with any new programs added.

However, these menus may not suit the way your staff work, so there are various things you can do:

  • Restrict access to functions so that each user only has access to the menus and procedures you want them to.
  • Append functions to standard menus.
  • Copy existing menus and modify them.
  • Create new menus.

For example, if you are happy with the standard menu structure, but simply want users to only see the menus and functions that are relevant to them, you do not need to alter the standard menu structure. You would just grant those users access to those functions - the other menu options would not appear on their screen. See How Function Access Works.

If you want to add functions to an existing standard menu, you can append them to the end of the menu.  See Adding to Standard Menus.

If you want to modify an existing menu further - for example to change the order of the menu items, you can make a copy of the menu function (see Making Copies of Functions), and make the changes to the copy.

But suppose you want a user to have a single menu that gives them access to the input functions of General Ledger, , and . You would create a new menu for them. Use Function Maintenance. You would put options on this menu to call up the input menus from each of the ledgers: in this case, %WGL1, %WSL1 and %WPL1.

Or suppose you wanted different users to have access to the input functions of different modules. You could create a menu that calls up all the input menus, and allocate access of different options to different users.  You could assign this menu as the main menu for those users. The main menu is the one displays when they first log in.  Do this in Users.

Default Menus

You can control the default menus users see when they log in.  There are three parameters to control this: one for a normal browser, one for the documentation menu and one for a PDA browser.  You can also override these defaults for individual users.
