Draft Certificates - Authoration Levels

Authoration Levels (Location) allows you to set the level of access to for draft certificate processing, for each user.

A user's authoration level determines whether he or she can enter or authore draft subcontractor payment certificates for orders on that .  A user cannot have level 1 and level 3 security set.  When setting up a user's authoration levels, standard security applies to both the user doing the set up, and the user being set up.  That is, both must be able to access the record for it to be created.

A "this period" limit can also be given for a user/contract. This means that the user cannot authore over that limit. 0 means no limit. The parameter LIMITCHK identifies the item IDs on the certificate that are checked against this limit. If the limit is exceeded then an error is given and that user can't authore the draft certificate.

"No Recall" can be set for a user/contract. This means that the user cannot recall certificates for that once they have moved to the next authority level. By default users will be able to recall any certificate.
