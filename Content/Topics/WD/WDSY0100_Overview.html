Report Servers and Queues - Overview

How to Setup Report Servers and Queues for OA

The aim of this document is to detail the steps required to configure OA for multiple report servers and queues.

  • Report queues are a means of providing more granularity to the management of reports. When a report is run, the request can be put into one of a number of named queues. This allows reports to be separated based on potential impact on the server, required response time etc.
  • Report agent families are a means of managing groups of background report generation agents (Progress batch processes), which may be on the main physical server or distributed across multiple physical servers. Each report agent family may process any number of report queues, allowing for a number of different deployment scenarios.
