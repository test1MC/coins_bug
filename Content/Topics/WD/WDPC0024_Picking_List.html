Picking List

The picking list feature is useful if you fulfil the majority of requests using your own . 

From the you can send the details of a request to the Picking List Workbench.  Staff can record which items are assigned to fulfil each request, and can then confirm the assignments so that the request can be completed as normal.

On asset groups and groups, you can identify where the items are to be picked from, and against users you can specify which picking areas they work in.

To record the items that have been picked:

  1. For assets, click and enter the asset you have allocated.
  2. For items:
    • If the full quantity is available, select the line and choose Fulfil from from the Choose Action list.
    • If the full quantity is not available, open the line using , and enter the quantity picked.
  3. If an item is not available, you can pass it back to the : select the line and choose Set to Not Available from the Choose Action list.
  4. If an item is not yet available but you expect it to be available soon, select the line and choose Set to Held by Picking from the Choose Action list.
  5. When you have completed the picking, you can print out the picking list using the button at the top of the screen. 
  6. Then, select the completed lines (which will have status T - Picked but not confirmed), and choose Confirm Picking from the Choose Action list. 

Configuration

Use the PC parameter PICKFLD to nominate one of the asset analysis sets to hold the picking areas.  (Asset groups and groups use the same analysis sets.)

Use Users to specify which picking area (or areas) a user works in.

When maintaining asset groups or groups, use the analysis set to identify where the item should be picked from.
