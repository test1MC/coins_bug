Text on Configurable Documents

Various parameters are used to specify text that is printed on configurable forms. These are configured on the form using the following fields.

On PORDER

RO_commit_msg Text for committed/authored orders. This uses the text from:

  • POCOMTXT if it is a purchase order.
  • SCCOMTXT if it is a subcontract order.

RO_confirm_msg Text for Confirmation Orders. This uses the text from:

  • POCONTXT if it is a purchase order.
  • SCCONTXT if it is a subcontract order.

 

RO_credit_msg Text for credit orders. This uses the text from:

  • POCRDMSG if it is a purchase order.
  • SCCRDMSG if it is a subcontract order.

RO_draft_msg Text for draft orders (when the draft print action is used). This uses the text from:

  • PODRFTXT if it is a purchase order.
  • SCDRFTXT if it is a subcontract order.

RO_version_msgText to print if a later version of the order exists. This uses the text from:

  • POVSNTXT if it is a purchase order.
  • SCVSNTXT if it is a subcontract order.

RO_voheadmsg__1 Heading. This uses the text from:

  • POADDTXT if it is an addendum/amendment purchase order.
  • POCANTXT if it is a purchase order cancellation.
  • SCADDTXT if it is a subcontract amendment order.
  • SCCANTXT if it is a subcontract order cancellation.

RO_voheadmsg__2 Additional Heading (2). This uses the text from:

  • POADDTX2 if it is an addendum/amendment purchase order.
  • POCANTX2 if it is a purchase order cancellation.
  • SCADDTX2 if it is a subcontract amendment order.
  • SCCANTX2 if it is a subcontract order cancellation.

 

TIP

On many sites where text has been set up for , the parameter value is double-spaced (for example ** V A R I A T I O N   O R D E R **).  If you want to continue to use double-spaced text, put white-space-collapse=false in the Additional Formatting for the field on the OA configurable document, to avoid the extra spaces between the words being lost.
