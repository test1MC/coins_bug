Buy-in Progression Setup

The Part Exchange Workbench - Buyin Progresssion tab on the Plot Summary of the Part Exchange Workbench shows on a single screen the steps that the buy-in of a part exchange property must go through, the expected dates for each step, and if the step has happened, the date that it happened.  To set up the list of steps, use PX Buyin Progression.

To configure the buy-in progression screen:

  1. Go to PX Buyin Progression.
  2. For each step you want on the Buy-in Progression screen, enter a line record. fills in the line number by default; you can change it if you want to insert a line between two existing lines.
  3. Each line can show either an actual date or an expected date or both. Specify where you want to get the each date from.

    4. Actual dates can be one of the following:

    • The date when a Sales Workbench Event or Part-Exchange Workbench Event occured. In the Source field, select Event Date or Plot Event and in the Event field enter or select the code of the event. If the event is a construction stage, you also need to enter the code for the stage.
    • Manually entered by the user.  In the Source field, select Manual Input.
    • Any of various key dates that relate to the buy-in (exchange deadline date, offer accepted date, offer deadline, offer sent, unconditional exchange date) or the associated plot (exchange deadline, unconditional exchange date). In the Source field, select the appropriate code.

    Expected dates can be one of the following:

    • Manually entered by the user. In the Source field, select Manual Input. To have enter a default date (which the user can update) based on the number of days after the reservation date of the associated plot, enter the number of days in the Plot Reserve Offset field.
    • Any of various key dates that relate to the buy-in (exchange deadline date, expected completion date, expected exchange date, expected occupation date) or the associated plot (exchange deadline date, expected build completion date, expected completion date, expected exchange date, expected occupation date). In the Source field, select the appropriate code. To use the date as a default and allow the user to update it, the Update field.
    • An expected date entered on the Sales Progression screen.  In the Source field, select Plot Expected Date Input, then enter the line number of the line on the Sales Progression screen.
  4. If you do not want the line to show a date in one column, set the corresponding Source field to Not Applicable.
