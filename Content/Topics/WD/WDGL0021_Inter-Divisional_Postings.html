Inter-Divisional Postings

If is set up to use inter-divisional accounts, then when a General Ledger batch has postings that cross between sectors, and there is an imbalance (in double-entry terms) within each sector, will automatically create additional balancing entries in an inter-divisional account in each sector.  This means that each sector will always have a self-contained trial balance.  This is an optional feature of using sectors.

If inter-divisional postings are used without sectored balance sheets, each sector will have its own balance sheet with an inter-divisional account.  One sector will normally contain all the control accounts, with an inter-divisional balance representing transactions processed on behalf of all the other sectors.

For example:

 

Account

DR

CR
Materials cost of 100 incurred in division 03 03.1213 100.00
Corresponding entry in PL control account (division 00) 00.7800  100.00
Balancing transactions to the inter-divisional control accounts 03.6D00 100.00
Balancing transactions to the inter-divisional control accounts 00.6D00 100.00

If inter-divisional postings are used with sectored balance sheets, each sector will have its own self-balancing trial balance, including its own share of the control accounts.  The inter-divisional accounts will only be used for other transactions which cross between sectors.  This also sectores the retained earnings at each year-end.

To set up inter-divisional postings:

  1. Go to GL Configuration. 
  2. Specify the code for the inter-divisional accounts; use underscores in place of the element that corresponds to the sector. For example: __.6D00.
  3. In GL Parameters, set the parameter SUMIDIV to Y.
