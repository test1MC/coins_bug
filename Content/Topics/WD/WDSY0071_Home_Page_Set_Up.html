Home Page Set Up

Each module menu has a home page, which allows you to provide users with information that is relevant to the module.  They also load quickly; this improves performance when changing modules.

You can configure the text that is shown on the home pages, to display the following information:

  • Module Text. You can use this for general text about the module, for example to explain its scope and purpose.
  • Environment Text. This text relates to the environment you are using.  For example "This is the Copy-of-Live environment which can be used for testing and training but any data input will be replaced overnight with a fresh copy from the live system".
  • Message board.  Authored users can easily update messages to draw users' attention to planned downtime, upgrades, input deadlines, new procedures, configuration updates, etc.  The messages are visible to all users of the module.
  • Key functions for the module.  Shortcuts to selected functions within the module (or related module).
  • Key documentation for the module.  Shortcuts to selected documentation functions relevant to the module. 
  • User-specific ite functions.  Each user can choose the functions to include on their view of the home page.  These ites will be available on every home page.
  • RSS feeds.  Authored users can "subscribe" to relevant news channels, and individual users can choose which ones they want to see.  The home pages only show the three most recent feeds from each channel. 

To set up the introductory text for a module home page:

  1. Either:
    • Click in the Module Text area of the home page or
    • Use the Module Specific Text tab of Home Page Maintenance to add a new record.
  2. Choose the module on whose homepage you want the text to appear.
  3. Enter the text using the HTML editor.

To enter messages for a module home page:

  1. Either:
    • Click in the Message Board area of the home page or
    • Use the Message Board tab of Home Page Maintenance to add a new record.
  2. Choose the module on whose homepage you want the text to appear.
  3. Enter the date after which the message will first show on the home page.  The default is today, in which case the message will be visible immediately.
  4. If you want, set an date for the message; it will no longer be displayed after this date.

    The message will appear with your user name against it.

To set up key functions and documentation for a module:

  1. Either:
    • Click in the Key Functions or Documentation Shortcuts area of the home page or
    • Use the Key Functions or Key Documentation tab of Home Page Maintenance.
  2. Choose the module on whose homepage you want the link to appear.
  3. Enter the function or topic code.

     

    The Function Description will appear as the text of the link.
  4. You can add extra text which will appear after the function description.

To set up an environment message for the system home page:

The homepage environment text is retrieved from a file outside of the database so that it is not over-written when environments are copied.

  • Create a file called homepage.txt in the $BASE/var/diag/ for the environment.

     

    The file can contain HTML mark-up.

Home page configuration

works out which home page to display based on the value of TopMenu in the URL.  If you want to force it to a particular value, you can do this by passing a parameter: "Module=xx" to the function.

The logo for the home page is named HPxx.gif (where xx is the module code), and is held in the images directory. 

If there is no module text set up, displays the module name followed by Home Page.

 

You can suppress the message board, key functions, favourites and documentation shortcuts on the home page if required: set NoWidgets=Y as a parameter on the homepage function.
