Sales Workbench Configuration

To use the Sales Workbench, you need to set up the following:

  • Lookup Tables:
    • Construction Stages allows you to set up the construction stages that you use to track the progress of a plot's construction.

    • Sales Status Codes allows you to set up and maintain a list of codes that can be used to identify the sales status of a plot or part-exchange.

      For example: Unreleased, Released, Reserved, Exchanged and Completed.

      You also need to fill in the parameters with the codes that represent certain specific statuses.

    • Reasons for Buying/Move allows you to set up and maintain a list of codes that can be used to identify why a purchaser decided to buy a particular property.For example,
      • JOB = Job Relocation.
      • BUS = Local Bus Service
      • SCHOOL = Schools Nearby

      This information will then be available to users for reporting, and when completing the relevant details in the Sales and Part-Exchange Workbenches.

    • Reasons for Cancellation allows you to set up and maintain a list of codes that can be used to identify why a purchaser decided to cancel the purchase of a property.

      For example:

      • MORT = Mortgage Problems
      • FIN = Financial Problems
      • CHANGE = Changed Mind

      This information will then be available to users for reporting, and when completing the relevant details in the Cancellations Workbench.

    • Purchaser/Prospect Status allows you to set up and maintain a list of codes that can be used to identify the current status in the market of a purchaser.

      For example:

      • FTB = First Time Buyer
      • NOCHAIN = No Chain
      • PARTEX = Looking To Part-Exchange.
    • Mortgage Status Codes allows you to set up and maintain a list of codes that can be used to identify the current status of a purchaser's mortgage application.

      For example:

      • MA = Mortgage Applied For
      • AP = Awaiting Approval
      • NOM = No Mortgage Arrangements.

      This information will then be available to users when completing the relevant details in the Sales, Part-Exchange and Workbenches.

      NOTE

      These codes are used for plots you sell, and for part-exchange buy-ins.  Set up codes that cover the needs of both.

    • Sales Staff allows you to set up and maintain a list of names and contact details for all sales staff, and to control the level of discount and incentive they are able to offer.
    • Guarantee Schemes allows you to set up and maintain a list of guarantee schemes and the companies that provide them.
    • / Group allows you to set up and maintain a list of codes that you can use to group and together for reporting.For example, you might use a group named ELECTRIC to report on additional lights, additional sockets and so on.
    • Standard allows you to set up and maintain the -level list of the standard that can be ordered on a plot.You must set up a list of at level.  You can then copy items to more detailed levels (such as or individual plots), and modify the standard cost and specification of the at that level.

    • Status Codes allows you to set up and maintain codes to indicate the current status of the that relate to a sale.

      For example:

      • NOTREQ - No required
      • WAITING - Awaiting Details from Customer
      • ORDERED - Ordered
      • COMPLETE - Installation Complete

      You also need to fill in the parameters with the codes that represent certain specific statuses.

    • Payment Methods allows you to set up codes for different methods of payment (for example: , Mastercard, VISA).These are used when recording cash using the cash event.
    • Incentive Categories allows you to set up and maintain a list of the discounts and incentives that are available.
    • Refund Terms allows you to set up and maintain a list of the terms that will be applied to the if the sale is cancelled.
    • If you record sales chain information, Chain Status Codes allows you to set up and maintain a list of codes to classify the status of a link in the sales chain.
