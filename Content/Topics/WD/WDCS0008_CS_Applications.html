Configurable Applications

The Application document format allows you to configure applications for printing.

Application documents show item lines from the certificate definition in the body, and can also show:

  • One row for each costing distribution record.
  • One row for each measured item record.
  • One row for each distribution record.
  • One row for each stage paid (if plot billing is used).
  • One row for each variation paid (if plot billing is used).

Define the layout of configurable documents using Document Designer in the System Module. See Configurable Documents for information on how to define configurable documents.

To define the format for printing applications

  1. Use the document types Applications (CS).

  2. In the Document field, enter a name for the application document (for example CSAPP1). You can define different documents to print applications with different layouts. When you print the application, gives you a choice of which document to use.
  3. In the IDs field on the Extras tab, enter a list of certificate item IDs, separated by commas, that correspond to any detail lines to be printed in the body of the application (see Defining a standard certificate).

    IDs: GRC,CON,RET,DIS,VAT,PAY

    The body line will be repeated for each Item ID:

    CumulativePreviousThis Period
    Gross Certified8,925.006,250.002,675.00
    Contras115.000.00115.00
    Retention446.25312.50133.75
    Discount211.97148.4463.53
    1,446.691.013.09433.60
    Payment8,266.785,789.062,911.32

    Lines with zero values will not normally be printed. To print lines even if all values are zero, put an exclamation mark (!) before the Item ID.

  4. In the header of the application, put information such as the customer's name and address, the code, the name, the application date and value. Also, if you want column headings, such as Current Valuation, these go in the header.
  5. In the body of the application, put a line that shows the details of the payment (cumulative, previous, and this period amounts).
  6. In the footer of the application, put any additional information that may be needed.

Tables available:

The following tables are available on all forms: cs_certificate, jc_job, cs_series, sv_job, ar_cust, cs_check, sv_cert

The following extra tables are available on specific forms:

Body - cs_certdet, cs_defndet

CERTDIST - cs_certdist

VATDIST - cs_vatdist

ITEM - sv_item, sv_measure
