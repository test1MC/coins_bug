Cash Transactions

Cash transactions in House Sales are entered as batches, in the same way as other financial transactions. 

The cash can be allocated to one of four types of account, using the analysis:

Account type

Analysis

Example
  • A plot
 (.phase).plot.transaction-category

2006.039.EX

You must enter the full plot code (for example, 039 not 39).
  • A part-exchange
 PX.part-exchange-reference.transaction-category PX.128.MS
  • A cancellation
 CA.cancellation-reference.transaction-category CA.24.RF
  • A pre-release reservation
 PR.reservation-reference.transaction-category PR.240.DE

NOTE

You can enter PX, CA or PR and use the lookup to find the appropriate reference and category.

When you enter the transaction, you can allocate it to a memo cash event.  (Sales staff would use a memo cash event to record the receipt or payment; financial staff would then enter the transaction and allocate it to the appropriate event.)

If you enter a transaction for a ground rent or service charge transaction category, you can allocate the transaction to pay off the charge, either using the Allocation to Charge tab, or afterwards using Allocate Cash to Charges.

Cash transactions - configuration

Transaction Categories allows you to set up the codes that identify different categories of transaction within House Sales Financials. These specify which General Ledger control accounts the transactions are posted to, and other settings.

Each category can apply either to plots or part exchange properties or both.
