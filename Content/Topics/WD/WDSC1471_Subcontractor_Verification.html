Subcontractor Verification

Verification allows three modes of submission: Match, Verify and Auto.

For subcontractors whose details are matched it does not matter which mode you use: the verification status will be matched and the ‘Tax Treatment’ shows whether the subcontractor is to be paid gross or net.

However if the submission contains subcontractor records whose details don’t match those on HMRC’s database, the results from the different modes are different.

  • If using Match mode, the subcontractors’ verification status will be failed, which means you can resolve any issues before re-submitting the details for verification.  Since the subcontractor has not been verified will not let you pay the subcontractor.
  • If using Verify mode, the subcontractors’ verification status will be unmatched, which means the subcontractor is HMRC verified and so can be selected for payment but would be subject to the higher rate of deduction.
  • Auto mode is provided for compliance with certain requirements of HMRC testing.  If using Auto mode, those subcontractors with a verification status of ‘New’ are submitted as a ‘Match’, and those with a verification status of ?Failed? are submitted as a ‘Verify’.  This assumes that you have corrected any problems with records that have a ‘Failed’ status before you run another Auto submission.  Because of this, we suggest you do not use ‘Auto’.

Because of this, you may wish to grant access to the different modes accordingly. 

The function codes are as follows:

%WSC2070UCOI1 - HMRC Match

%WSC2070UCOI2 - HMRC Verify

%WSC2070UCOI3 - HMRC Auto
