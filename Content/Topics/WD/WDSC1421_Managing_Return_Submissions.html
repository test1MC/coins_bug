Managing Return Submissions

You can use Monthly Return Submissionsto manage the submissions.  If everything is working normally, you should not need to use these functions.

You can:

  1. View the status of a submission.  This is shown on the main screen.
  2. View the details of a submission: click .
  3. View negative returns and mark them when you have notified HMRC (see Negative Returns).
  4. View any errors: click and look at the Errors tab.
  5. Restart a request.  If the submission times out (that is, if does not receive a response from the Government Gateway after a specified number of requests), the status will be set to 'Timeout'.  In this case will automatically create a delayed background entry, which will run a restart after a specified number of minutes. However you can delete this so you can restart manually.  As soon as you choose Restart, the status changes to 'Restarted'.  If you try to restart and the status is not correct you will get an error (so will not let you restart twice by mistake).
    • the request and select Restart from the Choose Action menu.
  6. Submit a request.  You should not normally need to use this, but if there is a fatal error with the submission or a problem that corrupts the data, you might want to re-submit the request after the problem has been sorted out.  Submit starts from the beginning, whereas Restart picks up from the most recent event.
    • the request and select Submit from the Choose Action menu.
  7. Output the XML file without submitting it to HMRC.  This may be useful for testing. 
    • the request and select Output XML from the Choose Action menu.

      The file will be in the IR-CIS-CIS300MR directory below the base directory specified by the SCUK/EGOVRDIR parameter, and will have a name of the form <attempt ID>pst1.xml.

  8. Load an XML file as if it were a response from HMRC.  This may be useful for testing.
    • Select Load XML from the Choose Action menu.

      will look for a file with a name of the form <attempt-ID>get1.xml in the IR-CIS-CIS300MR directory below the base directory specified by the SCUK/EGOVRDIR parameter.
