Sales Events

Sales Workbench Events allows you to set up the events you can run for a plot sale. These include events to release or rerelease a plot, record a reservation, complete a sale, and record various other events in the progress of a sale.

To set up sales events:

    1. Go to Sales Workbench Events.
    2. Click to add a new event.
    3. Enter a code for the new event.
    4. If the event is one that is not run manually from the Sales Workbench but is run automatically by a process like sending a letter, System Generated.
    5. Select the system event. This determines the type of event you are setting up. You need to set up at least one release, re-release, reservation, exchange and completion event. Also for you need to setup a cancellation event.
    6. Enter a full description for the event. (The short description is only used in .)
    7. If you want to allow users to run the event more than once, Allow Duplicates.
    8. If you want the event to be authored separately after it has been entered, Event Requires Authoration?. See Authored Sales Events.
    9. If you are setting up a conditional exchange event (system event code EXCHANGE) Conditional Exchange and enter the corresponding unconditional exchange event on the Conditional Exchange tab.
    10. If you only want to allow the event to be run if other specified events have already been run, enter the codes for those events in the Previous Events field. For example, to set up a "Reserve" event that can only be run if the "Release" event has already been run, enter the code for the "Release" event.
    11. If you want to prevent the event if other specified events have already been run, enter the codes for those events in the Stop Events field.

NOTE

If you want to allow or prevent the event depending on whether the status of the plot is "released", use the token $RELEASE. (You need to use this, rather than the RELEASE event code, because released plots can be dereleased.) For example, if you put $RELEASE in the Previous Events field, you will only be able to run the event if the plot is currently released.

  1. To restrict how many days before or after today users can enter for the date of the event, the Restrict Backdating or Restrict Forward Dating field (or both), and enter the number of days. For example, to stop users entering a date more than two days before today, set Allowable Days Backdating to 2. If both are set to 0, users will be forced to enter today's date.
  2. Select the sales status that this event will apply to the plot.
  3. If you are setting up a RESERVE event, from the Incomplete Reservation Sales Status list select the sales status code that the plot will have (in ) while the reservation is in progress but has not been completed.
  4. To prevent the event from changing certain sales status codes, enter a can-do list in the Stop Status Update field to specify the codes. For example, the pre-release reservation event might normally change the sales status to EB (earlybird), but not if the sales status is already released; to do this, you would put EB in the Sales Status Update field and RL (for released) in the Stop Status Update field.
  5. From the Minimum Construction Stage list, select the construction stage that the plot must reach before the event can be run. For example, you would set the legal completion event so that it cannot be run if the plot construction is not finished.
  6. Enter the default comments; this is the text that will appear in the Comments field when the event is run (the user can enter different text if they want).
  7. If the event is a pre-release reservation event, select the Reservation Type on the Prerelease Reservation Type tab; this is the reservation type that this event represents. To allow the user to fill in the reservation type when they record the pre-release, leave the field blank.
  8. If you want to update captions when the event is run, select the caption groups to update.

Sales Events - configuration

Set up which users can run the event, using Sales Event Security (see Event Security).

 

To copy the comments for the event to the Notes automatically, enter the event code as one of the values of the HS parameter COMNTVIS.
