Events

Workbench Events allows you to set up the events you can run for a . These include events to close or open the file, to record a visit to a , or to record an .

To set up events:

  1. Go to Workbench Events.
  2. Click to add a new event.
  3. Enter a code for the new event.
  4. If the event is one that is not run manually from the Workbench but is run automatically by a process like sending a letter, System Generated.
  5. Select the system event. This determines the type of event you are setting up. There are six system event types:
    • CLOSE - this closes the file. You should only set up one close event.
    • REOPEN - this reopens the file. You should only set up one reopen event.
    • VISIT - this records a visit to a . You could set up different types of visit event; for example, First Visit and Revisit.
    • - this records a general (one that is not recorded against a specific ). You could set up different events; for example, Telephone , Website .
    • PURCHASE - this records the purchase of a plot (usually this would be set as System Generated).
    • MEMO - this is for any other type of action, which simply records the date and comments.
  6. Enter a full description for the event. (The short description is only used in.)
  7. If you want to allow users to run the event more than once, Allow Duplicates. (You will probably want to allow most events to be run more than once, but if you have a "First Visit" event, you might want to prevent it being run again.)
  8. If you only want to allow the event to be run if other events have already been run, enter the codes for those events in the Previous Events field. For example, to set up a "Revisit" event that can only be run if the "First visit" has already been run, enter the code for the "First Visit" event.
  9. If you want to prevent the event if other events have already been run, enter the codes for those events in the Stop Events field.

    10. NOTE

    If you want to allow or prevent the event depending on whether the file is closed, use the token $CLOSE. For example, if you put $CLOSE in the Previous Events field, you will only be able to run the event if the file is closed.

  10. To restrict how many days before or after today users can enter for the date of the event, the Restrict Backdating or Restrict Forward Dating field (or both), and enter the number of days. For example, to stop users entering a date more than two days before today, set Allowable Days Backdating to 2. If both are set to 0, users will be forced to enter today's date.
  11. Enter the default comments; this is the text that will appear in the Comments field when the event is run (the user can enter different text if they want).
  12. If you want to update captions when the event is run, select the caption groups to update.

Events - configuration

Set up which users can run the event, using Event Security (see Event Security).

To copy the comments for the event to the Notes automatically, enter the event code as one of the values of the HS parameter COMNTVIS.
