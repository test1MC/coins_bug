Directing to the Correct Overlay File

Once the overlay TIFF file has been uploaded into the VSI-Fax server, it is necessary to configure so that it passes the correct overlay configuration to VSI-Fax.

  1. Ensure that the TIFF version of the overlay (.tif) exists in the overlay directory, which is specified by the PM/TESTDIR parameter (for example: $BASE/app/overlays/).
  2. Using a text editor, create a datafile with the same name as the overlay you are using.  This needs to be in the same directory as the overlays.  Following our example this file will be called order.dat. 

     

    uses this file to confirm which overlays are associated with which print file. This is especially important if there is more than one overlay for the print file (for example, continuation pages or terms and conditions pages).

    This file should contain a line for each overlay that is to be used, and the page from which it is to be used.  For example:

    • overlay.tif,1

      Where overlay.tif is the overlay file to be referenced and ,1 use this on the first page and all subsequent pages.

    • The following is an example where there is an overlay for the first page print for just one page (order.tif,1,1) and a second overlay for the second page onwards (overlay2.tif) and a terms and conditions pages as an attachment on the last page (tc.tif,,$). The $ identifies that this is an attachment.

      order.tif,1,1
      order2.tif,2  
      tc.tif,,$
