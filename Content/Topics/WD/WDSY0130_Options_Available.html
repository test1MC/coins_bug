Integrating with Exchange

Exchange Server 2010 Integration (using Web Services) allows you to integrate various features of with Microsoft Exchange.

  • You can copy tasks and appointments between the Activity Workbench and Microsoft Exchange.
  • You can copy contact details from to Exchange contacts.
  • You can copy emails about projects to Project Management.
  • You can use emails to run workflow actions. 

Copying emails and workflow replies requires public folders with an associated email address; replicating contacts, appointments and tasks is done by running a scheduled task on a Windows server.  This requires 10.23 or later; Exchange Server 2010 (or 2007 with service pack 1 or later).

See Exchange 2010 Integration (using Web Services).

 

If you are using Exchange Server 2003 or 2007, see the following:

  • Exchange Integration (using CDOEX) allows you to replicate tasks and appointments that originate in , to import emails for projects, and to import and process workflow emails.  This works using a scheduled task on the Exchange server.  (Requires v10.21 or later; Exchange Server 2003 or 2007.)

    See Exchange Integration (using CDOEX).
