Orders

The basic steps in processing orders are as follows:

  1. Enter the order (see Entering Orders )
  2. Commit the order (see Committing Purchase Orders)
  3. Print the order (see Printing and Committing Orders)
  4. If the order uses GRNs, enter the goods received ( advice) note  (see Entering Goods Received Notes (Materials and )). This generates a " on site" record.
  5. If the order uses allocations, allocate and confirm the sheets (see Weekly Sheets). This generates an accrual.

  6. Enter invoices.
  7. Match invoices (see Invoice Matching )

orders configuration

Charges for can be calculated using any of four different methods.  Various parameters need to be set:

  • Hourly
    • HRSUNITS - Units that represent hours
    • HRSDAY - The number of hours in a day
    • INCOTHHR - Whether to include 'other' hourly rates from the style
  • Daily/weekly
    • DLYUNITS - Units that represent weeks
    • DAYUNITS - Units that represent days
    • INCOTHWK - Whether to include 'other' weekly rates from the style
  • Monthly
    • MLYUNITS - Units that represent months
  • Lump sum
    • LUMPSUM - Units that represent lump sum payments
    • GLOBLUMP - Whether lump sum units apply to all commodities

You need to set up styles to cover the different charging patterns for , using Styles.  Set the default style for each item using .

Depending on the style, charges can apply on holidays or not.  Set up standard holidays using Holiday Calendar (-in ), and specific holidays that apply to individual or customers using Customer/Contract Free Days.
