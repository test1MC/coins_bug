Deleting Printer Spool Files and Other Temporary Files

You can configure to automatically remove printer spool files and other unwanted files.

  • In , files for all users can be deleted automatically by a scheduled program (sy906b.p).  This also deletes log files, temporary files and unknown files in the spool directory.
  • In , an individual user's files can also be deleted automatically when the user logs in.

Parameters control how long different types of file or record are kept before sy906b.p deletes them.  The files and records that sy906b.p deletes are:

  • print files.
  • print requests (pm_queue records).
  • Menu log history (ms_log records).  The log history for SYSAdmin can be retained for longer than for other users.
  • Report generation history (co_rptstatus) and the associated report output files.
  • Other unidentified files in the spool directory.

sy906b.p can also update the Document Management disk and keyword counts.

Configuration

If the PM parameter AUTODEL is set to Y, old reports generated by the users will be deleted when sy906b.p is run or when the users log in to . From v10.22 onwards, sy906b.p is run as part of the broker start script generated from COINSAdmin.   This automates the purging and, as it is done via cron, it will not interfere with the user login process.
