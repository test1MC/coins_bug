Scheduling a Report

Scheduling a report means that you can generate reports with the same selections on a recurring basis (for example: every day, or on the 15th of each month, or every second Wednesday), automatically.

You can also schedule other background processes, such as committing orders, in the same way.

When the date and time specified on the schedule is reached, places the request on the current queue, and if the next date and time is still within the date range of the schedule, puts a new scheduled item on the scheduled list.

To set up a scheduled report:

  1. Define the report options for the report in the usual way.
  2. On the Output Options tab, select Scheduled.
  3. You may want to change the report title to identify this scheduled report.
  4. Click .

    displays the Scheduler Update screen. 

  5. Enter the time of day you want to generate the report, and the range of dates over which you want the schedule to apply. Depending on how is configured, you may not be allowed to schedule reports during certain times; this restriction is shown next to the Time field.
  6. Select a schedule.  The schedules are pre-defined by your system administrator.
  7. Click .

    displays Report Status with the Scheduled tab selected.  This lists the scheduled report selections you can use (including the one you just saved), and shows the date and time when each of the reports will next be generated. 

To change the selection criteria or output options for a scheduled report:

  1. Click the link in the Description column.
  2. Make your changes to the selection criteria or output options.
  3. Click .

    returns you to Report Status, and does not generate the report. 

To change the name of a scheduled report, or the details of the schedule, click , type a new name in the Description field, then click .

Report schedules - configuration

Use Standard Schedules to set up schedules for reports. 

You can restrict the times at which reports can be scheduled using the PM parameter SCHRTIME, and which users the restriction applies to using the PM parameter SCHUSERS.
