Setting Up the Delivery Register

The Delivery Register allows you to record detailed information about deliveries of concrete and reinforcing steel.  See Delivery Register.

To use the Delivery Register, set the PO parameter USEDELRG to Y.

Locations

Locations are codes that identify the locations on a that you can record deliveries against. Set these up using Location Category.

Registry Types

The registry type determines which questions are used when recording information on the delivery register (currently you can only use one type for concrete, and one type for reinforcing steel).   

  1. Use Registry Type Maintenance to set up the codes and descriptions for the registry types.
  2. For each registry type, you can set up two groups of questions:

    • Test sets contain questions that relate to quality tests (these are intended mainly for concrete deliveries; the Concrete Test Register Report shows the responses to these questions)
    • Field sets contain other questions. 

    The procedure for setting these up is the same:

    1. Click the link in the Type column.
    2. On the appropriate tab (Test Set or Configurable Fields), add a new record.
    3. Click the link in the Test Set or Field Set column.
    4. Set up a record for each question.  For each question, choose the type of response you want (for example, a decimal or integer value, text, or one or more items from a list).  If you choose 'List', click the link in the Field column and set up the entries in the list.
  3. Set the PO parameters DREDCONC and DREGREO to identify which codes you have used for the registry types for concrete and reinforcing steel respectively.
  4. Set the PO parameters DRTCONC and DRTREO to identify which codes you have used for the test sets for concrete and reinforcing steel respectively.
  5. Set the PO parameters DRFCONC and DRFREO to identify which codes you have used for the field sets for concrete and reinforcing steel respectively.

Commodity Codes 

In order for a commodity code to use a register, it will need to be configured in Materials.

If you are using the Delivery Register, when you update a commodity the Registry Type field is available on the Main tab.

  • If you want to prompt for Delivery Register information when users enter a GRN for this commodity, select the registry type to use.
  • Leave the field blank if you do not want to use the Delivery Register for the commodity.

To update groups of detail commodity codes, enter a registry type code on the commodity header code, and on the Copy Header Data to Details screen, the Copy Registry Type? box.
