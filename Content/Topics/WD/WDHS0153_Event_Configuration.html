Configuring Events

There are three types of events relevant to :

  • General Events allows you to set up events that apply to plots.
  • Issue Events allows you to set up events that apply to issues.
  • Call Events allows you to set up events that apply to calls.

To set up events:

  1. Go to the relevant event maintenance function (see above).
  2. Click to enter a new event.
  3. Enter a code for the new event.
  4. Fill in the fields as follows:
    System GeneratedIf the event is not run manually from the Workbench, but instead is run automatically by a process such as sending a letter, this.
    System Event CodeEnter the code for the system event. This determines the type of event you are setting up. The possible events are:
    • Workbench events:
      • ACTIVE: Brings the plot back into warranty.
      • INACTIVE: Expires the plot's warranty.
      • RESALE: Records that the plot has been resold.
      • MEMO: Records other general information.
    • Issue events:
      • ACTION: Records that something has been done.
      • APPTMENT: Records that you have made an appointment and who is attending to it.
      • EMPLOYEE: Reassigns the employee to whom an issue is assigned.
      • MEMO: Records general information about the issue.
      • PRIORITY: Changes the priority of the issue.
      • RESOLVE: Records the resolution of the issue.
      • STATUS: Changes the status of the issue.
    • Call events:
      • INSPECT: Records an inspection.
      • MEMO: Records general information about the call.
    Full DescriptionEnter a full description for the event. (The short description is only used in.)
    Allow DuplicatesIf you want to allow users to run the event more than once, this.
    Previous EventsIf you only want to allow the event to be run if other specified events have already been run, enter the codes for those events.
    Stop EventsIf you want to prevent the event if other specified events have already been run, enter the codes for those events.
    Restrict Backdating, Restrict ForwardTo restrict how many days before or after today's date that users can enter for the date of the event, these fields, and enter the number of days. For example, to stop users from entering a date more than two days before today's date, set Allowable Days Backdating to 2. If both are set to 0, users will be forced to enter today's date.
    Default CommentsEnter the default comments; this is the text that will appear in the Comments field when the event is run (the user can enter different text if they want).
