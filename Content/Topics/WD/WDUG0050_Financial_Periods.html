In , are identified by a code of up to four digits. These will normally be the month and year of the end of the period, for example, 1205 or 606. Each has a financial date associated with it -- this date is the end of the period.

A period is either closed, open or current. If a period is closed, you cannot enter transactions to that period. If it is open, you can enter transactions. If a period is current, it is also open; this is the period which enters by default in some Period prompts.

For each module, only certain periods are open at any one time. You can only assign transactions to periods that are open for the current module. If you enter a period that is not open, displays the message Period Closed. If you enter a period that is incorrect or does not exist, displays the message Invalid Period.

Each module may have different periods open or closed. For example, Subcontractor's Ledger might have the January period closed on 1st February, and the February period opened on the same day, so that transactions can only be entered into one period at any time. However, might have the February period opened on February 1st, but the January period left open until February 10th, to allow late January invoices to be entered.

Periods can be opened or closed in any sequence, and open periods do not necessarily have to be consecutive. You can have up to 24 periods open (or 26 if you are using 13 periods per year), although since users could then post transactions to any of these periods, you would normally want to limit the number of open periods. Closed periods may be re-opened.

If periods in General Ledger or Status are closed, transactions to other modules will still update the general and costing ledgers. A closed period in General Ledger or Status only means you cannot enter journal transactions directly to these modules.
