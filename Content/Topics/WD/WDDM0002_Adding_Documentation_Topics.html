Adding Documentation Topics

Each page in the online documentation is a topic.  A topic is made up of different topic sections.  Currently only the following sections are available:

  • Overview.  Use this for the main body of an explanatory topic or as an introduction to a process topic.  Each topic must have an overview section (though it can be empty). 
  • Process.  Use this for step-by-step instructions.  This section is optional.
  • Configuration.  Use this for information about setting up and configuring the feature described in the main topic, and can include parameter help nuggets.  This section is optional.

When you save a documentation topic, creates a function record with the same code and description. 

To add a documentation topic:

  1. Go to Topic Maintenance.
  2. In the tree view, browse to the book you want to add to.

    NOTE

    If you want to add a new topic to one of the standard books, you must first create a user copy of the book.
  3. Using the Add Type drop-down, select Add new topic and:
    • Click to add the topic at the bottom of the list.
    • an existing item and click to add the topic above the selected item.

    • NOTE

      If you want to add a topic but not add it to a book (for example, if you want to link to it but don't want it to appear in the tree view), click List of Topics a the top of the Topic Maintenance tree view instead of browsing to a book, then click .
  4. Enter a code for the topic. This can be any code you choose, but do not begin it with % or +.
  5. The module defaults from the module of the parent book; you can select a different one if you wish. The module shows on the Search results.
  6. Enter titles for the topic. The Title shows at the top of the topic; the Menu Description shows on the menu - you might choose a shorter description for the menu.
  7. In the Body field, type the text of the overview section.
  8. Click .

     

The topic can consist of an overview section only. If you want, you can also add a process section and a configuration section. Currently the only difference is that configuration sections are shown with a border, and you can include parameter documentation; however you could use style sheets to modify the way the sections are displayed.

To add other sections:

 

  1. On the Topic Maintenance browse, click the Code link for the new topic.
  2. Click .
  3. Select either Process or Configuration from the Type selector.
  4. In the Title field, enter a heading for the section.
  5. In the Body field, type the text of the section.
