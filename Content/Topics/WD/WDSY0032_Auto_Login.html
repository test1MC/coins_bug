Auto Login

Auto login allows you to take advantage of Windows network security so that users can connect directly to without needing to go through a separate login process for . When the user clicks an icon or link that takes them in to , uses the domain name and user name of the Windows PC to log in. 

NOTE

Automatic login only works when using Internet Explorer. For an alternative method, see Remember User ID - Overview.

How it works

If auto login is enabled, login page is directed to an ASP that returns the domain and user name of the Windows user.  then searches for a user record that has that combination of domain and user name set up against it, and logs the user in with that user ID.

If a user logs in with an IP address that is not valid, or with a domain name and user name combination that has no match on any user record, displays the standard login screen and the user has to enter their user ID and password.

Auto login - configuration

To enable auto login, SY/WEBAIPS must specify the IP addresses for which auto login is allowed.

An ASP must be set up on the IIS server which returns the Windows user's user name and domain name.

In the Domain\User field on the user record of each user, enter that user's Windows domain name and user name.

NOTE

The Windows user names do not need to match the user names.
