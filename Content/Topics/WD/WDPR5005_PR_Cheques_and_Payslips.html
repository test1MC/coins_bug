Configurable Payroll and Payslips

The and payslip print format allows you to configure documents for printing payroll , payslips, or combined and payslips.

Define the layout of and payslips using Document Designer in the System Module. See Configurable Documents for information on how to define configurable documents.

To configure PR or payslips:

  1. Use the document type /Payslip (PR).
  2. In the Layout Type field, select:
    • to define the layout for ;
    • Combined to define the layout for combined and payslips;
    • Remittance to define the layout for payslips.

  3. The can be either at the top or at the bottom of the page. If the remittance lines take up more than one page, you will need to print a "VOID" form on the portion of the stationery on all pages after the first one. Configure a form for this and enter its name in the VOID Form field, then, depending on whether the is at the top or bottom of the stationery, do one of the following:

    • If the is at the top: in the Location field, select Top of Page. In the Continuation Header Form field, enter the name of the VOID form. Configure the Header Form to print the details.
    • If the is at the bottom: in the Location field, select Bottom of Page. Configure a form to print the details, and enter that form name in both the Footer Form and Continuation Footer Form fields. then replaces the footer with the VOID form after the first page.
  4. To print one per page, leave the Multiple per Page?box blank, and configure the header or footer form to print the details. To print multiple per page, select Multiple per Page?, and configure the body form to print the details.

  5. If you are using a tiered payslip (where, for example, tier 1 is used for payments and deductions column sets, and tier 2 is used for year-to-date and other item details), it has the following structure: header, body, tier 1 footer, tier 2 body, footer.  Give names to the tier 1 footer and the tier 2 body, and enter the number of lines for each.

  6. To indicate whether or not a is a duplicate, use the field RO_Duplicate.  This shows as blank on the original, and **********DUPLICATE********** on reprints.

The following special fields are available:

RO_SpelledAmount - The amount spelled in words all on one line based on USA standards.

RO_usaAdd1, RO_usaAdd2, and RO_usaAdd3: Use these each on a separate line to print the address in USA formatting and without a blank line in the middle of the address if the name/address only uses three lines instead of four.
