Cross- Year End

General Ledger allows transactions to be posted to two (24 or 26 periods, depending on how many periods per year are being used). To move the available periods for several forward by 12 months, run GL Year End (Cross-) (not necessarily at the end of a ; in fact it is more likely to be run mid-year). The procedure does not delete any information, it changes the current year on the GL configuration record. You have the option to carry budgets over to the new open year.

NOTE

Once you close a year (that is, move the periods on), you can no longer post to it; you can only report on it.

You can select the for which the year-end is to be run; will submit separate year-end runs for each .

To run GL Year End:

  1. Go to GL Year End (Cross-).

    You can only run the year end routine for a if the Status Msg column is not ed, that is:

    • All periods have been set up for the current year.
    • All module periods for the current year are closed.
    • All batches have been posted.
    • The clearing account balance is zero.
    • The year end process is not currently being run. 

    If any of the above conditions is not met, the Status Msg column is ed.  To see which condition has not been met, click on the relevant line. You will need to take the appropriate action before you can run Year End.

  2. the for which you want to run the GL Year End procedure.
  3. From the action menu, select GL Year End and click .
  4. Indicate that a backup has been made and that you are ready to run the Year End procedure.  If you want to copy the GL budgets over to the new year, the Copy Budgets? box.
  5. Click .
  6. Click Yes to confirm that you want to run the year end.

    When complete, displays a message to tell you how many have been successfully processed.
