Sector Security

GL Security Maintenance allows you to control which users are allowed access to which GL accounts. You can determine the access for individual users or for groups.

If you are using sectors it would be common to apply GL security to the GL element that is configured as "Sector".

Applying GL security as above will mean that users cannot access accounts outside their sector when using functions that access the General Ledger directly (for example, GL account ), but it will not prevent them from seeing transactions in other ledgers.  For example, the view of Invoices would not be restricted. 

However, if you are using Sector-Based Ledgers for a module and GL security is set as above, the security also applies to all transactions associated with the GL security settings; therefore users will only see, for example, Invoice transactions for the sector or sectors that fall within their GL security settings.

To apply sector security:

  1. Sectors must be enabled (see Sectors).
  2. In GL Parameters, set the parameter ACCTSEC to gla_eln (where n is the number of the element that identifies the sector).
  3. Use GL Security Maintenance to control which users have access to which elements (and therefore which sectors).
