Emailing Expense Remittances in COINSplus

In you can email Expenses remittances to employees.

Configuring to email EX remittances

  1. Set the EX/DEFPRNT parameter to EMAIL.
  2. Set up employees using Employees:
    • For any new employee entered, the default value of the Default Print Method field will be email.  If the Email field is not filled in (or if there is no @ sign in the field), displays an error.  The Default Print Method can be changed to Print if there is no email address available. 
    • For any existing employee, the Default Print Method field will be blank.  This means the preference in EX/DEFPRNT comes into effect; that is, email if there is an email address.   If there are employees you don't want to email, set the Default Print Method on those accounts to Print.
  3. Set up a function to print or email the remittances:
    • If you know that all your employees will have an email address, then amend your existing remittance print to add a third parameter Y (Email) and set the fourth parameter as Y (Autoemail).
    • If you know that there are some employees who wish to have printed remittances then you will need to set up an additional remittance print option, with the third parameter N (print) and the fourth parameter N (exclude email preference employees).   You will not be able to proceed to post a payments batch until all remittances have been printed, so both your email and your paper print remittances will get run each period.
