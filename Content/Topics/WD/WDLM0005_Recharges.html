Recharges

The recharges functionality allows costs to be captured through , Subcontract Ledger, and General Ledger. These costs can then be recharged against the landlord or tenant or they can be written off. The option to apply a markup or uplift to the recharge is also available.

General Ledger Input

Recharges can be entered using Journal Batch. The analysis for lettings recharge transactions is in the following format:

  • L.<PROPERTY>.<TENANCY> ? This is for coding the recharge to the landlord. The tenancy part of this analysis is optional.
  • T.<TENANCY> ? This is for coding the recharge to a tenant.

Input

The costing for invoices can be recharged to lettings. The costing analysis is the same as for GL input.

Subcontract Ledger Input

The costing for Subcontract Certificates can be recharged to lettings. The costing analysis is the same as for GL input. In addition the default costing for the subcontract order can also be configured to point to lettings.

Recharges - set up

To enable input of recharges in the General Ledger, you will need to amend the GL parameters ANALOPT and ANALDESC. Lettings recharges is the 8th entry in each of the parameters.
