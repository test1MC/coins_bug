Mark-Ups

Mark-ups are a way of adding a premium to an incurred cost which can then be re-charged to a customer.  allows you to vary these mark-ups depending on the cost category and a threshold figure (that is, two mark-ups per category are possible depending on the cost amount). The invoice class can be defaulted here also, and it is possible to mark a category so that any cost transactions of that category will be ignored when it comes to billing.

Further, you can default the mark-up figure depending on a hierarchy of customer, minor and major category, . So when determining the default mark-up figure for a particular cost transaction, the mark-up set up against a customer on the associated will override any mark-ups set up against the minor category of the , which in turn will override the major category, and finally the .

See other sections for descriptions of invoice classes and service fees.

Mark-ups are applied when the cost transaction is created. There is a facility to update mark-ups if anything has subsequently changed for any reason.

Mark-ups configuration

Mark-up configuration can be found under FM Setup on the 'Invoicing Setup' tab menu.

If the mark-up 1 limit is set, then a value should be entered for mark-up 1 (which applies when the amount is equal to or below this limit), and mark-up 2 (when it is greater than). If the limit is not set (equals zero), then mark-up 2 will always equal zero and will not apply.

  • Major/minor category and customer are mutually exclusive, and a minor category can only exist if a major category has been entered.
  • No mark-up figure can be negative, but it can be zero.
  • These figures can be manually overridden in the 'Ready for Invoicing' stage.

You can load mark-ups from a CSV file using Load Markups. The format of the CSV file is as follows (all field values are text unless stated):

  • Major Category (Optional)
  • Minor Category (Optional)
  • Customer (Optional)
  • Service Fee (Optional - numeric)
  • Service Fee SOR item (Optional)
  • Service Fee Desc (Optional)
  • Service Fee Inv Class (Optional)
  • Mark-up Limit (Optional)
  • Category[1] (*)
  • Ignore[1] (*)
  • Inv Class[1] (*)
  • Markup1[1] (* - numeric)
  • Markup2[1] (* - numeric)
  • Category [2] (*)
  • Ignore[2] (*)
  • Inv Class[2] (*)
  • Markup1[2] (* - numeric)
  • Markup2[2] (* - numeric)
  • Category [3] (*)
  • Ignore[3] (*)
  • Inv Class[3] (*)
  • Markup1[3] (* - numeric)
  • Markup2[3] (* - numeric)
  • Category [4] (*)
  • Ignore[4] (*)
  • Inv Class[4] (*)
  • Markup1[4] (* - numeric)
  • Markup2[4] (* - numeric)
  • Category [5] (*)
  • Ignore[5] (*)
  • Inv Class[5] (*)
  • Markup1[5] (* - numeric)
  • Markup2[5] (* - numeric)
  • Category [6] (*)
  • Ignore[6] (*)
  • Inv Class[6] (*)
  • Markup1[6] (* - numeric)
  • Markup2[6] (* - numeric)

(*) There can be a maximum of 6 cost categories in . Each CSV line of mark-up loaded must have one entry for each category that exists with either the invoice class/mark-ups or ignored flag set. 

You can update existing transactions with mark-up values using Set Mark-Ups.
