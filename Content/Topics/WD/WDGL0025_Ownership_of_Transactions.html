Ownership of Transactions by Sectors

When you enter a transaction (for example, a invoice), requires you to select an analysis (for example, a , overhead department, or asset).  This analysis will, either directly or indirectly, have a General Ledger account code associated with it, and the account code determines the sector to which that analysis belongs.

Posting to control accounts will always be to that sector, regardless of how the transaction may later be costed.  In other words, sectoration is not by costing distribution, but by the sector to which the "owning entity" belongs.

With Sectored Balance Sheets enabled, posting to control accounts will always be to that sector, regardless of how the transaction may later be costed.
