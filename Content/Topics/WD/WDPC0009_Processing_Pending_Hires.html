Processing Pending

The Process Pending screen allows you to specify how you will fulfil each of the items on a request.  At the top of the screen, the Assignments incomplete text indicates that there are request lines which do not yet have a specific component assigned to them.

NOTE

If a request line was for an asset group with a quantity greater than 1, the Process Pending screen expands this to separate lines; you need to make an assignment against each line.

Setting up re-hire auto-assignment

When you select a pending , will attempt to assign a automatically, based on the set up, if there are no assets in the relevant asset group that are not disposed of. If you choose Suggest , uses the same set up rules to propose a , but will do so even if there are assets that are not disposed of.

looks at the commodity code that is specified for the component on the web hierarchy ( Hierarchy). If no commodity code is specified, looks at the parent group, then the parent of that group, until it finds a commodity code. It then looks at the current trading agreement set up against that commodity code in SRM, and uses the supplier that is set up on that trading agreement as the for the component.

So, as an example of setting up the rules for assigning s, suppose you get most of your safety equipment from supplier A, but get high-visibility jackets from supplier B. In Hierarchy you have set up group 023 as Safety Equipment and group 0235 as High-Visibility Jackets. In the Commodity Code field for group 023, enter a dummy commodity code, say ZZ023; in the Commodity Code field for group 0235, enter a different dummy commodity code, say ZZ0235. In SRM Workbench, set up a company-level trading agreement (see Trading Agreements) for commodity code ZZ023, with a single supplier - supplier A; set up a company-level trading agreement for commodity code ZZ0235, with a single supplier - supplier B.
