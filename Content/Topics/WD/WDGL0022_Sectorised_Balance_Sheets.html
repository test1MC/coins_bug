Sectored Balance Sheets

You can set up so that the control accounts are split into sectors and each sector has its own balance sheet.

When posting to the control accounts, will override the sector element on the account code specified in the configuration.  For example, if the control account is configured as 00.7800 (and if the first element represents the sector), and the analysis for the invoice is to sector 03, would post to the account 03.7800.

If sectored balance sheets are used without inter-divisional postings, each sector will have its own share of the control accounts but will not necessarily have a self-contained trial balance.

If the two features are used together, each sector will have its own self-balancing trial balance, including its own share of the control accounts.  The inter-divisional accounts will only be used when a General Ledger batch has postings that cross between sectors, and there is an imbalance (in double-entry terms) within each sector.

To enable sectored balance sheets:

  1. Sectors must be enabled (see Sectors).
  2. Set the SECTBS parameter in Parameters, and the corresponding parameters in the individual ledgers, appropriately.

     

    CO/SECTBS is the "general" setting for whether you are using sectored balance sheets. The other parameters override this setting for the ledger they relate to (where available). So you could set CO/SECTBS to N, and have sectored balance sheet control accounts for by setting SL/SECTBS to Y. If the ledger parameters are blank (that is, they have never been updated), the setting in CO/SECTBS will apply. However you should check the parameter settings in each ledger to ensure any overrides are set as you want them.
