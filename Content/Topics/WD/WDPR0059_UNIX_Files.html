UNIX Files

creates ASCII text files during a submission attempt. The files are held in a subdirectory of the eGovernment root directory specified by the EGOVRDIR parameter. The subdirectory name is the same as the 'message class', which in the case of the end of year submission is IR-PAYE-EOY and is created automatically.

Each submission attempt is given a sequential number, and all files associated with that submission attempt have names beginning with this 8 digit number.

The files created are:

  • Log file (such as 00000001.log)
    A log of what has happened during the submission attempt which can be used to trace problems.
  • Sent Files (such as 00000001pst1.xml)
    XML messages sent ('posted') to the government gateway. The number following the 'pst' denotes the sequence in the protocol. Note that a file is only created when a new message content is required, so there are likely to be gaps in the sequence. You will always have a 'pst1' which will contain the P35 and P14 data, and a 'pst2' which will be the next message, but when polling, the gateway requires the same message to be sent repeatedly so new files are not created.
  • Reponse files (such as 00000001get1.xml)
    An XML message containing the response from the gateway.
  • Temporary File (such as 00000001.tmp)
    This file is passed as an argument to the java procedure that 'transmits' the XML. It contains the URL and the name of the file to send to the URL.
