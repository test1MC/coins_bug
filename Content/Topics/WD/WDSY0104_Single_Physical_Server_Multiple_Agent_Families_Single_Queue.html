Single Physical Server, Multiple Agent Families, Single Queue

In this scenario we have multiple agent families running on the same physical hardware server. Each agent family processes one or more report queues. A common usage would be for slow running reports (such as end of month reports) to use a particular queue (NASTY), with general reporting using another queue (GENERAL).

The diagram below includes a third queue reserved for more time critical reports (FASTPO) and shows how these queues may be apportioned across the agent families. All agent families will process reports for the FASTPO queue. The second agent family will also process reports in the GENERAL queue. The third agent family will process all queues (FASTPO, GENERAL and NASTY).
