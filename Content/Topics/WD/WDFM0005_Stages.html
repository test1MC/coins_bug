Stages

The stage setup will cater for the different steps and stages in the actual category and will depict the workflow. Each stage will have a function and therefore users and roles are granted access by their user groups or user profiles.

The stages that you enter will have a direct impact on the job as the stages dictate what the process of the job will be.

You can also have stages that are restricted on how they can be used, for example on used on a specific major category type. Some stages are linked to extra processing, where you mark the job as closed and it is ready for invoicing.

There are approx 37 stages available and these can be grouped by function or purpose.

Not all jobs will have all the stages and therefore you would select only the ones that follow the progress of the job from start to completion and the last one is “Job Closed”. This ensures that no more tasks or actions can be added to that specific job.

To set up stages:

  1. On the Stage Setup tab, click to add a new stage.Stage Setup Tab


  2. You must add stages in the order you want them to occur.  You must have Add as the first stage. 
  3. Select the Stage Type and enter text for the status you want the stage to set the to.

    NOTE

    Not all stages need to change the status.  For example, you may not want to change the status of the when a note is added.
  4. Choose whether this stage appears on the key stages screen. This will also be displayed on any reports and enquiries when doing the various stages
  5. Click to save the stage.

Each stage has a default comment that will be added to the history when that stage occurs.  You can change this text.

To change the history text:

  1. Click next to the stage.
  2. Change the text in the History field.

    To include the value of a field from the database, include the field name in double square brackets.  For example, to include the major category code, enter [[sw_job.sma_majcat]].  You can use the Database to find the field names.  If you omit the table code, will assume sw_job.

    History Text Example

  3. Click to save the stage.

After you have set up all the stages for the major category, you need to set the rules for the relationships between the different stages.  For example, before 'Confirming All Tasks Are Complete', you have to first 'Confirm All Tasks Have Been Planned'.
