Credit Notes

Credit notes are basically entered in the same way as invoices.   Choose a transaction type that represents credit notes, and enter the amounts as positive values (same as invoices).  However, when completing the batch header for a batch of credit notes, if the control total fields are set up to be input fields, enter the batch control amounts as negative values.

After you have entered the credit note, you can allocate it to offset invoice values.

To allocate credit notes to invoices:

  1. Go to Allocate Credits.
  2. Enter a new batch.
  3. Enter a new allocation transaction for each .
  4. If you are running a sector-based ledger, you must specify the sector to which you want to apply the payment.

    displays credit notes and outstanding transactions for the . 

    Before allocating (matching) a credit note to a transaction, you can check the invoice or credit note status and other details, by clicking the link in the Internal Reference column.  Click Back to return to the list of transactions.

  5. Select the credit notes and the invoices to match (allocate) them to. You can allocate a credit note to pay part of an invoice. The allocation amount must equal zero.

    NOTE

    This procedure does not allow you to leave a credit amount unallocated. If there is a remaining balance, it must be used to pay another invoice. The amount of the selected credit note(s) must match the amount of the selected invoice(s) that you want to pay. 
  6. Post the allocation batch. 

    generates a payment reference number for each payment allocated, whether it was made by a or a credit transaction. 

    The details of the payment batch are then processed in order to update the General Ledger.
