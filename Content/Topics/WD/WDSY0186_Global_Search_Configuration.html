Global Search Configuration

Global Search allows users to search for records in the database (for example, , customers, subcontractors or ), and also to find and run functions.  The search is configurable, so you can choose which tables, and which fields on those tables, to include in the search.

WARNING

Ideally you should make configuration changes with no one else on the system.

To configure Global Search:

  1. Before making any changes, stop the search index process: run syusrch_stop.sh in $BASE/bin.

    NOTE

    You do not need to stop the search index process if you are only making changes to the Label, Link, Open In, Foreground, Background, Detail, Icon, or Security RSP fields.
  2. Go to Global Search Configuration.
  3. Global Search comes with several standard search configuration records.  If you want to use one of these standard searches, open the record and the Use box.
  4. If you want Global Search to search for something that is not included as standard, add a new search configuration record.  Refer to the side-frame help for details.

    NOTE

    Global Search is intended for data that is relatively stable, such as or ; it has not been tested with transactional data such as invoices.  If you intend using it to search transactions, be aware that this will significantly increase the size of the database, and that the number of possible search results may make it impractical.
  5. When you have made the additions or changes you need, all the lines you have changed and run the Build Data action.
  6. the Initiale box.
  7. If you have changed the search conditions or want to remove existing indexed records, Delete.
  8.  Click .
  9. When the build data process has completed, restart the search index process: run syusrch_start.sh in $BASE/bin.
  10. Restart the webspeed agents, appserver agents and report agents.

    Whenever any of the tables that are included in the search are changed (records added, updated or deleted), the search data will automatically be updated to reflect the change.  You only need to rebuild the data if you change the configuration (specifically, if you change any of the following: Table, Condition Field, Key Fields, Condition Value, Field, ID Fields or Search Fields).

  11. To enable Global Search, set the SY/GLOBSRCH parameter to Y.

To configure the function search:

Function search uses a different set of data, which needs to be generated separately.  Use Build Function Search Data to generate the initial data, and run it again whenever changes have been made to functions or menus.
