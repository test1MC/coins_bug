Invoice Classes

Invoice Classes allow you to group sets of lines together on an invoice instead of printing them separately. The invoice class description will be displayed and the values of all the invoice lines with the same class will be totalled and displayed. Note: Invoice classes are only relevant to the 'SINGLE' invoice style.

Invoice Classes - configuration

Invoice Classes can be found under FM Contract Setup - 'Invoicing Setup' tab group.

Once configured, you can add these invoice classes to 'Schedule of Rates' (SOR) items and billable cost transactions.

For SOR items, the invoice class on a SOR item will be defaulted from the standard SOR item if it exists. To be able to change an invoice class in the 'Add/amend SOR items' stage, you must set it in the SORFIELDparameter and made it updateable in the particular SOR type.

For cost transactions, the mark-up configuration (see Mark-Ups) can be used to default invoice classes in the same way that mark-ups are applied.

Invoice classes on both SOR items and cost transactions can be updated in the 'Ready for Invoicing' stage.

Invoice documents

On FM Invoice configurable documents (see Configurable Documents), set the Invoice Class Form to the name of a form to display lines with an invoice class. Any lines with an invoice class will be grouped by that invoice class and printed using the fields configured for that form type. The fields available in this form type are: fmi_code, fmi_desc, fmi_totnett, fmi_totvat, fmi_totgrss.

If this form type is not set or the line does not have an invoice class, the lines will display individually as per the BODY form type.
