FM (FMAPP) Documents

FM documents (document type FMAPP) are used to print and invoices produced as part of the /certification process where all information within the process is summared at FM level (FM parameter INVGENER is set to Y). 

Define the layout of configurable documents using Document Designer in the System module. See Configurable Documents for information on how to define configurable documents.

Invoice Style

All FM and invoices have what is called an 'invoice style'.  This can be 'SINGLE', 'CONSOLIDATED', 'MAJOR CATEGORY', 'CROSS-'. 

For FMAPP the invoice style determines which are grouped on and invoices when Create {FM Application} and Invoice Generation processes are run. 

You can subgroup (subhead/subtotal) by invoice class and by item description (if not using entered order).

 

To configure a or invoice document:

  1. Go to Document Designer.
  2. Use the document type (FM) (FMAPP).
  3. On the Extras tab, leave Is Invoice uned if you are configuring a , it to configure an invoice.
  4. All invoice styles can be configured to be printed at detail (SOR item/cost transaction) or summary level (FM ) level.  the Level Summary box for a summary document: 
    • Summary level: You will get one page per customer (plus or major category or depending on the invoice style) and one invoice line per FM . 
    • Detail level: You will get a separate page per FM for each on the .  Each page then lists all items on that .  This is sometimes called a 'pricing sheet'.  Running at detail level, each page is effectively an FM invoice line but it will actually print the HEADER form, so you should configure this with FM fields and sw_invline fields, rather than just the normal sw_invhead fields (though you can use sw_invhead fields as well).
  5. Configure the BODY line with these special RO fields (sw_invline fields will not work):

    RO_FMAppCode - item code
    RO_FMAppDesc - item description
    RO_FMAppQty - item quantity
    RO_FMAppUoq - item unit of quantity
    RO_FMAppUoqValue - item unit value
    RO_FMAppValue - item value
    RO_FMAppUoqCost - item unit cost
    RO_FMAppCost - item cost
    RO_FMAppUoqMargin - item unit margin
    RO_FMAppMargin - item margin

  6. If subtotalling by item description (which allows you to group all SOR items and cost transactions of the same code/description together) then the following fields are used in the subtotal:

    RO_FMAppValueST
    RO_FMAppCostST 
    RO_FMAppMarginST
