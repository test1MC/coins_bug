Converting to Multi-Currency

is usually delivered with 1 as a multi-currency , but if you have an earlier implementation, you may have 1 set up as single currency (that is, the currency field is blank).  In this case you will need to define the base currency before you can process multi-currency transactions.  Also, any new you create on the system will have the base currency blank initially and will need to have the base currency set.

 

WARNING

If you have any doubts about the currency set up process, we recommend you run through the procedures in a "copy of live" system, and/or request assistance from a consultant, before performing the process on your main system.

To set up a to use multi-currency:

  1. Configure the using CO Configuration, but leave the Currency field blank initially.
  2. Set up the country codes -  use Countries.
  3. Set up the General Ledger account codes for the exchange gains and losses account, unrealed variance account and rounding account.  See GL Accounts.
  4. Set up the codes for the currencies you will use in that .  See Currency Codes.
  5. Set up the exchange rates for each currency.  See Exchange Rates and Conversion Rates.
  6. Set the base currency for the :

NOTE

To set the currency you MUST be logged in to as SYSAdmin rather than your normal user login.  This ensures that only the system administrator is able to set the base currency for a .
    1. Ensure that no other users are accessing the during this process.
    2. Change to the you want to set up.
    3. Go to Set Currency.
    4. Enter the code for the base currency of the - this is the currency that the will work in.

If you are converting an existing single-currency , and there are already transactions on the system, sets all transactions and and customer records to the base currency. This may take several minutes.  The process works through every account and every transaction present on the and calculates the various currency fields required.
