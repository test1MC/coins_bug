Validating the Configuration

Run CIS Scheme Configuration (in ) to check that schemes have been configured correctly.

If you have sectors set up, you must run this report, because it also initiales the fields on Sector Details Maintenance: it sets them to default to the parameters, unless the fields have already been filled in.

For each scheme, the report shows a line for each that belongs to the scheme, and for each sector within that that belongs to the scheme.  There is a gap between lines for different schemes.

    • The Submit column shows the expects you to use for submissions for that scheme.  If for the same scheme there are lines showing different 'submit ', that is a configuration error which needs to be resolved. Different schemes can have separate submission , but only one submission is allowed per scheme.
    • The column shows the included in the scheme.  The Linked column shows the other that are linked with the in the column (that is, those in the same group in LINKCOIR).  All in the scheme need to be linked, so any shown in the column for a scheme should be included in >LINKCOIR.  Within those , individual sectors can belong to different schemes.
    • The Name, Contact and Type columns show the contractor name, contact and company type as taken from the parameters or the sector address details; these must be the same for each combination of and sector within the scheme.

NOTE

If you are in any doubt about how to configure the schemes for your structure of CIS registration, contact for advice.  You need to ensure that the configuration is correct before you submit a monthly return.
