- Overview

Configurable validation allows you to store information about subcontractors or (for example, insurance policy details or non-insurance such as waste carrier's certificate or annual self-billing agreement, etc.), and to check at various points whether the information is missing or has expired; if so, can issue a warning or an error:

  • When entering or committing an order
  • When entering a certificate or an invoice
  • When entering a payment due date
  • During payment processing

The check can be set so that it warns you before the actual date, and gives you an error after the has expired.

There are different categories of :

  • Insurance - These require insurance-related information such as policy number and provider, as well as dates.
  • Dated Items - These items have dates but do not show insurance-related fields.
  • Qualification - These are for compliance items which simply need to be met once and do not expire.  These are more likely to be used on a subcontract than a subcontractor.

You can set up any number of , and can also set up as "memo only".  This means that you can store information against a without any validation being made against it. You can set up additional, user-defined fields against a , to record whether specific information has been provided; these can be set up as being required or as 'memo only'.

Process overview

  1. If you are using configurable for the first time, convert existing insurance data.
  2. Set up types.
  3. Apply them to existing subcontractors.
  4. Override the settings for individual subcontractors.
  5. Add to individual subcontractors.
  6. Check and update details on an ongoing basis.
