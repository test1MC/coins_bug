Compiling Function Security

In certain circumstances you can improve performance by compiling the user/function security settings.

If you have assigned a large number of user groups to each user, or if you otherwise have a complicated user access set-up, each time a user tries to run a function, has to check each user group in turn until it finds whether the user is allowed to run that function. By compiling the user/function security settings, the first time a user tries to access a function, evaluates whether they are allowed to run the function, and records this. Subsequently refers to this record to determine whether the user is allowed to run the function.

The SY parameter UFACCESS controls whether you are using this method.

One thing to be aware of is that if you make any changes to the user/function access settings, these will not take effect unless you re-set the UFACCESS parameter.
