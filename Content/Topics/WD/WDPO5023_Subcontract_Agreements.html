Setting Up Subcontract Agreement Processing

keeps track of the progress of subcontract agreements through the various approval and signatory stages by using a list of “events”.  See Subcontract Agreements.

To enable subcontract agreement processing, set the PO parameter SCTRCKAG to Y.  

If you want new subcontract orders to require an agreement by default, set the PO parameter SCAGREE to Y.

Agreement Types

Use Subcontract Agreement Types to set up a list of the different types of subcontract agreement you use (for example: DOM, JCT).

Agreement Events

Use Subcontract Agreement Events to set up agreement events, which are used to record things that happen in the process of the agreement.

The event codes themselves are user-definable, so your system can use any code you want for any event. For example, for the event “accepting an agreement”, you could use ACCEPT or OK. However, for automatic events, you must specify in PO Parameters which event code is used for which event.

Reminder Letters

You can set up a standard mail merge button on SC Agreements Workbench to allow users to send reminder letters to subcontractors (see Setting Up a Generic Mail Merge.  If you want to record automatically when a letter for an agreement has been sent, you need to set the Row Method to pgt-rsp.reminder1Sent (for the first reminder) and pgt-rsp.reminder2Sent (for the second reminder).  This will add the event specified in PO/AEV-REM1 or PO/AEV-REM2 to the agreement.  Users can filter SC Agreements Workbench based on whether there are letters due.
