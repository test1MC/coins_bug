Urgent Reports and Report Queues

Urgent Reports

allows you to give users the option to put print requests into an Urgent queue. will start a separate report process for each submitted request. The new process will process this one request then die. This urgent type should be used carefully as each new process takes up CPU and might have undesirable effects if it were to be overused.

We recommend you should reserve Urgent queues for use in emergency only (for example, where you have a number of reports all queued and you need something to run now).

Report Queues

Report Queues give you an alternative way of managing print requests. Report queues can have different production capabilities and performance. Queues are applied to normal (background/foreground) reports and with suitable setup will give you the ability to have urgent queues which may mean the Urgent type is not required.

Report Queues are "sticky" - remembers which queue you select when you run a specific report. The next time you run that report, will use that queue by default.

Configuration

To make the Urgent option available to the user, give them access to the function %WSY2130BCRR3.

Create report queues using Report Queues and specify on each user's record which queues they should be allowed to use and their default queue. Report queues are processed by report servers and the setup of report servers is required. If you do not setup any queues then a single unnamed queue will exist as before.

For more information about setting up report queues, see Report Queues.
