Changing Styles Using Style Sheets

If you want to change the and styles of

screens in ways that the palettes will not allow, you can set up an alternative cascading style sheet file to override the standard styles. 

To set up an alternative style sheet:

  1. Using a text editor, create a file to contain the style declarations.  For example: user.css.
  2. In the file, enter style declarations for the elements you want to change.  For example, if you want to set the background of the menu frame to white, and the font of the menu items to Courier New, enter:

    body.side{
        background-color:#ffffff;
        font-family:courier new;
    }

    NOTE

    The style sheet file could be a copy of the standard coinsstyle.css file that you have modified, or it could simply contain rules for the styles you want to change.
  3. Set the parameter SY/USRSTYLE to the path and file name of this file.
  4. You can also set up different style sheets for individual users; specify the address and filename of the CSS file using the Stylesheet field in Users.  You could use this to have different at each of your sites. 

    NOTE

    If the style sheet files and the style parameters refer to the same element, the setting in the parameters takes precedence.

As an alternative to creating a user-defined style sheet file, you could copy the standard style sheets to a bespoke directory and modify the style sheet files directly. 

To set up a bespoke style directory:

  1. Underneath the webserver base document directory there will be a coins directory. Create a new directory underneath this, and create a directory called style underneath that; for example: coins/bespoke/style.
  2. Copy all the contents of the Standard style directory into this user style directory. 
  3. Set the SY/WEBSTYLE parameter to the name of the parent directory of the style directory.  For example: bespoke.

    NOTE

    If you also have user-defined directories for scripts and images, set SY/WEBBASE to the parent directory of the images, scripts and styles directories, and set SY/WEBSTYLE to $WEBBASE.
  4. You can then modify the style sheets in the user directory, or add your own.
  5. If you want different styles for each , create different directories under the webserver base document directory (for example: coins/company1/images, coins/company2/images) and set the SY/WEBSTYLE parameter in each to point to the relevant directory.
