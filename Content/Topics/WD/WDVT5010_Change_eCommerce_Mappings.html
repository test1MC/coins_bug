Change eCommerce Mappings

If you are using the -etc hub for electronic trading, you will need to change the mappings between the BASDA codes and the codes in .

The -etc hub performs a mapping of all transactions from the suppliers' original format to a standard BASDA format. Within the BASDA format, code "S" represents the current standard rate of ,and typically the code "S1" represents the previous standard rate. When the rates change, this mapping will be updated so that code "S" still represents the current standard rate.

The sending of ecommerce transactions from the hub to your system will typically be suspended from approximately 14:00 on the day when the change of usage of BASDA code S from the current rate to the new rate takes place, until approximately 10:00 on the following working day. You should change the BASDA mapping within your system during that time.

If you require the suspension to operate for a longer period to enable you to make the necessary changes then this can be done on request. To request this please log a support call in the normal way at least two weeks beforehand. In the support call you will need to specify the following:

  • Date and time from which transactions should be suspended
  • Date and time from which transactions should be restarted

During the suspension period, you should update the mapping of codes for eCommerce as follows:

  1. Go to BASDA/ Mapping.
  2. Set the mapping for IN VAT/Tax Code S to the standard purchase code that uses the new rate; for example, P1.
  3. Set the mapping for IN VAT/Tax Code S1 to the standard purchase code that uses the current rate (the rate that will be superseded); for example, P11.

This will mean that any invoices received after you change the mapping will be mapped to the correct codes. Any invoices which are mapped to an incorrect code (for example, because of timing differences in the implementation of changes in the various systems) will load into and be registered on the without any problem. They will appear on a Exception Report which will be for information only. The figures from the XML are loaded to the . There should be no need to manually change the code on these transactions, although you can do so if you prefer.

You should review any credit notes that come in during the first few weeks after the rate change, to ensure that they use the same rate as the invoice to which they relate.
