Printing Purchase Orders

You can print purchase orders singly, print all unprinted orders, or select which orders you want to print.

The purchase order printing procedure can automatically commit the orders. In this case, checks your authoration limit and does not print orders over this limit.

  • Using the order workbenches, you can print individual orders or several selected orders at once.  You can also produce a draft print of the order or orders: this is marked as a draft, and does not commit the order.  If you print an order from the workbench that has already been printed, treats it as a reprint.
  • Using the separate 'print order' functions, you can print all orders or a range of orders.  Normally these functions will only print orders that have not yet been printed, but you can also choose to print previously-printed orders as well.  This procedure only prints purchase orders for the to which you have access (see the section on Contract Security in the System Guide for more information on access to ). You can also email or fax the orders, depending on the settings on the record.
  • You can also print alternative versions of the order (for example, a version without prices, or a order that shows only the changes). Separate functions must be set up to do this.

Purchase order printing

The parameter COMMETH determines whether printing purchase orders also commits them. The Maximum Order Value fields in Buyers define your authoration limit.

There are several PO parameters that determine which document to use for printing different types of order:

  • POFRM - The default document for material orders
  • You can specify alternative documents for other orders using the following parameters: ASSFRM (asset orders), OVFRM (overhead orders), RHFRM ( orders), SSFRM (steel orders), STFRM ( orders), WSFRM (workshop orders)
  • POPLFRM - The document to use for orders
  • POFRM-S - (Optional) The document to use for site orders.

The standard print functions use the configurable document specified by the appropriate parameter. If you want to be able to print orders using an alternative layout (for example, orders without prices or showing only changed lines), you need to set up separate functions. To do this, edit a copy of one of the existing "alternative form" functions (for example: %WPO1991FPOH) and enter the document code of the alternative document in the parameter list, as the value of the &sfs_form= parameter.

Emailing Orders

When printing an order and selecting Email output, the email will normally be sent from the /system email address. To override this and use the user's email address instead, add &CIWEmail=FALSE as a parameter to the OA function.

When "printing" multiple orders, you can email, fax or print the orders, depending on the 's preference (if the PO parameter DEFPRN is set to Y).
