Authored Sales Events

You can set up sales events that need to be authored before they are complete, so that one person runs the event but someone else has to authore the event before it takes effect.  For reservation events, you can set up the conditions under which the event requires authoration.

To set up a sales event (not reservation) that needs to be authored:

  1. In Sales Workbench Events, on the main Event Details tab, the Event Requires Authoration? field.

    displays the Authoration Event tab.

  2. Fill in the fields as follows:
    Sales StatusThe sales status to set the plot to after it has been authored.
    Authoration EventThe event to create in the Sales Workbench event history list when the event has been authored.
  3. Enter the default comments; this is the text that will appear in the Comments field when the authoration event is run (the user can enter different text if they want).

To set up a reservation event that needs to be authored:

  1. In Sales Workbench Events, on the main Event Details tab, the Event Requires Authoration? field.

    displays the Authoration Event tab.

  2. Fill in the fields as follows:
    Unauthored Sales StatusThe sales status to set the plot to if it requires authoration.
    Authored Sales StatusThe sales status to set the plot to after it has been authored.
    Authoration EventThe event to create in the Sales Workbench event history list when the event has been authored.
  3. You can specify the circumstances under which the event needs to be authored:
    All Reservations if all reservations need to be authored.
    Net Release if the reservation needs to be authored if the net sales price is less than the net release price.
    if the reservation needs to be authored if the is less than the default required amount.
    Exchange Deadline if the reservation needs to be authored if the exchange deadline has been changed from the default.
    Exchange Deposit if the reservation needs to be authored if the exchange deposit amount is less than the default required amount.
  4. Enter the default comments; this is the text that will appear in the Comments field when the authoration event is run (the user can enter different text if they want).

Setting Up Users for Authoration

To allow a user to authore events, you must specify which events they can authore.  For details of how to do this, see Event Security.
