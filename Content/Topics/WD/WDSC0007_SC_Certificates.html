Configurable Subcontractor Certificates

lets you design documents for printing subcontractor certificates. In this way you can print to your company's requirements and use pre-printed stationery.

Define the layout of configurable document using Document Designer in the System Module. See Configurable Documents for information on how to define configurable documents.

To define the format for printing certificates

  1. Use the document type Subcontract Certificate.
  2. In the Document field, enter a name for the certificate document (for example SCCERT1). You can define different documents to print certificates with different layouts. When you print the certificate, gives you a choice of which document to use.
  3. In the Body IDs field, enter a list of Item IDs, separated by commas, that correspond to any detail lines that are to be printed in the body of the certificate (see Defining a Standard Subcontractor Certificate).

    Body IDs: GRC,RET,DIS,VAT,TAX,PAY

    The body line will be repeated for each Item ID:

    CumulativePreviousThis Period
    Gross Certified47,000.0031,000.0016,000.00
    Retention2,350.001,550.00800.00
    Discount1,116.25736.25380.00
    7,681.415,024.912,593.50
    Tax10,883.447,178.442,593.50
    Payment40,268.7226,560.2213,708.50

    Lines with zero values will not normally be printed. To print lines even if all values are zero, put an exclamation mark (!) before the Item ID.

  4. In the header of the certificate, put information such as the subcontractor's name and address, the contract code, the contract name, the certificate date and value. Also, if you want column headings, such as Current Payment, these go in the header.
  5. In the body of the certificate, put a line that shows the details of the payment (cumulative, previous and this period amounts).  will print these details for every item line you define in the IDs list (see above). This uses the tables sc_certdet andsc_defndet. Use the following fields:
    • scd_prev for the Previous amount
    • scd_tp for the This Period amount
    • scd_cum for the Cumulative amount
    • scd_text for the item line description
  6. In addition, you can use the following hard-coded form types:
    • VAP – an optional line type to show VAP payment details. You could set up a different certificate form to print VAP certificate details separately from the main certificate print. This uses the tables vp_wbsdef, vp_unitdef, vp_subcc, jc_stndcc, vp_certdet and vp_subcon.
    • TCARD – shows timecard details for the payment. This uses the tables sc_timecard and jc_job
    • SCDET – shows details of the item line (which may have been restructured for detailed valuations). This uses the sc_itemdet table.
    • DAYWORKS – shows dayworks details for a payment. This uses the following tables: vp_wbsdef, vp_unitdef, vp_subcc, jc_stndcc, vp_certdet, vp_subcon, vp_dayworks.
    • CERTDIST, VATDIST – show the certificate distribution and distribution.
  7. In the footer of the certificate, put any additional information that may be needed.  Header and footer use the tables sc_certificate, sc_mast, sc_subcon, jc_job, sc_pay and sc_check
