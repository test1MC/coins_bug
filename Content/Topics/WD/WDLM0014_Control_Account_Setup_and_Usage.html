List of Lettings Management Control Accounts (Including Setup and Usage)

The letters and numbers in brackets refer to the Lettings Accounting Entries.

1 Tenant Control

This is set up on the Lettings ManagementModule Configuration screen. This GL account should balance to the total of the tenant balances in Lettings Management: it represents the unpaid amounts due from tenants; thus it is essentially a balance sheet debtor. It is posted to by the rent run (tenants) (Ai), manual charges/adjustments (E), recharge batches (G) and cash receipts/payments (B).

2 Rent Transaction Category Account

This is set up on the transaction category of type 'Rent' (in Transaction Categories). This represents rents due from tenants that have yet to be transferred to landlords; thus it is essentially a balance sheet accrual. It is posted to by the rent run (tenants (Ai) and landlords (Aii)) and manual charges/adjustments (E).

3 Landlord Control

This is set up on the Lettings ManagementModule Configuration screen. This GL account should balance to the total of the landlord balances in Lettings Management: it represents the amounts due to be paid over to landlords; thus it is essentially a balance sheet creditor. Rents are only credited to this account in the rent run once payment has been received from the tenant. It is posted to by the rent run (landlords) (Aii, Aiii &Aiv), manual charges/adjustments (E), recharge batches (G), the landlord payment run (D) and cash receipts/payments (D).

4 Management Fee Transaction Category Account

This is set up on the transaction category of type 'Management Fee' (in Transaction Categories). This represents management fees deducted from rents paid to landlords; thus it is essentially P&L income. It is posted to by the rent run (landlords) (Aii) and manual charges/adjustments (E).

5 Control

This is set up on the code. It is potentially posted to by the rent run (tenants (Ai) and landlords (Aii)), manual charges/adjustments (E) and recharge batches (G). For manual charges/adjustments and recharge batches the user can select the code. For the rent run, the code is taken from the transaction category where tenant and landlord codes are set. A default code is set up on the Lettings ManagementModule Configuration screen.

6 Tax Transaction Category Account

This is set up on the transaction category of type 'Tax' (in Transaction Categories). This represents withholding tax (deducted from non-resident landlords) which needs to be paid to the tax authorities; thus it is essentially a balance sheet creditor. It is posted to by the rent run (landlords) (Aiii) and manual charges/adjustments (E).

7 Internal Landlord Control

This is set up on the internal landlord. This represents the transfer of rents received to an internal landlord (similar to paying over the rent to an external landlord). This is most applicable where the landlord is a different group and this posting is inter-. It is posted to by the rent run (internal landlords) (Aiv).

8 Bank

This is set up on the bank account in . It is posted to by the landlord payment run (D), cash receipts and cash payments (B, C & D).

9 Unallocated Tenant Cash

This is set up on the Lettings ManagementModule Configuration screen. This is a balance sheet clearing account which should clear to zero when all cash has been allocated. It is posted to by cash receipts and cash payments (where tenant cash is not matched to tenant transactions) (B).

10 Deposit Control

This is set up on the Lettings ManagementModule Configuration screen. This GL account should balance to the total of the tenant deposits in Lettings Management: it represents the tenants' deposits held; thus it is essentially a balance sheet creditor. It is posted to by cash receipts (B), cash payments (B) and manual charges/adjustments (E).

11 Transaction Category Account

This is set up on each transaction category (in Transaction Categories). Different transaction categories represent different types of entry - some will be balance sheet debtors or creditors while others will be P&L revenues or expenses. Specific postings to certain transaction categories are covered elsewhere. There is a general facility to post to any transaction category from manual charges/adjustments (E).

12 Recharges Control Account

This is set up on the Lettings ManagementModule Configuration screen. This GL account should balance to the total of the recharges balances in Lettings Management. It represents pending recharges: that is costs which have yet to be either recharged or written off; thus it is essentially a balance sheet asset. It is posted to by costing entries from , Subcontract Ledger and (F), GL Journals (F) and recharge batches (G).

13 /Subcontractor

Other ledgers in financials allow entries to be costed to Lettings Management recharges. These other ledgers will include the setup of the relevant control accounts for the other side of these entries.

14 Recharges Writeoff Account

This is set up on the Lettings ManagementModule Configuration screen. This represents costs which have been written off rather than being recharged to tenants or landlords; thus it is essentially a P&L expense account. It is posted to by recharge batches (G).

15 Recharges Uplift Account

This is set up on the Lettings ManagementModule Configuration screen. This represents amounts added to costs recharged to tenants and/or landlords as a handling or administration charge; thus it is essentially a P&L revenue account. It is posted to by recharge batches (G).

16 Unallocated Landlord Cash

This is set up on the Lettings ManagementModule Configuration screen. This is a balance sheet clearing account which should clear to zero when all cash has been allocated. It is posted to by manual cash receipts and cash payments (where landlord cash is not matched to landlord transactions) (D).
