GL/JC Audit Data Download

Audit Data Download allows you to extract data files of GL/JC transactions and trial balances for audit purposes.

NOTE

This extracts the actual GL/JC batch lines, not the GL/JC transactions derived when the batch was posted.

Due to the large size of the output data set that can result (10M+ records) the extracts use direct extracts to files on the server. A single selection screen is used to collect the user-specified criteria for both GL/JC transactions and trial balance extracts:

  • - This specifies which to include in the extract.
  • Inclusive Date Range - This specifies which transactions are included in the GL Transactions Extract based on the in which they sit and the opening and closing balances to include in the Trial Balance Extract.
  • Date Posted (of GL Batch) - This specifies which transactions to include based on the date the GL batch was posted. The user either leaves this blank or specifies a date. If this field is left blank the report includes all transactions meeting the other criteria; otherwise the report will only include transactions with a GL batch posted date greater than or equal to that entered by the user. does not hold the time of posting, so restricting by time is not possible.

The GL Transaction Report filename is in the following format:

  • [Maximum GL Batch Posting Date]_[From Date]_[To Date]_[file suffix]_GL_Transactions
    For example: 2009-07-18_2008-07-01_2009-06-30_GL_Transactions

The Trial Balance Report filename is in the following format:

  • [From Date]_[To Date]_[file suffix]_Trial_Balances
    For example: 2008-07-01_2009-06-30_Trial_Balances

So for each of the extracts the user is able to specify:

  • Directory location - directory on server (NOT network locations)
  • File suffix - this is what the end of the filename will be after [To Date] - this will default to 'GL transactions' and 'Trial Balances'.
  • Delimiter (defaults to |)

OUTPUT FORMAT
The output format is a flat file using '|' as the field separator and "" to encapsulate text fields, with the first line containing the field names. The report has no other header or footer.

GL/JC Transactions

Code ; Name ; GL Batch Number ; Reversing Journal Indicator ; GL Batch Costing Line ; GL Account Code ; GL Account Description ; GL Account Type ; GL Account Creation Date ; Analysis ; Number ; Name ; Transaction Source ; Manual/Automated Journal Indicator ; Transaction Type ; Transaction Value in Base Currency; Transaction Value in Local Currency - if applicable ; Currency Code ; Foreign Exchange Rate - if applicable ; Effective Date ; Date ; Indicator - e.g. 0109 ; GL Batch Line Description ; GL Batch Header Description ; User ID ; User Name ; GL Batch Entry Date & Time Stamp

Trial Balance

Code ; Name ; GL Account Code ; GL Account Description ; GL Account Period Opening Balance ; Opening Date ; Opening Indicator - e.g. 0109 ; GL Account Period Closing Balance ; Closing Date ; Closing Indicator - e.g. 0109
