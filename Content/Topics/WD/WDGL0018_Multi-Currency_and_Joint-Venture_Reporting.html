GL Reporter – Multi-Currency and Joint-Venture Reporting

If you use multiple (kcos) to record the transactions of other group entities (subsidiaries, associates, joint ventures, etc.), you may need to produce single entity and/or "consolidated" reports. This is more complex if the functional currency (base currency) is different from one to the next - "consolidated" reports often need to use a common "presentation" currency. This may also require including varying percentages of joint venture(s).

The GL reporter allows you to:

  • Vary the base/alternative base currency for particular columns that use groups and make it different from the one chosen for the whole report.

  • Apply a percentage rate to the column value.  For example, in joint ventures you may only want to count part of the total value in the consolidated figures.

To configure a consolidated report:

  1. Use Group Maintenance to set up groups as required:

    • Set up a group that contains all the you want to consolidate across.

    • Set up groups for each combination of you want to show in a single column.  For each of these groups, all the in the group should use the same base currency or alternative base currency.

    • It is useful to also create a company group for each individual company. This allows company comparisons within a single report.

  2. Typically you would write the report in the that uses the presentation currency as its base currency.

  3. On each row,  in the Group field, enter the group that includes all the you want to report on.

  4. On each column, when adding any relevant decimal item (Opening Balance, Closing Balance and Movement) :

    • In the Group field, enter the group that includes the whose figures you want to add together.  These could be groups that include a single only, or they could include several that use the same base or alternative base currency. 

    • Specify which currency to show (the base currency or one of the alternative base currencies).

    you would use a group on the report rows that includes  1, 2 and 3.  You could then configure separate report columns for each :

    • Column 1 would specify a group that only included 1, and would use "base" currency.

    • Column 2 would specify a group that only included 2, and would use "alternative base 1".

    • Column 3 would specify a group that only included 3, and would use "alternative base 1".

    Alternatively, you could consolidate 2 and 3 in a single column, using a group that included both , and again used "alternative base 1".

To report a percentage of the total (for a joint venture):

When adding any relevant decimal item (Opening Balance, Closing Balance and Movement):

  1. Enter the percentage to report on in the Reported Percentage field.

To report in a different currency:

  1. Choose the currency to display (the base currency or one of the alternative base currencies). 

  2. You must also have a value in the Group field. 

When the report is run, If the reporting is included in the group, the column will be displayed using the base/alternative base currency set against the column, instead of the report currency.
