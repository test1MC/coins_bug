Valid Cost Codes

builds up cost codes by combining section, activity and (see Cost Codes). When generates cost codes automatically, either using Auto Generate Costcodes, or when a user enters a previously unused cost code, it checks that the combination of section, activity and is valid, based on classes and activity classes.

Each and each activity can be assigned to a class. These represent groups of or activities that belong together. For example, you might have a class of general that will be appropriate for all , and other classes for that are specific to different types of .

For each , section and activity, you can specify which classes are allowed.

For each section, you can specify which classes and which activity classes are allowed.

In order for a combination of , activity and section to be a valid cost code, the Class(es) lists on the activity, the section, and the must all contain the class to which the belongs, and the Activity Class(es) list on the section must contain the class to which the activity belongs.

NOTE

These rules do not apply to cost codes you create by copying standard or entering them manually.

To set up classes:

  1. Use Classes to set up the list of classes.
  2. When maintaining a , select the Class to which the belongs.
  3. When maintaining a , enter a can-do list of the classes that are valid on the , using the Classes field on the Cost & WBS Codes tab.
  4. When maintaining a section, enter a list of classes and activity classes that are valid on the section.
  5. When maintaining an activity code, enter a can-do list of the classes which are valid on the activity.

To set up activity classes:

  1. When maintaining an activity code, enter the activity Class.

    Activity classes are free-text codes; there is no pre-set list of the codes you can use.

  2. When maintaining a section, enter a list of activity classes that are valid on the section.
