Configuring the FAXSEND Parameter

When configuring the Company parameter FAXSEND in , there are various flags that can be added to the standard "vfx" command to provide further functionality to the Send Fax command. Some examples are given below, but for full details of the vfx command refer to the standard VSI-Fax User Manuals.

The standard send command for VSI-Fax is vfx; below is a list of the commonly used options from within :

-T string Send Time ([yy][mmdd]hhmm[am|pm])
-p priority l(ow)/m(edium)/h(igh)/u(rgent)
-C coversheet Coversheet to use
-s Silent: VSI-Fax will not return a Fax Request ID
-l length Page length; for example: A4

Email options - it is possible to request a fax status to be emailed from VSI-Fax when a fax is sent:

-m mode

Mode options - none/ok/fail/both/each

For example: fail will send an email to the address specified by -M if the fax fails; both will send an email if the fax fails or succeeds; each will send an email each time the fax fails..
-M mailaddr

Email address to send the notification mail to.

This can be

  • The System or Fax Administrator.
  • The user who made the fax request.  You can use a Session parameter such as $LOGNAME@companyname.com which picks up the user's ID, as long as userID@companyname.com exists as a valid email account on the appropriate Mail Server as configured in VSI-Fax.

If you are using for faxing you can use -M{EMAIL} to pick up the user's email address.  This means you do not need aliases like userID@companyname.com .
