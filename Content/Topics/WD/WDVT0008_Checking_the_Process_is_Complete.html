Checking the Return Processing is Complete

The following tabs of Returns Workbench all provide coded status columns to help you check that your  Return processing is complete:

 Returns – for all  Returns for  registrations submitted from this ;

Summary – for the latest  Return for each  registration submitted from this ;

Group Summary - for the latest  Return for each  registration (subject to security access to the submission for each registration).

 

Posting Status

  • Unposted - The posting batch has not yet been created.
  • Posting - The General Ledger posting batch has been created but has not finished posting.  This should normally be a transitory status unless there is a problem with the batch posting.
  • Posted (with Dig Deeper to General Ledger batch) - The batch transferring the entries from the control accounts configured against each code to the creditor/ debtor account configured against the Registration has been posted in the General Ledger.
  • Manually Posted - The necessary General Ledger postings have been dealt with manually.

Submission Status 

  • Unsent
  • Submitting
  • Failed
  • Complete
  • Manually Submitted - The Return has been submitted by means other than electronic submission direct from .  This allows for returns to other jurisdictions as well as for returns re-keyed directly onto the HMRC website.
