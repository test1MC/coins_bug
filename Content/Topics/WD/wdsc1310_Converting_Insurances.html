Converting From 'Old Style' Insurance

The 'old' style of SC insurance validation, using four insurance types, is only supported in .  This topic explains how to convert to the  checking supported in . 

You can convert existing data to the configurable format, so that the data is kept in sync and any reports you have will still work. 

To convert old-style insurance data:

  1. Set SC/EXTINS to Y.
  2. Use to create new  types that correspond to the existing four insurance types.
  3. Set SC/INSMAP to a pipe-separated list that corresponds to the existing four insurance types, using the names you have given them in , in the following order:

    Entry 1 = Public Liability;
    Entry 2 = Employer Liability;
    Entry 3 = First policy specified by SC/SCADDINS;
    Entry 4 = Second policy specified by SC/SCADDINS.

    For example: Public Liability|Employer Liability|Contractor All Risks|Professional Indemnity
  4. In , select each new insurance type, and choose Change Scheme Status from the Choose Action list.
  5. Set the status of the new versions to Default or Mandatory:  
    • Default means the insurance normally applies to the subcontractor but there can be exceptions.  The rule is applied to all subcontractors but can be removed for individual subcontractors. 
    • Mandatory means the insurance must be recorded against each subcontractor.  The rule cannot be removed. 

    This will add the new type to all existing subcontractors, and as long as the new policy name has the same name as one of the values in SC/INSMAP, loads policy data (insurer, policy number, limit amount and date) from the old insurance data (and updates the old insurance automatically if the new insurance type changes).

    runs a background report process to copy the data, so this may take some time.

  6. If you make a mistake in conversion (for example, if you set up an insurance type with a different name from that in the SC/INSMAP parameter) you will get no data.  In this case, set the status of the insurance type to "unset", correct the mistake and set it back to "default" (or "mandatory").
