Memo Data

Memo data is a term given to information that you can store, by , against each of the cost codes on a . A typical use of memo data would be to store budgets and forecasts that you want to use for comparison against actual figures for a . However, you can use memo data to store other types of information as well. Memo data allows you to store:

  • cost figures for each cost category
  • quantities
  • hours
  • revenue
  • text notes

You can set up different types of memo data (for example, to represent different budgets), and configure whether you can update the data for closed cost periods. Some specific procedures within (for example, valuations and forecasts) also store information in memo data.

You can enter memo data manually into , or import it from a spreadsheet application, such as Excel. If you are using an estimating system, you can also copy that data into a spreadsheet, which you can then import into . You can update budgets and forecasts by changing the values (unless auditing has been configured for a Status budget, does not keep a record of the previous figures once they have been changed). You can then use the memo data for reporting purposes. Memo data is available to the Horizontal Report, the configurable Status Report, and to OA reports.

You can control which memo data types individual users have access to, by setting memo data security.
