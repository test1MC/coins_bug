Configuring ETC to Store Images in OA Document Management

This topic explains the steps that need to be taken to store rendered images of XML documents from -etc in OA Document Management.

ETC Images in Document Management

  1. Ensure that basic OA DM functionality is working correctly; that is, ensure that you can upload an image against a PL Invoice and that it is then possible to view the image. If this is not working, refer to the How-To guide HOW TO ... CONFIGURE OA to enable the attaching of documents for information about how to set this up.
  2. The xsl style sheets must be available under the WebServer in order for the Invoice xml to be rendered correctly.
    They will exist in the directory that the EC parameter XSLLOC points to, usually $BASE/var/ECDir/style.
    Create a link directory under the WebServer base directory /coins/{environment name}/xsl to $BASE/var/ECDir/style.
  3. Copy all .xsl style sheets into the $BASE/var/ECDir/style directory. The standard ones we deploy can be copied from $BASE/app/xcode/db/xsl.
  4. Set the EC parameter XSLLOCOA to {Webserver Host Name}/coins/{environment name}/xsl
  5. Set the EC parameter OADMIMPT to Y.
  6. Set the EC parameter CIMDIR. (This must be set - although the value is not actually important when using OADM.)
