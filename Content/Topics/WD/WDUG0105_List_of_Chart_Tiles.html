Server Statistics
%WSYY001CMOE		 The live server's CPU statistics.  Showing a spot/instant utilisation and an average utilisation (10 min average but depends on system sar settings).

This is Application Monitor data and depends on the SY/APPMON parameter, which specifies which of your servers and environments is the main server/environment.

    
Webspeed Statistics
%WSYY002CMOE

This shows the number of webspeed agents running against the live environment, and of those how many were BUSY at the time of the snapshot (BUSY is where they are preparing and sending a page to a user at that point in time). The green line is the % of time that the webspeed agent has been idle during the snapshot period: 100% would indicate that the agents didn't respond to any requests; 0% would indicate they were always running and preparing pages and never sat idle waiting for the next request. So it is a measure of the percentage of time spent idle and waiting for requests to arrive.

This is Application Monitor data and depends on the SY/APPMON parameter, which specifies which of your servers and environments is the main server/environment.

    
Request Statistics
%WSYY003CMOE

This shows the total number of webspeed responses that the agents made and then divided in to the categories for the various time periods. So the number of responses under 1 second, the number of responses between 1 and 2 seconds, etc.

This is Application Monitor data and depends on the SY/APPMON parameter, which specifies which of your servers and environments is the main server/environment.

    
Database Statistics
%WSYY004CMOE		 This shows the number of record creates, reads, updates and deletes in the snapshot period.  The buffer hit % is the number of block reads (not quite the same as records) from memory as opposed to from disk as a percentage of total block reads (the higher the better).

This is Application Monitor data and depends on the SY/APPMON parameter, which specifies which of your servers and environments is the main server/environment.

    
Users Statistics
%WSYY005CMOE		 This shows the number of users logged in to the system (both concurrent users and role users).  Then of those how many are active i.e. have made a request within the snapshot period.

This is Application Monitor data and depends on the SY/APPMON parameter, which specifies which of your servers and environments is the main server/environment.

    
Report Statistics
%WSYY006CMOE		 This shows the number of report agents running, and of those how many are active and actually running a report (the rest are idle and waiting for the next report to run). It also shows the number of reports queued and waiting to generate.

This is Application Monitor data and depends on the SY/APPMON parameter, which specifies which of your servers and environments is the main server/environment.

    
Interface Statistics
%WSYY007CMOE

ESB Queue is the number of messages in the outbound ESB queue waiting to be sent out of .

ESB Error is the number of inbound transaction identified messages that have resulted in an exception/error and are sitting on the error queue waiting attention.

ETL Queue is the number of replication transactions waiting to be processed and replicated.

This is Application Monitor data and depends on the SY/APPMON parameter, which specifies which of your servers and environments is the main server/environment.

    
Disk Statistics
%WSYY008CMOE		 The % utilisation (full) of the various disks identified in AppMon.

This is Application Monitor data and depends on the SY/APPMON parameter, which specifies which of your servers and environments is the main server/environment.
