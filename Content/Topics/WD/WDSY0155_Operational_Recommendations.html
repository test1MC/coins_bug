Multi-Currency - Operational Recommendations

Do not allow transactions in other currencies

Although it is possible to have transactions (such as invoices) in a different currency to the currency of the account, we recommend you do not do this; set the flag to allow transactions in an alternative currency on an account to No.

Post batches before updating exchange rates

When transactions are written out to the system the currency conversion is made at that time. For example, where invoices are entered they are written to the system as unposted transactions, with the exchange rate as calculated at that time. When the batch is posted the items are set to posted but any foreign currency values are not adjusted.  Further transactions are written out at posting time, for example to the GL and ledgers.

If the exchange rate were to be changed between input and posting (for example, items left for some days before being posted) there could  be discrepancies between these two areas.  These would not be noticeable in day-to-day operation, but it is good business practice to ensure there are no unposted batches of transactions at the time that the exchange rates are updated. This task should typically be scheduled as part of the month end process, when unposted batches are cleared for other reasons.

Run Unrealed Exchange Variance reports regularly

Because the exchange rate movements can be significant, you should run the Unrealed Exchange Variance reports periodically to check the transactions held on the system in foreign currencies, to get early warning of significant differences, and to make provisions for them.
