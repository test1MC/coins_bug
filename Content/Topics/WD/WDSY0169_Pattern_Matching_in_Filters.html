Pattern Matching in Filters

Pattern matching is an alternative to using can-do lists in the simple filter (or in the Matches field of an advanced filter).

Pattern matching can be enabled throughout using the SY/PREFPAT parameter, or in an individual search or filter field by putting two backslash characters (\\) at the beginning of the search string.

If pattern matching is applied:

  • * (asterisk) means any number of characters.
  • . () means any single character.
  • , (comma) is a list separator (meaning OR).
  • - (dash) is a range separator. For example: \\A50-B50 will return records with codes in the range A50 to B50.
  • If the string begins with a caret (^) then the criteria is an exclusive list rather than an inclusive list.
  • If the values contain - , " or ' then they can be escaped with a backslash (\) or by enclosing the string in quotes.
    e.g. "A-000"-"A-999" would select a range of codes A-000,A-001,A-002....A-999.

NOTE

This only works for character fields.

You can combine the pattern matching wildcards. For example, \\A50-B50,*25 will return records with codes in the range A50 to B50 and all records with codes ending in 25.

This is different from the advanced filter, where:

From To Matches
A50 B50 *25

would only return records with codes ending in 25 that were also in the range A50 to B50.
