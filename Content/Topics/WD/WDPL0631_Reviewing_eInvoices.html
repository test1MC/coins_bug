Reviewing Electronic Invoices

The Loaded File Details (Review screen) shows a list of the individual invoices in a message.  It allows you to correct errors and review invoices that have been marked as 'held'.  You can get to this screen by:

  • Using the Load action to load a message on the first screen of Electronic Invoice Workbench.  If there are exceptions in the message, the Review screen opens straight away; if there are no exceptions gives you the option to open the Review screen.
  • Clicking against a loaded message on the first screen of Electronic Invoice Workbench.

To review invoices:

  1. Go to Loaded File Details.

  2. An invoice that, for whatever reason, cannot be moved to the will be flagged as ERR in the Status column: an error (or more precisely an exception). You can use the filter to see only those invoices that have been flagged as exceptions. By pointing to the Status column you can see what the exception is for a particular invoice. Erroneous purchase order numbers and duplicated invoice numbers can be corrected in the workbench.
  3. If the invoice is associated with a purchase order, you can see the order by clicking the PO Number link.
  4. To see the XML source of an invoice, click .  To see a human-readable view of the XML invoice, rendered using a stylesheet, click .  If the EC parameter SHOWINV is set to Y and you have dual monitors, the rendered view will appear on the second monitor.  The image has a Print button to produce a hardcopy of the image. 

  5. You can reject individual invoices using the Reject action.  This completely removes the invoice from the workbench.
  6. To see more detail about an invoice and, if possible, resolve any exceptions, click the link.  If an exception can't be resolved the invoice will remain in the workbench. 

  7. In the Details screen, you can:
    • View the invoice line details - click the Lines tab.
    • View the Invoice To and Deliver To addresses as sent in the XML message - click the Addresses tab.
    • Change the order number and the invoice transaction type. A unique invoice reference number for the and a valid will be required. Some may be able to send you costing information in the XML message; if so the costing distribution can also be entered using the data sent by the and preset defaults (see How To Change eInvoice PINVs to PINNs).  You can also change the distribution and the financial hold code (see How To Hold eInvoices).
  8. If an invoice is associated with a that is also linked to a subcontractor record in the same (they share the same account code), or the associated PO is flagged to be for operated , then you can note a Content component on the invoice. If the PL/LABCONT% parameter has a value, uses this to determine what percentage of the total invoice value will be allocated as a Content value.
  9. It is possible to hold back or retain an invoice in the workbench. These held invoices cannot be moved to the .  There are parameters that allow a whole selection of invoices to be put on hold based on the invoice value as they come into the workbench.

    • To hold an invoice, use the Hold action.
    • To release an invoice so it can be moved to the , use the Off Hold action. 
    • To hold or release all invoices in the message, click the Options button and use the Hold All or Off Hold All options. 
  10. When you have reviewed the invoices, click to return to the main Electronic Invoice Workbench screen, where you can create an invoice batch.

Configuration

The PL parameter ACCNODEFH determines how the account is sought. If set to PO and the invoice XML refers to a valid purchase order in the current , the account is drawn from the purchase order. The workbench will use the PL account number derived from the PO even if it is for a other than the one sending the message. In this case an exception is indicated. If set to MSG, the contractor's account code is thought to be known to the , or the ETC hub, and it is contained in the <BuyersCodeForSupplier> tag in the XML.

The transaction type is set initially from the BASDA- mapping and typically it is one that requires the invoice to reference a Purchase Order (commonly called INV or PINV transactions).

Hold codes for eInvoices

Set the following PL parameters so that a default hold code appears when creating the batch from the eInvoice workbench:

  • PL/DEFHOLD to Y
  • PL/DEFHCODE to the default code you require
  • If you want all registered invoices to be held, then set PL/HLDREGIN to Y. Registered invoices (that is, any that have not been able to be costed but are not in error) on the workbench will be put on hold.

Default values for the financial hold code are enabled by the PO parameters HLDOKCRN and HLDOKINV. The PL parameters EAUTOMHI and EAUTOMHC if set blank will overrule HLDOKINV and HLDOKCRN, treating them always as NO.

For costed invoices, anything above the Authoration limit (in ) will be put on hold, otherwise not.

The PL/INVINPER parameter is used to ensure that only transactions dated in the current period are allowed through the manual entry. The workbench respects this parameter as well.
