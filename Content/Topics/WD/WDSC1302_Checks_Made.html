Order Entry

Entering a subcontract order causes the information the subcontractor record to be verified; if any of the records triggers an error, it will not possible to enter the order.

Committing an Order

When committing a subcontract order causes, if any of the records on either the subcontractor record or the order triggers an error, it will not possible to commit the subcontract order.

If a record with the same type exists on both the subcontractor and the order it must be valid on both; it will not be possible to use information on the order to override information on the subcontractor record.  For example, if there as an expired public liability insurance record on the subcontractor and a valid public liability insurance record on the subcontract it will not be possible to commit the subcontract order.

Certificate Entry

When entering certificates, checks the status of the subcontractor and the relevant subcontract, and displays the appropriate warnings or errors.

If the test is marked as 'apply levy' and SC parameter LEVYITEM is set to a valid certificate definition line, then if the transaction date test fails, applies the rate to the certificate detail line.
