Format of Order Numbers

You can configure the format of purchase order numbers, so that they show information about the order.

Order numbers can be up to 15 characters long.

The simplest format for an order number would be a single numerical sequence for your entire . To do this, set PONUM1 to comp, and POFORM to 99999999 (or however many digits you want in the number, up to 15).

But you may want the number to show other information:

    • You can add the current number at the beginning of the order number.  Set APPENDCO to Y if you want to do this, and use APPENDDL to specify the character to separate the number from the rest of the number. Leave APPENDDL blank if you do not want to a separator.
    • You can show any one of the following as the base part of the order number:
      • For material, or subcontract orders: , location, group, type, or any one of the four analysis sets. Use the PO parameter PONUM1.
      • For overhead orders: department or one of the four General Ledger account elements. Use the PO parameter PONUM2.
      • For workshop orders: location or . Use the PO parameter PONUM3.
      • For asset orders: asset code or department code. Use the PO parameter PONUM4.
      • For orders: department or one of the four General Ledger account elements. Use the PO parameter PONUM5.
      • For orders: location/department or one of the four General Ledger account elements. Use the PO parameter PONUM6.

         

        For example, to include the code in the order number, set PONUM1 to . prefixes the number with the code. There will be a different sequence of numbers for each prefix; for example: 1/00001, 1/00002 and 2/00001, 2/00002.

         

        NOTE: To generate different sequences for each , set the above parameters to comp. 
    • To control whether the / separator is shown between the base and the sequence part of the number, use the PO parameter INSERTDL.
    • To indicate the type of order (for example, material, or subcontract), enter the prefixes against the following PO parameters:
      • ASORDPFX (Asset)
      • OVORDPFX (Overhead)
      • PLORDPFX ()
      • POORDPFX (Material)
      • RHORDPFX ()
      • SCORDPFX (Subcontract)
      • STORDPFX ()
      • WSORDPFX (Workshop)

For example, MAT-, OHD-, PLT- would produce order numbers such as MAT-00001, MAT-00002, OHD-00001, PLT-00001, OHD-00002 and so on.

  • To show the origin of the order (head office or site), enter a list of two prefixes as the PO parameter POSHOPFX. For example: H,S would produce order numbers such as H00001, H00002, S00001, H00003.

     

    To distinguish between head office and site subcontract orders, use the PO parameter SCSHOPFX in the same way.

You can combine any of the above, and all are optional. generates separate sequences for each different prefix. So if you set different prefixes for site and head office orders, and for , material and overhead orders, there will be six different sequences. If you have different prefixes for site and head office orders, and include the code, there will be two sequences for each .

Subcontract orders always have different sequences from material/ orders, even if you are not using any prefixes. The parameter SCFORM governs the format of subcontract order sequence numbers, and SCNUM1 governs the base.

You can change the next number in any of the sequences. See Number Sequences.

The prefixes for Order Templates are governed by the PO parameters POTMPFX (purchase order templates) and SCTMPFX (Subcontract order templates).

In addition, the PO parameter POPFXFST determines whether the order prefix is placed before the base, or before the sequence number.

So if POPFXFST = Y, the example order numbers above would be HM-1000/0005 and SP-Civil/B0102.

If POPFXFST = N, they would be 1000/HM-0005 and Civil/SP-B0102.

The PO parameter SCPFXFST has the same effect for subcontract order numbers.
