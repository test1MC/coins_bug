SC Ledger Basics - Subcontractor Warning Messages

Whenever you enter a subcontractor code, checks various conditions and may display a warning message. You can choose to ignore the messages and continue.

Possible messages are:

  • Outstanding authenticated receipt from date. If there are one or more authenticated receipts outstanding beyond a given date, the message shows the date of the oldest. The AGEAUTH parameter controls how long authenticated receipts can remain outstanding before produces a warning. The AUTHZERO parameter determines whether authenticated receipts with zero amounts are counted as outstanding.
  • n authenticated receipts outstanding. The NOAUTH parameter controls how many receipts may be outstanding before issues this message.
  • Subcontractor code is dormant. The Dormant field (in Subcontractor Maintenance) for this subcontractor is set to Y. may prevent transactions to this subcontractor. See Dormant Subcontractor Processing, for more details.
  • Unallocated cash exists for this subcontractor [...].
  • This subcontractor [...] is on hold. The subcontractor has been put on hold, to prevent payments being made to them (for example, if there is an unresolved legal problem between your company and the subcontractor).
  • Various conditions generate warnings or errors; see - Overview.
  • This subcontractor account is operated in a different currency.  There are subcontracts with currency different from that of the subcontractor. 

To continue after any message, click OK.

To produce a report on these and other conditions for subcontractors, use Defaulters Report (see Defaulters report).
