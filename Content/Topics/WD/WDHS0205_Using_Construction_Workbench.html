Using the Construction Workbench

Here is a summary of the process:

Plan the Build Schedule

  • Make sure plots have the appropriate construction associated with them; you can update the field from the Plots tab.
  • Enter the start (or end) dates and calculate the other dates by applying the standard . See Construction Workbench - Forecast Matrix.
  • Make any necessary adjustments to the for individual plots.
  • Take a baseline copy. See Copying Construction Forecasts.

Build Release

  • Release plots using the Build Release action on the Plots tab (or by running the Build Release event from the summary of an individual plot).
  • If required, you can make the build release subject to authoration.  See Authoring Sales Events. 

Update Progress

  • There are various options:
    • Print a sheet (using Construction Progress Tick Sheet), send to site and use the marked-up report for update on the Progress Tick Sheet tab. See Construction Workbench - Progress Sheet.
    • Updating Progress from the Forecast.
    • Updating the Progress by Construction Stage.
  • E/W if beyond Build Release

Monitor / Report

  • Print out a Gantt chart using Construction Gantt Chart.
  • The Plots tab highlights differences between the baseline build completion dates and the expected build completion dates.

Re-schedule / budget review

  • Save Baseline to historical copy

Configuration for Construction Workbench

Configuration

  • Set the relevant parameters using Module Parameters.
  • Set up build s using Construction .  The construction s will need to be appropriately configured (with different s for two storey houses, three storey houses, apartment shells, apartment fit outs, blocks of garages, show homes, etc).  This will also need to include a dummy construction for any non-sellable plots which are to be excluded from the construction workbench (and for which all stages will be marked as excluded).
  • Link house types and plots to the appropriate construction s.
  • Use Sales Workbench Events to configure events for build release and de-release, using the CSREL and CSDEREL event codes respectively.  The build release event can be configured to require authoration.  For information about how to do this, see Authored Sales Events.  This will enable releases to be authored either directly online by the authorer or by a clerk/secretary based on a signed Sales Events to Authore report. 
  • Set up any required  captions (such as for site works start and end dates) using Captions; specify which captions to show on the summary tab using HS/PRWBCG.
  • Set up any holidays using Construction Holidays.
  • Set up forecast types using Plot Forecast Types.
