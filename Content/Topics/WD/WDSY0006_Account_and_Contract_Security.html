Account and Security

You can control which or General Ledger accounts each user has access to. If a user is denied access to an account or , they will be unable to enter transactions, report or on it, maintain it, and so on. The account or will not appear on their lookups, and if they enter the account code or code, issues a message informing them that they are not allowed to access it.

Use Security and GL Security Maintenance.  Memo Security and Employee Security operate in a similar way.

To set up account or security:

  1. A parameter defines which field on the account or record is used as the basis of security.  For example, security could be based on the contract group field, or one of the account analysis sets.
  2. In the security maintenance function, for each user or group, you enter a can-do list of values for that field.  The user then has access to all accounts or on which the security field matches the list.

    For example, if contract security is based on the contract group field, an entry of !CIVIL,* allows the user access to all except those belonging to the group CIVIL.

  3. You can also specify a can-do list of individual accounts or that you want to grant or deny the user access to.

    For example, to allow the user access to all belonging to the group CIVIL except 1652, enter CIVIL in the Security List frame and !1652 in the Code frame.
