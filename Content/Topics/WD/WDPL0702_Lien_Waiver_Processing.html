Lien Waiver Processing

 

  ***NOTE: THIS IS A US SPECIFIC TOPIC.***  

Lien waivers can be generated from two modules: (PL) and Subcontract Ledger (SC). Outlined below is the information you need to know about processing lien waivers in these modules.

How to Process Lien Waivers:

Please note that some sections in this process refer to functionality within a specific module; others are shared by both modules. This is noted in the beginning of each title with the notation PL ONLY, SC ONLY, or SHARED.

    • For processing waivers in PL, you need to be concerned with the PL ONLY and SHARED sections.
    • For processing in SC, you need to be concerned with the SC ONLY and SHARED sections.

1. General Setup Maintenance

1.1 SHARED: Flag Company as USA Company

Lien waiver processing is only available for that have been defined as USA . Non-USA do not have any lien waiver data entry fields or menu items displayed. For clients using OA in the United States, the (USA) should already be set up correctly. If it is not, you can define a as a USA by entering USA in System > Company Setup > Configuration > Parameters under the Parameter ID COMPLOC.

1.2 PL ONLY: Configuration ?

The PL module defaults for lien waiver printing and warning messages are defined in > Setup > Configuration > Parameters. These defaults are assigned to each supplier that is created.

1.3 SC ONLY: Configuration ? Subcontract Ledger

The SC module defaults for lien waiver printing and warning messages are defined in Subcontract Ledger > Setup > Configuration > SC Parameters. These defaults are assigned to each subcontract that is created.

1.4 PL ONLY: Define Supplier Flags for Printing Options and Warning Messages

Each supplier has Lien Waiver flags on the Configuration tab in > Supplier Maintenance > Supplier Summary > Configuration tab.

There are two fields related to lien waivers on this screen:

  1. Lien Waiver Required - You may select one of three options: Selected, Always, or Never. For a detailed explanation of this functionality, see P4. PL ONLY: Summary of Options for Lien Waiver Flags.
  2. Warn for Outstanding Lien Waivers on Invoice Entry - If ed, this causes a warning message to appear in Invoice Entry if an outstanding lien waiver for the supplier is found.

1.5 PL ONLY: Define Flags for Printing Options and Defaults

Each or has Lien Waiver flags on the Configuration tab in Status > > Details > Configuration tab.

There are two fields related to lien waivers on this screen:

  1. Lien Waiver Required Default for Subcontractors - The Yes/No box defaults the subcontracts when a subcontract is created for the .
  2. Lien Waiver Required for Suppliers - You may select one of three options: Selected, Always, or Never. For a detailed explanation of this functionality, see 4. PL ONLY: Summary of Options for Lien Waiver Flags.

1.6 PL ONLY: Lien Waiver Overrides

Lien Waiver Overrides is a maintenance function found in > Administration > Lien Waiver Processing > Lien Waiver Overrides. It allows you to select a supplier and to always be generated for lien waivers. The existence of an entry in this function is what marks the supplier/ combination as 'Yes' for lien waivers. For more information on this function, see 4. PL ONLY: Summary of Options for Lien Waiver Flags.

1.7 SC ONLY: Define Subcontract Flags for Printing Options and Defaults

Each Subcontract has Lien Waiver flags on the Setup tab in Subcontract Ledger > Subcontract Maintenance > Subcontract Summary > Setup tab.

There are two fields related to lien waivers on this screen:

  1. Lien Waiver Required - This is a box that indicates if a lien waiver is required for this subcontract.
  2. Warn for Outstanding Lien Waivers on Invoice Entry - If ed, this causes a warning message to appear in SC Certificate Entry if an outstanding lien waiver for the subcontractor is found.

2. Steps for Lien Waiver Processing

2.1 Select Step 1A or Step 1B

Step 1A - PL ONLY: Data Entry

In the module, enter payments into a payment batch like you normally would and process them up through the posting of or payments. Once a batch is posted, it is ready for printing lien waivers. Unposted batches are not displayed in the Print Lien Waivers function.

- OR -

Step 1B - SC ONLY: Data Entry

In the Subcontract Ledger module, enter payments into a payment batch like you normally would and process them up through the posting of or payments. Once a batch is posted, it is ready for printing lien waivers. Unposted batches are not displayed in the Print Lien Waivers function.

2.2 SHARED: Printing Lien Waivers

Printing Lien Waivers is done in the PL module through > Payments > Print Lien Waivers, or in the SC module through Subcontract Ledger > Payments > Print Lien Waivers. The batch selection for this function is unique for each module, but the screens look very similar. The end result is that you choose the batch to be used for printing lien waivers.

The screen below displays the PL Print Lien Waivers batch selection screen.

Once the batch is selected, the Print Lien Waivers screen for the batch is displayed. Click the Batch Number to select a batch.

SPECIAL NOTES:

  • Since you can select posted batches to be used for printing lien waivers, you are allowed to reprint waivers if needed at a later date.
  • Lien waivers can be printed for payments made with and for payments.
  • One lien waiver is printed per or payment.
  • The generation of the Lien Waiver Receipts data is processed by using the report generator when the waivers are printed. This processing could take some time depending on the volume of transactions in the payment batch. It is always recommended that this type of processing be run in background mode.

2.3 Lien Waiver Receipts

The Lien Waiver Receipts function is found in > Administration > Lien Waiver Processing > Lien Waiver Receipts. It refers to the action of noting that a waiver has been received back from a supplier or subcontractor.

  • Lien waivers from both the and Subcontract Ledger modules are tracked in this function.
  • Clicking the folder icon for a lien waiver allows you to flag the waiver as received and to enter additional information for the receipt of the waiver.
  • Lien waiver numbers are automatically assigned when the waivers are printed. There is no Last Waiver Number counter stored in the system; the last number used is incremented.
  • The Lien Waiver Number is a link that takes you to a screen displaying the detail lines of the lien waiver.
  • There are no Add or Delete options for this function. Entries are automatically added to this function when lien waivers are printed.

2.4 Outstanding Lien Waiver Report

The Outstanding Lien Waiver Report allows you to see a list of lien waivers that have not been flagged as received from the supplier or subcontractor. The same report can be run from either > Reports > Outstanding Lien Waiver Report or Subcontract Ledger > Reports > Operational > Outstanding Lien Waiver Report. 3. California Lien Waivers

The state of California requires a special lien waiver format that does not print all the BODY information found in the general waivers normally provided with OA. The system comes with some California waivers. These waivers have a document code that begins with '%CAL'. If you need to custome these for a particular client, the custom waiver document code needs to start with 'CAL'. If OA prints a waiver with a document code that begins with %CAL or CAL, then BODY lines are automatically suppressed.

5. Lien Waiver Processing Troubleshooting

  1. I've posted my and printed lien waivers, and I'm missing some waivers because the Supplier flags, flags, or Overrides are set incorrectly. What should I do?

Since the flags are interpreted when the waivers are printed, you can just correct the flag settings and print the waivers again.

  1. I've printed lien waivers, and too many waivers are printing because the Supplier flags, flags, or Overrides are set incorrectly. What should I do?

Since the flags are interpreted when the waivers are printed, you can just correct the flag settings and print the waivers again.

  1. I see that I have entries in Lien Waiver Receipts for payments that did not need a waiver. How do I remove the unwanted entries in Lien Waiver Receipts?

This most likely happened because the Supplier flags, flags, or Overrides are set incorrectly. Since the flags are interpreted when the waivers are printed, you should correct the flag settings and print the waivers again. The system is smart enough that it should remove extra waiver receipt entries that do not belong. Note that if a waiver is marked as received, it is locked in the system and not removed.

  1. I need one lien waiver to print per . When I print , all the for a given supplier are combined into one and only one waiver is printed. How do I print one waiver per ?

Lien waivers are currently designed to print only one waiver per . To print one waiver per , you must enter the payments so that a single prints for each .
