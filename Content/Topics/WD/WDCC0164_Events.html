Events

Within , you need to record key events that occur during the process. For example:

  • For a call, these might include a site inspection or recording a response from the customer.
  • For an issue, these might include changing the priority or status of an issue.

Some of the events are optional; others may be required in order to correctly manage the business process involved. There are some events you can only record if certain other events have already occurred (for example, you might want to make sure an out of warranty letter is only sent after the out of warranty event has been run), and normally these events will not be available until the prior event has been recorded. In addition, creates some events automatically, for example when you print a letter.

Once events have been recorded, you can see them in the History tab on the summary page of the relevant record.

To record an event:

  1. From the Summary page for the record you want to add an event for, click the Options button.

    displays a drop-down menu which includes the events that are available to you.

  2. Click the event you want to record.

    (Alternatively, click the History tab, use the Add Type drop-down to choose the event you want to add, and click .)

    displays an Add Event page. The fields available will depend on the specific event, but usually include at least a field for the date when the event occurred and a field for you to enter comments about the event.

  3. Enter the details of the event.
  4. If you want to create a follow-up action for yourself or another user (for example, to remind a colleague to phone a purchaser, or to record an appointment you have arranged), the Follow Up?box and fill in the details on the Follow Up Action tab. See Actions.
  5. Click to save details of the event.

Configuring events

Each type of record has a different set of events available, with a different maintenance function:

  • General Events
  • Issue Events
  • Call Events

See Configuring Events.

Standard Event Comments allows you to set up standard text that users can include in the Comments field when they record an event.

You also need to set up event security to control who can run each event; see Event Security.
