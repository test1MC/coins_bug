Linking Documents to Tables

To allow users to attach documents to a record type, you need to set up a Document Management link for the corresponding database table. This enables the "Attach Documents" icon on the relevant page.

 

Setting up documents:

Set up the tables you wish to attach documents to:

    1. Go to Document Management Links.


    2. For each table you want to attach documents to, enter the table name.
    3. In the Base URL field, enter unless you are not using Document Management to store the attached documents. 

      NOTE

      This only works if the page has the table you want to attach documents to as one of the header tables. If the table you want to attach documents to is a body table, you will need to add a column to the browse to show a button next to each record.

To identify the base table name, click on the page header title, such as - Invoice Details.

To identify the base table name, navigate to the page required, such as - Invoice Details. On the page use the keyboard shortcut of CTRL-Shift-Z

 

Configuration

Create a Unix directory as the base directory for documents that have been attached; for example, "attached" under ../var/data. Ensure this directory has full rwx permissions - chmod 777 attached. Use Disk Maintenance to specify the path to this directory. 

Ensure that the upload script and directory are set correctly: see File Uploads.
