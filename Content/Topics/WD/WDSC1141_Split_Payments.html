Split Subcontractor Payments

Sometimes payments to subcontractors need to be separated either for or individual certificates. You can control payments to subcontractors such that payment by subcontractor, by or by transaction are possible. You can set up defaults on subcontractor and subcontract records, which will default down to the certificate record where it will also be updateable.

To process split payments:

  1. In Subcontractor Maintenance, on the Payment Method tab, set the Payment Type field as follows:

    Subcontractor = payment for each subcontractor -- this will be default value and also the value used if the feature is suppressed.

    = Separate payments for each .

    Separate = Separate payments for each transaction (certificate).

  2. When you raise a new subcontract order (or a new subcontract record), the Payment Type will default to the setting from the subcontractor. You can change this. If you change the Payment Type on a committed order, the subcontract record will be updated accordingly.
  3. You cannot change the payment type when raising a ; you will need to update the subcontract.
  4. When you enter a certificate, you can specify how to pay that certficiate using the Payment Type field on the certificate record.
  5. When you freeze the payment batch, creates one payment record for each certificate or as required by the payment type set on the certificate.
