Major Categories

 

Major Categories are linked to a Contract. Each major category defines the method of invoicing and the workflow or stages for all the jobs within that category.

A Major Category is the main group name of a type of work or job that needs to be carried out for the contract, i.e. service maintenance. It is the first line in the hierarchy of a job.

A major category also defines the method of invoicing for all the jobs within that category. The categories would include lump sum, schedule of rates, repeat invoices.

For example you may wish to bill the client for the contract based on the number of contacts made and the method used, i.e. by email, by phone. If the method was by phone then the cost plus basis may be used for the cost plus an additional admin charge. Another example is for a repeat invoice, where the client wants to spread the cost of the boiler service over the year.

The minor categories relate to the type of job, or it could be the geographical areas. For example we have a major category for service maintenance which incorporates the annual fire alarm test and the boiler service. The minor category would be each service. It could also be location, as the cost for repairs may vary in different districts of Hampshire.

Fields

Description
Code

The code for the type of major category. (Trainer’s Notes - This can be contract specific however it is recommended to keep the codes the same throughout. For example, Repeat Invoicing = REPINV. You have up to 8 characters.)
Description A meaningful description of the Code.
Invoice Type

This determines how the jobs will be invoiced.
Default Job Value Enter the default amount of the job in this category.
Maximum Value The maximum amount of the job.
Uplift (%) The percentage amount to add.
Claims Handling Element (%)

Enter the amount if a handling charge is required. (Only for cost plus jobs.)
Order not to exceed jobs Used with transaction invoicing types. When selected in the Tick Box field, opens two additional fields, which are mandatory.
Order Not to Exceed SOR Item Code

Defines which SOR code to reduce the invoice amount to the maximum
Order Not to Exceed SOR Item Description The Description of the SOR code used
Quoted Jobs Used with transaction invoicing types. These show the Quoted Value Job. (These cannot be used with the applications & certification.)
  1. Save the Major Category.

For each major category you should set up at least one of the following - minor category, stages of work, responses times, schedule of rates and the Invoice Details.
