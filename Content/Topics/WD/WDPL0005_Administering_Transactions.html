Administering PCard Transactions

The administrator reviews all transactions to ensure that all the necessary information is present, completes the costing and distribution, and can then create a PCard transaction batch.  If individual card holders are authored to enter the initial details, they can correct errors and mark them as reviewed.  If not, the administrator will need to correct any errors in the transactions.

Administering Transactions

  1. Go to PCards Workbench - Administrator Mode.
  2. If you look after more than one administration , select the from the Administration list and click . Screenshot: PCard Workbench (Administrator Mode) 

    NOTE

    If individual card holders are responsible for reviewing transactions, you can set the filter to "Reviewed Only".
    • If a transaction contains errors, the Status column shows Error.  You must correct any errors before you can mark the transaction as "Reviewed".  You can see a detailed explanation of the error or errors by pointing to the error indicator.
    • If there are warnings that relate to a transaction, the column to the right of the Status column shows a red exclamation mark.  You can see the warning text by pointing to this warning indicator.
    • You can see an image of the transaction receipt by clicking the button in the View Receipt column (or by clicking the card network logo from the transaction detail screen). Transaction Receipt Image
    • You can see the XML message for a transaction by clicking the button in the View column.
  3. To fill in any missing transaction details or correct errors, click .

    The Notes field (immediately below the Line Item Details) shows any comments from the card holder.

  4. Complete the allocation details for the transaction:

    Screenshot: Allocation Section

    • Confirm the for which the purchase was made.
    • Ensure that the account is correct (and is not dormant or closed).
    • Confirm the analysis or enter the correct , overhead department, asset or workshop code.
    • If the uses WBS codes, confirm or enter a valid code. This code will be defaulted into the WBS column against any line item detail line where no WBS code is already present.
    • Enter the amount (Level 1 only).
    • the Allowed box (this will be uned if the transaction is for a merchant category that is not normally allowed).  This confirms that you are aware of the transaction.
  5. Enter or complete the detailed costing analysis: Screenshot: Breakdown Section

    • The Commodity Code on the detail lines defaults from the commodity code in the Allocation section (which  identifies based on the merchant category code).  You can change the codes on individual lines to match the goods purchased.  All (non-blank) lines must have a valid commodity code.
    • If the uses WBS codes, the WBS code on the detail lines defaults from the WBS code in the Allocation section (on transactions from level 3 merchants, may have identified the WBS code from the buyer reference, if one was entered).  You can change the codes on individual lines to match the actual work breakdown.
    • You can add new lines (for example, to distribute commodities to different WBS codes).
    • Ensure that the total of the amounts is equal to the Net Amount.

      NOTE

      The important thing is to enter the correct amounts.  does not calculate the line amounts from the quantity and price, and does not validate the Unit or Per fields.
  6. Enter the full distribution.
  7. Click to save your changes.
  8. To hold a transaction (so that it is not included in the PCard batch), select it, choose Mark as Held from the Choose Action list and click .

    You might hold a transaction if, for example, you want to investigate it further or speak to the card holder; eventually, though, you must allow the transaction. 

When you have corrected any errors, you can create a batch of transactions.

Configuration

Administration users need to be set up as administrators for one or more administration ; use PCard Administrators.
