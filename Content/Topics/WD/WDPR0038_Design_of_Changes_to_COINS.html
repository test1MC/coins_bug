Design of Changes to

Enabling online submission requires changes other than simply how the information is transmitted.The following design changes were made:

  • now holds extra information previously hand written on to forms.
  • It now allows correction of in year mistakes.
  • It now handles multiple P14s for an employee in a single year. They can now have test copies, an original and amendments. Online submission works by sending an amended return with the movements; the differences from the original.
  • It now validates the return content pre-submission. Note that the additional validation is more exacting than previously required, for example a can only contain letters, hyphens and apostrophes; spaces and any other character would cause the return to be rejected.

The data that makes up a P14 is now held in allowing you to see a true version of what was submitted, and to over-ride the values derived from elsewhere in . The P35 and P60 print routines now allow you to print data as is, or from a particular return.

If you are working with a return for the current tax year then will work out the values - taking them from employee items and NI details. Working on a return from a previous year this will only happen when you create the return, and you will be allowed to amend any of the values on the form.

The return has statuses similar to a batch in :

  • On creation it will be Incomplete.
  • When the P35 details have been entered you can run a routine that will validate the content and set the status to Complete.
  • Once complete you can submit the document, initially changing the status to Submitting, and then to Submitted on HMRC acceptance.

Completion for a large payroll may be time-consuming, so to minime the number of times the validation needs to be run, you are prevented from changing values once the return is complete. To change values you must un-complete the return.
