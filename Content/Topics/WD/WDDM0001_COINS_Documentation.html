Documentation

The Documentation system has been developed to meet the following requirements:

  • To provide documentation for in addition to the side-frame field and function help.  It allows us to use common information (for example, standard text, glossary definitions and parameter explanations) in both the online documentation and the side-frame help.
  • To make the maintenance of documentation more accessible to staff, allowing more people to be involved in producing it.
  • To allow clients to add to and amend the documentation provided by , to suit their own way of using .
  • To provide a framework that clients can use to produce their own documentation.
