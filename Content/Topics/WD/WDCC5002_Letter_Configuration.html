Letter Configuration

 

In you can generate letters to send to one or more individuals or companies. You need to set up a mail merge document in Microsoft Word, using a datafile that generates. On some letters, you can include information from the relevant captions.  

To configure a letter:

  1. Go to Letter Configuration. Letter Configuration Browse

  2. Use the filter to display the letter type you want to add (, Call, Issue or Task).Search Filter

  3. Click to apply the filter
  4. Click to add a new letter.
  5. Fill in the fields Main Tab

    Field

    Description

    LetterThe code to identify the letter.
    DescriptionA description of the letter.
    DatafileThe location (usually on your PC) and filename that you want to use for the mail merge data file. When you first set up a letter this will not exist; will create it.
    DocumentThe location and name of the Word master document. When you first set up a letter, this will not exist.
    MacroThe code of the DDE macro that opens Word from COINSplus. This is not needed for COINS OA.
    OA/COINSPlus UsageWhich interfaces the letter will be used in: COINSplus, COINS OA, or both.
    Automatically Merge Document

    When running a mail merge letter from OA, whether the unmerged or merged document is loaded into Word.

    Ticked = The merged document is loaded into Word.

    Unticked = The unmerged document is loaded into Word.  This gives you the opportunity to edit the letter first then manually run the mail merge.

    Parameters Tab

    The Parameters tab allows you to enter information that is specific to the type of letter you are configuring.

    Field

    Description

    Event CodeThe code of the event that will be added to the relevant history when the letter is generated. If you leave the event blank, the history will not record that the letter has been generated.
    Contact SelectionWhether the user is able to select at run time which contacts to send the letter to.
    Contact TypesThe types of contact you want the letter sent to.  Use $PURCHASER for the original purchaser.
    CommentsComments to add to the event history.
    OptionsHow many letters you want to produce (a single letter, one per address, or one per person).
  6. Click to save the letter.

    The letter description will appear on the Options menu for the relevant workbench.

    Options Menu

  7. Run the letter event to generate a sample data file that you can use to set up the Word document. 

    8. saves the datafile you specified, but since the Word document does not yet exist, it produces an error.

  8. Create a mail-merge letter in Word, using the datafile as the data source. Note that there are various merge fields for the different people on the record; the combined name fields, like Combined_Names and Combined_Full_Names, will be different depending on whether the letter is configured to create a single letter, one per address, or one per person.
  9. Save the document.
    • For, save the document with the file name and location you entered in the Document field.
    • For , depending on how is set up, either:
      • Place the document in the /word/ subdirectory of the environment on the server; or
      • Place the document in the location you specified.
      The file name must be the same as the name you entered in the Document field. 

Now when a user chooses the letter from the workbench Options menu, generates the mail merge and opens the Word document in a new window.

NOTE

The first time a user generates a letter, they will need to download an ActiveX; to do this they will need to have set up as a trusted site. See Setting Internet Explorer to Allow ActiveX Controls.
