Setting Up PCard Merchants

Merchants are the providers of goods and services as identified on the PCard transactions.  needs to be able to map the merchant on a transaction to a account code.

  1. Use PCard Aliases.
  2. Set up a default line (without specifying a ) to specify which each merchant name corresponds to.
  3. If necessary, add lines to map the merchant names to different codes in individual .
  4. If a transaction with a merchant name that is not in this list arrives on the PCard Workbench, the field will be blank.  When the user selects the appropriate and saves the transaction, will add this as new mapping line in PCard Aliases.

You may also want to set up one or more Miscellaneous records in , to allow users to allocate transactions even if the actual has not been set up.

 

Each merchant has a Merchant Category Code (MCC) that describes the goods or services they provide.  You can map these to detail commodity codes, to provide with a default analysis for transactions that relate to the merchant.

  1. Use PCard Merchant Category Codes to map merchant categories to commodities. You can use the bulk add/update feature to import the master list of Merchant Category Codes. 
  2. For each merchant category, you can specify a maximum transaction threshold; if a transaction exceeds this amount, a warning will be displayed.
  3. You can also specify that a particular MCC is not normally allowed (for example, if you would not expect card holders to purchase goods of that type).  Transactions for merchants with these merchant categories produce a warning on the PCard workbenches which the administrator must specifically "allow" through.

Merchant Levels

There are three different levels of merchant; this relates to the transaction information they provide.

Level 1: The incoming XML transaction message only gives the gross amount of the transaction.
Level 2: The XML message gives the gross amount and the amount; also (if the card holder provided one) a point of sale reference. The point of sale reference can be a code, an overhead department, or a /WBS code. can then use these to work out the costing analysis for the transaction.
Level 3: The XML message gives the gross amount, the amount, the point of sale reference (if the card holder provided one), and also the line item details.
