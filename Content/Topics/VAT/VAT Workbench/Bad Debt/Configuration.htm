VAT Bad Debt Configuration

The aim of this topic is to detail the steps required to configure and use VAT Bad Debt Relief functionality in .

Prerequisites

 

 

 

The VAT Bad Process involves transferring amounts between various control accounts - please ensure Finance/VAT manager have approved the process and nominated the correct General Ledger accounts.

Before running the Bad Debt process – please ensure all cash is allocated.

Parameters

There are several new parameters that need to be configured:

Parameter

Notes

CO/BDACCT

GL Account code for bad debt VAT suspense account.

For transactions flagged as bad debt this account will be used instead of the VAT suspense account on the VAT configuration record.

PL/BDACCT

GL Account code for bad debt control account.

For transactions flagged as bad debt this account will be used instead of the standard control account. Can be sectored.

SL/BDACCT

GL Account code for bad debt control account.

For transactions flagged as bad debt this account will be used instead of the standard control account. Can be sectored.

SC/BDACCT

GL Account code for subcontract ledger bad debt control account.

For transactions flagged as bad debt this account will be used instead of the standard subcontract ledger control account. Can be sectored.

CS/BDACCT

GL Account code for ledger bad debt control account.

For transactions flagged as bad debt this account will be used instead of the standard ledger control account. Can be sectored.
