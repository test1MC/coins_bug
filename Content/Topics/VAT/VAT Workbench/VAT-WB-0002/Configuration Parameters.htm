VAT Workbench Configuration

 

 

UK-specific VAT parameters.  

 

UK Vat Parameters can be found in:

 

VAT > Setup > Configuration > UK VAT Parameters

 

UK VAT Parameters allows you to set parameters that govern the electronic submission of UK VAT returns to HMRC.

 

There are 10 specific UK Parameters for VAT workbench. These parameters cover how the Workbench communicates with the Government Gateway Transaction Engine (this is a centralised registration service for e-Government filing services allowing the exchange of electronic data. Used for PAYE year-end returns as well as for CIS monthly returns and verification requests).

VAT Workbench Parameters:

 

Some of Parameters used for the government Transaction engine are also used in the Payroll and or Subcontract modules. These Values can be used for the VAT workbench

Parameters further details

 

Parameter

Description

VTUK/EGOVMODE - VAT Return Submission Mode

The submission mode; this must be set to 0 for live operation.

0 = Live operation.

1 = Test in live.  This service is not currently supported by HMRC, and will not be supported until further notice.

2 = Vendor test.  For use only, Do Not Use.

3 = Put XML Only. This allows you to create an output XML file but not to transfer it to HMRC. This can be used for test purposes.

VTUK/EGOVRDIR - VAT Return XML root directory

The base directory for XML (Extensible Markup Language - a markup language based on HTML, often used for creating structured data files) submissions to the Government Transaction Engine - For example, $BASE/var/egovxml.

The ASCII text files that creates during a submission attempt will be held in a subdirectory of this root directory. The subdirectory name is the same as the 'message class', and is created automatically. These are:

  • HMRC-VAT-DEC - Live Submissions
  • HMRC-VAT-DEC-TIL  - Test in Live Submissions

All users who will send the Internet submissions will require permissions to read and write from this directory.

If you have more than one submission company, you may want to create separate subdirectories for each submission company, to make it easier to locate the XML for each company.

For example: $BASE/var/egovxml/company16

VTUK/EGOVTYPE - VAT Return Submission Type

VAT Return HMRC Submission Class.  

Must be empty or set to HMRC-VAT-DEC for Live submissions.

Must be set to HMRC-VAT-DEC-TIL for Test in live submissions.

VTUK/JAVAHOME - VAT Return Java root directory

The root directory of the Java installation on the server.

The report which will provide after running the Daltrey* test (a test that can run to confirm that your server can communicate with the Government Transaction Engine (a centralised registration service for e-Government filing).

VTUK/JAVAJRE - VAT Return Java jre directory

The Java Runtime Environment (JRE) subdirectory of the Java root directory.

The report which will provide after running the Daltrey test (a test that can run to confirm that your server can communicate with the Government Transaction Engine. Coins Support run test if required.

VTUK/POLLINT – VAT

Return Polling interval (Seconds)

The interval (in seconds) between request events by to check whether the Government Transaction has a response from HMRC to submissions.

To comply with HMRC guidelines on polling, recommend you set this to 10 seconds or more.

VTUK/POLLMAX - VAT Return Maximum number of polls

The maximum number of times that will re-check for a response via the Government Transaction Engine before timing out.

If has not received a response after this number of attempts, the status of the submission is set to "Timeout" and creates a new delayed request (see POLLNEXT).

To comply with HMRC guidelines on polling, recommend you set this to 10.

VTUK/POLLNEXT - VAT Return If timeout minutes to re-poll

The number of minutes waits before creating a new polling request.

If polling times out (see POLLINT and POLLMAX), creates a new (delayed) request with a start time based on the value of POLLNEXT.

suggest you set this to 10.

VTUK/RPTQUEUE - VAT Return Submission Report Queue

The report queue to use for electronic VAT Return submissions to HMRC.

uses the OA background report generator for electronic submissions to HMRC.  It is Possible to set up more than one queue, to give different priority to different types of request

VTUK/VTRXMLNS - VAT Return Verification XMLNS

The XML namespace for VAT Returns,

that is: http://www.govtalk. gov.uk/ taxation/vat/vatdeclaration/2
