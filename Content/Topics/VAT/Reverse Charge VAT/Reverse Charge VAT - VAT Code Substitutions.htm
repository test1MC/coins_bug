Reverse Charge - Code Substitutions

A new function is available to hold the Code Substitution rules.

NOTE

This is only shown for UK (that is, where CO/COMPLOC = UK).

This is only shown for in which CO/COMPLOC is not US or CA.

Normally this table would include two entries for each Source code with two different Target codes. In the case of Reverse Charge or Acquisition , one of the Target codes would be set as Output while the other would be set as Input. In the case of Partially Recoverable both target codes would be set as Input. In some rare cases it may be required to have just one or more than two entries for the same Source code.

 Code Substitutions Screen

Fields

Field
Description

Source Code

A valid code for Input (it may be one set up as input or as both input and output but not as just output)

Target Code

A valid code.

Calculation

How the  amount on the new  distribution line is calculated. This can be one of:

  • Copy Source Amount
  • Calculate from Target Net Amount at Target Rate
  • Calculate from Source Net Amount at Target Rate
  • Remaining to be distributed

Net Amount Percentage

A number between -100.00 and 100.00 (held and displayed to two decimal places).

Output/Input
This must be set to either input or output. This must be compatible with the setup of the Target Code.
Description
Text to explain the purpose/usage of this substitution entry.
