Overview

Previously, the Default Daily Hours field in Payroll Company Configuration was uned, because time was input into COINS via Load Timesheets. However, there was a problem when transferring timesheets from SE because Daily Hour values were not being populated. Also, if the Default Daily Hours field was ed, the effort to modify all of the templates in use would be quite large and pose a risk for error.

The option to distribute Daily Hours during Transfer SE and Transfer Daily Reports has therefore been added to is available in Payroll and Service Management. This allows COINS OA users to implement Daily Reports (mCDRs), transfer timesheets from SE  that include populated Daily Hour values, and be able to use existing templates for Load Timesheets.

The changes to functionality are discussed in this document. It is relevant for the USA and Canada only.
