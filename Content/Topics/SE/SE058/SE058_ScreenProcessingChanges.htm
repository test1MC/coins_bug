Inception Date

The criteria in which the Inception Date cannot be changed (it is disabled) in Update mode are:

  • The has been selected for Renewal.
  • The has been renewed.

Validation Updates

Since changes to the Inception Date are carried to all appropriate fields throughout the , certain validation processes have been updated to avoid errors that could occur. These validation processes are described in this section.

Amount - and First Billing Date

The goal of this validation update is to retain the original date order, but still pass all required validation.

The and First Billing Date are changed for all Location/Service Department records where the equals the old Inception Date (there could be more than one, since the can reference multiple Locations/Service Departments).

If the has subsequent Amount record(s) on file for a Location/Service Department with aan that will conflict with the change made to the first record, any subsequent records are also adjusted. All subsequent records are moved one day forward until there are no conflicts in the date.

Billing Schedule – Scheduled Bill Date

There is no validation on the Scheduled Bill Date referenced within the Billing Schedule. Users must adjust any data.

Revenue Schedule –

There is no validation on the referenced within the Revenue Schedule. Users must adjust any data.

Renewals – Start Date

The Start Date is changed for the first Renewal record where the Start Date equals the old Inception Date. Users must change the End Date.

NOTES:

  • Any Planned Inspections and Covered Asset Inspections with a date outside the range of the Renewal Period are removed. Users must adjust the tasking data.
  • The Cost Schedule and Cost Budgets contain validation to prevent non-zero values for out of the Renewal date range. Users must remove any values covered by the new Inception Date prior to changing the Inception Date.

Term – Start Date

The Start Date is changed for the first Term record where the Start Date equals the old Inception Date. The End Date is automatically recalculated when the Start Date is changed, and the Duration remains unchanged.

If the resulting dates overlap other Term records, users must adjust the other records to provide no overlap in coverage dates.

Type –

The goal of this validation update is to retain the original date order, but still pass all required validation.

The is changed for the first Type record where the equals the old Inception Date.

If the has subsequent Type record(s) on file with aan that will conflict with the change to the first record, any subsequent records are also adjusted. All subsequent records are moved one day forward until there are no conflicts in the date.
