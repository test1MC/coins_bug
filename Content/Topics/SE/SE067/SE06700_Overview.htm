Overview

The ability for users to track the reason a customer may void a is now available in Service Management. It requires a new SE/VOIDREAS parameter to be set to Yes. This functionality can also help users identify customers who are voiding and whether a customer may need training on entry.

Configuration

The following SE parameter was added for this functionality.

Parameter

Description

VOIDREAS

Require Void Reason Code when Voiding (Y/N)
