Import Covered

A new An import called Import Covered is available in Service Management to import tasking data. In addition to the ability to import Covered , it can create master Location when the is not on file. This function also supports a Workspace area.

Import Data Menu

A new An Import Covered submenu is available under the Import Data menu in Service Management > Administration, as shown in Figure 1: Import Agreement Covered Assets Menu. This new submenu allows you to create Covered , as well as master Location records when the is not on file. It includes two functions:

  • Import - This provides you with an Example CSV file. The columns in the Example CSV file are configurable, so that you can rearrange or remove them as needed. See Section 1.2 Import for more information.
  • Workspace - The Workspace browse has column set capability, so that you can view the elements of data you have imported. See Section 1.3 Workspace for more information.

Import Covered Menu

This import is designed as a multi-step function:

  1. Import the data into a temporary Workspace.
  2. Use the Workspace browse to make corrections to the data. You can also use the Delete button to remove individual rows of data.
  3. Select options from the Options menu to validate the data (Validate All Records), post the data (Post All Records), and clear out all data (Clear Workspace).

Import

The Import screen (Figure 2: Import Screen) allows you to import data for Covered from a CSV file. This data can then be imported into the Workspace, where it can be edited, validated, and posted.

Also found on this screen is a link to an Example CSV file. The columns in the Example CSV file are configurable, so that you can rearrange or remove them as needed. In order to create new records, the CSV spreadsheet must contain the Location fields. Since the Location has three mandatory fields, the three columns required to create Location on the CSV spreadsheet are: Code, Type, and Description.

Import Screen

Create Location

The Import selection screen has a Create Location field, which is also included as a field in the column set. If this field is ed and the corresponding Location is not on file, the column is set to YES. The Validation routine under Options then recognes the Create Location value and responds accordingly.

Validation must ensure that the two location created for a Location have the same Code, Type, and Description within the Workspace.

Workspace

The Workspace browse has column set capability, so that you can view the elements of data you have imported. Here you can edit, validate, and post the imported data. A sample browse is shown in Figure 3: Workspace Browse.

The Status column is found in the Workspace browse. A record is flagged if it does not pass validation to qualify as a valid Covered or a Create Location .

Workspace Browse

Options Menu

The Options menu at the top of the Workspace browse contains the following options: Validate All Records, Post All Records, and Clear Workspace, as shown in Figure 4: Validate All Records Option. Each option is explained below.

Validate All Records Option

Validate All Records

Use this option to perform validation of the Workspace records.

The Status of a validated record is shown as a green circle with the tooltip appearing as "OK", as shown in Figure 5: Green Circle Status.

Green Circle Status

Post All Records

All records in the Workspace must be valid in order for the Post to process. If at least one record is not valid, an error message is shown (see Figure 6: Validation Error Message) and posting is aborted.

The Post process removes all Workspace records upon successful completion.

Validation Error Message

Clear Workspace

This function lets you clear the Workspace of all data.
