Days To Invoice Report

The new Days To Invoice Report is a standard report showing the number of days needed to invoice work after it has been completed (including weekend days). It is a metric used to determine the efficiency of each Service Department.

To accommodate that can be billed multiple times, the report counts the number of days between the first Dispatch Start Date and the first Invoice Date, as well as the number of days between the last Dispatch Start Date and the last Invoice Date. If there is only one invoice for the , the second value is more relevant since the billing was done after the work was complete.

Menu Item

A new Days To Invoice Report menu option is available on the Service Management > Reports menu, as shown in Figure 1: Report Menu.

Selection Screen

When you select Days To Invoice Report from the SE Reports menu, a selection screen appears. This screen contains a Selection tab, as shown in Figure 2: Report Selection Screen.

The Billing Complete Date From/To option lets you exclude orders that are still in the billing phase.

Report Selection Screen

Report Sample

After you enter the criteria on the selection screen, a Days To Invoice Report is generated. The report data are listed by /Location/Customer Name, as shown in Figure 3: Report Sample.

  • In order for a to appear on the report, it must be marked as Schedule Complete and have at least one posted invoice.
  • Scheduled Billing are excluded from the report (regardless of the selection screen) if the contains only that particular Billing Detail Type, which has no schedule/dispatch data.
