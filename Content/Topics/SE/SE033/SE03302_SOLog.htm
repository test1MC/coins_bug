Log

The Log tab of the function has been changed to is a drop-down menu, as shown in Figure 1: Service Order Log Menu. This menu has 3 options: All Log Entries, Problem Code Log, and Resolution Log. These options are discussed in the sections that follow.

Log Menu

All Log Entries

The All Log Entries menu option displays information for both Problem Codes and Resolutions, as shown in Figure 2: All Log Entries.

All Log Entries

Problem Code Log

The Problem Code Log menu option provides you with a quick way to see any data related to a Problem Code, as shown in Figure 3: Problem Code Log. The records on this browse are shown with a Log Type of either ProbCode or ProbCon. A Consolidated column helps to identify whether a source Problem Code Comment has been consolidated.

Problem Code Log

Filters

A named filter has been added to on the Problem Code Log browse with has the following options: All, Source, Consolidated.

  • Source - The Log Type is ProbCode and displays the original source comments only.
  • Consolidated - The Log Type is ProbCon and displays the consolidated comment(s) only.
  • All - Both Log Types are displayed.

Consolidate box and Buttons

An Exclude Consolidated Comments box and three Consolidate buttons are found at the bottom of the browse screen. Each is explained below.

Exclude Consolidated Comments

This box allows you to exclude comments that have already been consolidated during previous consolidate processing. This option works best for New and Append Consolidate processing.

CONSOLIDATE (NEW)

When this button is clicked, a new Log Comment is created, which organes either all or non-consolidated ProbCode entries in Date/Time order.

There is no limit on the number of consolidated comments in this scenario. You can prepare multiple consolidated comments, each containing the desired text.

CONSOLIDATE (APPEND)

If a Consolidated Comment already exists, either all or non-consolidated ProbCode entries in Date/Time order are organed at the end of the existing text.

This option is only available if there is one consolidated Problem Code comment on file (ProbCon) that has a Comment Type equal to the SE/CMPROBCN value.

Users are responsible for deleting duplicate text.

CONSOLIDATE (REPLACE)

If a Consolidated Comment already exists, the comment text is replaced with either all or non-consolidated ProbCode entries in Date/Time order.

This option is only available if there is one consolidated Problem Code comment on file (ProbCon) that has a Comment Type equal to the SE/CMPROBCN value.

When one of the above buttons is clicked, a new Log Entry is created with a Log Type of ProbCon.

  • If the Problem is for a particular , then the ’s description is included at the beginning of the text in parentheses.
  • A consolidated comment does not include the Problem Code or the Problem Code description.
  • The consolidated comment can be edited, including the Comment Type.
