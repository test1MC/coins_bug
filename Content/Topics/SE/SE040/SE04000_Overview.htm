Overview

Since Service Management uses a common Costing , it could not reference a specific Line of Business and/or Profit Center. As a result, users could only create multiple Cost Codes where the G/L Elements were hardcoded, and the Cost Codes for a could only be masked based on the data available in Status. In a typical situation, Cost Codes used the Natural Account from the Standard Cost Type file, combining it with the other G/L Elements from the /Section.

This process has been improved, so that users now have the ability to Users can set up G/L Account Masking directly within Service Management. The Masking information within Service can be used during Cost Code creation, thus blending the G/L Elements defined in Service with the Natural Account in the Standard Cost Type file.

Changes made to improve this process are described in this document.
