The A new SE parameter, ABTAXPCT, in COINS OA allows an Billing Sales Taxable % Method. When this is set to Y, the % Taxable field is visible in the , regardless of whether the Revenue Tax Code is referenced.

Parameter

Description

ABTAXPCT

Allow Billing Sales Taxable % Method

Note: The % Taxable field will be seen in the when the parameter is set to Y, regardless of the Revenue Tax Code referenced.

A new Scheduled Billing Taxable % field is available under Scheduled Billing in > Locations tab > Allowable Work Details > Billing & Sales Tax tab, as shown in Figure 1. It is only visible and enabled when Scheduled Billing is selected as Revenue Taxable and parameter SE/ABTAXPCT is set to Y.

The value in this field allows for two decimal positions; it enforces the value to be positive and to not exceed 100. (In COINS Ti, this value is a whole number that cannot exceed 100.) The value of 0 in this field is recogned as 100 percent.

Billings

COINS Ti supports separate Taxable Amount values on a transaction through the use of a separate Taxable Amount field. However, in COINS OA, every transaction is taxable, and the value of the tax is based on the Tax Code referenced. The Generate Scheduled Billings and Generate Time of Service functions now allow you to create two separate Scheduled Billing Detail Lines (see Figure 2): (1) one line holds the taxable portion; and (2) the other line holds the non-taxable portion.

  • The taxable portion references the ’s Revenue Taxable Tax Code and is the first line created.
  • The non-taxable portion references the ’s Revenue Exempt Tax Code. Any rounding result is applied to the Taxable Detail Line. The sum of the two lines must equal the value of the Scheduled Amount.

  • The Generate Scheduled Billings function uses the Taxable % entered in the , since both the and the detail lines are created as one transaction. The Generate Time of Service (TOS) function (see Figure 3) uses the Taxable % visible on the ’s Allowable Work.

NOTE:

The Taxable % has no effect when a Scheduled Billing Detail Line (Figure 4) is manually entered. You must enter the applicable Amount and Revenue Tax Code on each Scheduled Billing Detail Line added.

Print Invoices

If you wish to use the Taxable % process, you must have your Invoice Documents modified. This is due to the Print Invoices function only printing one line for all Scheduled Billings Detail Lines on a if the ’s Allowable Work Taxable % is not 100 (or 0).

RO fields now on the Invoice header contain the Invoice Taxable Amount (RO_sei_TaxableAmt) and Invoice Nontaxable Amount (RO_sei_NontaxableAmt). You may want to have your Invoice Documents adjusted to print the breakout of Taxable and Nontaxable amounts.

  • If the Invoice Line’s Tax value is <> 0, the Line’s Taxable Amount is calculated. Note: Retention is not considered as Taxable.
  • The difference between the Line’s Amount and the Taxable Amount is reported as the Nontaxable Amount.
