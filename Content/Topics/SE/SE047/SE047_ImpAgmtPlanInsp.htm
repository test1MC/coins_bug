Import Planned Inspections

A new An import called Import Planned Inspections is available in Service Management to import Planned Inspections. This function also supports a Workspace area.

In order for this Import to be able to run, the Details, Locations, Allowable Work, and Cost Codes must already be on file.

Import Data Menu

A new An Import Planned Inspections submenu is available under the Import Data menu in Service Management > Administration, as shown in Figure 1. This submenu allows you to create Planned Inspections. It includes two functions:

  • Import - This provides you with an Example CSV spreadsheet file. The columns in the Example CSV file are configurable, i.e., you can rearrange or remove them as needed. See Section 1.2 for more information.
  • Workspace - The Workspace browse has column set capability, so that you can view the elements of data you have imported. See Section 1.5 for more information.

Import Planned Inspections Submenu

This import is designed as a multi-step function:

  1. Import the data into a temporary Workspace.
  2. Use the Workspace browse to make corrections to the data. You can also use the Delete button to remove individual rows of data.
  3. Select options from the Options menu to validate the data (Validate All Records), post the data (Post All Records), and clear out all data (Clear Workspace).

Import Screen

The Import screen (Figure 2) allows you to import data for Planned Inspections from a CSV file. This data can then be imported into the Workspace, where it can be edited, validated, and posted.

Also found on this screen is a link to an Example CSV file. For more information on this CSV spreadsheet, see Section 1.2.1.

Example CSV Spreadsheet

The columns in the Example CSV file are configurable, i.e., you can rearrange or remove them as needed. In order to create new Planned Inspection records, the CSV spreadsheet must reference the mandatory fields required to create an Planned Inspection.

The spreadsheet (and Workspace) only reference the Planned Inspection header information (see Figure 3), including , Location, Service Department, and Renewal. It does not reference monthly columns since the monthly columns reflect each ’s Renewal Date information and are therefore different between . When posting the imported data, a monthly schedule is generated and based on the Standard Schedule referenced.

Workspace

The Workspace browse has column set capability so that you can view the elements of data you have imported. Here you can edit, validate, and post the imported data. A sample browse is shown in Figure 3.

The Status column is found in the Workspace browse. Also, a record is flagged in an Alert (red flag) column if it does not pass validation to qualify as a valid Planned Inspection.

Workspace Browse

Options Menu

The Options menu at the top of the Workspace browse contains the following options: Validate All Records, Post All Records, and Clear Workspace, as shown in Figure 3. Each option is explained below.

Workspace Options Menu

Validate All Records

Use this option to perform validation of the Workspace records. A record is flagged if it does not pass validation to qualify as a valid Planned Inspection.

The Status of a validated record is shown as a green circle with the tooltip appearing as "OK", as shown in Figure 3.

Green Circle Status

Post All Records

All records in the Workspace must be valid in order for the Post to process. If at least one record is not valid, an error message is shown (see Figure 5) and posting is aborted.

The Post process removes all Workspace records upon successful completion.

Validation Error Message

Clear Workspace

This function lets you clear the Workspace of all data.

RO Fields

Since some Planned Inspections may not be set up to generate the Covered Inspection data, you must individually manage these . Nine new RO fields (Figure 11) have therefore been added to the function and the to help you do this. They can help identify in which the Planned Inspection values and Covered Asset values are different.

All of these RO fields are found in a column set at the bottom of the screen. They contain Planned, Tasked, and Remaining totals in columns for # of Pieces, Hours, and Amount($).

RO Fields

  • The RO fields are the sum for all Locations, all Service Departments, and the current Renewal (based on today).
  • The RO fields support parameters for Location, Service Department, and Renewal.
