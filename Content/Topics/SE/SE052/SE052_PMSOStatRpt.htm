PM  Status Report

The COINS OA Service Management module now has a standard report to show the status of Planned Maintenance (PM) backlog. The new PM Status Report lets you see the detailed or summared status of Planned Maintenance for a specific Period.

Note:

This report is not fiscal sensitive. The hours performed are obtained from detail, not from posted/incurred hours.

Menu Item

A new PM Status Report option has been added to the Service Management > Reports menu, as shown in Figure 1: Report Menu.

Report Menu

Selection Screen

When you select PM Status Report from the SE Reports menu, a selection screen appears. This screen contains a Selection tab, as shown in Figure 2: Report Selection Screen.

The total number of selected for inclusion serves as the denominator for the On Time percentage analysis in the Report totals.

Report Selection Screen

Field Definitions (Selected Fields)

On Time Completion Number of Days

Enter the number of days between the ’s Scheduling Complete Date and Effective Date. The default value is 30.

If the number of days is equal to or less than the On Time Completion Number of Days, the is considered to be On Time.

The report totals provide percentage of completed on time.

Show Scheduling Complete

Indicate whether you wish the report to list only the PM that are marked as Scheduling Complete.

If set to N (not selected, which is the default value), the report lists only the PM that are still “open” for scheduling.

If set to Y (selected), the report also lists PM that are marked as Scheduling Complete.
