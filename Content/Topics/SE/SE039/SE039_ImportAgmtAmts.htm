Import Amounts

A new An import called Import Amounts is available in Service Management to import Amounts. This function also supports a Workspace area.

In order for this Import to be able to run, the Details, Locations, Allowable Work, and Cost Codes must already be on file.

Import Data Menu

A new An Import Amounts submenu is available under the Import Data menu in Service Management > Administration, as shown in Figure 1: Import [%=Variables.Se Agreement caps%] Amounts Submenu. This submenu allows you to create Amounts. It includes two functions:

  • Import - This provides you with an Example CSV file. The columns in the Example CSV file are configurable, that is, you can rearrange or remove them as needed.
  • Workspace - The Workspace browse has column set capability, so that you can view the elements of data you have imported.
Import Amounts Submenu

This import is designed as a multi-step function:

  1. Import the data into a temporary Workspace.
  2. Use the Workspace browse to make corrections to the data. You can also use the Delete button to remove individual rows of data.
  3. Select options from the Options menu to validate the data (Validate All Records), post the data (Post All Records), and clear out all data (Clear Workspace).

Import

The Import screen (Figure 2: Import Screen) allows you to import data for Amounts from a CSV file. This data can then be imported into the Workspace, where it can be edited, validated, and posted.

Also found on this screen is a link to an Example CSV file. The columns in the Example CSV file are configurable, that is, you can rearrange or remove them as needed. In order to create new Amount records, the CSV spreadsheet must reference the mandatory fields required to create an Amount.

Generate Billing and Revenue Schedule

This box allows you to mark Amount records for the automatic generation of Billing and Revenue Schedule records when Import posting is complete. (The Generate Billing and Revenue Schedule function under Import Data can also be used when needed.)

In order for the Billing and Revenue Schedule to generate correctly, you must make sure that: (1) the Amount record has a corresponding fields value set; and (2) the Effective Date, First Billing Date, and Amount import values are set.

Generate Billing and Revenue Schedule Function

In addition to the Generate Billing and Revenue Schedule box discussed above, Generate Billing And Revenue Schedule is provided as a separate function under the Import Data menu in Service Management > Administration, as shown in Figure 3: Generate [%=Variables.Se Agreement caps%] Billing And Revenue Schedule Function. It can be used when schedules are not generated while posting the Amount.

Generate Billing and Revenue Schedule Function

As usual, you can also still generate Billing and Revenue Schedule records through the Generate Billing Schedule and Generate Revenue Schedule actions on > Operational Setup > Amount, as shown in Figure 4: [%=Variables.Se Agreement caps%] Amount Actions.

Workspace

The Workspace browse has column set capability so that you can view the elements of data you have imported. Here you can edit, validate, and post the imported data. A sample browse is shown in Figure 5: Workspace Browse.

The Status column is found in the Workspace browse. Also, a record is flagged in an Alert (red flag) column if it does not pass validation to qualify as a valid Amount.

Workspace Browse

Options Menu

The Options menu at the top of the Workspace browse contains the following options: Validate All Records, Post All Records, and Clear Workspace, as shown in Figure 6: Workspace Options Menu. Each option is explained below.

Workspace Options Menu

Validate All Records

Use this option to perform validation of the Workspace records. A record is flagged if it does not pass validation to qualify as a valid Amount.

The Status of a validated record is shown as a green circle with the tooltip appearing as "OK", as shown in Figure 7: Green Circle Status.

Green Circle Status

Post All Records

All records in the Workspace must be valid in order for the Post to process. If at least one record is not valid, an error message is shown (see Figure 8: Validation Error Message) and posting is aborted.

The Post process removes all Workspace records upon successful completion.

Validation Error Message

Clear Workspace

This function lets you clear the Workspace of all data.
