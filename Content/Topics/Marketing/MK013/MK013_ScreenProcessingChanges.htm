MK Personnel Workbench

Personnel Detail Tab

A number of new fields are now available on the Personnel Detail tab, as shown in Figure 1. These fields behave differently depending on the HRLINK parameter value:

  • If HRLINK=Y, fields are read only in , and data are synchroned one-way from the corresponding fields in the HR Personnel Workbench.
  • If HRLINK=N, fields are lookups and do not link with the HR Personnel Workbench.
  • The Director, Supervisor, Manager, and Works To fields use Personnel.
  • The HR Group, HR Sub Group, and Analysis Sets fields use HR lookups.

Personnel Detail Tab

Field Definitions (Selected Fields)

Director

The employee's director.

  • If HRLINK is Y, only employees with Director ed in Job Assignment of the Organation tab (HR module) appear in this field.
  • If HRLINK is N, any personnel can be used as Director.

Supervisor

The employee's supervisor.

  • If HRLINK is Y, only employees with Supervisor ed in Job Assignment of the Organation tab (HR module) appear in this field.
  • If HRLINK is N, any personnel can be used as Supervisor.

Manager

The employee's manager.

  • If HRLINK is Y, only employees with Manager ed in Job Assignment of the Organation tab (HR module) appear in this field.
  • If HRLINK is N, any personnel can be used as Manager.

Works To

The 'works to' person for this employee, if relevant, such as a safety representative or project manager.

HR Group

The group to which the employee belongs. Groups are a way of structuring people in your company. For example, you could have different geographical areas which people can belong to.

HR Subgroup

The subgroup to which the employee belongs. Subgroups are related to groups and can help you structure your organation. If you had a group called North Region, you could have a subgroup of North East Region.

Regenerate Function

Menu Option

The parameter MK/HRORG allows for some fields in HR to be synched with the same fields in . Previously, this synch only happened when the already existing parameter (MK/HRLINK) was set to yes. This resulted in a tedious process requiring users to choose the fields to be synched in Parameters and then save each linked HR Personnel.

To resolve this, a function called HR Hierarchy Fields is now available under the > Administration > Regenerate Data menu, as shown in Figure 20.

HR Hierarchy Fields Menu Option

Selection Screen

When you select HR Hierarchy Fields from the Regenerate Data menu, the selection screen appears, as shown in Figure 21.

Regenerate Data Selection Screen

If the Confirm field is ed, the HRLINK parameter is checked first.

  • If HRLINK is set to N, a one line report that reads “HRLINK parameter is set to No, no changes are necessary” is generated.
  • If HRLINK is set to Y, the following process occurs:
  • The report reads through each Personnel. If the Personnel is linked to an HR Personnel:
  • All of the fields listed in the HRORG parameter are updated, as well as the fields that HR currently syncs when an HR record is updated.
  • A line is written to the Status column on a report showing that this was done (see Figure 21).

Sample Report

A sample of the report generated after running the HR Hierarchy Fields function is shown below in Figure 21. The following columns are found on the report: HR Int. Ref., Int. Ref., , First Name, Second Name, Status, and Message.

HR Hierarchy Fields Report

This report may show the following statuses:

  • Synced
  • Personnel not linked to an HR Personnel, no changes were made. (The HR Personnel column will be blank.)
  • HR Personnel Hierarchy fields are blank, no changes were made.
  • Personnel fields were updated from HR Personnel fields.
