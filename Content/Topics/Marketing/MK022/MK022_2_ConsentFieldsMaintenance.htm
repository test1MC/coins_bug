GDPR in : Consent Fields and Maintenance

SY Data Consent Categories

Several fields can be added to a Contacts record in order for you to record which type of correspondence a contact can receive. The Data Consent Categories function allows you to determine the data for which you wish to gather information. This function was originally only an HS function found in House Sales > General Setup. It is now an SY function that can also be found in > Setup > General Setup. A sample screen is shown in Figure 1.

You can use this function to set up methods of contact (such as Email, Fax, Cell Phone, Work Phone) and whether the contact can be contacted via that method.

Data Consent Categories Screen

To accommodate the information for both House Sales and , the columns were changed as follows:

  • Replaced the Mapping column with the HS/CC Mapping column.
  • Added a column labeled CRM Mapping with two choices: Do Not Mail and <Not Applicable>. After one of these records is marked Do Not Mail, the remaining records are marked as <Not Applicable> and cannot be changed.

MK Data Consent Migration

A new Data Consent Migration function is now available in > Administration. The selection screen allows you to set the values of the Consent fields (set up in Data Consent Categories) for all Contacts, as shown in Figure 2.

Data Consent Migration Screen

[Various Consent Fields]

For each field on this screen, you can select how you wish to handle consent information. The choices are:

  • Unknown
  • Yes
  • No
  • Do Not Mail

Source of Consent

Use this field to select the source for the data consent of the fields above. The options in this field are set up in MK Data Consent Sources.

SY Data Consent Sources

A Data Consent Sources function was originally only an HS function found in House Sales > General Setup. It is now an SY function that can also be found in CRM > Administration. It allows you to set up definitions for data consent sources (the places where you learned what a Contact's preferences were). A sample screen is shown in Figure 3.

A list of these sources are found in the Consent section of > Contacts > Contact Details, where they can be used to record the source of the consent data.

Consents Audit History Tab

A Consents Audit History tab is now found in the Main section of the new Contacts layout, as shown in Figure 4. Each time the value of a Consent Category or its source is changed, an audit of the change is shown on this screen.
