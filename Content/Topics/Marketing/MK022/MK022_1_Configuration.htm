GDPR in : Configuration

The MK parameters listed in the table below are now available for General Data Protection Regulations (GDPR) functionality.

Parameter

Description

MK/CONHP

Contacts Workbench Hide Parameter. This allows selections of fields from the Contacts WB to hide.

MK/CONSNOTEW

Error/warning if no data consent notes are entered.

MK/CONSUSE

Use Consent functionality. This includes a Consent Section on the Contact Details page and shows related maintenance files. If this is set to N, the standard “Do Not Mail” fields remain on the page.

CONHP Parameter

The MK/CONHP parameter allows you to select which fields on the Contacts Workbench should be hidden. The default is blank (no fields are hidden). It has a predefined list of the following fields that you can select to hide/not hide:

  • Start Date (pcc_start)
  • Date of Birth (pcn_dob)
  • Fax (ppc_fax)
  • Identifying Comments (ppc_indentifier)
  • Origin (ppc_origin)
  • Personal Assistant (ppc_passist)
  • Job Group (ppg_seq)
  • Role (ppj_seq)

CONSNOTEW Parameter

The MK/CONSNOTEW parameter allows you to choose whether to receive a hard stop (error) or a message (warning) if the consent functionality is in use and the data consent notes are blank. The default is a warning. The parameter is ignored unless the MK/CONSUSE parameter is set to Y.

CONSUSE Parameter

The MK/CONSUSE parameter allows you to turn the consent functionality on and off. The default setting is N.
