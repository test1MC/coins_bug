Field Definitions

User ID

The ID of the COINS user.

Name

The name of the user or group with this ID.

Security List

A can-do list of the codes from the control field that define which records this user is allowed to access. This field uses the MK/PISEC parameter (see Section 1.2) and, depending on the value set for this parameter, the lookup for this field is reloaded.

Code

A can-do list of record codes that allows you to grant or deny the selected user access on a record-by-record basis when entering transactions. If left blank, the user is allowed access to all records.

MK/PISECDEF

Access to for a user with no record in > Setup > Security > Security depends on how this parameter is set:

  • If set to Yes, the user does not have access to any .
  • If set to No, the user has access to all .
  • If left blank, there is no Security.

MK/PISECTYPE

Select one of the options below to identify the type of security to be used:

  • Field Security
  • Full Record Security

MK/PRFLDSEC

Use this parameter to select values for the Maintenance fields that should have Security. You can select multiple values from the Available column and move them to the Selected column, as shown in Figure 3. Any values in the Selected column will have security.

This parameter relates to parameter MK/PISECTYPE when its value is set to FS (Field Specific Security).
