Parameter Changes

This section describes any changes that were made to MK parameters to satisfy the requirements of mCRM and OA Webservices.

CLOSEDEF

OA Webservice Changes

The Default Close Date to Today's Date is now determined by the Y/N value of the MK/CLOSEDEF parameter when the Stage is changed to Closed-Won/Closed-Lost/Inactive.

App Changes

When a user changes the Substatus on the App, and MK/CLOSEDEF is set to Y, the Close Date defaults to Today’s Date.

CONTGRAD

OA Webservice Changes

The Webservice now does the following:

  • Checks the MK/CONTGRAD parameter; if it is set to N, the Grade field is hidden.
  • Provides data for the Grade lookup.
  • Displays the Grade field for Contacts.
  • Allows the App to now submit the Grade field when submitting Contacts.

App Changes

  • The App now downloads and stores the Grade field.
  • The Grade field is now available on the Contacts screen.
  • The Grade value can be selected from a lookup if it needs to be changed.
  • The App hides the Grade field if the MK/CONTGRAD parameter is set to N.

DISUNGRP

All MK OA Webservices now must only display records that the logged in user is allowed to access.

ESTCLOSE

The Webservice now references the MK/ESTCLOSE parameter, which allows the correct label to be displayed to the App for the Estimated Close field.

EXCDENTITY

OA Changes

The App can now reference the MK/EXCDENTITY parameter.

App Changes

Two OA Webservices now allow the App to download Work Group data; one service downloads the full list, and the other service downloads a filtered list. The App must download both lists (sets of data).

Depending on the type of record being updated, processing is as follows:

  • Company record: The App now checks the MK/EXCDENTITY mobile configuration value within a comma-separated list of values. If one of the values in the list is “ci_company”, the App uses the Work Group data (i.e., from either the full list or filtered list).
  • record: The App now checks for “pi_project” in the MK/EXCDENTITY mobile configuration value.
  • Contact record: The App now checks for “pp_contact” in the MK/EXCDENTITY mobile configuration value.
  • Office record: The App now checks for “ci_office” in the MK/EXCDENTITY mobile configuration value.

PISECDEF

The MK/PISECDEF parameter ensures that all MK OA Webservices must now only display records that the logged in user is allowed to access.

PISECTYPE / PRFLDSEC

OA Changes

The App can now reference the MK/PRFLDSEC parameter.

OA Web Service Changes

Two OA Webservices can now return records to the App.

  • If MK/PISECTYPE is set to Full Record Security (RS), the Webservices do not display any records that the user is not allowed to access.
  • If MK/PISECTYPE is set to Field Security (FS), and field security should be applied to some records, this new field should be set to True.

App Changes

  • The new mobile configuration item MK/PRFLDSEC can be downloaded and stored. It contains a comma-separated list of OA field names.
  • If any records received have field security, the fields contained in the MK/PRFLDSEC list are hidden.
