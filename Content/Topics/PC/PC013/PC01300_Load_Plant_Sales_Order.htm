Importing Sales Orders/Requisitions

Load Sales Order is a new function that allows you to load sales order (or requisition) from a CSV file. This provides a way of getting any large requirement into ; for example, when an external service takes design drawings and produces a parts list required to make formwork, falsework, scaffold, or site accommodation.

Load Sales Order – Main Tab

The CSV file can contain columns for Item Code, Quantity, Per, Price, Date, Date, Responsible Person, WBS Code and . You indicate on the CSV Columns tab which columns are in the CSV file and their order. The Item Code and Quantity must be present in the CSV file. If any of the other columns are missing, you can enter default values for them. The settings are stored and recalled for each supplier entered on the Main tab, as are the settings if the supplier is left blank.

Load Sales Orders – CSV Columns Tab

The Item Code must correspond to a valid item, which could be the item code, shortcut code, number or RFID/Barcode, as long as it gives a unique match. If the item is hidden or inactive, the validation will depend on the setting of the parameters STOPHIDD and STOPINAC.

If the Per column is not specified, the default group Per is used. If there is no default Per, an error is returned.

The is available if PC/FULLCOST is YES.

If PC/WFORD parameter is populated, the Trigger Workflow box is available for Load mode. If Trigger Workflow is ed, the workflow starts and the requisition is set to '05' status - Awaiting Approval.

You can enter the customer details for the requisition on the 'Customer/ Details' tab.

Load Sales Orders – Customer/ Details Tab

Normally the whole file will need to be error free to load, but the Load when File has Errors box allows you to override this.
