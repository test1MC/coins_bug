Shift Patterns

Shift Patterns is a new Central Repository option to define shift patterns for people and equipment. You can use these in the Asset Scheduler if you need to schedule specific timeslots.

The initial browse allows you to maintain the shift pattern records.

Shift Pattern Maintenance
Field
Description

Days, Start

Enter the number of days the shift pattern covers; if it starts on a particular day of the week you can select that. For example:

  • For a weekend shift pattern covering Friday evening to Monday morning, set Days to 7 and Start to Friday, then set up four days of shifts.
  • For a two-week shift pattern of early and late shifts Monday to Friday, set Days to 14, and Start to Monday, then set up shifts that start on days 1 to 5 and 8 to 12.
  • For a repeating shift pattern of three days on and one day off, set Days to 4 and Start to None, then set up three days of shifts.

Groupings

You can define groupings by entering a comma-separated list of group names.

Groupings allow you the person requesting the to specify a shift pattern, but restrict the shifts by group. For example, to with a triple shift pattern – Early, Late and Night – but specify that they only want the Early and Late shifts of the pattern. Groupings are optional; if you define groupings here, the Groups field is available when setting up the shift records.

Restrict Use

You can restrict a shift pattern to use with equipment or people – by default they can be used with either. For example, if equipment is being used 24 hours per day but operated in three shifts, you might set up a pattern with early, late and night shifts, but restrict it to equipment because that pattern is not appropriate for people.

When you have added a shift pattern record, the link on the Shift Code column takes you to the Shifts of Patterns screen where you can set up the individual shifts within the pattern. The Day field indicates which day in the pattern the shift starts on.
