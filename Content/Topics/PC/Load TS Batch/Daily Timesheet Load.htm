Daily Timesheet Load

If the new PC parameter LOADDLY is set to something other than blank, new fields are available on the Daily Timesheet Load selection screen.

Daily Timesheet Load Selection Screen

Fields

Field
Description

Batch Number

You can select an existing batch to load the timesheets to, or select New to create a new batch. If you select an existing batch, the Transaction Date, Tax Point Date and Depot fields are not relevant and are disabled.

Transaction Date

The date for transactions in the batch.

Tax Point Date

The tax point date for the batch.

Depot

The depot for the batch.

Post

Whether to post the batch automatically if it passes validation. This is only relevant if a batch is selected.

File Format

The load file can now have two additional fields at the end, to take the item type and quantity. It is possible to load files without the new fields.

The first line in the CSV file is the header line and will not be imported.

The fields in order are:

  • Asset/Item Code
  • Operator Code
  • Cost Type
    • 1 -
    • 2 - Asset
    • 5 - Workshop job card
    • 7 - Customer
  • Cost To - Customer, , Asset or Workshop job card
  • Activity
  • Activity description - If left blank, the description defaults from the selected activity
  • Date
  • Duration
  • Per
  • Rate - If zero (starting with "z"), the system rate will be used
  • Charge (If used)
  • Address Code (if used)
  • Quantity (if used - only required on non-asset lines)
  • Item Type (if used - if item type not populated, the load function assumes it is an asset and populates it from the asset type)
