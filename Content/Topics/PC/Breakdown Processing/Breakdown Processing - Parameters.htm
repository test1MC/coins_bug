Breakdown Processing - Parameters

The following new parameters affect breakdown functionality.

Parameter

Description
BDWNEVNT

The workshop event code that will be assigned to created for breakdowns.

BDWNMAIL

A list of users or roles to email when a breakdown is registered.
BDWNSTAT

The asset status to use during breakdown.

If this status excludes the asset from charging, charging is suspended until the outcome of the breakdown is processed and the number of Free Days is entered.
INSPSTAT

The status code of assets that need inspection after they are returned.

If you wish to inspect certain groups of assets after the , set the return status for the groups to this status. In the Maintenance Desk you can filter to assets that need inspection and choose to make them available or create to work on them.
WAITSTAT

The status code of assets waiting for decision. This status is used when the person doing first line assessment is unsure and refers the decision (whether to make the asset available or create ) to someone else.

If this is populated, a third action – Awaiting Decision – becomes available on the Maintenance Desk - Assets tab, to allow you to set the status of assets that are to be inspected after they are returned.
