FA/DLOADFLD - A list of the fields in the depreciation

A list of a fields in the depreciation load file, in the order they appear.

A list of the fields in the load file in the order they appear.

The following fields can be loaded:

  • faa_code – The asset the depreciation relates to. There will be an error if this asset does not exist, purchase is not posted, or asset has been disposed.
  • fdp_depntp – The value of depreciation in the current period.
  • fdp_wdv – The new written down value of the asset.
  • fdm_code – Optional field – if not supplied the value from the asset will be used.

One of fdp_depntp or fdp_wdv must be supplied. If both are supplied then fdp_wdv will be used.

If secondary depreciation is used, the fields can be given extents (for example: fdp_wdv__2). If secondary depreciation is not used, any extent will be ignored.

If the file contains additional fields, specify these using blank entries in the list of fields; these columns will be ignored in the file.
