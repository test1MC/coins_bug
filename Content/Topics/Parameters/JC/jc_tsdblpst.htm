JC/TSDBLPST - Double posting lines for timesheets

Whether to combine timesheet postings for employees into a single cost transaction per analysis with 0 value (Y or N). If so, a second cost transaction is created for all employees per analysis with the combined values.

This allows for "double posting", where the timesheet posting program generates a debit costing line with employee name and hours only. When the transactions are posted, the costing lines are generated one line per employee with name and detail of the days worked, and with days in the Quantity field of the costing transaction, but with zero cost. An additional transaction is generated to each cost code used, with an aggregated description, such as 6 Employees, and the total aggregated cost.
