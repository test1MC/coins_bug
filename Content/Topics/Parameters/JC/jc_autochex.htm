JC/AUTOCHEX - Auto Create Cost Codes - Exceptions

Exceptions to the "auto-create cost codes" policy (as specified by AUTOCH).

AUTOCHEX specifies that are exceptions to the AUTOCH policy.

  • If AUTOCH is set to N then auto-creation of cost codes is switched ON for these exceptional .
  • If AUTOCH is set to Y auto-creation of cost codes is switched OFF for these exceptional .

To specify exceptional using AHOTCHEX, enter a pipe-separated list containing a maximum of two attribute pairs (field name and can-do list), the first pair selected from List A below and the second pair from List B below:

List A

Select only one of the following (if required):

  • rcm_num={CAN-DO list of client account numbers}
  • jgr_group={CAN-DO list of contract group codes}
  • jty_type={CAN-DO list of contract types}
  • jcl_loc={CAN-DO list of contract locations}
  • job_fore={CAN-DO list of contract managers}
  • job_anal1={CAN-DO list of contract analysis set 1 codes}
  • job_anal2={CAN-DO list of contract analysis set 2 codes}
  • job_anal3={CAN-DO list of contract analysis set 3 codes}
  • job_anal4={CAN-DO list of contract analysis set 4 codes}

List B

  • job_num={CAN-DO list of contract numbers}

    NOTE

    If List B is used in conjunction with one of the entries in List A, the CAN-DO list of in List B must not use the exclusion technique (that is, use of the ! character) that is normally supported by the CAN-DO method.

For example: jgr_group=CIV|job_num=V1002 specifies within group CIV, and including V1002.

This technique is similar to that employed by security.

Typically, but not necessarily, AUTOCHEX will be a -specific parameter since it refers to data held at level.
