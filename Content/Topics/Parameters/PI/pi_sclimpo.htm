PI/SCLIMPO - Limit one PO against a Subcontract Package

Whether to prevent users raising more than one order for a subcontract package, if you are not using the full functionality.

Y = You can only raise one order per package.

N = You can raise multiple orders per package.

NOTE

If you are using the full functionality (that is, creating documents, comparing quotes and selecting a winner), you are automatically limited to one order per package.
