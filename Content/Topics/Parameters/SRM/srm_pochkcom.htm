SRM/POCHKCOM - Check the Commodities being entered

How COINS responds if a user enters an order for a commodity that is not listed as one of the capabilities of the office against which the order is being raised.

This would be useful if you use capabilities to record which commodities each supplies, and want to prevent users raising inappropriate orders.

blank = No action.

E = COINS issues an error and will not allow the commodity to be ordered.

W = COINS issues a warning but allows the commodity to be ordered.
