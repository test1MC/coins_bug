SRM/APPTYPES - iPortal Approval Types

A comma-separated list of iPortal approval types that will place a Company Restriction on a vendor's record in the Company Information Workbench.

Builder's Profile can be configured to maintain approvals at whatever level the user requires. This approval information is sent to iPortal and then forwarded to the relevant COINS system. On receipt of this approval information, COINS looks at the approval types in the XML message assesses whether a Company Restriction should be imposed.

An example setting for APPTYPES might be: Commercial,SHEQ,Financial. If a vendor record is received with any of these approval types set to "NonApproved" then a Restriction is imposed. If all of these types are received as "Approved" then any Restriction is removed.

If a Company Restriction is in place:

  • An asterisk with a red background shows to the right of the company name in the Company Information Workbench.
  • If the CI parameter RSTDISP is populated, the Group Status shows the value stored in that parameter rather than the actual Group Status Code. (The actual Group Status Code remains unchanged.)
  • A tooltip shows the reason for the restriction.
