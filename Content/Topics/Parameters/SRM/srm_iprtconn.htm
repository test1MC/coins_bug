SRM/IPRTCONN - Connection Mode

How COINS connects with the iPortal (Foreground or Background).

Background = (and if the background task srmu002.p is running), data that is waiting to be sent to COINS is processed in the background, and users will not experience any delay when visiting or refreshing a screen on the iPortal Monitor. The New Companies and Amended Companies will display whatever data has been processed.

Foreground = COINS contacts the iPortal more or less every time an iPortal Monitor screen is visited or refreshed. If there is a lot of information waiting to be sent to COINS, users may experience a long delay whilst this data is processed and stored in COINS.

We recommend that during the initial testing phase this parameter should be set to Foreground. Once all issues have been resolved, this can be set to Background.
