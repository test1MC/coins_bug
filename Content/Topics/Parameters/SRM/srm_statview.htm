SRM/STATVIEW - Set Status Codes by View - Select Level

The level at which you want to amend status codes.

blank = You are not using the option to amend status codes at a specified level. The Company Information Workbench shows -specific and group status codes; in SRM, the tabs that companies appear on depend on the Group status.

n = The "status by view" level, that is, the level at which users can amend the status codes. Users at a different view level will not be able to amend the codes. Where different companies have been given different status codes at the same level, the higher level view will show the lowest status. For example, the Crawley branch has given Southern Brick Supplies a status of Preferred, but the Slough branch has given it a status of Approved. The South Regional office will see Southern Brick Supplies with a status of Approved, because that is lower than Preferred (see SCD-PREF, SCD-APP).

NOTE

If you are using one of the "status by view" settings, you must define your organational structure using Organation Structure.
