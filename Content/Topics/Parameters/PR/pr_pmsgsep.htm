PR/PMSGSEP - Payslip messages separator

Specify the separator to be used between two payslip messages, such as a pipe ("|") separator.

You can use ASCII codes for otherwise invisible characters which could be lost when saving the parameter, for example CHR(32) for a space or CHR(13)CHR(10) to start a new line. If this parameter is not set, the two messages are separated by " , ".
