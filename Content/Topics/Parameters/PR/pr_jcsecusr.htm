PR/JCSECUSR - Users to apply PR security to in Contract Status

A can-do list of user IDs to apply Payroll security restrictions to on Status cost transaction records. This allows you to hide employee information on cost transactions from users who should not be able to see them.

If the user is in the JCSECUSR list and has access to the employee record, then the employee number, description and composite description fields are displayed as normal.

If the user is in the JCSECUSR list but does not have access to the employee, they can still see the transactions, but the employee number is blanked out, and the description and composite description are replaced by the text "Payroll timesheettimecard".

This means that you can avoid the overhead of checking employee security for users if it is not relevant. Note that if the user is not in the JCSECUSR list (or JCSECUSR is blank), and the user had access to the cost transaction records, the employee details would be displayed as normal.
