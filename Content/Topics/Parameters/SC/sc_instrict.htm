SC/INSTRICT - Insurance checks include SCADDINS types

This allows you to ensure that all four standard insurance types are validated consistently.

This is only relevant if you are not using extended insurance validation (see EXTINS).

Originally COINS only handled two insurance types (Public Liability and Employer Liability); this was extended to the four 'standard' types using the parameter SCADDINS. However the payment processing was not checking these four insurance types consistently for being valid.

Y = COINS checks the validity of all insurance types the same way.

N = COINS checks the insurance in the same way it did before. This is provided to allow users to continue using existing settings.
