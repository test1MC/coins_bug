SC/INSMAND - Subcontractor insurance details mandatory

This allows you to make certain fields mandatory when entering details for a new subcontractor.

If you are using (EXTINS = Y) you can set up any number of types (that is, you are not restricted to the four fixed insurance types). In this case, if INSMAND is set to Y then for any with the category 'insurance', the provider, policy number, and dates must be entered. For dated items, the date must be entered.

If not using ( only; EXTINS = N) then INSMAND can have up to four entries (in a pipe-separated list):

  • If entry 1 is set to Y then the public liability date must be entered.
  • If entry 2 is set to Y then the employer's liability date must be entered.
  • If entry 3 is set to Y then contractor's all-risk date must be entered.
  • If entry 4 is set to Y then professional indemnity date must be entered.
