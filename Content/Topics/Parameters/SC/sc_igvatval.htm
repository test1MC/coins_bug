SC/IGVATVAL - Ignore VAT Validation on Subcontracts

( only) Whether to bypass the Code validation on creating or updating subcontract records (Y or N).

If IGVATVAL is set to Y, then when creating or updating a subcontract record, COINS:

  • does not check that the code exists;
  • does not check that the code is active;
  • does not check that a code is entered (that is, it allows blank) if the subcontractor is registered;
  • does not check that any code entered is exempt or zero rated if the subcontractor is not registered.

If IGVATVAL is set to N, COINS makes all the above checks when creating or updating a subcontract record.

You could set this to Y during initial setup, when loading all subcontracts into the system, if you do not know the actual details at the time.
