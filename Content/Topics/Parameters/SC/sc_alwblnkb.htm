SC/ALWBLNKB - Allow blank Billing Type in PO Entry and Subcontract Maintenance

Whether to include the "as for subcontractor" option in the Billing Type selection list (Y or N).

This only applies if SC/OVRBTYPE = Y.

Y = The subcontract will by default just be controlled by the billing type (authenticated receipt, invoice or self-billing) on the subcontractor, but in an individual subcontract you can choose to set the billing type if it is different from the billing type on the subcontractor record.

N = Users need to choose the billing type for each subcontract. You might want to do this if you have a mixture of billing types on some subcontractors, and want users to check the billing type for each subcontract.
