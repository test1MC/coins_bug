PO/WARNUNPR - Warn (OA) if order is unpriced (Commit/Print)

Whether COINS issues a warning if a user tries to commit an order that contains unpriced lines (Y or N).

Y = The Commit PO's (Batch) function and the Commit Orders choose action check whether there are any zero-value lines on the order.

N = COINS does not check for zero-value order lines.

COINS displays the warning at the time of selecting the order(s) for committing. Note that if you use the batch programs that commit orders, then any orders with unpriced lines will not be committed.
