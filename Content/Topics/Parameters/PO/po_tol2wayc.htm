PO/TOL2WAYC - The Tolerance Check derives the Expected Amount from the Invoiced Quantity (2 Way Matching)

How the expected amount is derived when matching a non- order line for an order which does not require GRNs.

If TOL2WAYC is set to Y and PO parameter TOLMETH is set to blank (that is, "By Line"), then the expected amount is calculated based upon the entered invoice quantity.

If TOL2WAYC is set to N then calculations are made in accordance with the previous rules.
