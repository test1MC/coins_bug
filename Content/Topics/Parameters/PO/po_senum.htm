PO/SENUM - Order Number Base: for Service Order related Orders

The format for automatically-generated numbers.

PONUM1 = The number is generated using the pattern defined by the PO parameter PONUM1.

SE = based Sequence.

SO = based Sequence.

The number is generated as per the following pattern:

{kco}{delimA}{prefixA}{ServiceOrder}{delimB}{prefixB}{sequence}

Explanation of pattern:

{kco} Only present if the PO parameter APPENDCO is set to Y

{delimA} Only present if the PO parameter APPENDCO is set to Y and the PO parameter APPENDDL is not blank

{prefixA} Only present if the PO parameter SEORDPFX is non-blank and the POPFXFST is set to Y (or blank)

{Service Order} The as entered

{delimB} Only present if the PO parameter INSERTDL is set to Y

{prefixB} Only present if the PO parameter SEORDPFX is not blank and the POPFXFST is set to N

{sequence} Unique sequential value with (and Prefix if present). The length of the sequential part of the number is determined by the PO parameter POFORM.

It should be noted that, in the case of numbering a Material Order raised against a , we do not make a distinction between a Head Office or a Site raised order.
