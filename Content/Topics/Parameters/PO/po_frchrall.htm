PO/FRCHRALL - Force Full Allocation (Hourly) on these contracts

A can-do list of for which you want COINS to automatically set the accrual method on the order to "full allocation".

In order to capture weekly operator timesheet data, hourly-rated orders raised by need to be set to full weekly allocation mode.

If a order is raised against a in this list, and that order contains hourly-rated (typically operated ), COINS ignores the accrual method set up against the and sets the accrual method for this order to G, thereby requiring a GRN to be input (but see AUTHRGRN) and full weekly allocation. Note that this will only happen on orders that contain at least one item that is being on an hourly operated basis.
