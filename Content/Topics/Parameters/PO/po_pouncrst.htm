PO/POUNCRST - Restrict uncommit to buyer's own Orders

Whether to apply security to the Uncommit Orders procedure, to prevent uncommitting of orders raised by another buyer.

Y = Users will only be able to uncommit their own orders (that is where they are the buyer).

This applies to all orders except subcontract orders.

This can be overridden for individual users (for example, SYSADMIN) by setting up a user procedure parameter. The Procedure field should be set to: po123 and the Parameter(s) field set to: CANUNCOMMITANY.
