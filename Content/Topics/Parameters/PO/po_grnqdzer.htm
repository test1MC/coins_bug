PO/GRNQDZER - Defaults the GRN Quantity Delivered to Zero (pipe list)

Conditions for preventing the defaulting of the amount in the Quantity Delivered field in GRNs.

This parameter takes a pipe-separated list and when the order linked to the updated GRN meets one of these conditions the Quantity Delivered field will default to zero (won't be defaulted to the amount).

COINS can use the following conditions for defaulting the Quantity Delivered field to zero:

  1. List of Purchase Order types (attribute POTTYPE), for example "N" - Normal, "C" - Call-Off and etc.
  2. List of main types of order (attribute POHMPO), for example M for Material orders, P for Orders, W for Workshop, I for orders, A for Asset, O for Overhead orders.

Examples:

POTTYPE=C, B|POHMPO=I – the GRN Quantity Delivered field will be set to zero (blank value) if the purchase order related to this GRN has the order type "C" or "B" or is a order.

POHMPO=I – the GRN Quantity Delivered field will be set to zero (blank value) if the order is a order.
