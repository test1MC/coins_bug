PO/SCSENUM - Order Number Base: for Service Order related Subcontract Orders

Defines how COINS generates the base part of numbers.

Set to SCNUM1 to use the value defined in PO/SCNUM1.

Set to SE or COMP and the PO Number is generated using the following pattern:

[kco][delimA][prefixA][][delimB][prefixB][sequence]

[kco] - Only present if PO/APPENDCO is set to Y

[delimA] - Only present if PO/APPENDCO is set to Y and the PO parameter APPENDDL is non-blank

[prefixA] - Only present if PO/SEORDPFX is non-blank and the SCPFXFST is set to Y (or blank)

[] - The as entered

[delimB] - Only present if PO/INSERTDL is set to Y

[prefixB] - Only present if PO/SEORDPFX is non-blank and the SCPFXFST is set to N

[sequence] - Unique sequential value with (and Prefix if present). The length of the sequential part of the number is determined by PO/SCFORM.
