PO/AUTCHK - Invoice Auto-matching Directives (pipe list)

When matching invoices received electronically, which of the optional checks do you want COINS to use to determine whether an invoice line and a PO line match?

COINS can use the following checks to determine whether lines match:

  1. PO number
  2. 's product code
  3. 's ticket number
  4. Delivery date
  5. Quantity
  6. Unit price
  7. Invoice line value (taking discount into account)
  8. WBS code (used in 2 way auto-matching criteria)

COINS will always check the purchase order number, but 2 - 7 are optional. This parameter takes a pipe-separated list indicating which of these checks to include or omit. For example: ARTCODE=Y| SUPREF=N| DELDATE=N|QTYEXP=Y| PRICEEXP=Y| AMTEXP=Y|WBSCODE=Y.

NOTE

You can also configure automatic electronic invoice matching for individual (at office and/or company level) using Organisation Setup.
