PO/CHMIN - Prompt for Minor Codes on PO Header

Whether buyers can globally override the minor code of associated with all line items on a purchase order.

Where the minor code (the final character or characters of the ) is used as an identifier to report exceptions (such as Buying Error, Schedule Error, Addenda, Site Loss and so on) this allows buyers to change the value of the minor code on all items within the order.

Set CHMIN to Y, and CHROOT to the number of characters at the start of the cost code that will not be replaced. We also recommend you have the "ALWCHOV1" parameter (Allow Overrides in PO entry) set to "Y", so that you can amend any costings that are not valid.)

In order header input, buyers can now input a set of characters that will replace that part of the commodity not referred to by PO parameter CHROOT for every commodity line on the order.

If required, a buyer can still override individual lines.

Additional notes

  • The order could include commodities linked to a variety of "Roots" but each of these would have the same range of "minor codes" available, so adding one of these to the end of the code should always result in a valid .
  • You cannot use Minor Codes and Default functionality (setup against an Order Type) on a Purchase Order (COINS will display an error message on [F1] of the Order Header.)
  • You cannot enter a Minor Code on a template order.

  • If updating an existing PO that previously did not have a minor code, COINS will apply the new Minor code to all existing PO lines. If the existing PO did have a Minor Code, it will only apply the new Minor Code to lines that had the old Minor Code (that is, lines that had not been purposely overridden).

    If the existing PO did have a Minor Code, and this is blanked out when updating the Header, all the existing lines will be regenerated without any reference to a Minor Code (except for those that have purposely been overridden).

  • You cannot use Minor Codes when copying POs or Template POs.
  • - when creating a VO, the Minor Code from the original order is not brought forward, (it can always be re-entered). Any Minor Code will only affect new lines, not lines brought forward from the previous/original PO. (Any subsequent change to the Minor Code will only affect these new lines).
