PO/ECATWBS - eCatalogue WBS list length

The maximum number of WBS codes that will be sent to the e.

When raising an order using the e for a that uses WBS codes, a list of WBS Codes and descriptions is sent to the e server when the user first connects. For each product ordered the user can then pick relevant WBS codes from a list inside the e application (rather than going through all order lines again when back in COINS). There is an upper limit for the length of messages to/from the e server, so a long list of WBS codes can exceed this limit. If COINS attempts to send a list that is too long then the user will be told that COINS cannot connect to the e. A diagnostic message will appear in the e message log (as specified by the EC parameter TRACECAT).

ECATWBS allows you to avoid this error. For any given , if the number of active WBS codes for the exceeds the parameter value, no WBS list will be sent and the user must revisit order lines when back in COINS to assign WBS codes.

No check is done if ECATWBS is blank.
