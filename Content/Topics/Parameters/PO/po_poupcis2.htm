PO/POUPCIS2 - POs update CI Commodity lists in detail?

Should purchase orders update Central Repository commodity lists in detail? This is only relevant if POUPCIS1 is set to Y.

Y = The commodities raised on the purchase order will automatically update the respective commodity list as detail commodity codes against the in the Central Repository. If the relevant header code does not exist, it is also created.

N = Only the header codes, and not the detail codes, will be created.
