PL/ETOLAMT - Invoice Line Value Tolerance for e-Invs

The tolerance for eInvoice lines.

Valid entries: Any decimal value to two decimal places, usually a low value. The amount entered is in pounds so to include pence you need to include a decimal point. For example: 1.50 incorporates a tolerance level of 1.50.

If the PL/AUTOINVM parameter is set to 2 (that is, 2-way auto-matching is enabled) then invoice lines not greater than the PO Line value plus the tolerance amount and not less than the PO Line value less the tolerance amount will be considered as matched.
