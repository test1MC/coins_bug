PL/AUTOINVM - Automatic Invoice Matching

Whether you are using two-way matching in the Electronic Invoice Workbench.

N = The Electronic Invoice Workbench does not attempt any matching of invoices, it simply creates registered invoices.

2 = For those orders configured not to use GRNs, the Electronic Invoice Workbench will attempt to match the invoice messages, and if successful, will create costed invoices. The Electronic Invoice Workbench still creates registered invoices for invoice messages that cannot be matched.

The value of this parameter can be overridden for specified or departments using the PL/AMJOBS and PL/AMDEPTS parameters.
