PL/NONPOWBS - Costing in XML for Non-PO eTransactions

Whether Electronic Invoice Workbench should locate a valid WBS code embedded in the Order Number field of the XML message.

Setting this parameter to Y indicates that Electronic Invoice Workbench interprets the third element of a PO number (split by the / character) in an eInvoice message to be a WBS code. A blank value assumes N.
