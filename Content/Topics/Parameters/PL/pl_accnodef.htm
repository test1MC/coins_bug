PL/ACCNODEF - Account Number Default in Invoice Workbench

How COINS determines the account number of invoices received electronically:

PO = COINS uses the account number stamped on the purchase order.

MSG = COINS uses the account number found in the 's XML message <SuppliersCodeForBuyer> tag and does not attempt to override it with any account number found on the purchase order.

This parameter can be overridden on a -by- basis by giving the <AccountNumberDefault> tag in the eCommerce Configuration HUB record a -specific value.
