PL/EAUTOMHC - eCreditNote Automatch Hold Code

The hold code for eCreditNote transactions that are matched automatically.

Valid entries: Any valid PL HOLD CODE

In 3-way matching the PO parameter HLDOKCRN indicates whether a credit note transaction should be put on hold after matching. If YES, COINS uses the hold code from PL/EAUTOMHC. A blank entry in EAUTOMHC indicates that COINS overrules the HLDOKCRN flag.

If COINS is not set up to put transactions on hold after matching, any existing hold codes (typically these will be REG) will be removed.
