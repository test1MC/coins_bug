PL/EAUTOMHI - eInvoice Automatch Hold Code

The hold code for eInvoice transactions that are matched automatically.

Valid entries: Any valid PL HOLD CODE.

In 3-way matching, the PO parameter HLDOKINV indicates whether an invoice transaction should be put on hold after matching. If YES, COINS uses the hold code from PL/EAUTOMHI. A blank entry in EAUTOMHI indicates that COINS overrules the HLDOKINV flag.

If COINS is not set up to put transactions on hold after matching, any existing hold codes (typically these will be REG) will be removed.
