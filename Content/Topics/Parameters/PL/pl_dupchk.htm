PL/DUPCHK - PCard Duplicate Check Directives (pipe list)

When PCard transaction is received, which of the optional checks do you want to use to determine whether this transaction is a potential duplicate?

can use the following checks to determine whether some PCard transactions are identical:

  1. Card Number
  2. Transaction Date
  3. Merchant
  4. 's reference
  5. Buyer's reference
  6. Transaction Value

will always check the Card Number, but 2 - 6 are optional. This parameter takes a pipe-separated list indicating which of these checks to include or omit. For example: TRANDATE=Y|MERCHANT=Y|SUPREF=Y|BUYERREF=N|TRANVAL=Y

If the parameter is blank or any of the tokens (TRANDATE, MERCHANT, SUPREF, BUYERREF, TRANVAL) are not present, then no duplicate check is carried out.
