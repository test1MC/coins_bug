PL/FIRSTFLD - First Field in Invoice Entry

When entering invoices and credit notes, the user has the choice of entering the code or the PO number in the first input field. If this parameter is set to one of the following settings, the setting will determine which field will appear first. The user can still use the Return "key" to toggle between the two available fields if required:

AVM_NUM = Code (Account Number)

OHD_NUM = PO Number

If the parameter is left blank, the first input field will always prompt for the code.
