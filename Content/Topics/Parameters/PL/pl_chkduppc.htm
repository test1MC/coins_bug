PL/CHKDUPPC - PCard Transactions Duplicates Checks

Whether or not COINS should perform a Duplicate Check during PCard load process (potential duplicates will be moved to the PCards Workbench - Quarantine).

If the parameter is blank, then no duplicate check is carried out, otherwise COINS will use specified checks to determine whether the transaction is a potential duplicate or not.

COINS can use the following checks to determine whether some PCard transactions are identical:

  1. Card Number
  2. Transaction Date
  3. Merchant
  4. Supplier's reference
  5. Buyer's reference
  6. Transaction Value

COINS will always check the Card Number, but 2 - 6 are optional, and could be specified in this parameter.
