PL/UNREGCHK - OA only - Unregistering Invoices Check Invoice Matching

( only) This controls how COINS responds if you try to unregister an invoice that has already been matched against an order.

ERROR or blank = COINS issues an error message and does not allow you to unregister the invoice; you need to manually undo the matching first.

WARNING = COINS issues a warning message, allows you to unregister the invoice, and automatically unmatches the invoice.

NONE = COINS does not issue a warning, but allows you to unregister the invoice, and automatically unmatches the invoice.
