PL/DISPPO - Retain PO Number When Entering Invoices

This applies if Invoice Entry is set up to prompt for PO Number first (that is, the PL parameter FIRSTFLD is set to "ohd_num"):

Y = When entering several invoices in a batch, COINS defaults the order number from the previous invoice (this helps if you are entering several invoices against the same order).

N = COINS does not default the order number.
