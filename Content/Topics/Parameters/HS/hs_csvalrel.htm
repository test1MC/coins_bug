HS/CSVALREL - Error/Warn if stage completed > build release stage

How COINS responds if a construction stage is marked as complete but is greater than the current release stage for the plot.

If CSVALREL is not blank, then when a construction stage is recorded as complete through input of a construction stage event (either directly or indirectly), COINS validates that the stage is no greater than the current build release stage for the plot. If this validation fails, COINS responds depending on the setting below:

Error = The user will be prevented from completing the input and will receive a message "Error: Stage cannot be completed ? this plot is only build released to stage <stage>".

Warning = The user will be able to complete the input but will receive a message "Warning: this plot is only build released to stage <stage>".

blank = COINS does not validate the construction progress against the build release.
