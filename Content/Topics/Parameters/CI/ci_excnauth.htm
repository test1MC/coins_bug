CI/EXCNAUTH - Authentication token for Exchange Server contacts replication

The authentication token for Exchange contacts replication.

NOTE

This applies to the 'old' method of contact replication using WebDAV. It is not required if using EWS. See CI/EXCNMETH.

The Authentication Token is an encoded version of the user name and password for the replication user account. This is passed with every single replication request between COINS and the Exchange server. The information is encoded using a weak algorithm and can be reversed, so it is important that a special user account be created and that this account is given restricted access.

Use Generate Authentication Token to generate the token. See How to Configure Exchange Contact Replication for more information.
