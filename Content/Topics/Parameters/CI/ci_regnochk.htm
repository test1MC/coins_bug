CI/REGNOCHK - Enhanced duplicate Co Reg No checking

Under what circumstances do you want to use enhanced checking? This enhances the check controlled by the parameter REGNOADD.

REGNOADD only informs you of duplicate s if they have been entered in the same format. REGNOCHK, however, returns an error message indicating that this already exists.

None = No enhanced checking.

Add = Check only when adding a new company.

Both = Check when adding a new company and when running the Check for Duplicate Companies Report.

Report = Check when running the Check for Duplicate Companies Report.

Note that use of this parameter may slow down the performance of the Check for Duplicate Companies Report, depending on how many companies are in your database.
