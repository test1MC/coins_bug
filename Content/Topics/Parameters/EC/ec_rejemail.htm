EC/REJEMAIL - Trigger for invoice rejection email

When the eCommerce Invoices produces rejection emails.

CREATE = Rejection emails are produced when the Create action is used.

REVIEW = Rejection emails are produced when a batch of eInvoices is reviewed in the workbench.

REJECT = Rejection emails are produced when the Reject action is used.

The parameter may be set to REVIEW,REJECT in which case two emails will be produced for each invoice in error (once when reviewed and once when rejected). Emails are only produced according to the rules in Validation Error Codes. The same applies if the parameter is set to REVIEW,REJECT,CREATE.

Leaving the parameter blank or setting it to NONE overrides the validation rules however they are set and no email is produced. The AUTOREJECT functionality is not affected by this parameter.
