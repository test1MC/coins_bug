VP/ACCTRADE - Source of trade code to use for accruals

The source of the trade code to be used by the VAP specific accruals calculation.

COSTHEAD = Use the trade from the .

ORDER = Use the trade from the order.

For trades included in parameter VP/MUTRADE the accrual is based on budget less certified; for other trades the accrual is based on total orders less certified amounts.

If this parameter is set to COSTHEAD then accruals on which have a default trade which is a multi-order trade but order values from orders which have a different trade on the subcontract record then the accrual calculation will assume that the budget is the maximum total liability and will deduct certified amounts from this.

If this parameter is set to ORDER then accruals on which have a default trade which is a multi-order trade but order values from orders which have a different trade on the subcontract record then the accrual calculation will assume that those order values from the other trade are in addition to the budget so that the maximum total liability is the budget plus thus additional order values and will deduct certified amounts from this.

The recommended setting for most situations is COSTHEAD.
