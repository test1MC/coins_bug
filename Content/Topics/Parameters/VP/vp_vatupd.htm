VP/VATUPD - Update VAT code on certificate header

Set to N unless you are using mixed certificates (that is, if some costed lines do not come from VAP). If you set both this and VP/VATFIRST to Y, you must manually override on all certificates.

This is one approach if you treat retentions as able, rather than taking on the amount before retention. For an alternative approach, see VP/VATDIST.
