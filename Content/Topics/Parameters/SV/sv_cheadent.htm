SV/CHEADENT - Variation Resources Cost Head Entry Method

The method used to enter resource breakdowns.

Warning: This should not be changed if data is already being processed on a .

NCHENT = No entry: users enter resources by specifying the commodity type and commodity code. COINS determines the from the commodity code.

CHONLY = only: users enter resources by specifying only the (not the commodity code). This would be useful if your users know which to use, and you don't want to set up lots of commodity codes.

OVERWR = Overwrite: users enter resources by specifying the commodity type and commodity code. COINS determines the default from the commodity code, but the user can change it. This would be useful if you use that are more detailed than those on the commodities.
