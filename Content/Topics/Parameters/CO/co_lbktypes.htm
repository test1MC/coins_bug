CO/LBKTYPES - Transaction types requiring large buckets

A can-do list of transaction types for which you want to specify large bucket sizes.

A bucket is a temporary storage area that acts as a "pool" of available batch numbers. COINS uses this pool to ensure that batch numbers are not lost if, for example, a user creates a batch and then deletes it. COINS tops up this pool when the number of available batch numbers drops below a pre-defined level. Normally this level is fixed, but you can set a different level for the specified transaction types using CO/LBKMIN.
