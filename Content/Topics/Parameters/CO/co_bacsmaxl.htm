CO/BACSMAXL - Max number of detail lines to have in BACS file

The maximum number of detail lines to have in a file.

This allows you to limit the size of the output file, if required for operational reasons.

If the number of lines in the file (excluding the header and footer) exceeds the limit specified by BACSMAXL, COINS will start a new time-stamped output file.

NOTE

The relevant BACSFILE parameter must include [TIME=HHMMSS].
