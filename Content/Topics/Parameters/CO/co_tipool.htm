CO/TIPOOL - Pool COINS Ti connections

Whether to use Ti connection pooling (Y or N).

Y = If an agent needs to connect to Ti for the very first time it does so upon request. It establishes a connection, logs in as the current user, does its work, and keeps the connection. If a second request for Ti comes to the same agent, it will resume the already open connection.

N = If an agent needs to connect to Ti it does so upon request. It establishes a connection, logs in as the current user, does its work, and closes the connection.
