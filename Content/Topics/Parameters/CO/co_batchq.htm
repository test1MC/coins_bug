CO/BATCHQ - Batch processing queue

The queue on which batch processes will be run.

The batch posting process is a regular report with a generate program that actually does the updates to the ledger. The user does not get to choose the queue on which the process is executed.

This parameter allows a system administrator to decide which queue is best suited to process batches. Typically this might be the local queue running on the main database server and may be a queue to which the user does not normally have access.

If BATCHQ is blank, then the user?s default report queue is used.

The batch posting process and any subsequent posting report or a batch listing report also use this parameter to select the queue.

NOTE

If the CO parameters BATCHDM and BATCHDMQ are set, these dictate where the posting report is run.
