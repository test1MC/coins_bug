SY/_GITSFBRANCH - Git Safe repository branch name

It is a valid branch name and used to the installed 'custom data' git repository branch is pointing to the required branch for this environment.

Examples: test or live or qa

This parameter is used in conjunction with the logical _GITSAFE parameter. If _GITSAFE = TRUE, the data identified by the _COINSDATAID parameter will be dumped and loaded to and from disk. Said data will be further secured in the git branch identified by this parameter.

GitSafe also secures overrides and customisation data. These are +* and $* respectively. For example: +WSYBSUR function and $WSABDCE function would be dumped and loaded from disk. Said data will then be further secured in this git branch.
