SY/WEBINFO - Users to receive exception info

A list of users who receive enhanced error messages with additional information including program/method details of where the error occurred.

This information is useful to developers, but could also be a security risk, so WEBINFO allows you to control who has access to the additional information.

The default, blank, means that no one gets the extra information. This is the safest setting.

* would send the extra information to all users.

The intention is that you would usually leave this blank (the safest option) and perhaps set it to a single user ID if you needed to see the specific information for a user.
