SY/LDAPTIME - LDAP Authentication Timeout

The LDAP authentication timeout (in seconds).

The default timeout period used by COINS LDAP authentication when communicating with your network's LDAP servers is dependent on the network configuration of the COINS server. In some cases, where LDAP servers are listed in your network domain name resolution services but are not responding to authentication requests, it can cause login times to be extended.

To resolve extended login times, you can set an overriding LDAP authentication timeout period, in seconds, using this parameter.

Ensure that you are aware of your network's performance before altering this parameter. Setting the value too low may result in authentication failures on poor performing networks and/or LDAP servers. For reliable, healthy networks and/or LDAP servers values down to two seconds can be used.
