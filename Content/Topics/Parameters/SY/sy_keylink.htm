SY/KEYLINK - Tables for deeplink by key

A list of database tables for which you can specify deep links to COINS that contain key index strings instead of ROWIDs.

Deep links that contain key indexes allow external systems to create a link to COINS for a specific record. For example, in the link:

http://195.40.14.50/cgi-bin/ wsdevrtb.cgi/ wocoins.p ?TopMenu=%25WJC &kco=1 &jc_jobRowid=$1|1 &MainArea=%WJC2300SJOB

the jc_jobRowid is replaced with $ and the unique key to the record (in this case kco=1 and job_num="1").

If the table is not in the KEYLINK list, the user will get an error. This prevents a user from constructing a query with a key that gets around security, employee security, or GL security, for example.

* = Users can link to any record.

blank = This feature is switched off.

Function security still applies in all these situations.
