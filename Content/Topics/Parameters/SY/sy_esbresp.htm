SY/ESBRESP - ESB response ID reference

The path of the ID reference in the XML returned from the send service when sending ESB messages out of COINS. This allows you to record in COINS some reference from the receiving script as proof that the message was received.

If ESBRESP is set, the response is expected to be an XML document, and ESBRESP holds the path of the ID to return to the ESB ID field in COINS. For example if the response is

<root><id>12345678</id></root>

then a value of /root/id would retrieve the 12345678 string from this response and store it in the ESB id in COINS.

If ESBRESP is blank, the return value of the script in ESBSEND will be stored in the ESB ID field.
