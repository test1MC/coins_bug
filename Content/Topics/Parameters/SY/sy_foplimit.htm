SY/FOPLIMIT - Maximum lines in single FOP table

The maximum number of rows COINS will use in a FOP table when generating PDF output. Change this to prevent memory errors.

Larger tables require more memory. To prevent memory errors when generating PDF files, you can:

  • Change the value of the FOPLIMIT parameter to a smaller number. When the table reaches this number of rows, COINS begins a new table. This reduces the amount of memory required. We recommend you start with a value of 1000.
  • Change the structure of the report to a paged report (for example, one that produces a new page per ). COINS produces a new table for each paged sort item (for example, each ).
  • Increase the amount of memory available by increasing the maximum memory setting in fop.sh (for example, -Xmx96m is 96Mb).
