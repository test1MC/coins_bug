SY/LANGLOAD - Which languages should build a string list

A list of the languages for which COINS generates translation strings. New strings will only be created in the translation list if they are found for these translations.

This is to cater for the situation where you have a multi-language system but different people use different languages in different modules.

If LANGLOAD = *, then when running in a language (non-blank) then the translation strings are created for every new string you come across. This is good for when you are building your language translation but unnecessary for when you are finished.

However, if you are building your translation for a second language then it will still be doing this extra processing for users with the first language, which is unnecessary. Setting LANGLOAD to the second language allows you to suppress the extra effort when not required and just target it at the language(s) that you want.
