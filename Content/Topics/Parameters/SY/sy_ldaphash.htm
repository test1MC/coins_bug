SY/LDAPHASH - LDAP Hash Algorithm

The LDAP password hashing algorithm to use.

There may be times when using LDAP Directory user authentication that the COINS server is not able to communicate with the corporate LDAP Directory servers published by your network. To avoid any failed COINS login attempts, COINS can retain the last successfully-used password and use it when no LDAP servers can be contacted.

If this parameter is not set (has an empty value), the default, then COINS will not cache any user passwords. In this instance, if COINS loses connectivity with the LDAP Directory servers, then users will not be authenticated and COINS logins will fail.

The value of this parameter must be chosen from a valid list of supported one-way hashing algorithms. Select the hashing algorithm that suits your specific corporate security policy. Once set and on a successful login attempt, COINS will encode the user's password, using the selected hashing algorithm, and store the encoded password on the user's COINS account. In the event that the COINS server loses connectivity with the LDAP Directory servers, this cached password will be used to determine whether or not the user can access COINS.
