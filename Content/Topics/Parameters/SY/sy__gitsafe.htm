SY/_GITSAFE - Git installed and custom secured

Main control switch to quickly enable or disable the GitSafe features in a particular environment where GitSafe has already been fully configured.

Examples: YES or NO

If _GITSAFE parameter is TRUE, the data identified by the _COINSDATAID parameter will in turn be dumped and loaded to and from disk. Said data will then be further secured in git.

GitSafe also secures overrides and customisation data. These are +* and $* respectively. For example: +WSYBSUR function and $WSABDCE function would be dumped and loaded from disk. Said data will then be further secured in git.

If _GITSAFE parameter is FALSE then no data, overrides nor customisation, nor data identified by _COINSDATAID will be secured to disk and subsequently git.

The _GITSAFE parameter is also be used to control access to the GitSafe menu item under System.
