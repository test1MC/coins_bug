SY/MSGREV - Review messages before send

Whether to put outgoing messages in a queue to be reviewed.

Y = Messages are placed in a queue and must be released (using My Messages ()) before they are sent. If you set this to Y and find that your outgoing messages are not being sent, check the permissions on the startoa.sh script.

N = Messages are sent immediately using the ESB Outbound Messages process. Users do not have to review/release the messages, but the ESB Outbound Messages process provides a log of all messages sent.

You can override this default for individual users in Users.

NOTE

This is only relevant if you are using the ESB Outbound Messages process; that is, if WFMTXN is set to Y.
