SY/ENVNAME - Override environment name

Whether to allow an environment name derived from the installation directory to be overridden.

  • If an environment name is specified, it is used for the <Environment> tag in produced and consumed ESB messages. Note: An environment name is required if a DR environment is not called live but needs to be called live at a disaster event and be able to process live webservice requests.
  • If the parameter is blank, the environment name is derived from the installation directory as before.
