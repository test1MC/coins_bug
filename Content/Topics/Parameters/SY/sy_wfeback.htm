SY/WFEBACK - Send workflow emails in background

Whether you want emails generated via the House Sales workflow functionality to be sent in background.

By default emails are sent in foreground; this can prevent transactions from being entered in COINS if your email server is not responding. Setting this to Y will queue the emails in the COINS database for delivery.

If you wish to enable this option you will also need to setup the procedure syu041.p to run in background via a Unix crontab job. Once started this will continue to run and check for new queued emails which need to be delivered.

You can setup the procedure syu042.p to shutdown the background email process at the end of the day.
