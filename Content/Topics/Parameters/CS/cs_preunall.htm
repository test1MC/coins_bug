CS/PREUNALL - Prevent posting of Unallocated Cash

Whether to prevent a Cash Entry batch from being posted if it contains any unallocated cash.

Y = When the user clicks on the Cash Entry Allocation screen without allocating all of the cash receipt, COINS displays a warning message saying that all cash must be allocated.

N = When the user attempts to post a cash batch that contains unallocated cash, COINS displays an error message saying the batch cannot be posted as it contains unallocated cash.
