SW/RESAMEOP - Reschedule Task to same Operative

When rescheduling a task, should COINS try to reschedule it with the same (Y or N)?

Y = The reschedule appointment webservice will try to reschedule the task with the same as was originally booked.

N = The reschedule appointment webservice will create a task for the same trade but this will need to be scheduled to the appropriate later.
