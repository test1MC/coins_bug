SW/ALLTSKRE - Allow Task Reassignment to a Different Operative

Whether to allow task reassignment to a different even after the task is received; that is, Task Received is ed (Y or N).

Y = The reassignment is allowed unless the task was already started (in other words, either the Started or Complete date is not empty).

N = The system behaves as originally; that is, it does not allow task reassignment to a different after the task is received.
