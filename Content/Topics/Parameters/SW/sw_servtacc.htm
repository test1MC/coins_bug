SW/SERVTACC - Server time accuracy for time validation (seconds)

Server time accuracy for time validation in seconds; for example, value 180 means 3 minutes.

This is used when there is a check that the entered time should be in the past. COINS does not validate against now time, but against (now time + SERVTACC value). This is especially useful when COINS receives data from outside.

If this parameter is not set, COINS defaults the value to 5 minutes.
