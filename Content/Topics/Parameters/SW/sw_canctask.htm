SW/CANCTASK - Default behaviour when cancelling task via mFM or a web-service

This determines the when cancelling a task from mFM or a web-service.

If blank then the task or appointment will be cancelled based on field settings (swt_cancelled and swt_appt_cancelled). Any other setting will automatically cancel the appointment rather than the task based on the rules listed below:

Blank = Default .

CANCAPPT = Cancel appointment.

MFMAPPT = Cancel appointment only if from mFM.

WSAPPT = Cancel appointment only if outside mFM.

NAAPPT = Cancel appointment only if no access recorded.
