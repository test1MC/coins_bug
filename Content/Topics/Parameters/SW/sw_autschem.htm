SW/AUTSCHEM - Automatically schedule Emergency Jobs

Whether to schedule emergency automatically (Y or N).

If this is set to Y, then when a is added with a response code which is marked as an Emergency Response and with a type; then COINS will attempt to automatically schedule the tasks which were added. COINS looks at each in turn whose Primary Trade matches the trade of the task, looking for marked as Preferred first. If a has availability within the Target Start and Target Complete DateTimes on the , the task will automatically be assigned to that . If the is not available, COINS looks at the next .

If no are configured to pick up emergencies for this , area and trade, COINS displays a warning indicating this. If the AUTSCHWF parameter is set, the specified workflow will be started.

If there are configured, but none of them are available (either due to booked or other tasks scheduled), COINS displays a warning indicating that the could not be automatically scheduled. If the AUTSCHWF parameter is set, then the specified workflow will be started.
