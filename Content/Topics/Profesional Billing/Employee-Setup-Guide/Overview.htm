Components

The following are key elements of how the system works (menu options are under Contract/Job Status – Timesheets unless otherwise specified):

For every employee:

Menu option

Purpose

User Maintenance

Set up employee as a user (System module), with appropriate kco and function security access

HR & Payroll

Set up and manage employees. Make sure Payroll costing goes to correct GL department.

Contract Security

Every employee added needs to have the standard security configured (under JC Set Up Maintenance)

Contract Manager

If the employee is responsible for Project Management and/or Billing on (using the Invoice Workbench) they need to be set up as a Contract Manager (under JC Set Up Maintenance)

JC Timesheet Employee

Set up new employees, allocating an Employee Grade and an Employee Department

 

Relevant Masterfile data:

Menu option

Purpose

Employee Grades

Employee cost rates are managed by Grade, not by individual. Timesheets cannot be entered without an employee being allocated a suitable Grade. Grades also determine the available Timesheet Activities for Timesheet Entry.

Employee Departments

These departments control timesheet approvals (by department).

Timesheet Activities

This will mostly be a static table that does not need amending. It defines

(a) the mapping of Timesheet Activities to , so that when entering a timesheet, the JC Cost Code is created from the entry of the SHO (JC Section), the “Project Stage” (JC Activity), and the Timesheet Activity (which then populates the Cost Head),

(b) the double entry postings (based on masking rules) when the timesheet batch is posted,

(c) determines allowable contracts for that activity

SHO Workbench

This workbench is part of Professional Billing. Creating a SHO Line will create the relevant JC cost codes, and will therefore control which Timesheet Activities are available for timesheet entry.

Product Maintenance

This is part of Professional Billing. It is relevant to timesheet entry because it shows how PB Product codes are mapped to Timesheet Activities. When creating a SHO Line, a product code is specified, which then defaults into the SHO line the relevant mappings to both the Cost Code and the relevant Timesheet activities (note that one product can map to several Timesheet Activities).

Purpose

Employee Security

Departments

There are three concepts of Employee Department:

  • General Ledger Departments (GL Element 1). These are relevant when posting Timesheets, as the double entry coding for the department is NOT derived from the JC Employee Department, but derived from
  • The Timesheet Contract (for the debit), and
  • The JC Employee record “Analysis” code (for the credit). The employee analysis code is mostly a T* contract which then maps to the relevant GL department.
  • JC Employee Department codes (employee’s department, and employee’s approving department – normally the same code). These are used for Timesheet Approvals (US only), and possibly for reporting JC timesheet employees by department.
  • HR/Payroll department and GL costing rules (to GL Element 1) - the Payroll costing determines General Ledger departmental payroll costs.

Timesheet postings may move costs from one GL department to another, or they may create intercompany postings:

  • Department cross charge within a company (kco):
  • If the Timesheet contract is mapped to a different GL department than the employee’s home GL department (defined via the Analysis field on the JC Timesheet employee record), then costs will be transferred. All contracts except T* may create a cost transfer.
  • If the “non T* Contract” belongs to the same GL department as the employee, then the net effect for the GL department will be zero. 
  • T* contracts are associated with zero cost rate OH* timesheet activities and will not generate any cost transfers.
  • Cross company timesheet entries:
  • When entering a timesheet, it is possible to select a different , and then select a client contract from within that company. The department that owns the client contract gets charged with the timesheet costs, and the employee’s home GL department gets the credit.
  • Balance sheet intercompany entries are also made in each (kco).

Grades.

Each employee is allocated a Grade. The Grade determines:

  • Permitted Timesheet Activities. For example CONS grade has CON* activities but not TEC* activities, and the TEC grade has TEC* activities but not CON* activities. 
  • The cost rate for each permitted Timesheet Activity.
  • Every Grade has all the zero cost rate OH* Timesheet Activities
