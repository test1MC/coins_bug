Contract Security

There are a few users e.g. Finance and Timesheet Administration, who will have access to all contracts.

Contract security should follow the structure shown below. Note that the Security list is based on the Contract Type.

Client contracts are set as C01 or C04. Example of US Job set up where Job Type is C04.

All Internal contracts X*, I*, SUP*, S* (S* are pre sales contracts) have the Contract Type set as INT. All DEV* contracts have the Contract Type set as PDR.

The Code Txxn is the employees department contract for OH* timesheet activities. Set thus, employees cannot access another departments T contract.

The Code T is entered for any employee that does not enter timesheets. Employees that have left, and user groups (except TSENTRY) have a blank code. Setting the filter as shown below will show active employees for the logged in kco:

Security is set up by User, not by User Group. User Groups should be blank, except for TSENTRY.

User Group TSENTRY:

  • TSENTRY settings should not be changed.
  • TSENTRY provides automatic and appropriate permissions when selecting cross company client contracts.
  • User specific permission overrides should not be set to invalidate the TSENTRY permissions.
  • The reason behind this set up is so that each territory doesn’t have to worry about users in other territories accessing non client contracts via timesheet entry.

Contract Manager

Under JC > Set Up > Set Up Maintenance > Contract Manager screen – create the new employee as a Contract Manager and allocate their coins user id. Best to do this for all employees, because the Contract Manager control is based on allocation of the Contract Manager within either the Contract Maintenance screen, and/or within the SHO Workbench.
