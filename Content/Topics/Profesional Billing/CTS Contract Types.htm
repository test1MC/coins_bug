How It Works - Client Contract/Job & SHO & Project Stages

There are different Contract/Job Types available.

The contract/job is effectively the client. There are potentially three contracts for each client - N1nnn, N4nnn or N5nnn, where nnn is a common 3 digit number for the client. The first letter is typically for the territory: N for North America, C for UK, M for Middle East, A for Australia.

  • C1nnn/N1nnn is for new implementations, or major implementations in existing clients.
  • C4nnn/N4nnn is for ongoing work with existing clients
  • C5nnn/N5nnn is not for timesheet entry, and no longer used for recording Support effort. It is only for recurring revenues.

If permission has been given to access a "foreign territory" client contract/job and SHO, you can do so – by first selecting the Company (kco), then the contract/job within the selected kco. Contract/job security needs to be enabled for the user to post time to that "foreign client contract/job & SHO".

The Contract/Job Section is the SHO. The sales function create and populate an Excel template for a SHO, which is then used to set up the SHO record in by a SHO set up administrator (Julie Clark for the UK, Ireland, Australia, Middle East. <tbc> for the US).

Account Managers or Project Managers should advise employees of the SHO to be used for the timesheet entry.

The SHO Budget (Revenue, Cost and Hours) will have been uploaded into Contract/Job & SHO. The Hours uploaded appears as the Current Budget in timesheet entry.

If your hours entered exceed the budget figure, data entry will still be allowed and the excess hours will appear in exception reporting.

The budget load creates the permitted Timesheet Activities available for that SHO (by creating the relevant Cost Codes)
