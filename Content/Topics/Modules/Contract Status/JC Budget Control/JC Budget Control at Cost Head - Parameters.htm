JC Budget Control at - Parameters

The following PO parameters govern the when comparing order values against budgets. Two existing PO parameters have been renamed to more accurately reflect their purpose:

  • BUDCHCOM - has been re-named BUDCCCOM and its description changed to "Budget Type for Cost Code budget validation when committing PO".
  • BUDCHCAT - has been re-named BUDCCCAT and its description changed to "Validate Cost Code Budget at Cost Category level".

Four new PO parameters have been introduced:

  • BUDCHCOM - Budget Type for budget validation when committing PO.
  • BUDCHCAT - Validate Budget at Cost Category level.
  • BUDCHWRN - Issue Warning rather than Error if budget exceeded.
  • BUDCCWRN - Issue Warning rather than Error if Cost Code budget exceeded.

If the new parameter BUDCHCOM is set to a valid JC budget type, an order cannot be committed if any affected by the order would cause the related budget to be exceeded.

If the (newly renamed) parameter BUDCCCAT, or the new BUDCHCAT, is set to Y, the check against the budget will be at the level of the cost category.

If either of the new PO parameters BUDCHWRN and BUDCCWRN is set to Y, orders that would cause the budget or cost code budget (respectively) to be exceeded can be committed, but a warning is issued.
