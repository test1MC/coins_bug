End of Year Procedures

This chapter explains the procedures to follow for the End of Year process.

We recommend that you follow these procedures in the order that they are listed; for example, ensure that all ‘Prior to Year End’ is completed before you move to the ‘At Year End’ procedures.

NOTE

The fixes detailed on page 6 must be in place prior to completing the 2016-2017 payrolls and prior to processing any 2017-2018 payrolls or RTI processes.

Prior to Year End

Order New P60 Stationery

Order P60 stationery together with any other stationery that you require. Refer to HM Revenue & Customs forms Order line. This should be available as a contingency in case HMRC has any technical issues during submissions.

NOTE

HMRC have not yet released details of the new format for the P60 so please refer to their website for any updated details. We expect that the only change will be that the ASPP field will be replaced with the ShPP field. (16 Feb 2016)

The P60 format we support is the P60 (Single Sheet) (201x-1x). http://www.hmrc.gov.uk/ebu/p60snglsheet.pdf

This is a single sheet laser form and you will require a laser printer.

Print Sample P60s – Alignment Test

Using this year’s P60 (Single Sheet) (201x-1x), print ten P60s to the printer that you intend to use for this year’s P60 print, to check alignment, page throws, and font size.

You will need to test the alignment with an actual P60 (single Sheet) (201x-1x) form and not a printed copy of the above .pdf form.

    Go to Payroll menu > Monthly Payroll > Maintenance > Print P60s.

    Select the relevant Form (and tax reference if you are using multiple tax references), and the Year you are submitting for (only the current tax year will be displayed this time around).

    On the Employee Selection tab, select a range of employees for test purposes, then click the button.

The report will be produced as a PDF file and will be available on the Report Status workbench.  You can then print the PDF file and confirm that it aligns with the pre-printed stationery.

 

Set Week 53 Indicator to Yes

This is only applicable to Weekly Payroll (including two-weekly and four-weekly) and is the only variance between monthly and weekly.

If it is necessary to run a payroll for week 53 pay period 2016/2017, you must ensure that the ‘Week 53 year’ box has been ticked. Do this before starting the pay process for week 52, or sooner if possible.

To set the week 53 indicator:

    Go to Payroll menu > Company Setup > Company Configuration.

    Tick the Week 53 box.

    Go to Payroll > Weekly Payroll > Setup > Periods Maintenance, and make sure that a valid week 53 period exists if required.

    If you need to add a period, click Add to insert an additional line and populate it with dates and other values as appropriate. Please ensure that the Week 53 dates are different to those set up for Week 52.

 

RTI Submissions – FPS and EPS

    On Payroll > Global Functions > HMRC Workbench > RTI tab, create an FPS, and the Final Submission for Year box. 

The Questions and Declarations tab is now no longer required and has been removed.

    Submit the FPS as normal.

    Create an EPS, and the Final Submission for Year box.

    Complete the required fields on the Recoverable Amounts Year to Date tab.

The Questions and Declarations tab is no longer required and so has now been removed.

    Then submit the EPS.

You have now completed your End of Year submission.

Print P35 Summary

Print the P35 summary for your records.

    Go to Monthly Payroll > Reports > P35 Report.

With the P35 being submitted electronically, there is no requirement to include leavers.

Print Relevant General Ledger Entries

Print the relevant entries for income tax and NIC deductions and any other statutory values such as SSP/SMP from the General Ledger.

    From the home page, select the General Ledger module.

    Select the Report menu and select the Operational Report submenu.

    Select the GL Transaction Report option from the menu.

The GL Transaction Report screen is displayed.

    On the Main tab, enter the dates for the Start and End periods.

    On the Account Selection tab, select the accounts required, using the Lookup buttons to select from the list.

You can select multiple accounts: use the Shift or Control keys and click to select the rows, then click the Tick icon (Return Selected) button.  

You will return to the GL Transaction Report screen.

    Click the Next button to run the report.

Additional reports that you may wish to run include Transaction Type and Detailed Source Reports, which can be filtered as required to the relevant transactions.

Reconcile Income Tax and NI Contributions

Use the P35 column totals and the General Ledger reports from the previous two steps to reconcile the payroll reports to your financial ledgers.

It is recommended as best practice that you carry out this reconciliation task at the end of every month as part of your normal monthly routine. This will highlight at an early stage if there are any adjustments that you need to make, either in Payroll the General Ledger or both.

It is common that payroll personnel are not responsible for the resultant entries in the financial ledgers. However, it is good practice to keep a permanent file note of any unusual circumstances or adjustments that you encounter in payroll processing during the year. This may help the person whose job it is to carry out reconciliation tasks as well as being a reference point for any subsequent Inland Revenue enquiries.

Mid-year implementations of payroll require special care as this requires manual population of detailed year-to-date NI information as well as summary totals in the relevant year-to-date pay items.

These manual data population exercises should be checked carefully against a full P60 listing, P35 column totals and General Ledger reports at the time of conversion.

Leavers appear at the end of the P35, and for current employees the directors are reported first.

Check that Pay Items Are Set Correctly

NOTE

This procedure also deletes Pay Item history records when they exceed the number of years to be retained as indicated by the Years History field on each Pay Item. When all Pay Items for an employee have been deleted in this way, the employee record is also deleted. Before running this procedure, please make sure that the Pay Item History fields are set appropriately. To avoid employee deletion set at least one common Pay Item to 99 years.

Prior to running the Year-End routine, to ensure you have a pay item with 99 history years assigned, go into pay item details via Payroll menu > Global Set up > Pay Items.

Years History should equal 99 to ensure employees with this item in their pay sequence are not deleted due to zero history on all items.

Please note that the “Year End Clear” tickbox is a very important flag also in terms of year-end clearing. Statutory items such as Taxable, NI-able, and NI paid and Tax paid should have the box ticked as this is cumulative only for the tax year and should be reset to zero for the new tax year as of period 1.

Company items such as Holiday accrual (if run by calendar or year other than fiscal year) and staff loans reducing balance and calculated should have the box unticked as these balances should be brought forward to the new Tax Year. At this stage the Year End can be run.

At Start of New Tax Year

NOTE

This information is a guide to the fields that require updating within . It should be used in conjunction with the information issued by your tax office.

The Tax tables are global to ALL companies and ALL payrolls but are specific to a Tax Year. The tax tables are retained for current and prior years; this ensures that payrolls that are still in the prior year and requiring calculation can still be processed based on the relevant tax years figures. This means you can run year end for monthly payroll and advance the new rates for 2017/18, while weekly payroll (pre year-end) can still use 2016/17 rates for final pay-period calculation purposes.

Amend the Tax Tables National Insurance and SSP/SMP/SAP/SPP Tables

The Tax and NI tables have been combined into one function in current versions of .

    Go to Payroll > Global Setup > Annual Details for Tax Year.

    Ensure the new NI, Tax and Statutory Payment rates are as shown in the example screen shot below:

    Enter the new rates for NI and Tax by clicking open and completing the relevant fields.

uses the date in the Payroll Configuration screen assigned to the pay frequency to derive the year applicable for tax rates, and thus refers to the appropriate year’s Tax NI, SSP and SMP/SPP/SAP table at the point of payroll calculation.

    Once the rates have been entered click Save to accept the details.

 

Run the Tax Code Increases Procedure

NOTE

The basic personal allowance for 2017-2018 will be increased to £11, 500 and the tax code for emergency use will be 1150L W1 or 1150L M1 or 1150L X
As a result there will be a general uplift of tax codes with suffix ‘L’ which will be increased by 50.

Run the Tax Code Increases procedure to increase the suffix by the appropriate amount.

The Tax Code Increases procedure is a company-wide procedure by pay frequency, so make sure all payrolls have run the end of year clearing procedure above before performing this step and selecting the required pay frequency to update.

    Enter the increases.

    When all the rates have been entered click next to run.
