OA Expenses - Claims Workbench

The Claims Workbench allows you to enter details of approved expense claims, and to process claims, which can be either claims that were entered using CAS Expenses, or claims that were entered manually using the Claims Workbench.

Claims Workbench

Fields

Batch

The number of the expense claim batch.

Description

A description to identify the batch.

User ID

The ID of the user who entered the expenses batch.

Status

The current status of the expenses batch.

Transaction Date

The date on which the batch was created.

The period end date of the financial period that all the transactions in this batch will be posted to.

Txns

The number of transactions in the General Ledger batch that is created when the expenses batch is posted.

GL Batch

The batch number of the General Ledger batch that is created when the expenses batch is posted.

GL Batch Status

The batch status of the General Ledger batch that is created when the expenses batch is posted..

Claims Workbench Actions

The following actions are available for completed claim batches:

  • Post - This runs a background process that creates the necessary payment transactions and adds them to the unfrozen payment batch. If no unfrozen payment batch exists it creates one.
  • Validate - This checks for any errors in the claim batch. You might use this if the posting report shows there were errors. The validation report provides more detail about any errors.
  • Summary - This produces a financial summary report of the expense claim batch.
