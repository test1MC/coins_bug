OA Expenses - Parameters

Expenses Parameters allows you to update all the existing Expenses parameters. Some of these are only relevant to COINSplus Expenses.

The following new parameters have been introduced:

WEBINVLD

Whether to allow invalid costing in a batch. If set to Y, this allows claims submitted using a web service (for example, claims from CAS Expenses) to contain an invalid costing analysis which can then be corrected in OA Expenses.

WEBAPOST

Whether to allow claim batches submitted using a web service (for example, claims from CAS Expenses) to be posted automatically when they are received. If allowed, a setting on the claim batch message determines whether the batch will be posted.

SRCTXNTP

A list of transaction types to use for claims that are created using a web service. This can be a single transaction type or an attribute list if you want to separate sources. If a list is used the different sources are the attributes and their transaction type the values; for example source1=TT1|source2=TT2.e.g. CAS=CASEX.

The following existing EX parameter is used differently in OA Expenses:

ERRCTRL

If set to Y, users enter totals manually, and issues an error if the control totals and claim details do not match. If set to N, calculates the totals from the details.
