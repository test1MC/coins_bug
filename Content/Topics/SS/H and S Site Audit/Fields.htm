Fields Tab

Several new features have been made available on fields. 

Mobile Form Field Maintenance

Field
Description

Width and Height

In OA on the generated page, a field height over 1 means you will get an editor box, and the width divided by 20 is used to determine the number of column spans the field takes up.

Append

This allows you to append a field on the generated page, so that it appears on the same line as the previous field. This is feature only applies to OA as in OA the whole audit is presented on a single page; this allows you to make best use of the space. 

Comments Box

This allows you to control whether a comments box is available with the field:

  • Yes - A comment box will appear on the OA audit form.
  • Optional - The comment box will be hidden unless the user s a box to open it.  This would save screen space if there are lots of places where there may be a comment.
  • None - No comment box appears.

Field ID

The field ID is used to identify the field for reporting and in calculations. It can be explicit, or use underscores to mask it with the section element.  You can enter checklists with combo, integer or radioset fields.

Checklist

This can be the code for a checklist or a number in { } to indicate which checklist to use, from a list specified on the element.

Make Mandatory Values

This allows you to specify the values for a field that should make input for other fields mandatory.  When this is populated you are offered a list of the other fields in the same section and can which will be mandatory, so you need to define all the fields before this can be set.

Mandatory Fields Setup

Regular Expression

A regular expression that the field input must conform to, and an error message to display when it doesn’t. This allows you to validate data on the form as it is entered. For example, the regular expression [A-Z]{4,} will display an error if you do not enter at least four upper-case letters.
