Document Designer – Mobile Forms

When defining a Mobile Form Document, on the Extras tab, specify the mobile form that the document will print, and save the document.  This creates Extras fields for each section where you can define the document forms to use for each section of the mobile form. 

Mobile Form Document Extras Tab

For each section you can define a header and a body form.  For sections using elements (sy_mobfel) the body table is the element and field values are accessed with RO fields:

  • RO_sme_value__n – the value entered for the nth field in the section
  • RO_sme_text__n – the comments for the nth field
  • RO_sme_xlist__n – the checklist items selected for the field.

For each section you can also specify forms for Images, Workflow, Comments and Checklist.

There are Header and Body forms for Images Section and Workflows Section. 

  • If you want all the images in one section, populate the Images Section values.
  • If you want images to appear after the section of the form that they are linked to, use the Image forms against each relevant section.  The same principle applies to workflows.
