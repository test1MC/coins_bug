Follow Up Email

A new report process allows you to send chase emails on actions assigned to other people, where there has been no movement on the issue for a specified interval (that is, the time since the last note on the task).  You can also specify the range in terms of days to task due date, so you can select near due or overdue. A note will be added to the task that a chase email was sent. 

The subject line and body of the email message can be entered on the Output Options tab.  You can use tokens in curly brackets braces { } to insert values such as the date when the task was first assigned. 

When deciding if the task is overdue, uses the base task due date and complete flags, not the value on the notes.  This assumes tasks will only have one assignee and follow ups are not multi-threaded.
