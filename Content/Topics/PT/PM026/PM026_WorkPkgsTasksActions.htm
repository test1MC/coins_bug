Production Tracking Functionality: Work Packages/Tasks/Actions

The Work Packages/Tasks/Actions function is found in Project Management > [Working Project] > Production Tracking. It allows you to do the following:

  • Create and maintain the Work Packages for a Project.
  • View the status of the Work Packages by including calculated Task and Action data.
  • Link to associated Tasks or Actions of the Work Package.

This function is divided into the following tabs: Work Packages, Tasks, Actions, Cost Codes. All of these tabs are discussed below.

Work Packages Tab

You can use the Work Packages tab (Figure 1: Work Packages Tab) to create and maintain Work Packages for a Project.

Delete Work Packages

You can use the Delete button to delete Work Packages with Tasks that do not have any installed transactions (open scheduled Actions can be deleted).

Export Work Packages

The Export button generates a spreadsheet of selected Work Packages for a single Project using a predefined, fixed set of columns.

Download/Upload Work Packages

To download and upload Work Packages, you must use the Spreadsheet Workbench found in Project Management > [Working Project] > Production Tracking. For more information on this Workbench, see Section 0.1 Production Tracking Functionality: Spreadsheet Workbench.

Tasks Tab

The Tasks tab (Figure 3: Tasks Tab) allows you to maintain a Task list for all Work Packages of a selected Project. It allows you to easily record installed quantities against a Task, move a Task to another Work Package (even after the Task has earned values, i.e., installed quantities), and link to associated actions of Tasks.

The following actions are found at the bottom of the Tasks screen:

  • Change Cost Code (see Section 0.1.2.4 Change Cost Code)
  • Change Work Packages (see Section 0.1.2.5 Change Work Packages)
  • Complete Tasks (see Section 0.1.2.6 Complete Tasks)
  • Schedule an Action (see Section 0.1.2.7 Schedule an Action).

PLEASE NOTE: The remaining four fields below are concerned with the Action (not the Task), which is automatically created after the Task is saved. Once the Action is created, these fields will be empty.

Quantity Installed Current

The quantity that is currently installed. When this value is entered, the Quantity Installed to Date value is automatically calculated and displayed on the screen.

Quantity Installed to Date

The quantity that is installed to date. When this value is entered, the Quantity Installed Current value is automatically calculated and displayed on the screen.

Installed Date

The actual date that the quantity was installed. This date defaults to the install date set in the Task header, but it can be overridden.

  • If Quantity Installed Current has a non zero value, the Installed Date must be entered.
  • For the Task function only, the Installed Date cannot be previous to any other action with an installed quantity for this task.

Complete Flag

  • If the Complete Flag box is ed, the system fills in the installed quantity with the unearned quantity, and the installed date from the session parameter.
  • If the Complete Flag box is uned, the Quantity Installed Current, Quantity Installed to Date, and Installed Date are blank. It does not reflect whether the Task is complete.

Delete Tasks

To delete Tasks, select them on the Tasks tab and then click the Delete button.

  • You can delete Tasks that do not have any installed Actions.
  • If you delete a Task that only has scheduled Actions, the scheduled Actions are deleted.

Export Tasks

The Export Tasks button on the Tasks tab lets you generate a spreadsheet of selected Tasks for a single Project (see sample spreadsheet in Figure 5: Sample Export Tasks Spreadsheet). It exports a predefined fixed set of columns. Note: Action quantities are not included in the column set for importing.

Change Cost Code

The Change Cost Code action allows a Task to be assigned new cost codes, even after the Task has earned values (installed quantities). When you select this action, the Change Cost Codes popup screen appears, as shown in Figure 6: Change Cost Codes Screen.

When a Task is assigned a /Cost Code via the Change Cost Code action, Status must potentially be updated.

Whether or not the Task that was changed has Earned values, the system checks the total Earned value for all the tasks of the Cost Code and compares that to the Planned hours of the Cost Code. If that Earned/Planned value has changed, Status must be updated.

NOTE: The /Cost Code is not stored on the actions; the actions point to the Task to get the /Cost Code.

Change Cost Codes Screen

The following occurs when the Change Cost Code action is executed for a Task that has earned values and is split into a new Task pointing to a new Cost Code:

  • The original Task record(s) has the plan quantity set equal to the earned quantity.
  • The new Task(s) have all the same fields as the original Task, with the exception of the plan quantity and /Cost Code.
  • The plan quantity is equal to the original Task plan quantity minus the original Task earned quantity. The /Cost Code is set from the input screen.

Change Work Packages

The Change Work Packages action performs a mass update to assign selected Tasks to the same Work Package. When you select this action, the Change Work Package popup screen appears, as shown in Figure 7: Change Work Package Screen.

Change Work Package Screen

Complete Tasks

The Complete Tasks action allows you to select a set of Tasks and mark the records complete. This creates Action records that make the Tasks 100% complete (Action records for the unearned quantity). When you select this action, the Set the Install Date popup screen appears, as shown in Figure 8: Set the Install Date Screen. This date defaults in from the Installed Date on the Tasks screen but can be changed.

Set the Install Date Screen

After the Complete Tasks action is finished, the Tasks screen is refreshed so that the earned values are updated.

Schedule an Action

The Schedule an Action action allows you to select a set of Tasks to create a scheduled Action for each record (with a scheduled quantity other than zero). When you select this action, the Schedule an Action popup screen appears, as shown in Figure 9: Schedule an Action Screen.

Schedule an Action Screen

Actions Tab

You can use the Actions tab (Figure 10: Actions Tab) to edit Actions for selected Work Packages and Tasks, view Actions and the status of scheduled Actions, and close scheduled Actions.

The following actions are found at the bottom of the Actions screen:

  • Close - Installed in Full (see Section 0.1.3.1 Close - Installed in Full Action)
  • Close - Not Installed (see Section 0.1.3.2 Close – Not installed Action)
  • Move to a New Day (see Section 0.1.3.3 Move to a New Day Action)

NOTE: If a scheduled Action is created/saved and the Task or Work Package that the Action belongs to is constrained, a warning message is displayed.

Actions Tab

Field Definitions (Selected Fields)

Work Package

Select the Work Package from the lookup list. It can only be edited in Add mode.

Task

Select the Task from the lookup list.

Scheduled Quantity

The quantity scheduled for the Action.

Responsible Party

The name of the person responsible for the Task.

Target Date

The scheduled target date for the Action. It must be entered if Scheduled Quantity is non zero.

Install Scheduled

This box is always uned.

Note: If this box was ed, the Installed Quantity would automatically be set equal to the Task Plan Quantity. The action's status depends only on the Installed Quantity field and is Closed once the Installed Quantity equals the Task Plan Quantity.

Installed Quantity

The total quantity already installed for the Action. This value cannot be greater than the Plan Quantity of the Task.

Install Date

The actual date that the quantity was installed. If Installed Quantity has a non zero value, the Install Date must be entered.

Constraint

Any constraint that should be set up for the Action. If there is a Scheduled Quantity and the Installed Quantity <> 0 and less than the Scheduled Quantity, the Constraint must be filled in.

Close - Installed in Full Action

The Close - Installed in Full action allows you to easily close a number of fully installed, scheduled records at the same time. It sets the Installed Quantity to equal the Scheduled Quantity and sets the Install Date from the input value. When you select this action, the Close - Installed in Full popup screen appears, as shown in Figure 11: Close - Installed in Full Screen.

Close - Installed in Full Screen

Close – Not installed Action

The Close - Not Installed action allows you to easily close a number of scheduled records at the same time, such as records that were not installed. When you select this action, the Close - Not Installed popup screen appears, as shown in Figure 12: Close - Not Installed Screen.

Close - Not Installed Screen

Move to a New Day Action

The Move to a New Day action takes the selected open records, closes them, and re-creates them for a new day. When you select this action, the Move to a New Day popup screen appears, as shown in Figure 13: Move to a New Day Screen.

You cannot do this for Closed or Unscheduled Actions.
