Configuration

A new Company parameter called HRPRLEAV has been added to the Human Resources module. When set to WORKBENCH, you can enter the from the HR Personnel screen, even if the CO/PRHR parameter is set. If left blank, the HR Personnel acts as it currently does.

Parameter

Description

HRPRLEAV

HR to Payroll Employee - Allow Personnel to be updated when PRHR has been set to sync PR fields to HR fields.

The options are:

  • [Blank] - No change in functionality and is not updateable if PR fields sync to HR.
  • WORKBENCH - Linked to PR Requests Workbench. You can enter the Personnel field in HR and send a request to Payroll to start the process before the employee has left. If the request is accepted, the Employee is updated with the Personnel .

When the HR/HRPRLEAV parameter is set to WORKBENCH, and a Personnel is entered, a record is created in Payroll > [pay frequency] > HR Requests, and the Employee is not set until the PR Request has been accepted or an Employee has been entered in PR Employee Maintenance. Entry of the from Payroll automatically closes out the PR Request, if it is on file.

Screen and Processing Changes

HR Personnel Workbench

The shown on the HR Personnel Workbench > Personnel Summary > screen (see Figure: [%=Variables.leaver employee%] Screen) is open for update if the following are true:

  • CO/PRHR parameter has been set to sync PR fields to HR,
  • the HR/HRPRLEAV parameter has been set to WORKBENCH, and
  • the linked PR Employee has a Status of Not a or Leaving this Period.

HR Requests Screen

Approval

When a HR Request is Approved in Payroll > [pay frequency] > HR Requests (date is selected and Approve action shown in Figure: HR Requests Screen is run), the following occurs:

If the Employee has not already left and has a Status of 2 or 3, then:

  1. A message is displayed to the user.
  2. The Request is Rejected.
  3. The Request is archived.

If the Employee has not already left and has a Code of 0 or 1, then:

  1. The Employee Code is set to 1.
  2. The Employee Date is set to the date on the request.
  3. The Request is Approved.
  4. The Request is archived.

Rejection

When a HR Request is Rejected in Payroll > [pay frequency] > HR Requests (date is selected and Reject action shown in Figure: HR Requests Screen is run), the following occurs:

  1. The Request is Rejected.
  2. The Request is archived.
  3. Alerts are updated.

PR Employee Maintenance

When you select a Code of 1 or greater (see Figure: PR Employee Maintenance), and an open HR Request is on file, it is used to default the into Payroll > Employee Maintenance > Employee Details > General tab, as shown in Figure: PR Employee Maintenance.

Once saved, and the Code is 1 or greater, the system checks to see if there are any open HR Requests for that employee. If the matches, the request is accepted. If the does not match, the request is rejected.
