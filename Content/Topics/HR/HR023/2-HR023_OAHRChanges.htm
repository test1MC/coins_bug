OA HR Changes

This section describes the OA HR functionality related to Leave of Absence (LOA) processing.

HR Layoff Lookup Code

You can create a Lookup Code for Layoff in the HR Leaver ReasonLookup Type list in OA Human Resources > HR Maintenance > Lookup Codes > Leaver Reason, as shown in Figure 1: HR Leaver Reason Lookup Code. This Lookup Code is used to make an employee inactive without terminating benefits. To create a Layoff code:

  1. Add a Code name that you wish to use for Layoff in the HR Leaver Reason list (the sample screens use the name "Layoff", but you can use a different name).
  2. the Layoff box (see Figure 1: HR Leaver Reason Lookup Code), then save the Code.
  3. NOTE: The Layoff box will only display if you have OA HR connected to Ti PR by setting the TI* parameters in Company Parameters (see sample settings in Figure 2: TI Company Parameters).

HR Employment History

To allow for Leave of Absence information, the Hire/Term/Rehire/Other column has been relabelled as Hire/Term/LOA/Rehire/Other on the OA Employment History screen in Human Resources > Personnel Workbench > [personnel link], as shown in Figure 4: Employment History LOA Column.

Employment History LOA Column

HR Personnel Summary

After the Rehire Date is entered in the Ti PR Employee File (see Section 2.1.2), the Personnel HR Status in OA Human Resources > Personnel Workbench > Personnel Summary > Profile tab is set to ACTIVE (shown in Figure 5: OA HR Profile Information), and Termination information on this screen is updated automatically.

OA HR Profile Information

When HR Status is Leave of Absence (based on the HR/TIPRLOA parameter), LOA Date and Reason fields are created and added to the Ti PR Employee File and the OA HR Personnel Workbench. A Term Reason is also displayed.
