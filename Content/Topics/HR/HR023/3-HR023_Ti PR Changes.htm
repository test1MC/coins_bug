Ti PR Employee File

This section describes the Ti PR functionality related to Leave of Absence (LOA) processing.

Employee Layoff Date

OA HR Employment History is updated with a record type of HRLOA and uses the Layoff Date and Term Reason entered in Ti Payroll > Employee File, as shown in Figure 1: Ti PR Employee File Screen. After entering the Layoff Date and Term Reason, the Active flag in Ti is set to N, and the Ti PR Employee Layoff Date is sent to OA as a separate piece of information (rather than as the Term Date).

Ti PR Employee File Screen

Employee Rehire Date

When an Employee returns from a Leave of Absence, the Rehire Date is entered on the Ti PR Employee record in Ti Payroll > Employee File, as shown in Figure 2: Ti PR Employee File - Rehire Date. The Layoff Date and Term Reason fields are removed in both Ti and OA, and the Active flag is set to Y. When Ti PR -> OA PR -> OA HR syncing occurs, a Rehire Employment History record is automatically created.
