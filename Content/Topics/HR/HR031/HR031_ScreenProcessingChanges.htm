Employee Benefits for Prevailing Wage Calculations Report

A new Employee Benefits for Prevailing Wage Calculations Report has been added tois found in Human Resources. It calculates the cost of company provided benefits. The report can be used for analysis, as well as determining the cost of employee provided benefits when subject to Prevailing Wage calculations.

This document discusses this report in more detail.

Location

The Employee Benefits for Prevailing Wage Calculations Report can be found in Human Resources > Reports > Benefits.

Report Selection Screen

The report selection screen for the Employee Benefits for Prevailing Wage Calculations Report (Figure 1: Report Selection Screen) consists of two report-specific tabs (Average Hours Selection and Selection) and the standard Personnel tab, all of which are discussed in more detail below.

Report Selection Screen

Average Hours Selection Tab

The Average Hours Selection tab lets you determine whether to include average hour calculation in the report.

  • If ed, an Average Hours field appears on the screen, as shown in Figure 2: Average Hours Selection Screen ([%=Variables.tick%]ed). This field allows you to enter the average number of hours to be calculated on the report.

Average Hours Selection Screen (ed)

  • If not ed, the Average Hours Start Date and Average Hours End Date fields appear on the screen, as shown in Figure 3: Average Hours Selection Screen (not [%=Variables.tick%]ed). These fields allow you to enter the starting and ending dates for the report to search for employee hours for the average hours.

Average Hours Selection Screen (not ed)

Selection Tab

The Selection tab lets you enter the plan types to display on the report. Up to 20 plan types can be printed on the report.

To View

Indicate whether you wish to view All or New on the report.

  • If All is selected, the As Of and Salary As Of fields appear on the screen, as shown in Figure 4: [%=Variables.Enrolment caps%] Tab - All. These fields allow you to include all employees based on the Personnel selection criteria, regardless of the Start Date.

Tab - All

  • If New is selected, Start Date and End Date fields appear on the screen in addition to the As Of and Salary As Of fields, as shown in Figure 5: [%=Variables.Enrolment caps%] Tab - New. These fields allow you to include all employees based on the Personnel selection criteria and the Start/End Dates.

Tab - New

Plan Types

Select the Plan Types you wish to display on the report, up to a maximum of 20.

Include 401k

Whether to report on the 401k plan types.

  • If ed, the 401k Plan and 401k Plan Start/End Date fields appear on the screen, as shown in Figure 6: [%=Variables.Enrolment caps%] Tab - 401K ([%=Variables.tick%]ed). These fields allow you to select eligible plans that are listed and enter the Start and End Week ending dates to report on.

Tab - 401K (ed)

  • If not ed, no additional fields appear on the screen, as shown in Figure 7: [%=Variables.Enrolment caps%]Tab - 401K (not [%=Variables.tick%]ed). The 401k plans will not be included on the report.

Tab - 401K (not ed)

Personnel Tab [US Region only]

The Personnel tab uses the standard selection screen criteria. However, it also includes a Show SSN field, as shown in Figure 8: Personnel Tab with SSN Options. This field lets you indicate how you wish to display the Social Security Number (SSN) on the report. The options are:

  • Full SSN - Displays the full SSN.
  • Last 4 Digits - Displays only the last 4 digits of the SSN.
  • None - Does not display the SSN.

Personnel Tab with SSN Options
