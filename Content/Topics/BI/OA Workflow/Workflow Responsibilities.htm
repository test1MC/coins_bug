Workflow Responsibilities

Once Roles have been defined within the system, their responsibilities within the workflow may be configured. Workflow Responsibilities are used to convert roles set in a workflow to actual user IDs.

In Role Maintenance a role is associated with a type and up to 3 variables (These variables are defined within a workflow).

In Responsibility Maintenance a series of records are created that map possible values of these variables to one or more user IDs. When the role is used in a workflow stage and the values match then the user IDs are used.

Field

Description

Type

This code is referenced when setting up a workflow role. Use Workflow Responsibility to set up different responsibilities depending on the value of a variable defined in the workflow.

Keys 1 – 3

The appropriate values of the Role key.

The value in these key fields corresponds to the variable set in the equivalent key of the role that is linked to this responsibility (see role maintenance).

If the value of this variable set at the workflow stage where the associated role is used matches this responsibility value (along with the other two key fields) then the user ids configured on this responsibility are used in that workflow stage.

Role

The person who must action this stage

Users

The user or users who are responsible for this role in a workflow.  These must be users’ setup already within the system.
