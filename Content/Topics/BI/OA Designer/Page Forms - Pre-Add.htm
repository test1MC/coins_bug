Page Forms - Pre-Add

A Pre-Add form is shown prior to the add form. It can be used to prompt the user for key data prior to the main form. This might allow certain key fields to be pre-selected prior to an add and require less field validation to be used on the main page. Or it might be used to select a record type before creating the record itself allowing a different page layout to be selected or fields to be hidden.

After completing the preAdd form the user is taken to the add form proper with the fields entered in the preAdd available in the URL.

You can perform validation on the contents of a preAdd page. The method preAddValidation is called after the preAdd page has been processed, just before the actual add page is requested.

The fields on the preAdd page will have been assigned to the fields in the t{&TABLE} record in the RSP. You should check this record as required and/or the URL contents using getVar() and throwError() as appropriate.
