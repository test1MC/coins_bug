Workflow Template (multiple Roles)

 

 

In this template example there are multiple roles which are defined as different authorisation levels required for a Purchase Order dependant on the value of the Order.  As the value of the order is important in this example it should be one of the variables which are defined on launch of the order. Therefore we need to capture this in the initial calculation. Once defined the variable poh_orderval is available to be used within calculation syntax.

Workflow Template definition (TRPROJ3)

Refer to the workflow template   TRPROJ3.  See details below:

Within the First Fork statement there is calculation syntax allocated which sends the workflow down the correct 'flow route'.    Calculation syntax should equate to a TRUE / FALSE (1/0) statement.  For further information on Calculation syntax refer to your designer Module 1 course notes. If the calculation equates to TRUE (1) then the Link will execute otherwise the option will not be available.

You will also note that within this workflow template the option to either Commit (P1) /Reject (P2) is dependent upon the value of the order. If the value is below the amount check for the authorisation level the option to Commit (P1) will appear however, if the value of the order exceeds this limit (5K), the option to send an email (E3) of the Order up the chain to authorisation level 2 will appear instead of the Commit Option (P1).   The Option to Reject (P2) will always be available as there is no calculation condition against that option.

The path for this workflow template shows the 'flow route' available.
