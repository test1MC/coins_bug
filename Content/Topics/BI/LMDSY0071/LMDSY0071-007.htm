Workflow Template Change

Stage decision makers

When leaving a stage you must always make allowances for all decision possibilities. Usually there are at least two decisions Yes/No scenarios, however there maybe conditions as to which options are available at decision time.  In this scenario we will be allowing the Level 1 authoriser to be able to commit the order or reject the order.

On the Stages Tab, Select the line P1 and click Copy

 

Change the Stage Label to P2

Change the description to Reject Order

Change the Calculation to method('poh-rsp.RejectOrder')

Set the Next field to E2

Click

Select the line for stage E1 and click Copy

 

Set Stage to E2

Change the Description to Email Buyer to notify of Rejection

Change the subject to Order {poh_ordno} has been rejected

Change the Next to END

Now we need to modify stage S1 to include the second option:

Select Stage S1 and click

Scroll to the Link section and add the Option for P2.

 

If we select the Diagram tab again and refresh the page, we should now see that there are two possible paths the workflow could take. You can drag the stages around to make them more readable.

 

 

Purge the workflow

Then re-launch 

If a decision has to be made by the user, the option to complete from the front page of the Activity Workbench is only possible if a default link is specified within the stage.

 

If this field is left blank, the user must open the task via the link and complete the task individually. In this example the selection on the Activity Workbench now provides the user with two options; to either Commit or Reject the Order. No default link is specified so a choice must be made between the two.
