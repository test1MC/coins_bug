Stages – Building the basic structure

A quick reminder of the design for this Workflow:

Stage 1 - The business process is that someone at a level higher than the buyer must authorise an order before it can be issued (S1)

Stage 2 - The authoriser will be given the ability to commit the order (Process P1)

Stage 3 – An email will be sent to the buyer to advise them that the order is approved – an Email Stage (E1).

The Body of our email is going to give the buyer a link back to their order, so that from the email they can go directly into and view the specific order that has been authorised.  To do this we need to Launch notepad and bring back the text file of the Order hyperlink that we saved earlier (or go into the details of any purchase order and capture it again).

The Text will look something like:

http://195.40.14.50/cgi-bin/oatrainbi/woframe.p?TopMenu=%25WPO&pvCILevel=1&program=wou005.p&MainArea=%WPO1010SPOH&%WPO1010SPOH=&kco=10&poh_mpo=M&POLive=TRUE%20&initFunction=%2525WPO101ABPHL&RaisePOFromReqn=&OptionOrderMenu=S&po_hdrRowid=0x000000000014c704&tip_type=MATERIAL&UpdateMode=POHeader&CIWTemplMode=&CILevel=&PILevel=&job_num=&jph_phase=&jwb_code=&REQLive=

First, delete the leading characters up as far as, but not including /woframe.p and replace them with, http://$[$cURL$]$

The reason we have to replace the beginning of the URL is that it is currently hardcoded to a particular server and environment.  Should you copy this workflow to another environment or change the server configuration in any way, the workflow link will break.  This new prefix will ensure the current server and environment is dynamically created each time.

Having fixed the hardcoding of the server, we also need to make sure that the Purchase Order in the email is also dynamically changed depending on the actual order being processed.  We need to make changes to the fields for the RowId, kco, poh_mpo, and tip_type.  Replace as follows:

http://$[$cURL$]$/woframe.p?TopMenu=%25WPO&pvCILevel=1&program=wou005.p&MainArea=%WPO1010SPOH&%WPO1010SPOH=&kco={kco}&poh_mpo={poh_mpo}&POLive=TRUE%20&initFunction=%2525WPO101ABPHL&RaisePOFromReqn=&OptionOrderMenu=S&po_hdrRowid={po_hdrRowid}&tip_type={tip_type}&UpdateMode=POHeader&CIWTemplMode=&CILevel=&PILevel=&job_num=&jph_phase=&jwb_code=&REQLive=

These replacement values correspond to the initialise variables we set up earlier.  Any fields that will ultimately be replaced within the workflow stages should be defined as variables.
