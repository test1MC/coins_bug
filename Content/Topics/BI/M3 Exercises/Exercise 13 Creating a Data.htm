Exercise 13 - Creating a Data Mart

This exercise takes you through the steps to create a Data Mart.  Some important things to keep in mind when creating Data Marts:

  • A Data Mart is company specific. If a cross company Data Mart is required it is probably best to create a 'Reporting Company' as the source.
  • A Data Mart is a stored snap-shot of information so a decision has to be made on when the Data Mart can be run and the frequency of the data.
  • To get the best results in a Data Mart the information should be gathered at a static period (i.e. overnight).
  • Snapshots of the data are a quick way to view figures and report on information at consolidated views.

The minimum requirements for setting up a Data Set are:

  • Data Set Code and Description.
  • Data Set Query.
  • Data Set table name.
  • Data Set fields

The data mart exercises are designed to be run on the Sample FirstLearn environment in the training room.  Special Care must be taken if the training is done outside of this environment as customers may have a large number of contracts available and the data may be unmanageable and excessive disk space may be used if care is not taken to control the number of records being extracted.

Data Marts is found on the OA & BI reporting menu under the sub-folder called Data Warehouse

 

Select the Data Marts option and click the New icon.  Remember that Data Marts are kco specific so ensure you are in the right company when you click new!

Enter the data Mart code – use the same name used in the Report Data Mart Field

Enter a description for the data Mart

Data Mart Tab

  • Data Mart table - dw_fact.
  • Extract – as a variable {RS_glp_fdate__2}.
  • This will be the date on the update page of the populate report. It is the key to how the Data Mart is stored. (i.e. Will it be stored on a Daily/Weekly/Monthly Basis). Keys Tab

Keys are created as a way to drill down through the Data Mart.  They should always correspond to the number of entries in the key.

  • The suggested best practice is to use three letter acronyms for key names (ie: COM for COMpany, GRP for GROUP – This makes identification easier).
  • Remember to always create a drill down summary in steps without jumping levels. (E.g. you could not create TOTCUSREF without creating a TOTCUS level).
  • The default tab is which Tab you want the key to default to.  Options are the 'Browse,Form,Time,default'
