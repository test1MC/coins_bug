Luckins Notes

General

Current version of Luckins supplied data is – LUCKINS DATAPLUS.

It is the USER’S responsibility to load the Luckins files to Unix files in the user’s Home Directory (user being the user who is to do the Luckins load to )

The Luckins and Steel Stock modules are mutually exclusive in the same company. If switching Luckins on, then you must ensure that the Steel Stock parameter is switched off

Luckins data is loaded into the stock tables. (N.B. these records would not normally be used as real stock items.)

Configuration

Stock Configuration > Parameters > Luckins.

(No other configuration is required)

Parameter

Description

LUCKINS?

Set to “Y” to switch the Luckins facilities on.

LUCKENT

Luckins provides several prices. This parameter stipulates which of these prices you want to use as the base cost price; that is the price you would apply all the discounts and agreements to and ultimately use when placing orders.

For the system to automatically use the Lowest price(greater than zero), leave this parameter blank (recommended).

LUCKLOW

If you have left LUCKENT blank to use the lowest price, then you need to set this parameter to “Y”.

LUCKPCE

You can set one of the standard stock sales prices to be labeled as the LUCKINS price. Entering the price number here will label that price with the word Luckins on the stock record. As we do not normally use these stock records as real stock items this is somewhat irrelevant so this parameter can be left blank.

LUCKVAR

The system is able to report on incoming new price variances. Set the level of variance here over which you want exceptions reported.. (Select the Luckins Variance Check enquiry to see these variances). (e.g. 10% would report on all products who’s price has increased by 10% or more over the existing price).

Preparation

Prior to the initial load of Luckins Data, it is necessary to establish a table of allowable units of price/measure. Unfortunately Luckins does not supply this information so we have to enter these units manually. In addition, does not use these units when placing orders etc, so we have to map the Luckins units to the standard units. (a list of Luckins units of measure is included in these notes.)

Loading the Luckins Data Files.

It is the user’s responsibility to load the luckins data files from CD or disc, to the user’s home unix directory.

The loading process is the same whether doing the initial load or the weekly update.

Luckins supplies nine files as follows:-

A0701234.CTL Control File

B0701234.CRV Item Details

C0701234.CRV Supplier Index

D0701234.CRV Product Range

E0701234.CRV Commodity Codes

F0701234.CRV Price Qualifiers

G0701234.CRV

H0701234.CRV

I0701234.BLN Bulletin Details

It is not absolutely necessary to load the A or the I file but you may as well load them for information.

Take care to load the files in the following sequence:-

A, I, C, D, E, F, G, H, B

You have to load each file discreetly by repeating the Load Luckins Data option and entering the individual file names at the prompt.

As the data is loaded relevant reports are generated.

Additional products (Non Luckins)

You are able to add non Luckins records to the database. It may be that you are aware of products which are not included in the Luckins data or you may not have selected certain data sets from Luckins in the first place. You are allowed to create your own product range records and your own product records. The system recognizes these items and automatically numbers them. In so doing it guarantees no duplicate keys are created. You are also able to add your own Manufacturer records.
