Adding an Account Line

The Account Line will retrieve specific data from the General Ledger account record. These accounts are already created and maintained in the General Ledger module.

The Account Maintenance browse screen lists all the information regarding the accounts and what information/data can be used.

Account Lines will display information from the General Ledger account record. For example you may want the balance of the account, the description of the account or the account number. These would all require an Account Line to be selected.

You select which accounts to include in the report by entering an account row, and filling in a ‘From’ and ‘To’ range, or a ‘can-do’ list in the Matches field. You can select accounts by the whole account, sector, expense code, analysis set etc. You can also use Report Tokens to allow the user to specify which accounts to include when they run the report. If an account line is shown in detail, a line will appear in the report for every individual account that is included in the selection. If the line is shown in Summary (detail flag un-ticked), only one line will be displayed showing the balance of the entire account range selected. Therefore, it is possible for the report definition to just contain one Account line (selecting all accounts) and if ticked as Detailed, will display a row in the report for every account.

Select the Account Line from the dropdown list. Select the Add button.

Enter additional information for the Report Row.

The new line is displayed showing the details required for input. Not all fields will be required to be entered.

In our example we are entering details for the account number. The fields required for input are defined in the table below.

Field:

Description:

Line:

The line number of the report row.

Description:

The description of the data you want to be displayed. This will print in a ‘Row Description’ column but not others such as ‘Account Description’.

Level:

The Level you wish to use. Account lines at the same level will be displayed together in a column on the report. This can be used for totals. In our example we are using just one level

Account:

The General Ledger account number or range of accounts for this line. Filtering can be used on each part of the full account code depending on the General Ledger accounts configuration. Matches or ‘can do’ lists can be defined to include tokens that are entered by the user at report run time to filter accounts and values to be included in the report.

Sector:

A sector or range of sectors to retrieve account balances for the report.

Expense:

An expense code or range of expense codes to retrieve account balances.

Main Headings:

The Main Headings to retrieve account balances e.g.  5 = Fixed Assets, 6 = Current Assets, 7 = Creditors etc.

Statutory Analysis:

The Statutory Analysis code or range to retrieve account balances

Company Group:

The group of companies to report on. 

This allows the user to write reports which access companies other than the ‘current’ company.  I.e. you may be writing a report in company No. 001, but wish to report on companies 001, 002 and 003. Specify the company group to select from here. Note that it may therefore be desirable to create a company group for each individual company (in the System module of ) as well as a group of ‘all companies’.  That way, you could specify a report that produced either report rows or columns of each company, plus a consolidated view.

Currency Type:

The base currency for conversion if you are using more than one currency.

Percent Line:

The line number to reflect as a % of another row e.g. Profit as a % of Total Sales.

Enter the row number of the report that you would like to show this line amount as a percentage of.  Note that the row number that you refer to must already exist in the report definition.  (i.e. it must be above this current line in the report).  i.e. row 10 could not contain a reference to row 11 in its percentage line field. However, row 11 could contain a reference to row 10.

Note that the Column tab lines must also be flagged as a % line for these calculations to work. Otherwise the report will show the value of the row, rather than its % value.

Divide Row:

The line number to divide a row by another row. 

Sort:

The order to sort the report.  This allows you to sort by account, account code element or analysis set so that accounts will show in an order different to the full account code sequence. You can have up to 5 levels of sort.  e.g. 1 = Account, 2= Analysis Code, 3 = Branch, 4 = Dept, 5 = Detail. 

Header/Footer Level:

The level corresponds to a set of header and footer column fields configured in Report Columns. The header/footer allows you to specify which header/footer to use at the relevant break for the corresponding sort option. These header or footer lines are then referenced on the Column tab of the report.

e.g. if you put a 6 in the header/footer level, then it would use the columns you had set up for show on levels H6 and F6 when displaying the header and footers.

Print Line:

Print the line on the report or just process to accumulate report values.

Print Zero:

Print zero balances on the report.

Reverse Value:

Display negative values as positive and vice versa.

Detail:

To print balances in detail or a summary. Detail mode will show all account codes in the selected filter ranges for account codes with one line for each account code. Summary mode will total all account balances on this report as one line.

Print When:

The type of balances to be printed on reports i.e. debits, credits.

Account Type:

The type of accounts to be included in the report i.e. Balance Sheet only, Profit and Loss only, or All Accounts.

Save Into:

This field is used for calculations and total rows of accounts which will then be printed and used on total lines. The unique name to group values for saving into this row i.e. the name of the accumulator. This allows you to add or subtract from a subtotal in a report (which you can ‘name’ to call upon later).

e.g. In a Profit and Loss Report, Sales might be added into an accumulator named for ‘Sales’ and Cost of Sales might be added into an accumulator named ‘CoS’ and these two would then be totalled into an accumulator called ‘GP’ for Gross Profit. Multiple Account lines and/or total lines can ‘save into’ a total.

Operand:

This works with the ‘Save Into’ field for calculations. Determines whether the values are added to or subtracted from the accumulator value i.e.  + or -

Class:

The font name and size for the report e.g. Arial or Bold, 10pt or 12pt.

Variant:

A variant to be used for Header or Footer Total.

Extra:

The format for the variant.

Enter all the required fields for the details for the account line of data that you require to be added to the report. Note that in the screen shot below the fields for Account, Sector, Expense, Main Headings and Statutory Analysis will vary from system to system depending on the General Ledger configuration.

You may want to enter some filters and selections in the Lookup fields for specific data to be displayed in your report, i.e. a specific range of accounts.

  1. Select the codes that you require from the list.

  2. Select the Save button.

    The Account Line will be entered and a new Account Line will be opened to enter more information for other accounts.

  3. Select the Undo button if you do not want to enter a new Account Line.

    You can also use the filters to add strings of codes to match, rather than using the From field and To fields for the Lookup filters. 

You can have as many Account Lines as you require for the report.

The report will still show a blank page if run and you will not be able to view the report in the Report Status browse screen. We have explained what we want to see, however have not detailed how we want to see it.
