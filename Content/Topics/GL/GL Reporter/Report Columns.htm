Report Columns

The Report Columns tab will enable the data to be displayed in the report. This takes the data in the Report Rows tab and displays it using the Report Columns tab. Each report row can have its own set of columns, or share a common set of columns.

There are many different Column Type to choose, however these are group into 5 main groups.

Column Type:

Code

Description:

Character:

CH

Columns used for data that is Alpha, Alpha Numeric or Account Numbers. e.g. Account Full Description

Literal

LI

This will display exactly what has been entered into the Column Description Field.  e.g. Trading Account

Decimal

DE

Columns that are using figures, or values that are decimals. These can be used as calculations.

Integer

IN

The data is numerical and is not decimal, i.e. period numbers

Date

DA

Columns where the data displayed are dates.

You must select the right Column Type to ensure that the data and formatting is displayed correctly.

Column headings are entered by using a literal column type and positioning the header using a negative number for the row value. To make it easier to review the report these columns should be defined first.

Adding a Character Type

There are two types of Character types to the Report. The first will enable you to select information from the General Ledger record. The second is to show the exact details you have entered on the Description field in the Report Rows tab.

Adding Account Details

This will be a Character Column type.

  1. Select the Account Number option from the dropdown list and click the Add button.

    The Line will be available for you to enter the details for the display.

  2. Enter the details into the relevant fields.

    Field:

    Description:

    Label:

    The label to identify the Data

    Format:

    The format you want the data to be displayed

    Show On:

    Relates to the Level of the Data Rows and also the type of Column. This can be a comma separated list if more than one value is required. The

    Row:

    The number of rows down on the page to start the data

    Column:

    The number of columns across the page to start the data. The starting position is (approximately) the number of characters across the page. Note that if the format of a column is wider than the difference between the two Col fields, the following columns will not be aligned.

  3. Enter the format that we are using for the data. The Account Number is X(9), which means it will display 9 digits.
  4. Enter the Show On, which in our example id D1.

    The Show On field determines which line types the column will be printed on. Use a comma-separated list of one or more from the following:

    B

    Blank Rows

    Dn

    Account Rows

    Fn

    Sort footers

    Hn

    Sort headers

    Jn

    Contract rows

    Sn

    Standard text rows

    X

    Additional text rows

    Where n is a number that represents a level. For account and contract rows, you can think of levels as variations of the same type of line on which some of the columns may be different. For example, if you want one set of account lines to show closing balance and movement, and a second set to show budget and forecast, give the two row types different levels, and put the relevant level number after the "show on" code for each column. For header and footer lines, the levels correspond to the sort levels as defined on the row. A dot instead of a number means "show on all levels for this type of line".

  5. Click the Save button.

    The report can be run and show the details on the report.

Adding an Account Description

Either the Account Description or Account Full Description can be used as a column.

  1. Select Account Full Description option from the Item field drop down list and click the Add button.

    A new line is created for the Account Full Description line in the report.

  2. Enter the Account Full Description details in the relevant fields.  The fields are defined in the table below.

    Field:

    Description:

    Line:

    The line number for this Report Column. 

    Label:

    The description for the column. 

    Type:

    The data type of the report column being added will be CH for a character field.

    Format:

    The display format for the column value.  Standard Progress display formats are used e.g. x(20) = show 20 characters

    Show On:

    The line types to show for this report column.

    D - Relates to account rows and should be followed by the relevant level number from the row setup e.g. D.

    Row:

    The row number to display the value.  For headers i.e. column headings use negative numbers.

    Col:

    The column number to display the value.

    Alignment:

    The alignment of the column e.g. text defaults to the left.

    Class:

    The font name and size for the report e.g. Arial or Bold, 10pt or 12pt.

    Variant:

    A variant to be used for Header or Footer Total.

    Extra:

    The format for the variant. 

  3. Click the Save button.

    A new line is displayed; click the Undo button.

Adding Row Descriptions

The second way of adding descriptions is to use the descriptions you have already entered. These are the descriptions in the Report Row tab for the data.

You can use the details entered rather than the actual names of the accounts. This is more widely used when consolidating the accounts to show a summary rather than the full details.

  1. Select the Row Description option from the dropdown list and click the Add button.

  2. Enter the details in the relevant fields for the row’s data to be displayed.

  3. Select the Save button.

    If the wrong formatting is shown you can truncate the characters that are displayed in the description column. You may have to open the line and change the number of characters to be displayed.

    The report can be run and the details are available to be shown.

Adding a Literal Report Column

A Literal Report Column (LI) will display text entered in the Label field of the column type e.g. Trading Accounts

  1. Select Literal option from the Item field drop down list and click the Add button.

    A new line is displayed for you to enter the description of the account line.

  2. Enter the Literal line detail in the relevant fields.  The fields are defined in the table below.

    Field:

    Description:

    Line:

    The line number that identifies this column in the setup.  The line numbers are also used in the calculation field.

    Label:

    The unique text required to be displayed in the report.

    Type:

    The data type of the report column will be LI for a literal field.

    Format:

    The display format for the column value.  Standard Progress display formats are used e.g. x(20) = 20 characters

    Show On:

    The line types this column should be shown on may be one or multiple types for a line.

    S - Relates to standard text rows and should be followed by the appropriate level e.g. S1, S2 etc.

    Row:

    The row number to display the literal line.

    Col:

    The column number to display the literal line.

    Alignment:

    The alignment of the column e.g. text defaults to the left

    Class:

    The font name and size for the report e.g. Arial or Bold, 10pt or 12pt.

    Variant:

    A variant to be used for Header or Footer Total.

    Extra:

    The format for the variant.

  3. Click the Save button.

    A new line is displayed for you to enter the next Literal Line. 

  4. Click the Undo button to remove the line.

Adding an Integer Report Column

An Integer (IN) Report Column is used to display numbers without decimal places.  For example the line type Period Number, Period <= Reporting Period etc.

  1. Select Period Number option from the type field drop down list and click the Add button.

    A new line is created for a Period Number in the report.

  2. Enter the description for a Period Number to be printed on the report.
  3. Enter the Period Number details in the relevant fields.  The fields are defined in the table below.

    Field:

    Description:

    Line:

    The line number that identifies this column in the setup. 

    Label:

    Enter text to describe the information to be displayed in this integer line e.g. Period Number.

    Type:

    The data type of the report column will be IN for an Integer line type.

    Format:

    The display format for the integer value e.g. 9999 value.  Standard Progress display formats are used. 

    Show On:

    The line types this column should be shown on.

    Row:

    The row number to display the value.

    Col:

    The column number to display the value.

    Item:

    This will display a description of the column type being added.

    Offset:

    The GL period offset to be applied. This will be a positive or negative number that will determine which GL period should be used to determine the values for this row. i.e. a value of ‘-1’ will be the previous GL period to the period selected when the report is run. A value of -12 will be 12 previous periods and +1 will be the next period.

    Company Group:

    The Company Group to report on. 

    Alignment:

    The alignment of the column e.g. text defaults to the left

    Class:

    The font name and size for the report e.g. Arial or Bold, 10pt or 12pt.

    Variant:

    A variant to be used for Header or Footer Total.

    Extra:

    The format for the variant.

  4. Select the Save button. 

    A new line is displayed.

  5. Select the Undo button to remove the line.

Adding a Date Report Column

A Date Report Column enables dates to be included in your reports to enhance the report information.  Data Types available include Today’s Date, Period End Date (long) and Period etc.

  1. Select Today’s Date option from the type field drop down list and click the Add Button.

    A new line is created for Today’s Date in the report.

  2. Enter the description for the date to be printed on the report e.g. Today’s Date
  3. Enter the Today’s Date line details in the relevant fields.  The fields are defined in the table below.

    Field:

    Description:

    Line:

    The line number that identifies this column in the setup. 

    Label:

    Enter text to describe the information in this line e.g. Today’s Date

    Type:

    The data type of the report column will be DA for a Date line.

    Format:

    The display format for the column value.  Standard Progress display formats are used. 

    Show On:

    The line types to show in this column.

    Row:

    The row number to display the value.

    Col:

    The column number on which to display the value.

    Item:

    This will display a description of the column type being added.

    Mask:

     

    Period:

    A selectable list of the valid General Ledger periods (1-12) plus TP for ‘This Period’ and TY for ‘This Year’.

    Offset

    The GL period offset to be applied. This will be a positive or negative number that will determine which GL period should be used to determine the values for this row. i.e. a value of ‘-1’ will be the previous GL period to the period selected when the report is run. A value of -12 will be 12 previous periods and +1 will be the next period.

    Company Group

    The group of companies to report on. 

    Alignment:

    The alignment of the column e.g. text defaults to the left

    Class:

    The font name and size for the report e.g. Arial or Bold, 10pt or 12pt.

    Variant:

    A variant to be used for Header or Footer Total.

    Extra:

    The format for the variant.

  4. Click the Save button.

    A new line is displayed for you to enter the next Date line.

  5. Click the Undo button to remove the line.
