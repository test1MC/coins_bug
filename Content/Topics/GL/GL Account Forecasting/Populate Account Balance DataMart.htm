Populate Account Balance DataMart

The GL Account Balance Data Mart (%WGLGLA001) has been changed to extract forecast amounts for future periods in Data Mart.

By default the forecast data is populated from the GL Forecast(s) entered in the into which the Data Mart is extracted, irrespective of the Forecast status. The Budget Type defaults from the GL/FBUDTYPE parameter.

If GL/USEBPFXT=Y and GL Forecast(s) exist in the company, a new box appears below Budget Type: Use only Completed Forecast. If this is ed, only completed forecast(s) are used when populating the Data Mart.

The Data Mart Movement column has been renamed to Amount and is used for storing Rolling Forecast amounts of forecast periods.

The Data Mart Quantity column is used for storing the Rolling Forecast quantity of forecast periods.
