Set up an Asset Trigger

To set up a trigger:

Go to Fixed Assets > Setup > Maintenance > Asset Trigger Definitions.

The value in this field is automatically filled in based on the Data Type selected, but it can be changed.

Trigger Type

The trigger type. This, combined with the Value, determines how the expiry value for the trigger is reset. The types available depend on the data type.

Trigger Type

Value

This is the reset value, which determines what the Expires field on the trigger is reset to. The entry in Trigger Type governs how the trigger is reset.

The trigger has a current value and a "trigger point" - a value at which it expires/is triggered. Which of these changes on reset is affected by the trigger type. For example:

  • If you are doing a 10k service on a car you will want the current reading to be unaffected by reset and the trigger point to be updated (from 50K to 60K for example). You would do this with a PL (plus) type trigger - there is no "Reset To" and "Value" (10k) will determine the reset trigger point value (new trigger point will be current + Value).
  • For a 500 hour use service, you may want the trigger point to be constant (500) and the current reading to reset. This would be done with a GE type, and "Reset To" would be zero.

Triggers can be reset to a fixed value (like "0" or "yes"), to the current value of the trigger plus or minus a fixed number, or to a number of days, months or years after today's date.

Reset To

The value to which you want to reset the current value when the trigger is reset (for EQ, GE or LE type triggers).

Main Trigger

Link this trigger as a secondary to primary trigger.

This can be used in situations where the event can be fired for more than one reason; for example, service every 10k or 6 months.

Validation

How responds when someone selects the asset.

The trigger can be set up to give either a warning or an error message when the asset is selected (for example when entering a dispatch or changing an item on hire). How do you want this trigger to behave?

  • None. does not give a warning or an error.
  • Warning. issues a warning message but allows you to select the asset. Use this, for example, if the trigger relates to a service or inspection that is not mandatory.
  • Error. will not allow you to select the asset. Use this, for example, if the trigger relates to a service that must be done before the asset can be dispatched.

In workshop procedures, all errors are treated as warnings. This means that you can still book the asset into the workshop.
