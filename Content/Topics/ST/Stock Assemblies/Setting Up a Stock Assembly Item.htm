Setting Up Assembly Items

A new tab, Assembly, is available when maintaining item in Item Maintenance. Here you define the type of assembly (how the item behaves), a table of parts and optionally a calculation.

Stock Item Maintenance - Assembly Tab

Fields

Assembly Type

This can be one of:

  • Normal. This is the default (for items with no assembly components). It can also be used with assembly components for items that are pre-assembled (for example, roof trusses); only the assembled item will appear on the issue.
  • Site. This is used for items with assembly components if you want to charge for the individual components. When the assembly item is entered on the issue, the required components are added to the issue, and these are what are charged for.
  • On Issue. This is used for items with assembly components if you want to charge for the assembly item. When the assembly item is entered on the issue, the required components are added to the issue, but the assembly item is the item charged for and the customer won't see the detail.

Component Browse

The component browse allows you to specify the individual components that make up the assembly item.

Assembly Item Component Browse

Component Browse Fields

Item

The item used as part of the assembly.

Description

The description of the item.

Fixed

The fixed quantity. If the component item is not used in the calculation, this is the quantity of the component item per single assembly item. If a calculation variable applies to the component item, and the fixed quantity is non-zero, the calculated value is multiplied by the fixed quantity to give the quantity of the component item.

Variable

The calculation variable that determines the quantity of this item.
