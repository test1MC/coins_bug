Configuration

CO/TAXTYPE Parameter

Parameter CO/TAXTYPE must be set to SALES in order for the Sales Tax functionality discussed in this document to work correctly.

ST/TXBYITEM Parameter

A new parameter called TXBYITEM has been added to the module. When set to Y, dual taxability options are switched on.

Parameter

Description

TXBYITEM

Whether to enable functionality to exempt certain items from sales tax.

Exempt Tax Code Field

Issuing Process

The issuing process for defaulting a Tax Code now works as follows:

  • If Item is not exempt on purchase and issuing to a , the ’s Exempt Tax Code is used.
  • If Item is not exempt on purchase and not issuing to a , the Location's Exempt Tax Code is used.
  • If Item is exempt on purchase and issuing to a , entered Analysis dictates a Tax Code. A Tax Code is used from Status > > specific > Cost Code Maintenance > Tax Code for used Cost Code and Category.
  • If Item is exempt on purchase and not issuing to a , the Location's Sales/Use Tax Code is used.
  • You can override the default Tax Code on the line.

Transfer

Transfer currently has no Sales Tax on Location Transfers and its processing therefore remains the same.

Other Screen and Processing Changes

PO Orders

When using Orders in Procurement > More Orders, the Item Code now defaults as a blank value but is overridden to the appropriate Tax Code from the Location based on the Item setting after saving:

  • If Item is not exempt on purchase, the Location's Sales/Use Tax Code is used.
  • If Item is exempt on purchase, the Location's Exempt Tax Code is used.

PO Add Order - Single Form Entry

When using Add Order - Single Form Entry in Procurement > More Orders > Orders (Single Form), the Item Code now defaults in the appropriate Tax Code from the Location based on the Item setting.

PO Import Options

If the supplied data for a PO Import has blank Sales Tax, the Item Code now defaults in the appropriate Tax Code from the Location based on the Item setting.

Enterprise Service Bus (ESB)

If the supplied data for an ESB has blank Sales Tax, the Item Code now defaults in the appropriate Tax Code from the Location based on the Item setting.

Copy Order from Template

When copying an Order from a template, the Item Code is overwritten with the appropriate Tax Code from the Location based on the Item setting.
