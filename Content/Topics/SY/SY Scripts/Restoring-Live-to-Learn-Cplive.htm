Restoring Live to Learn/Cplive

This script typically is used when the data on a non-live environment needs updating either for testing purposes or before an upgrade.

The script does not impact any downtime on Live.

Please note this only copies the databases and does not copy any attachments or document management files.

 

The first thing the script checks is that the two environments are on the same version.

It then asks you to confirm you are happy to restore the backup over the environment (always check the date of the backup you are restoring)

Once you are happy to proceed it then carrys out the following :-

  • Unzips the backup
  • Shuts down the non-live environment
  • Restores the Live backup to non-live – please note you have to confirm to overwrite the data – again check date and time
  • Update parameters – Y to apply
  • Restarts the non-live environment

Once completed you should now check that both environments have the same fixes installed.
