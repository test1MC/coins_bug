Dynamic Queries - Query Security

By default, any query defined will be available for users who have access to run the dynamic query functions.

If any of the queries are for more sensitive data, security can be set up for those queries. If you create a function Q<QueryCode>, for example QJOB for the JOB query, then access to this function can be granted like any other function using groups etc. If a query is denied then it will not appear in the dynamic query browse and the user will not be able to run it. It will also be removed from the list of available queries in Report Builder (see 1, Dynamic Queries - Report Builder).

If a user happened on a URL that took them to the dynamic query (see 1, Dynamic Queries - Functions) then they would receive a security error and the /report would not work.
