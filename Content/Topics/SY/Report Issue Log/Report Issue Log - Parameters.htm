Reporting Issues - Parameters

The following new SY parameters control the issue reporting functionality.

Parameter

Description
ISSREP Whether email, workflow or both should be launched when an issue report is raised. The default is none, in which case the reporting button is not available.
ISSMAIL The email address (or addresses) to which to send the report, for example the address for your support desk or support application. Required if ISSREP is Email or Both.
ISSWF The workflow to launch on raising an Issue Report. Required if ISSREP is Workflow or Both.
ISSWFKCO The company number in which the workflow has been defined.
ISSFUNC

The function used for the input form that opens when the user selects the issue report option. If this is blank, the default function is %WSY2580FXXX.

The title or tooltip of the button is taken from this function. If you need to change the default title “Report Issue”, create a + function and change the context description. Alternatively create a custom input function/form and set the desired description, and enter the function code here; this will be used instead of the standard.
ISSCLASS

A list of classifications to be shown on the Report Issue form; for example:

Suggestion, Issue, Query

If this is blank then a fill-in field will be shown. If entered then a drop-down selector containing the list will be shown.
