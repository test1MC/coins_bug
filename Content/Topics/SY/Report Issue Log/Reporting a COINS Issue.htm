Reporting a Issue

If you want to report a problem with , you can create an Issue Report.

If the issue relates to a page, you can create an Issue Report from the button in the help frame.

Report Issue Button - Help Frame

If the issue relates to a report, you can create an Issue Report from the Report Status workbench, using a button on the row detail of the report.

Report Issue Button - Report Status Detail

The issue report is opened in a over the current page and the User, Function and additional System Detail fields are prefilled.

Report Issue Screen - Function

Report Issue Screen - Report

Report Issue Fields

Field
Description

Classification

The type of issue you are reporting. The classifications are configurable.

Description

Your description of the issue you are reporting.

User

Your user ID. You can change this if necessary.

Attachment
You can optionally attach a document (for example, a Word document containing further information and/or screenshots). For reports the report PDF, Excel and log files are automatically included in the email or workflow instance.
Function
The current function or the report function.
System Detail
System information, which includes:
  • The browser agent string
  • Whether the page is a user-modified (+) page
  • Any errors recorded on the user’s console (recent error messages)
  • The URL of the current main area window

Or the report parameters if you are reporting on a report.

Version
The current version.

You must complete the classification and description section and can override the user field if required.
