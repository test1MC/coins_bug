Setting up a Scheduled Task for Outlook Integration

If the integration is being run as a console program (rather than as a Windows service), it will run once and then terminate.  To enable regular replication, the replication software needs to be set up as a scheduled task, running as the replication account. Use the Console Schedule Task Setup screen during the installation to set up the task schedule.

  1. In the Account Name, PW and Domain fields, enter the name, password and domain for the replication account.
  2. Choose how often you want the scheduled task to run.
  3. Click Create to create the schedule.

If the account that the scheduled task will use does not exist, click the Launch Task Scheduler button and set up the account.

The installation routine will create the initial schedule, but if you need to make any changes to the schedule later, use the Launch Task Scheduler to make the changes using the Task Scheduler.

The installation process will also create a desktop shortcut that allows you to run the integration program directly. On the Polling Interval screen, select how often you want the integration program to poll the Exchange server:

Click Next. The final screen allows you to confirm that you want to install the application.

When you click Install, the installer creates the necessary files, sets up the schedule and creates a desktop icon to run the integration as a console program.
