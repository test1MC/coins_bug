Mail Assistant - OA Configuration

NOTES

The following configuration is required to allow OA users to use the Mail Assistant.

  1. A OA concurrent user is required for each Mail Assistant user.
  2. The role %MAILADDIN must be available in the environment which  Mail Assistant is intended to work with.

  3. On the user profile of each  user who will use  Mail Assistant:
    • the Mail Add In role.
    • Set the Replication Mail Server field to the AWBServer value in the Replication Software configuration file.
    • Set the Replication Mailbox field to the User Principal Name (UPN).
      • To check the UPN for Exchange on premises please speak to your Active Directory administrator, or log in with the impersonation account and run the following in Command Prompt “whoami /upn”.
      • For Exchange Online, UPN is the e-mail account.
  4. If necessary, set the SY parameter EWSEURL to the external EWS URL.

    There is an issue that can occur with the Microsoft add-in framework where the authentication token returns a token based on the external Exchange Web Service URL, but the URL provided by the authentication mechanism is the internal EWS URL. This results in the request failing. To overcome this, set the parameter to the external EWS URL. will then retry the request using this external URL if the first request fails.

  5. Give the user access to the Mail Assistant functions. These begin %WSY1650 and control which workbenches the user has access to when using Mail Assistant.

    The user will need to be assigned the correct role: Mail Add in. (This can be done through the user’s profile > Groups > Mail Add In).

    They will also need to be a NAMED USER.
