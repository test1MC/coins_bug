Background Process Control – Parameters

The following existing parameter is used:

ID

Description

WEBLOG

This parameter should be configured to contain a list of token and value pairs for each of the log files that are to be viewed with the log viewer.

Webspeed agents should have the agent name and the log file. Use specific names for the agents (rather than WebSpeed=); this allows different WebSpeed agents to be found in the log files.

For example:

oalive=$BASE/var/diag/ oalive.server.log, oaliveas=$BASE/var/diag/ oaliveas.server.log, SRCH=$BASE/var/diag/ syusrch.log

The following key tokens should be used:

  • WFMON – WF monitor
  • ESBW_XXX – ESB sender (XXX is the name)
  • ESBR_XXX – ESB reader (XXX is the name)
  • ESBE – ESB Engine
  • ETCW – ETC sender
  • ETCR – ETC receiver
  • ETCRO – ETC router
  • SRCH – Global Search

If the corresponding token for the log file is not present then the log file will not be shown.
