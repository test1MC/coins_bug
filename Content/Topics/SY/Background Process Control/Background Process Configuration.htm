Background Process Configuration

The new function Background Process Configuration (%WSY1330SMNP) allows you to configure the required background processes.

Background Process Configuration Screen

The Processes tab shows the single process background processes, together with a selector for each of the multi-process background processes (ESB send/receive) which controls the visibility of the ESB Send and ESB Receive tabs.

Field
Description
boxes
Whether the process is required.

Status

Shows the current status of the process (green if the process can be found to be running; red if it is not running). The status shown is evaluated at the point the page is refreshed.

Text next to the status indicator shows the date that the process started (or the time, if it started today).

Auto

Indicates whether the process will be started when the database brokers script is started.

Poll Interval

A setting on some of the processes; this is the number of seconds it will pause between looking for things to do.

Start/Stop
These allow control of the process.
Log
Allows you to view the log file (providing it is configured in SY/WEBLOG).

The ESB Send tab contains additional settings for each ESB sender process.

Background Process Configuration - ESB Send Tab

It also shows the status, and has START, STOP and LOG buttons.

Field
Description

Poll

The polling time in seconds (defaults to 10).

Endpoint(s)

The endpoints that will be sent by this process.  This is a CAN-DO list of endpoint names.

Retries

The number of retries that will be tried before stopping the queue (defaults to 6).

Multi
The number of messages that will be batched together and sent if there are consecutive messages for the same endpoint (requires ESB configuration to define the messages that can be batched).
Auto
Whether the process should be started at DB broker start.

The ESB Receive tab contains additional settings for each ESB receiver process.

Background Process Configuration - ESB Receive Tab

The ESB receiver is used ONLY when SONIC ESB is in use. It reads messages SONIC ESB queues and processes the message payload with a service.

Field
Description

Poll

The number of seconds to wait before polling for more messages.

Endpoint

The SONIC endpoint to read messages from.

Queue

Whether the SONIC endpoint is a queue or a topic.

Reply To
If set then the Sonic Queue which will receive the responses to the message.
Auto
Whether this process should be started automatically with the DB brokers.

If you start or stop any of the processes, you will need to refresh the page for the new status to show.  It can take some time for the process to start and stop.
