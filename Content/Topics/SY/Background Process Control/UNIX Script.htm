Background Process Control – UNIX Script

A new script is now created by COINSAdmin and placed in the $BASE/bin directory.

coinsctl is the control script for all background processes

It is called when the start and stop buttons are selected on the screens and it consults the coins.cfg configuration data to decide what to do.

It has the following syntax:

coinsctl START|STOP|TEST|LOG [<TYPE>|ALL|AUTO [<NAME>]]

The first parameter is an action:

START and STOP will start or stop the process.  For stop this involves removing a controlling .lk file which the process checks periodically and the absence of which causes the process to exit.

TEST will list the matching processes but will do nothing

LOG will view the tail of the corresponding log file(s).

The second parameter is the process type:

<TYPE> is the process type e.g. WFMON, ESBW, ESBR, ESBE, ETCW, ETCR, ETCRO or SRCH

Alternatively, ALL can be used and all processes will be actioned.

Alternatively, AUTO can be used and all auto starting processes will be actioned.

These last two are used for automatic start and stop with the DB brokers.

The third parameter is optional for ESBW and ESBR and allows you to specify the particular sender or receiver to be address.

Examples:

List all processes

coinsctl TEST ALL

Test WFMON

Test ESBW_APPSTUDIO

Test ESBW_MOBILE

Test ESBW_NORMAL

Test ESBR_NORMAL

Test ESBE

Test ETCW

Test ETCR

Test ETCRO

Test SRCH

coinsctl TEST AUTO

Test WFMON

Test ESBW_APPSTUDIO

Test ESBW_MOBILE

Test ESBW_NORMAL

Test ESBE

Test ETCW

Test ETCR

Test ETCRO

Test SRCH

ESBR_NORMAL is omitted because it is not set as auto start.

 

coinsctl TEST ESBW

Test ESBW_APPSTUDIO

Test ESBW_MOBILE

Test ESBW_NORMAL

Just the ESBW processes are shown

 

coinsctl TEST ESBW NORMAL

Test ESBW_NORMAL

Just the ESBW_NORMAL process is shown

 

If you replace the action with START or STOP then the processes listed would be started or stopped.

If you replace the action with LOG then the tail of the log file is shown.
