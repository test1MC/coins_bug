Set Up Currency Codes

Use Currency Maintenance to set up codes for the currencies you will need to work with. You need to set up currencies separately in each company.

 

 

Field

Description

Currency

A code for the currency, for example, a 3-character International Standards Organisation (ISO) code.

The code for the Euro must be EUR.

Description

The name of the currency.

Cheque Wording

The wording for the currency that will be printed on cheques; for example: 'Dollar', 'Pound'.

Plural Form

The wording for the plural form of the currency.  This is the wording that will be printed on cheques (when they are configured to print the amount in words); for example: 'Dollars', 'Pounds'.

Sub Unit

The sub unit of an amount as it will appear on a cheque; for example: 'Cents', 'Pence'.

Accuracy

The accuracy to which currency amounts should be calculated. This prevents calculations producing inappropriate divisions of the currency.

For example:

  • for Japanese Yen, the accuracy is 1.000, because there is no subdivision of the Yen.
  • for Pounds Sterling, the accuracy is 0.010, because one pound is divided into 100 pence.
  • for currencies that allow ½¢, the accuracy would be 0.005.

Format

The display format.

You can use this, for example, to prevent currencies without subdivisions (like pence or cents) showing as decimal values.

Exchange Variance

The General Ledger account to which a realised variance for this currency (for example, the difference between the invoice and the payment when converting from foreign currency to base currency) is posted. Also known as exchange gains and losses.  This should be a P&L account.

Unrealised Variance

The account to which journals to represent unrealised variance at the end of a financial period are produced using the Unrealised Variance Report.

Rounding

The account to which differences due to rounding are posted.  This should be a P&L account.

Detail

Whether the exchange variance is posted to the General Ledger in detail:

 = Post in detail (transactions are posted).

 = Post in summary (batch totals only are posted).

Divide

 = When calculating currency conversions, divide by the exchange rate (this is the normal way of expressing currency conversions).

 = When calculating currency conversions, multiply by the exchange rate (this is used only occasionally - for example, if the base currency is the US Dollar, it is the convention to express the GB Pound as a multiply rate, and all other currencies as a divide rate).

(This is not used by the Euro conversions due to the triangulation rules.)
