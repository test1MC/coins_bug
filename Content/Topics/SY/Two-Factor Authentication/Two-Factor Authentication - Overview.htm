Two-Factor Authentication - Overview

There is a new an optional feature to require users to authenticate with a normal user ID and password but also with a second method which involves a time-limited passcode that is sent to the user via SMS or email.

This greatly enhances the authentication of users in to and is particularly relevant if the system is available on the public Internet.

The existing authentication methods are preserved. If using LDAP then the user ID password authentication will continue to be with LDAP. If using authentication then the user ID and password will be checked with .

The extra authentication is intended to be used with the Remember User ID – Overview feature introduced in v11.05. This allows a user’s authentication to be stored securely in private data in the browser so that they need to authenticate with user ID and password (and now with passcode) less frequently than each time they access .

The password hashing algorithms have also been strengthened.

NOTE

Authentication for mobile apps remains a simple user ID and password.
