Two-Factor Authentication - Parameters

The following new SY parameters govern two-factor authentication:

Parameter Description
AUTH2EXP

The number of seconds that a two-factor authentication passcode will remain active. If omitted then 60 will be assumed.
AUTH2IPS

The range of IP addresses that will be included for two-factor authentication.

Access from this range will be made to authenticate with a passcode. If this is set to * then all logins will require it. If blank (default) then two-factor authentication will not be used.
AUTH2KEY

The length of the two-factor authentication passcode key (default is 6, maximum is 9).
AUTH2SMS

Whether to use SMS as the primary method for two-factor authentication (Y or N).

If set to Y then a suitable SMS plan will be required and the notification of passcode will be sent via SMS. If the user does not have a number defined, email will be used. If the user does not have an email address or number, they will not be able to gain access.
