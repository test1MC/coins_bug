Implementing Secure Sockets Layer

Whether it is due to the assurance of the additional security provided by Secure Sockets Layer or to conform to corporate applications policy, LDAP Directory user authentication can be configured to demand SSL/TLS sessions with a verified server certificate.

This task covers the implementation of secure sockets layer for the communications channel between and the Corporate Directory server.

This task is optional. However, implementing an SSL connection to the Corporate Directory server will encrypt all communications between and the Corporate Directory servers.

When activating Secure Sockets Layer, please take care to note the Expiry Date on the certificate authority certificates being installed. The installed certificates must be updated prior to their respective expiry dates. Failure to do so will result in refusing access to all user accounts configured to use the LDAP Directory user authentication mechanism.
