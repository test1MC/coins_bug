Installing Certificates for

Once all required Certificate Authorities certificates have been transferred to the server, you must make them available to so that is able to trust the Secure Sockets Layer connection to the Corporate Directory servers.

Perform the following steps using the administration account on your server. You must be logged into the server using the server’s root account.

  1. Log into the server using the administrative account login details.
  2. Create a directory to store the approved set of Certificate Authority certificates. recommends the creation of: /usr/local/etc/coins/certs

  3. Move the certificate files into the certificate repository and enter the certificate repository.

  4. If you have extracted the certificate files from a Microsoft Active Directory server, then you will need to convert the certificates from DER encoding to PEM encoding. If the certificate files are already PEM encoded, then you can skip to step 5, otherwise convert each certificate by:

    Replace <certfile> with the name of the certificate file (without the .cer extension) and repeat this process for each of the loaded certificate files.

  5. Now you have PEM encoded certificate files. For to be able to successfully match the directory server’s certificate to a certificate file in the repository, each certificate must be referenced by its unique hash key. This requires the determination of the correct hash key for each certificate file and the creation of a symbolic link to that hash key.

    The symbolic link file should be named as the output hash value (from the first command) with a .0 file extension. Repeat this process for each of the PEM encoded certificate files in the certificate repository.

  6. The file permissions on the certificate files need to be altered to allow the coins account to access them.

    Once all certificates have been installed, converted and renamed ready for use by , we recommend that you test the certificates to ensure that the secure sockets layer can be initiated between and the LDAP directory servers.
