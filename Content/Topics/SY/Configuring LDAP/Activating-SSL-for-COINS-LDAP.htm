Activating SSL for LDAP user authentication

After you have installed and verified the corporate LDAP Directory servers’ certificate authority certificates, you are in a position to activate secure connections from to your corporate LDAP Directory servers.

 

Do not proceed with this task if you have not been able to successfully complete the steps outlined in Testing the installed certificates. If the installed certificates are not correct, then may refuse access to all accounts that have been modified to use the LDAP Directory user authentication mechanism.

 

Complete this task using OA.

  1. Log into the OA using an administrative account that has the ability to alter system parameters.
  2. Navigate to System module > System Setup > SY Parameters.
  3. Find the LDAPCERT parameter and update it such that it refers to the path of the certificate repository, as created in Installing Certificates for COINS

Once the entered value has been saved, will attempt secured communications with the corporate LDAP Directory servers.

If you need to disable secured communications with your corporate LDAP Directory servers, simply clear this parameter so that it has no value.
