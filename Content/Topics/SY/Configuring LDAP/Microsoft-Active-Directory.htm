Microsoft-Active-Directory

This sub-task covers the process required to extract the signing Certificate Authority’s certificate from an Active Directory server.

  1. Using a domain or local administrator account, log into the Active Directory server.

  2. From the Windows Start icon, select “Run” and enter: mmc

    Press Enter to launch the Microsoft Management Console. If prompted for User Account Control verification, accept it with a “Yes” response.

  3. From the displayed console, select File → Add/Remove Snap-in... as shown below:

  4. From the Add or Remove Snap-ins window, highlight Certificates and then click the Add button

  5. Select Service account and click the Next button

  6. Select the Local computer option and click the Next button

  7. Highlight Active Directory Domain Services and click the Finish button.

  8. The Certificates - Service (Active Directory Domain Services) snap-in should now be added to the console. Click the OK button to accept the added snap-in.

  9. Open the Certificates tree on the left of the screen and navigate to NTDS\Personal→ Certificates. You need to identify the Issuer of the certificate used for the SSL connection. If there are no certificates listed, as shown below, then it appears that Windows uses the first available Server Authentication certificate found on the local computer. If you see a certificate listed, skip to step 11.

  10. Repeat steps 4 to 6, this time selecting Local Computer at step 5. Find the first certificate within Personal → Certificates that contains an intended purpose of Server Authentication

  11. Identify the certificate issuer as shown below:

  12. Navigate to Certificates – Service (Active Directory Domain Services) → NTDS\Trusted Root Certification Authorities →Certificates and locate the Issuer’s certificate

  13. Double-click on the Issuer’s certificate and in the opened window click the Details tab.

  14. Click Copy to File and then select Next on the opened window.

  15. Select DER encoded binary X.509 (.CER) and click the Next button

  16. Enter a file location on disk to use to store the certificate and then click the Next button. Use a meaningful name for the certificate file so that it is easy to identify

  17. Review the export request and click the Finish button.

    If the export was successful an acknowledgement message will be displayed. The generated certificate file will contain the Certification Authority’s certificate (or CA cert for short). This file must be transferred to your server for use by in making a secured connection to the Active Directory server.

    If there are many Active Directory servers within your domain, then this process must be completed for each Active Directory server that is required to use. You will only be required to extract Issuer certificates where the Issuer certificate is different from ones previously extracted.
