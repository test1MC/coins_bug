Configure LDAP Directory User Authentication

This section provides instructions for system administrators on the steps required to implement LDAP Directory user authentication from within .

Traditionally, has maintained its own user accounts, each with an associated authentication token (password). From v11.04, has been enhanced to allow the authentication of users by utilising their Corporate Directory domain password.

This method of user authentication only requires the successful configuration of the server for communications with the corporate LDAP Directory servers. Users do not need to be logged into your corporate domain to be able to access .

 

Always test this process in an up to date, non-live environment before attempting it on a LIVE environment.
When performing this process on a LIVE environment we recommend that you have a working, complete backup of the LIVE environment.

 

Prerequisites

The user performing the tasks in this How To must have:

  • Administrative (root) access to the server

  • “root” access in OA

  • Access to OpenLDAP libraries for the server’s operating system

  • Access to the OpenSSL package for the server’s operating system (only if using SSL)

  • Access to the Corporate Directory signing certificate authority’s (CA) certificate (only if using SSL)

  • Access to change the name server – for example editing the resolv.conf file

  • Ability to change firewalls to allow ports 636 and 53 inbound and outbound

Microsoft Active Directory requirements:

  • Support for LDAP v3 clients

  • Support either Simple binding or SASL DIGEST-MD5 binding

  • Support for LDAP (port 389) and LDAPS (port 636)

  • Internal DNS IP addresses

  • Internal AD Domain Name

COINS Cloud requirements:

  • Site to Site VPN or MPLS required
