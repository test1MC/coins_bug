Assume User ID

Location : System > User Maintenance > Assume User ID

Assume User ID allows support users to assume the identity of another user provided they have appropriate security settings. This is intended for support users to log in as another user to check what that user is experiencing.

Field

Description
User ID The ID of the user.
This field links to:
Assume User
User Name The name of the user or group with this ID.
Prime Group The first group for this user.
Other users who belong to this group (that is, have this group in their group list) will be able to see this user's reports, post their batches, etc., if they have a higher security level than (or - depending on configuration - the same security level as) this user.
The parameter SY/EQACCESS determines whether users with the same security level can post each other's batches.
See Batch and Print Security.
Email The email address of the user.

 

By following the link in the User ID column, you will be presented with an encoded URL that you can paste in to an InPrivate browse window which will log you in as that user.

 

The link is valid for 30 seconds only. Your name is recorded in the log file for each action you perform whilst logged in as the other user. You can assume the other user's ID without logging them out as this type of access allows a duplicate login.

 

SYSAdmin user and root group are both banned from being assumed. SYSAdmin user can assume any other user.
