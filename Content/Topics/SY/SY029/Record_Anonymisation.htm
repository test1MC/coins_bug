Anonyming Records

Various regulations (including the European GDPR) require businesses not to keep data that can identify individuals for longer than is necessary. Clear User Personal Data is a new function that allows you to anonyme user ID values on records that are more than a set number of years old. The actual user ID will be replaced with 'COINS'.

How Anonymation Works

The new SY parameter GDPRUIDY specifies the number of years to retain user ID records. Records that are older are anonymed; that is, the user ID or user name on those records is set to ‘COINS’.

A user record with the user ID ‘coins’ must exist (which it does on standard environments).

User records on which the last login is more than SY/GDPRUIDY years ago are deleted if they are locked and are not a named user.

So, for example, if you have been using for 20 years, you could have 20 years’ worth of records with JSMITH stamped as the user who last updated them. If you run Clear User Personal Data with GDPRUIDY set to 10, ten years’ worth of transactions that have JSMITH stamped on them will have this replaced with ‘COINS’, and 10 years’ worth will keep JSMITH for now.

Once JSMITH leaves (and IT lock the account and remove the named user setting) and it gets to ten years since they last logged in, their user record will be deleted. By this point there will be no records that refer to JSMITH.

If a user record is deleted, any “satellite” data is also deleted; for example:

  • User

  • User

  • Buyer in Procurement

  • Service

  • JC Employee

  • FM

Because the process takes a long time to run, Clear User Personal Data allows you to specify a maximum run time in Hours. If not all user ID values are anonymed during the run, the process will remember where it stopped and will continue from there next time it is run, if it is run less than a month later. If the next run is a month or more after the initial run, the process will start from the beginning. If all user values have been anonymed, but it hasn’t been a month since the initial run, the process will not anonyme user ID values until a month has passed since the initial run.

The report content details the number of records processed for each table updated in that run. It also displays the last revised value for the last table it was processing before the report hit the run time limit.
