Login

When a user first logs in to (or if the security token has expired or if they are accessing from a non-specified IP address such as a VPN), they will see the normal login page and will be required to authenticate with . If using LDAP authentication then this authentication might in turn be delegated to LDAP.

If the user is accessing from an IP address defined in the WEBRIPS list, then if they log in to successfully, a security token will be built and encrypted, and stored on the user's browser as local storage data.

The next time they request the login page (on the same computer and browser, and from an IP address specified in WEBRIPS), checks this security token, and if it is valid the user will be allowed access to without further authentication.

When the security token expires (or if required the WEBRPWD is changed) users are redirected to the normal login page and are required to re-authenticate, at which point another security token valid for the number of days specified in WEBRDAYS is created, encrypted and stored.
