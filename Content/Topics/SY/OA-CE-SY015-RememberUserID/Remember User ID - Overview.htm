(A similar feature was previously available only for IE and required a specific IIS server.) It is assumed that appropriate Windows user authentication, screen locking etc. is in place because (as was the case with the existing IIS pass through authentication) if someone is able to access an unlocked Windows screen then they will also have access to for the user logged in on that PC.

The user needs to periodically authenticate with (this can be with LDAP/Active Directory if configured) but between these authentications will recogne who they are let them straight in without requiring additional repeated authentication.
