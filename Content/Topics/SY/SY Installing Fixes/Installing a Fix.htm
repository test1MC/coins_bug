Installing a Fix

 

The Install Fix option allows you to install fixes that have been sent to your site by or downloaded by you from the website. It is therefore crucial that you have the version of that you are using.

Each fix has a unique number. supply fixes in the form of “Fix File or Service Packs” which can contain one or more fixes. 

Deploying any fix causes a recompile of all necessary fixes installed and the relevant time should be allocated accordingly. All Service Packs are compiled and based on you having already installed the previous ones.

 

It is a requirement that all fixes are deployed and tested in a NON-Live environment before deploying to a Live environment.

 

Installing any fix to the Live environment should be performed when all users are logged out of or at a quiet period, i.e. out of office hours (See Delayed Fixes)

The Fix File or Service Packs are simply tar archives and have a standard naming format, i.e. coins_version-fix_number.tar. In our example V1022 = Version Number, 63365 = Fix Number and .tar = the file type. 

Installing a fix is a two-stage process

  • Loading the fix into the software repository.
  • Deploying the fix.

However, you can do these at different times. You can load several fixes, and install them later.
