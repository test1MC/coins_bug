Download the Fix

 

All fixes are installed on the server. When you are installing a Fix or Service Pack you will be required to download these onto a local directory. This can be your C:\ drive and you are recommended to create a Folder called Fixes on the C:\ drive.

This will enable you to always locate the files once they have been downloaded from the website or from an email attachment direct from Support Desk.

The Client Area on the Website shows a list of all the fixes and service patches available. This is a secure area and you will need to have an allocated User Id and sign on to access this area.

From the website  www.coins-global.com select the Client Area from the Tool Bar. Enter your User Id and your password in the relevant fields. Select the Sign In button.

The Client Area screen will be displayed. Select the Service Packs & Fixes option under the Software Releases menu

Select the hyperlink required for your version to show the current Fixes and Service Packs available to download.

Fixes are released throughout the month. Any Service Packs will always include the Fixes released the previous month and any additional Fixes required to be installed. You must always install the previous Service Packs before you deploy any fixes for the current month.

The Fixes are graded using a code, Red = Urgent, should be deployed as soon as possible, Amber = Important Non Urgent, to be deployed during the month and Green = Can be deployed during the month or in the month end Service Pack. Service Packs are always Red as these should always be installed.

The details of each Fix or Service Pack are listed in a table format and the headings are defined below

Description

Fix = A Brief Description in Bold including the Main Module it relates to and a short description

Service Pack = The Month of the Pack.

Product Group / Module

Fix = The Main Module and areas the fix will impact

Service Pack = Download Service Pack

Date

The date the Fix was released

Fix No

The unique number of the Fix or Service Pack, which should be recorded to use when we are deploying them

Locate the Fix or Service Pack that you require and make a note of the number. Select the download using the fix number hyperlink.

A Dialog Message box will be displayed asking “Do you want to open or save this file?” Select the Save button

The Save As screen will be displayed to enable you to save the Fix File or Service Pack. In our example we are saving the file on the C:\drive under a specific folder called Fixes.

The Fix File or Service Packs are tar archives and have a standard naming format, i.e. coins_version-fix_number.tar. In our example V1022 = Version Number, 63365 = Fix/Pack Number and .tar = the file type. 

Select the Save button. The Download Complete screen will be displayed. Select the Close button

Select the Close button and you will return to the Fix Table to download the next one you require.

Once all the Fix or Service Pack downloads are complete you are required to transfer these to the server and ensure they are in the correct format.

We recommend that you create another folder or sub folder called “Archived” or “Installed”. When all the latest Fixes and Service Packs have been installed and complete, move all the Fix Files and Service Packs to the Archived/Installed folder.  

Only the current Fix files and Service Packs that are waiting to be installed should be in the Fixes folder.
