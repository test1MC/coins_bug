Synchronised Fixes

 

The COINSadmin tool will enable you to synchronise the fixes installed between two environments. For example Fixes will be deployed automatically so the target environment you select will have the same fixes installed as the source environment. (The source environment is the original environment to synchronise from).

 

It is strongly recommended that you deploy and test all fixes in one non-live environment BEFORE you synchronise the fixes to deploy them in a live environment.

Log in to the COINSadmin area and locate the Administration Main Menu screen.

Select the Environments option using the Arrow keys. Select the Return key.

The Environment Menu screen is displayed. Select the Administer an Environment option using the Arrow keys. Select the Return key.

Select the target environment you want to install the fixes using the Arrow keys. Select the Return key.

The Environment Menu screen is displayed showing your target environment. Select the Install menu option using the Arrow keys. Select the Return key.

The Deployment Menu screen is displayed showing your target environment. Select the Synchronise Fixes menu option using the Arrow keys. Select the Return key.

The Select Environment to Synchronise screen will be displayed. Select the source environment required using the Arrow keys. This environment will be synchronised to the target environment you have already selected. Select the Return key.

 

If you have the target and source environments the wrong way round, select the F4 key to return to the Administration Main Menu screen to start again.

will display a message asking “Do you wish to continue?” Select the Yes option using the Arrow keys.

Select the no option if you have the target and source environments the wrong way round, or you have selected the wrong environment for the source. You will return to the Select Environment to Synchronise screen

The Synchronise Fixes screen will be displayed. Enter the details in the relevant fields.

User Name

Your Name. The Fix Status enquiry uses this data to show who installed fixes

Multi User Compile

The Default is Y = Yes the database brokers are running. Change to N = No if they are not running.

Select the Return key. The Fixes Compile screen is displayed.

When the two environments have been synchronised the new screen will be displayed showing the fixes that have been synchronised.

Select the F4 key to return to the Administration Main Menu screen. You may have to select the F4 key more than once.

You can see what fixes have been deployed by using the Fix Status Enquiry.

You will now be required to trim the webspeed agents.
