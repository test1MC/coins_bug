Transfer a Fix File or Service Pack or Multiple Fix File or Service Packs

 

In some cases you may wish to transfer the Fix File or Service Packs and place them into the System Administrator’s home directory in the UNIX operating system. To enable them to do this and be able to transfer the Fix File or Service Pack between two computers / servers they would use an FTP (File Transfer Protocol) program such as FileZilla, or other similar software/shareware.

In this example we are going to explain the process using the Windows XP command prompt. However you will need to modify the steps based on the FTP program that you are using.

 

We would suggest that you read through this section first before you progress with the transfer, to enable you to collate the information, file names and locations required. Otherwise the system will time you out whilst trying to locate them, at each stage, and you will then have to start from the Run screen again

The Fix File or Service Pack has already been saved on the C:\ drive in a specific folder called Fixes.

Select the Start button at the bottom left of the screen to display the Start Menu.

Select the Run option from the list of options.

This will display the Run screen for you to input the details in the relevant fields.

Input the command “ftp xxxx”, where xxxx is the Server’s Name or the IP Address. To find the Server Name or the IP address you could ask your System Administrator. Select the OK button

The C:\WINDOWS\ Dos screen will be displayed. The curser is flashing on the User field. Enter your User Name in the User field and select the Return key.

The Password field will be displayed for you to enter your password. The field will remain blank as you enter your password. Select the Return key.

The screen will display the ftp> prompt for you to enter the file transfer mode.

 

Make sure that the file is transferred via a binary transfer. This will ensure that the Fix File or Service Pack is in the right format for it to run on the UNIX environment.

To alter the current FTP to a Binary mode, enter bin at the ftp> prompt. Select the Return key.

The screen will display the message that you are “Switching to Binary mode”. The ftp> prompt is displayed on the next line to enter your instruction.

We need to inform the system where the current Fix File or Service Packs are stored. These are in our Local Change Directory, under a folder called Fixes. The system prefers to have shortened instructions, like Text Talk. We can use the shortcut lcd = Local Change Directory.

At the ftp> prompt enter lcd c:\fixes. Select the Return key.

The message will be displayed “Local directory now C:\Fixes” and the ftp> prompt is on the next line to enter your instruction.

Now that we have established where the current Fix File or Service Packs are held, the next stage is to inform the system where we want to transfer them to, which is our current UNIX Server.

To locate where this is enter “pwd” at the ftp> prompt and select the Return key. (pwd = Print Working Directory).

The location of the Print Working Directory will be displayed on the next line. In this example it is “/home/coins”. The ftp> prompt is on the next line to enter your instruction.

We recommend that you create a folder called fixes in the /home/coins directory to keep continuity. This stage would only have to be done once, as on the C:\ drive.

Enter “mkdir fixes” at the ftp> prompt and select the Return key.

A message is displayed stating “/home/coins/fixes created”

We want to be able to change this to the Fixes folder. To change the folder, enter “cd fixes” at the ftp> prompt and select the Return key.

A message will be displayed stating “Directory successfully changed”.

To check that you are in the correct folder enter “pwd” at the ftp> prompt and select the Return key to display the directory you are in.

This will display the “/home/coins/fixes”. The ftp> prompt is on the next line to enter your instruction.

From here the process if different for transfer a single Fix File or Service Pack and multiple Fix Files or Service Packs.

Transfer a Single Fix File or Service Pack

Once we have defined where either Fix Files or Service Packs that we want are held and also where we wish to transfer it to, the next step is to actually transfer it. (Should you have Multiple Files refer to the next section)

To do this we need to enter the full details of the name of the file, as there may be more than one held in the Fixes folder. The instruction code to transfer is put.

At the ftp> prompt enter “put TheNameOfTheFile.tar”. For example put v1022-63365.tar. Select the Return key.

The system will go and find the file. The message will be displayed “PORT command successful. File receive OK”.

There is other information available regarding the file size and the number of seconds to transfer it. The ftp> prompt is available on the next line to enter your instruction.

Should you enter the wrong file name the message “Vxx.yy-nnnnn.tar: File not found” will be displayed. The ftp> prompt is available to rekey the correct file name.

Once the file has been transferred we can leave the C:\WINDOWS\ Dos screen. At the ftp> prompt enter “bye”. Select the Return key.

The screen will display “Goodbye” on the next line and close the window automatically.

We have transferred the Fix File or Service Pack from the local C:\drive to the UNIX Server, in the right format, for you to deploy at a later time. (See the section on Deploying a Fix)

For Multiple Transfer of Fix File or Service Packs

Once we have defined where either the Fix File or Service Packs that we want are held and also where we wish to transfer them to, the next step is to actually transfer them.

However one thing to check before you start using the mput option is the Interactive mode is Off. If this is in a status of ON the system will ask you to confirm each file that is being transferred. You will be required to enter “y” for every file you are transferring.

To check the status of the Interactive mode, enter “prompt” at the ftp> prompt - this will display the status of the Interactive mode.

To change the Interactive Mode enter “prompt” at the ftp> prompt - this will change the status of the Interactive mode.

To do this we need to know what files we are transferring. It could be all the files in the Fixes folder, or just specific ones, i.e. all files relating to a specific version number - v1022, v1021. The instruction code to transfer multiple files in the system is mput and wildcards are used. 

At the ftp> prompt enter “mput *” - this will transfer ALL the files that are saved in the Fixes folder.

Alternatively if all the files are for a specific version at the ftp> prompt enter “mput v1022-*” - this will transfer all the files that start with v1022- and any alphanumeric after. Select the Return key.

The message will be displayed “PORT command successful. File receive OK” for each of the files that have been transferred. There is other information available regarding the file size and the number of seconds to transfer each one.

The ftp> prompt is available on the next line to enter your instruction.

Once the file has been transferred we can leave the C:\WINDOWS\ Dos screen. At the ftp> prompt enter bye. Select the Return key.

The screen will display “Goodbye” on the next line and close the window automatically.

We have transferred the Fix File or Service Pack from the local C:\drive to the UNIX Server, in the right format, for you to deploy at a later time. (See the section on Deploying a Fix)
