To Load a Fix into the Repository

Once all the files have been transferred in the right format (Binary) and in the right place (UNIX Server) we can now load them into the repository. The repository is the central holding place for all the versions.

All fixes must be loaded into the repository before being loaded into the environment.

Log in to the COINSadmin area and locate the Administration Main Menu screen.

Select the Configure COINSadmin option from the Administration Main Menu screen, using the Arrow keys. Select the Return key.

The Configuration Menu screen will be displayed.

Select Load Repository option from the menu using the Arrow keys. Select the Return key. The Load Release Archive Menu screen will be displayed.

From here the process is different for loading a single file into the repository or loading multiple files.

Loading a Single Fix File or Service Pack into the Repository

To load a single Fix File or Service Pack from the Load Release Archive menu screen.

Select the Load CUI Fixes into Repository option from the menu using the Arrow keys. Select the Return key.

The Archive File screen is displayed. There is one field that requires input. This is the File field.

Enter the full path of the file that you wish to load into the repository. In this example we have entered /home/coins/v1022-63365.tar.

Once you have entered the full path select the Return key.

The field will change to a no-input field (greyed out). A message prompt is displayed at the bottom of the screen “Load Fixes”. 

It is a good discipline to check that you have entered the right path and file name that you wish to load to the repository.

When you checked the details enter Y = Yes and select the Return Key

The system will display the following message in the middle of the screen “Information Load Successful” and the cursor is highlighting the OK option.

Select the Return key and the system will return to the Load Release Archive menu screen

The file has been loaded onto the repository ready for deployment.

Loading Multiple Fix File or Service Packs into the Repository

To load multiple Fix File or Service Packs from the Load Release Archive menu screen you will need to select the appropriate option.

Select the Bulk Load of Fixes into Repository option from the menu using the Arrow keys. Select the Return key.

The Bulk Load Fixes screen is displayed. There is one field that requires input. This is the Directory field.

Enter the path of the files that you wish to load into the repository. In this example we have entered /home/coins/fixes. Once you have entered the path detail select the Return key.

The field will change to a no-input field (greyed out). A message prompt is displayed at the bottom of the screen “Bulk Load Fixes”. 

It is a good discipline to check that you have entered the right path that you wish to load to the repository.

When you checked the details enter Y = Yes and select the Return Key

The system will display the following message in the middle of the screen “The following fixes have been loaded........” and details of the files are listed.

The cursor is highlighting the OK option.

It is recommended that you check the files to make sure that these are the correct ones have been loaded into the repository. 

Select the Return key and the system will return to the Load Release Archive menu screen.

The files have been loaded onto the repository ready for deployment.
