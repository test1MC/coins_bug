Overview of Functionality

Server-based Exchange integration allows you to:

  • Replicate contacts – this is a one-way process.  Contacts created in can be replicated to a public folder on the Exchange server.  This folder may then be included as an address book in Outlook.
  • Replicate users' appointments and tasks - this is a two-way process. Diary events and journals created in OA will be automatically copied to Exchange. Any updates made in Exchange to these items will also be copied back to .  Appointments and Tasks created in Exchange can also be flagged for replication back to by the use of a “” category.  This method is an alternative to client-based integration of appointments and tasks which is explained in the guide How To Set Up Outlook Integration.
  • Import emails for a project.  Emails put into a public folder (by moving, copying, drag and drop, or by including the public folder’s email address in the TO or CC address list) are imported into as marketing journals and can be viewed in Project Management and Marketing.
  • Import workflow replies. Workflows can be configured to send emails to users, with links for moving the task to a different stage. These links allow users to send replies to a workflow email address; reads these replies and updates the workflow task as instructed. This means workflow users can control the workflow without having to log into .
  • Import messages from monitored folders.  Any number of public folders can be configured for monitoring.  When the Exchange Integration utility is run, each of these folders is checked for any incoming messages, and the specified program is run to handle the inbound message. Once processed, the message is either moved to an archive folder, if specified, or deleted.
  • Process sent emails from the Mail Assistant and load the emails and attachments into .

Importing emails and workflow replies requires public folders with an associated email address.

Replication is done by running a scheduled task on a Windows server; this may be the Exchange server or another server on the network.
