Import of Project Emails

Project emails are imported from public folders on the Exchange Server.  Each project must have its own public folder and these must all be under the same parent folder.  These folders may also be given an email address to allow emails to be sent directly to the folder.  The folder name must contain the Project Number as viewed in the Project Workbench:

The folder name can either be the Project Number only (e.g. “0204”) or a combination of both the Project Number and a label.  If the folder name contains more than just the Project Number then the project number part of the folder name must be enclosed in square brackets.

The following are valid folder names:

  • 0204
  • Merrion Centre [0204]
  • [0204] Merrion Centre
  • 1/0204 Merrion Centre – Berkshire – Fred Smith [0204]

Example public folder tree:

  • All Public Folders
      • Projects
        • Merrion Centre [0204]
        • Heathrow T4 [0205]

The replication account should be given permission level of at least “Editor” as this account will need to delete items owned by all users. 

 

Archive Folders

In addition, a second set of project folders may be used to archive emails that have been imported (if archiving is not used then emails are deleted after import).  The archive folders do not need an email address.

Example public folder tree with archive folders:

  • All Public Folders
      • Projects
      • Inbound
        • Merrion Centre [0204]
        • Heathrow T4 [0205]
      • Archive
        • Merrion Centre [0204]
        • Heathrow T4 [0205]

The replication account should be given permission level of at least “Editor” on the archive folders.

The Default permissions should be set to “Reviewer”.
