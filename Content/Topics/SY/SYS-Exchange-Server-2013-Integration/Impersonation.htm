Impersonation

For replication (of activities and sent emails) it is necessary for the replication account to be able to pass data between and the individual users' Exchange/Office 365 mailboxes.  The method to allow this is called “impersonation”.

See http://msdn.microsoft.com/en-us/library/bb204095.aspx for more information.

To enable impersonation:

  1. Create a role to allow our user to impersonate.
  2. Launch the EAC and browse to permissions > Admin roles.
  3. Click the “+” button to add a new role.

  4. Complete the fields as follows:
    Field
    Description
    Name

    User Impersonate.

    Description

    Allows this user to impersonate other user when restoring mailboxes.

    Write Scope

    Default.

    Roles
    Add the Role name “ApplicationImpersonation”.
    Members
    Add the User “The User you would like to use”.

Alternatively, you can do this using Exchange PowerShell :

New-ManagementRoleAssignment -Name:”User Impersonation” -Role:ApplicationImpersonation -User:coins_exchange

See the MSDN article mentioned above for examples of assigning the permission to all users or a group of users.
