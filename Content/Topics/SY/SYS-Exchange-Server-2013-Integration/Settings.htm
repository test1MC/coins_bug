Integration Application Settings

The installation process will create a configuration file in the working directory (for example, C:\_Exchange\wrk).  By default this file will be called _exchange.settings, but you can override this using command line arguments (see Replication Software - Command Line Arguments).

The configuration file consists of one or more environment sections with parameter values in each section. Identify an environment section using the environment name enclosed in square brackets (for example, “TEST” in the sample file below).  The replication process will run for each environment in the configuration file in turn.

Sample configuration file:

[TEST]
AppServer=AppServer://coins.server.internal:5192/asappqafb
LogLevel=3

EWSURL=https://exchange.server.local/ews/Exchange.asmx
COINSUser=dsmith
UserName=
Password=
Domain=
AutoDiscoverEmail=
 
ContactsReplicate=Y
ContactsFolder=/contacts
 
AWBReplicate=Y
AWBServer=TEST
 
WorkflowEmails=Y
WorkflowFolder=
WorkflowMailBox=coins_workflow@test.com
 
ProjectEmails=Y
ProjectFolder=/projects/inbound
ProjectCopy=/projects/archive
 
MonitoredFolders=Y
 
SentEmails=Y

Parameter values and their meaning are as follows:

Parameter

Description

Default

AppServer

Appserver connection string as advised by the system administrator. Mandatory.

For example: AppServer=AppServer://coins.server .internal:5192/asappqafb

 

LogLevel

Logging level.  1 – basic, 2 – intermediate, 3 – verbose, 4 – debug.

3

EWSURL

URL for the Exchange Web Services application as advised by the Exchange administrator. For example: EWSURL=https://exchange.server.local/ews /Exchange.asmx

Either EWSURL or AutoDiscoverEmail must be specified.

 

AutoDiscoverEmail

Email address to use for auto-discovery of the EWS server.  If specified then this will be used instead of EWSURL.

 

COINSUser

A valid user in the environment used for recording record additions, deletions etc.  Mandatory.

 

UserName

The Active Directory replication account (see Active Directory Replication Account).  Useful for debugging.  If omitted then the login credentials are taken from the logged in user.  Optional.

 

Password

Password for the user specified in the UserName parameter.  Optional.

 

Domain

Domain for the user specified in the UserName parameter.  Optional.

 

ContactsReplicate

Enable replication of Contacts. Y/N value.  Optional.

N

ContactsFolder

Relative path to the public folder used to hold contacts. Required if ContactsReplicate is set to Y.

 

AWBReplicate

Enable replication of Activity Workbench tasks and appointments. Y/N value. Optional.

N

AWBServer

The code for the replication process. This can be any value, but is compared to the Replication Mail Server field on the user record. Only users with the same value in the Replication Mail Server field will be replicated. Required if AWBReplicate is set to Y.

 

WorkflowEmails

Enable import of Workflow emails from a public folder or mailbox. Y/N value. Optional. If set to Y then either WorkflowMailBox or WorkflowFolder must also be set.

N

WorkflowFolder

Relative path to the public folder used to hold workflow emails for import. Ignored if WorkflowMailBox is set.

 

WorkflowMailBox

The mailbox name for inbound workflow emails.

 

ProjectEmails

Enable import of project emails from public folders. Y/N value. Optional.

N

ProjectFolder

The relative path to the public folder which is the parent folder of the individual project folders for import.  Required if ProjectEmails is set to Y.

 

ProjectCopy

Enables archiving of imported messages. If set then an archive tree of folders is set up to mirror the import tree of folders, with corresponding folders for each project and messages will be moved from the project folder to the archive folder after importing. If not set then messages are deleted after importing. Optional.

 

MonitoredFolders

Enables processing of monitored folders. If set to Y then all monitored folders defined in OA will be checked for inbound messages and these will be passed to OA for processing.

N

SentEmails

Enables sent emails that have been attached to a record via the Mail Assistant to be loaded into .

N
