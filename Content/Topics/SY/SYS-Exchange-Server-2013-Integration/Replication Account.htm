Active Directory Replication Account

A replication user account is required to do the actual replication. The OA Outlook Integration Application connects to Exchange using the replication user account, and impersonates (see 1, Impersonation) the actual user to retrieve email messages.

Create a domain user account in Active Directory. Pick a meaningful name, for example, "coins_exchange".

Account settings:

  • User must change password at next login - uned
  • Password never expires - ed

The password is used on the task schedule (see 1, Scheduling). “Never expires” is the preferred setting, otherwise the replication will stop working at some stage with little indication of why (there will be nothing in the replication log file as the process will not run). If your policy requires you to allow the password to expire, you will need to ensure that you change the password on the task schedule before it expires.

To help with setup and debugging, the account should be a member of Remote Desktop Users. If you want to run the replication immediately rather than through a schedule, this allows you to log in as the _Exchange account and run it with no login credentials specified.

Make sure the Logon as batch job policy is set for the user. This policy is accessible by opening the Control Panel, Administrative Tools, and then Local Security Policy. In the Local Security Policy window, click Local Policy , User Rights Assignment, and then Logon as batch job.
