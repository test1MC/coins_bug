Configuring Global Search

Global Search (see Global Search) allows users to search for records in the database (for example, suppliers, customers, subcontractors or ), and also to find and run functions. The search is configurable, so you can choose which tables, and which fields on those tables, to include in the search.

WARNING

Ideally you should make configuration changes with no one else on the system.

NOTE

Data for the menu/function search needs to be generated separately; see Configuring Function Search.

To configure Global Search:

  1. Before making any changes, stop the search index process: run syusrch_stop.sh in $BASE/binclick the Stop button at the top of the screen.

    NOTE

    You do not need to stop the search index process if you are only making changes to the Label, Link, Open In, Foreground, Background, Detail, Icon, or Security RSP fields.

  2. Go to System > System Setup > Global Search Configuration.
  3. Global Search comes with several standard search configuration records. If you want to use one of these standard searches, open the record and the Use box.

    NOTE

    The standard search configuration records include searches for high-volume transactional data, which are intended for use with the extended search capability only. If you do not have the extended search capability (that is, using an external search engine) configured, you will not be able the Use box for these searches.

  4. If you want Global Search to search for something that is not included as standard, add a new search configuration record. Refer to 1, Global Search Configuration - Fields for details.

    NOTE

    Global Search is intended for data that is relatively stable, such as suppliers or ; it has not been tested with transactional data such as invoices. If you intend using it to search transactions, be aware that this will significantly increase the size of the database, and that the number of possible search results may make it impractical.

  5. When you have made the additions or changes you need, select all the lines you have changed and run the Build Data action:

    1. Tick the Initiale box.
    2. If you have changed the search conditions or want to remove existing indexed records, Delete.
    3. If you want to create search data for all , enter * in the field. Otherwise, enter a list of the numbers of the you want to create data for.
    4. Click NEXT.
  6. When the build data process has completed, restart the search index process: run syusrch_start.sh in $BASE/binclick the Start button at the top of the screen.
  7. Restart the webspeed agents, appserver agents and report agents.

    Whenever any of the tables that are included in the search are changed (records added, updated or deleted), the search data will automatically be updated to reflect the change. You only need to rebuild the data using the Build Data action if you change the configuration (specifically, if you change any of the following: Table, Condition Field, Key Fields, Condition Value, Company Field, ID Fields or Search Fields).

  8. Set the following parameters that relate to Global Search:
    ParameterExplanation
    SY/GLOBSRCH

    To enable Global Search, set GLOBSRCH to Y.

    Optionally, you can prevent individual users from using Global Search: set the Search On field on the Preferences tab of the user record to No.

    SY/SRCHOPEN

    Use SRCHOPEN to specify whether the links from the search results open in the main area, in a frame or in a new window. Note that this default setting can be overridden for individual users or search configuration records.

    SY/SRCHMONUse SRCHMON to specify the email addressto which any error messages encountered during the global search process will be sent.

The scripts to start and stop the indexing process can be added to the database start and stop scripts. See Global Search - Indexing Scripts.
