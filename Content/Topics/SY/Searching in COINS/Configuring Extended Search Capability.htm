Configuring the Extended Search Capability

The extended search capability requires an external search engine to be installed. Currently the only external search engine supported is Apache Solr.

The technical team will install Apache Solr and the appropriate version of Java, do any initial configuration, and set up access permissions for the Solr console.

The BI team will build the data for the first time. The process is outlined below.

Before enabling the extended search capability, any existing Global Search data must be removed.

  1. Run syusrch_stop.sh in $BASE/bin.
  2. In Global Search Configuration:
    1. Click the Stop button at the top of the screen.
    2. Select all the configuration records.
    3. Run the Build Data action.
    4. On the Build Data screen, the Delete box, then click Next.
  3. Set the following SY parameters:

    Parameter

    Description

    SRCHENGN

    Search Engine. The default is blank which indicates that standard searching will be used. Currently only one other option is available – “Solr”.

    NOTE

    Do not change this parameter until the Global Search data has been removed (see above).
    SRCHURL

    The base URL for the Apache Solr instance; for example: http://localhost:8983/solr/

    SRCHCOL1

    The collection name for data. Collections are similar to database instances – each has a schema and holds data. It is possible to configure multiple collections corresponding to different environments (LIVE, TEST etc).

    WEBLOG

    This needs to be updated to include syusrch.log:
    SEARCH=$BASE/var/diag/syusrch.log,...

  4. Configure the search records for the tables you want to include in the extended search; see Configuring Extended Search - Fields.
  5. Initiale the extended search data. In Global Search Configuration:
    1. Select all the configuration records.
    2. Run the Build Data action.
    3. On the Build Data screen, the Initiale box, then click Next.

    NOTE

    The initial build of the data must be done while the system is not being heavily used.
