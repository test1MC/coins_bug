Extended Search and Copied Environments

If you make a copy of a environment that has Extended Search Capability configured, and want the Extended Search Capability to be available in the copied environment (for example, a copy-of-live environment), consider the following (assuming the search engine is Solr):

  • The Solr database needs to be kept in sync with the database so if you copy an environment the Solr database should also be copied.

  • Solr will need to be shut down for the copy.

  • The SY/SRCHURL parameter (which points to the Solr database) needs to be changed on the copied environment (otherwise the coped environment will be updating the live Solr data).
