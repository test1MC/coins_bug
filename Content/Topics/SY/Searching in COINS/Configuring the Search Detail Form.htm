Configuring the Search Detail Form

Desktop global search includes a small details page for each record. The standard naming for these is %WSYD<TLA>. For example, the detail page for ap_invoice has the following:

  • Function: %WSYDAIN
  • Page: %WSYDAIN – this is a page with Detail form only

If you want to provide a detail form for a record that is not already available, or want to show different details, create the appropriate function and page, or copy and modify an existing one. It is best practice to show all fields that are indexed. Child records can be shown using a field with View As set to List Frame that points to a browse page.

Enter the code for the detail form in the Detail field of the configuration record (see Global Search Configuration Records).
