Security Configuration

Access is governed by the control field, which can be the code, one of the other validated fields, such as location, or any one of the four analysis sets.

Two Status parameters control how Security will be implemented:

JC/JOBSEC

Which field uses to control security.

When determining whether to allow a user access to a , checks the value of the field specified here against the settings for that user in Security Maintenance.

Enter one of the following:

  • job_num for code
  • jgr_group for group code
  • jty_type for type code
  • jcl_loc for location code
  • job_fore for manager
  • rcm_num for customer
  • job_anal1 for analysis set 1
  • job_anal2 for analysis set 2, etc.
  • or leave blank for no security.

JC/COSECDEF

Whether a missing record in Status > Setup > Setup Maintenance > Security > Security means 'None' or 'All'.

  • Y = If there is no record for the user in Status > Setup > Setup Maintenance > Security > Security, the user has access to no .
  • N = If there is no record for the user in Status > Setup > Setup Maintenance > Security > Security, the user has access to all .
