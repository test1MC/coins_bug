Batch and Print Security

Each user has a security level, which controls whether they have access to other users' batches and print requests. In many modules, transactions are entered in batches. stamps each batch with the User ID of the user who created it. Other users are prevented from changing batches, with the following exception:

  • A user may see and update another user's batch if they have a higher security level than the other user and if the other user's prime group is one of the groups the user has access to. The security level is indicated on the user record (9 highest, 0 lowest). By giving a supervisor the same group code as their clerks but a higher security level, the supervisor can change and post the clerks' batches.

If you also want clerks to be able to change and post each other's work, set the system parameter EQACCESS to Y, using Parameter Maintenance. The same hierarchy of group access is also applied to amending, viewing and reprinting reports in Home > Report Status.
