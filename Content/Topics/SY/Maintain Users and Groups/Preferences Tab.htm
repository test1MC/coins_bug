Preferences Tab

This tab allows you to set preferences for this user.

 

 

Field

Description

Text Buttons

Whether to show text, icons or both on buttons.  This only affects selected buttons.

Default take the value of the SY/PREFTEXT parameter

 

Recall Filters

How the filter preferences for this user should be recalled from previous sessions.

 

Full = all filters are recalled

Day = recall the filters from today only

Session = only recalls from the current session

None = does not recall filters

 

Default takes the value of the SY/PREFFILT parameter.

Recall Advanced Filters

How the advanced filter preferences for this user should be recalled from previous sessions.

This allows you to prevent advanced filter selections, which may result in a slow performing screen, from being retained.

The options are:

Full - Recalls all filters.

Day - Recalls the filters from earlier today only.

Session - Only recalls from the current session.

None - Does not recall filters.

Default - Uses the system default setting.

 

The SY/PREFADV parameter determines the system default.

Recall Filters in Lookups

How the filter preferences for this user should be recalled from previous lookups.

The options are:

Full - Recalls all filters.

Day - Recalls the filters from earlier today only.

Session - Only recalls from the current session.

None - Does not recall filters.

Default - Uses the system default setting.

 

Default takes the value of the SY/PREFFILT parameter.

Recall Lookup Advanced Filters

 

How the advanced filter preferences for this user should be recalled for lookups from previous sessions.

 

This allows you to prevent advanced filter selections, which may result in a slow performing screen, from being retained.

 

The options are:

Full - Recalls all filters.

Day - Recalls the filters from earlier today only.

Session - Only recalls from the current session.

None - Does not recall filters.

Default - Uses the setting from the SY/PREFLADV parameter

Recall Default Tab

How the default tab to show should be set for this user.

The options are:

Full - Always shows the most recent tab that this user visited.

Day - Shows the most recent tab that this user visited today.

Session - Shows the most recent tab that this user visited during the current session.

None - Always shows the first tab on the screen.

Default - Uses the system default setting as set by the SY/PREFTAB parameter

Store Filters Query Limit

The time limit in milliseconds above which filter settings for browses will not be stored.

This is used to prevent a combination of advanced filters and sort orders, which have taken too long to return a set of records, from being stored and used by default the next time the user visits the page.  Instead, the default filter will be used.

If left as zero, the system default value as determined by SY/PREFQLIM parameter is used.

Report Email Default

The default setting for the Email box on report selection.

Set this to Yes if you want reports this user generates to be emailed to their email address by default.

Default - Uses the system default setting as set by the SY/REMAIL parameter

Column Sets

Whether this user should be allowed to set up and use different columns on browse screens, where available.

Column sets are alternative groups of columns to display on browse screens. They are not available on all screens, but if they are available and the user is allowed to use column sets, a Columns button gives access to the column sets configuration function, and a drop-down selector at the top right of the screen allows the user to choose which column set to display.

Default means use the value of the System parameter SY/COLSETS.

Unload Page Warning

Whether to show a warning if the user tries to move off a page where unsaved data might still be present.

Scroll Show All

Whether to scroll "show all" pages.

Search On

Whether the global search feature is available for this user.

Search Open In

How links from the global search function should open (in what type of window/frame).

Date Format:

Select the preferred date format from the dropdown.

Stylesheet:

User-specific stylesheet if required.

This can be used override styles in the standard stylesheet and company-specific stylesheet; for example, to provide a contract/site style where the user s on a specific site.

Email Workflow

The workflow templates that should send actions to this user as an email (if configured).

Enter a comma separated list of the workflow templates or Enter * for all workflow actions to be sent as emails.

Desktop

Current Desktop

The desktop menu to be shown. If blank then the desktop is suppressed.

Each user can have a different desktop, or several users can use the same desktop.  Note that if the user has access to the Desktop tab in User Preferences, they will be able to modify the desktop, and any changes they make will apply to any other users who share that desktop.

Alternative Desktop

A comma-separated list of desktops to which this user has access (including the user's current desktop).

If this field is populated, a selector at the top of the user's desktop will allow them to switch between the alternative desktops.

Initialise

The calculation program that should be run for this user on initialisation of the desktop.

Desktop Image

The background image for the desktop.

This background only applies to this user's view of the desktop; if users share the same desktop, they can have different background images.

 

The images that are available as desktop backgrounds are in $BASE/custimg/bg/.  You can load images using System > System Setup > Background Images

Timed Logout

The number of minutes of inactivity before the user is logged out of the system.

Main Menu

The main menu that this user should be presented with on login. If left blank, then the standard main menu will be shown.

The main menu is the list of modules that appears in the module selector in the menu frame. The user will be taken to the first module in this list.

Documentation Menu:

The main menu that this user should be presented with by default, when they select the documentation module. If this is left blank then the standard documentation menu will be shown.

PDA Menu

The main menu that this user should be presented with, by default, on login on a mobile device. If this is left blank then the standard main menu will be displayed.

The main menu is the list of modules that the user can select from. The user will be taken to the first module in this list.

RSS Channels

The RSS channels that this user subscribes to.

RSS allows you to view news headlines and other information from different websites in one place.  In , these can be displayed on the module home pages, or viewed using RSS Reader.

If all boxes are blank, the user can view all channels in RSS Reader, but no channels will be included on the home pages.

The channels must first be set up using RSS Channels in System Setup.
