Install the OpenEdge SQL Client Access software

The Progress OpenEdge SQL Client Access installation package is delivered as a self-extracting ZIP archive. Please ensure that you have the correct installation package for your operating system.

The Progress OpenEdge environment does not natively support ODBC database connectivity. The OpenEdge SQL Client Access product provides the operating system drivers and software necessary for your computer to connect to the database using standard ODBC compliant connections.

This sequence of steps will instruct you on how to install and configure the Progress OpenEdge SQL Client Access software on a Microsoft Windows based computer.

The following steps will be performed using Windows Explorer. Alternatively, for advanced Windows users, the Windows Command Prompt can also be used.

  • Create a temporary folder to hold the un-packed installation files. For this purposes of this “How-To” guide, the folder C:\OESQL has been created.
  • Copy the OpenEdge SQL Client Access installation package into the temporary folder .
  • Invoke the installation package by double clicking the filename .
  • Extract the installation files into the current folder.
  • Find the setup.exe program within the extracted installer files and invoke it, by double clicking the filename
  • The Installing OpenEdge setup program will launch. After it has performed some initial installation configuration, the first setup wizard window will be displayed as shown below.

  • Select Next to continue.
  • The Serial Number and Control Codes wizard window will be displayed as shown below.

  • Enter the Serial Number and Control Code of the SQL Client Access product that is appropriate for your installations operating system.
  • Once you have entered the Serial Number and Control Code for the SQL Client Access product, press the Accept button. The OE SQL Client Access product should be added to the Product(s) to be installed list, as shown below.

  • If it is not, then you have incorrectly entered the Serial Number or Control Code, or you are attempting to install the incorrect product for you operating system.
  • To continue the installation, select the Next button.
  • The installation wizard will now prompt for the destination and working path directories as shown below.

  • For the purposes of this “How-To” guide, we will assume the defaults. You may however choose to alter these paths. If you do alter the installation destination, please remember what you have changed it to as it will be required later in this “How-To” guide.
  • Once you have made any required changes, select the Next button to continue.
  • The installation wizard will move onto the selection of the setup type as shown below.

  • It is recommended that you perform a Complete installation.
    Select the Next button to continue the installation.
  • The installation wizard will now ask for a Program Folder to hold all created Start Menu shortcuts as shown below.

  • It is recommended that you append the OpenEdge version number of the installation to ensure that future versions of OpenEdge products can be clearly identified within your Start Menu.
  • Once you have decided on the Program Folder, select the Next button to continue the installation.
  • The installation wizard will now provide a summary of the OpenEdge SQL Client Access product installation as shown below.
  • The summary provides you with the ability to make any last minute changes to your installation selections before performing the actual install.
    Verify that the installation disk requirements (Space Required) is less than the amount of disk space that you have available to you (Space Available).
  • If you do not have enough disk space to complete the installation, it will fail.
  • Verify your entries and select the Next button to perform the installation of the OpenEdge SQL Client Access product.
  • The installation will now proceed. Once it has finished, and assuming that you encountered no errors along the way, you will be presented with the completion window as shown below.

  • Select the Finish button to complete the installation and end the installation wizard, closing the installer window.
  • Remove the temporary folder that you created in the steps above. Now that the installation has completed, you no longer need the installer files.
    Ensure that you are not using the temporary folder, for example: you have a command prompt that is currently in the temporary folder, and remove it.

You have now successfully installed the OpenEdge SQL Client Access product. You are now ready to configure ODBC System DSN’s that can access your environments.
