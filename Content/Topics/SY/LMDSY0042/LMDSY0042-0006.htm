Create a System DSN

System DSN’s register ODBC data sources with the Microsoft Windows operating system. Application software will typically expose System DSN’s as data sources that can be chosen when data is required by the application.

On the Microsoft Windows platform, DSNs are registered in different areas within the system registry for the different application bit widths. If installing the 32 bit SQL Client Access product on a 64 bit Windows operating system, the default ODBC Data Sources Administrator cannot be used to configure the System DSN. Instead you must manually invoke the 32 bit ODBC Data Source Administrator from C:\Windows\SysWOW64\odbcad32.exe.

This task explains how to configure a System DSN, using the OpenEdge SQL Client Access software that was previously installed.

The following steps will be performed using a Microsoft Windows computer.

  • From the Start Menu, open the computer’s Control Panel.
  • Within the control panel, open Administrative Tools.
  • Within the Administrative Tools, open Data Sources (ODBC).
    Note: If the 32 bit SQL Client Access product was install on a 64 bit Windows operating system, you must manually invoke C:\Windows\SysWOW64\odbcad32.exe
    The ODBC Data Source Administrator window will open, as shown below.

  • Select the System DSN tab as shown below.

  • Select the Add button. The Create New Data Source window will open as shown below.

  • If you do not see the Progress OpenEdge <version> Driver ODBC driver, then the installation of the Progress OpenEdge SQL Client Access software was not successfully completed and you cannot continue with this task.

  • Highlight the Progress OpenEdge <version> Driver line and select the Finish button.
    The ODBC Progress OpenEdge Wire Protocol Driver Setup window will open as shown below.

  • Complete all of the display fields as follows:

Data Source Name

A unique identifier for this data source. The value entered here will be shown when choosing a data source in an application, so ensure that you provide adequate detail to identify this data source.

Description

A more detailed description of the data accessible via this data source.

Host Name

The fully qualified domain name (FQDN) of your server, provided by your COINS System Administrator.

Port Number

The port number to use to connect to your database, on the COINS server, provided by your COINS System Administrator.

Database Name

Should always be coins.

User ID

Your ODBC user identifier provided by your COINS System Administrator.

A completed example is shown below:

  • Select the Advanced tab and alter the Default Isolation Level so that it is set to READ_UNCOMMITTED as shown below.

  • Return to the General tab and select the Test Connect button. A new Logon window will open as shown below. You will be prompted for your ODBC password, which was provided to you by your COINS System Administrator.
  • Enter your password and select the OK button.
  • If all of your connection settings are correct, then window shown here will be displayed.

If you receive any errors, please contact your System Administrator for assistance in determining the cause of the displayed error(s).

  • Select the OK button to close the Test Connect window.
  • Select the OK button on the ODBC Progress OpenEdge Wire Protocol Driver Setup window. This will return you to the ODBC Data Source Administrator window where your new data source will be displayed, as shown below.
  • At this point, if you need to configure more data sources, maybe for different COINS environments, you can repeat these steps from step Select the Add button. The Create New Data Source window will open as shown below.If you do not see the Progress OpenEdge <version> Driver ODBC driver, then the installation of the Progress OpenEdge SQL Client Access software was not successfully completed and you cannot continue with this task.. If you are finished, proceed to the next step.
  • Close the ODBC Data Source Administrator by selecting the OK button.

You have now successfully configured ODBC data sources for your environment(s). These data source should now be accessible from within other applications, like Excel, for extracting data directly from your COINS environment.
