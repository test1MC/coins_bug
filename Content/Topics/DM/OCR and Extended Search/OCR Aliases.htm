OCR Aliases

Alias Groups is a new function that allows you to associate text labels on a scanned image with  database fields, so that when processes a scanned image, it can automatically build a template for that image layout (see 1 - Document Management OCR Invoices - OCR Layouts). Currently this only works for invoices.

The initial browse allows you to create the alias group. For invoices, create a group named OCRTBL:AP_INVOICE.

OCR Alias Groups Browse

The link on the Group field opens the Alias Details browse.

Fields

Field
Description

Source

The database field you want to create an alias for.

Target

The text of the label that identifies the data on the scanned image. For example, the label Inv Date might indicate that the following text is the invoice date (ain_idate).

Format

How you expect the information on the scanned image to be formatted. For example: XX999 (two characters followed by three digits) or ‑>>>>>>>>9.99 (a decimal value).

The format allows the OCR process to judge if what it has read alongside the target is what is expected. If it fails, it will look up to three lines below the target for text or numbers that match the format.

Data Type

The data type of the data (usually the same as the data type of the field). Values are IN for integer, DE for decimal, DA for date, and CH for character.

Allow Spaces

Whether spaces or other separator characters are expected within the text read from the image.

The OCR process treats text surrounded by white space as a “box” of text. If this is ed, the OCR process will append additional “boxes” to the right of the initial “box” until it matches the expected format.

You can set up multiple aliases for the same database field. For example, the invoice date might be “Inv. Date”, “Invoice Date” or “Date of Invoice” on invoices from different suppliers, so you could set up three different aliases.

NOTE

If you find that the OCR process often misinterprets specific characters, you can set up additional aliases to compensate. For example, if “ic” is interpreted as “k”, you could set up an alias to associate ain_idate with “Invoke Date” as well as with “Invoice Date”.
