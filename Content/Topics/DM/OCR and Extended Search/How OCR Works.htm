How the OCR Process Works

The Optical Character Recognition process initially inspects the file to identify the alias value (the label) for ‘avm_vatregno’. The adjacent text (to the right or below) is then tested against the format of the field. If the value passes the format check it is then used, in conjunction with the current company, to locate the invoice supplier.

If an OCR Layout Exists

If an OCR layout for the supplier already exists (either manually created, or if the supplier has already had an invoice processed using OCR and a layout has been generated automatically), the existing layout will be used to locate the values from the image. This reduces processing time, by saving the need to scan the image for the text as set out in the alias table.

If No OCR Layout Exists

If no OCR layout exists for the supplier, one will be created using the data from the supplier record. The new layout will be given the name as it appears in the ap_vendor table (field avm_name).

The OCR process goes through the remaining alias fields, and wherever possible, matches what is on the image to the field on the invoice record. Only fields that are on the ap_invoice (PL invoice, credit or journal transaction) will be populated by OCR.

No data validation occurs during the OCR process. All fields automatically populated should be checked during normal invoice processing and posting.

The new layout can then be viewed (and edited if required) from the OCR layout screen in Document Management Links to check for accuracy (see 1 - Document Management OCR Invoices - OCR Layouts).

If No Supplier Is Identified

If no supplier is found, a blank ap_invoice record is created with only the automatically-generated internal reference number for the invoice or credit note. All other fields will be left blank for the user to complete.
