Importing Scanned Images

  1. Prepare the files to be imported by placing them into a network scanner location.
  2. Run Import Scanned Images.

    The options during scanner import are:

    Field
    Description

    Batch Name

    The name of the batch (co_batch record) to create in COINS. You can normally leave this as the value that the system generates, but you are free to use any value you wish.

    Index to batch

    A generic field which is normally set to yes (Y). Setting a value of no (N) will cause all files imported to be held in the same location, causing multiple users to try to find their work in the same pile of images.

    Scanner Location

    This is the location as set in the network scanner location configuration screen (Document Management > Setup > Configuration > Network Scanner Locations).

    Pool

    The target pool where the files you intend to import will reside. When using OCR, this is only a temporary location, as the files will pass straight into batch.

    Blank Page Action

    If your files are multipage PDF files, or multipage tiff files, this allows COINS to detect if a blank page has been used to separate the individual invoices in the file. If set, the action of New Document should be selected, and generally, the ‘Max Dark %’ should be left as the default value of 1.1. If the individual invoices are duplex scanned (both sides scanned at once), then the duplex option should be chosen. When scanning in duplex, there should be two successive blank pages to cause the file to be split.

    Split Multi-page Documents

    If the file imports is a batch of single page invoices, this option separates the file into its individual images to be processed separately.

    OCR to P/L Invoice

    This instructs COINS to process each image using the OCR engine as set in the parameters above. Currently, only tesseract is supported.

    P/L Invoice Document Group

    If document groups are used, you can choose which group this batch of invoices will be stored under. The default document group (if set) will be applied.

  3.  Applying the ‘Next’ action triggers a report.

      You can check the status of the report at any time using Report Status.

    Once complete, the report displays the values read against the Invoice that has been created.
