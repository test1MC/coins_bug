Document Management OCR Invoices - Parameters

The following parameters control the OCR functionality.

Parameter Description
OCRCAT

A list of document categories that can use OCR; for example: ap_invoice.

Currently only attempts to create a layout for ap_invoice documents. For any other categories, full-page OCR is run for the purpose of storing the information into SOLR.
OCRDATA

[New in 11.08] The fully qualified path to where supporting files for the OCR engine are located. By default, this should be /usr/local/share/tesseract/.
OCRENGINE

The OCR engine to use. Set this to tesseract.
OCROFFSET

[New in 11.08] The percentage of leeway allowed around each keyword box. Typically, this would be less than 1 percent, but an absolute maximum of 5 percent is permitted. On most images, 5 percent would be too large.
OCRPATH

The full path to the OCR executable; for example: /usr/local/bin/tesseract.

OCRTPDIR

The full path to the directory where the images used by OCR templates are stored.
