Screen and Processing Changes

Document Category Maintenance

Fields have been added to the Document Groups screen.

Fields

Field
Description

Template

An optional email (based on a system email template).

Upload

Whether to trigger the email when a document is inserted or updated against this group.

Check Out

Whether to trigger the email when a document is checked out from this group. (Some tables/categories, for example ap_invoice, cannot have this action applied.)

Check In

Whether to trigger the email when a document is checked into this group. (Some tables/categories, for example ap_invoice, cannot have this action applied.)

Delete

Whether to trigger the email when a document is deleted from this group.

Users/emails

A list of user IDs and/or email addresses to send the email to. Field names in curly brackets can be used; the field will be replaced by the database value. You can used field values from the dm_document record or the table associated with the document category. This will make it possible to configure who an email is sent to based on values on the associated record (for example, the buyer on an order).

NOTE

This has no effect on the workflow as that uses standard workflow responsibility functionality.

Email Templates

A new function, DM Email Templates , allows you to set up templates for the emails to send when documents are uploaded, checked in or out, or deleted.

This is similar to the standard Email Templates maintenance (see Email Templates in the online documentation), with the following additional fields:

Field
Description

Auto-Text

If this is ed, by default the subject line will include the following text:

Document <Description> (<doc_docid>) was <Uploaded|Deleted|Checked in|Check Out> By <User> on <date/time>

plus any text defined on the email template.

Alternatively, provided Auto-Text is ed, you can add the token [AT] anywhere in the subject or body text and the auto-text will be placed there.

Document Link

If this is ed, the email body will contain a URL that links back to the document in DM.

Alternatively, provided Document Link is ed, you can add the token [DL] anywhere in the body text and the document link will be placed there.

Use of {} Fields in Subject and Body

Field values can be used by surrounding the field name with curly brackets {}. Field values from the following tables can be used.

  1. Document
  2. Associated record
  3. System user
  4. company

If a field name exists on more than one table, it will return the value from the table highest in the list.

Changing Document Group

If the document group of a document is changed from or to a group that triggers email on upload/check in or check out, an email is triggered. If both groups trigger emails, only the email for the ‘to’ group is triggered.
