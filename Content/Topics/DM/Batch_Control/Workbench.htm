Document Management Batch Control Workbench

Batch Control is available on the Document Management Input menu.

Batch Control Workbench

The workbench consists of three sections: a browse of documents in the batch, a document display of the highlighted row, and batch selection options. Batch selection allows you filter by batches that you created (either via the scan client, or by importing documents from a network scanner), or by date, so the required batch can be more easily located.

Changing the Order of Documents

The display order is the order in which the operator will see the documents in the index screen. You can change the order of the documents in the batch using the Up and Down buttons; for example, to reorder documents that may not have been scanned in the correct order.

Deleting Documents

You can select one or many records, and delete them in one go. Deleting is permanent, but a warning message is shown to confirm the action.

Changing the Document Category

To make indexing the documents in the batch simpler, you can set the Document Category for individual pages in the batch. All the documents in the batch could be a specific category (such as PL Invoice), or they could be a mixture of categories.

  • To set the category for an individual record, update it using the Open button.
  • To set different categories on several records, select the records to update, and use the Concurrent button. Choose the appropriate category for each record then click Save. Selecting the row will still show the image on the right of the screen.
  • To set several records to the same category, select the records to update, and use the Multi button. Choose the category then click Save.

Once the categories have been set, it is possible to index the documents.

Indexing Documents

The workbench makes the processing of the batch simpler. You can view the documents before indexing, and Actions provide several ways to index some/all of the documents in the current batch.

Index Batch

This takes you to the index screen, to index all the documents in the selected batch. All the records in the batch must be set to the same Category, or an error will be displayed. The batch will then be indexed as usual.

Index Category

This option is useful if a batch contains a mix of document categories. This creates a new batch from the current batch, taking all the documents that are the same category as the selected record. This new batch will then be used in the index screen.

If all the documents in the batch are the same category, then a new batch will not be created, but the current batch (with all the same categories) will be passed to the index screen.

This process expects only one row to be selected, to indicate the category that you want to index. If more than one row is selected (even if they are all the same category), an error is displayed.

Index Selected

You can select specific records that you want to index; all the selected records must be the same category. This creates a new batch with the selected documents only and takes you to the Index screen.

If the selected records have different categories, an error is displayed. If the selected records include all the documents in the batch, then the current batch is used, and a new batch is not needed.

Moving Documents

You can move documents in a batch to other batches, and even into new batches. Select the documents in the current batch, then select the Move Document action. This displays the Move Documents to this Batch screen, from which you can create a new batch, or move the documents to an existing one.

Splitting and Merging Documents

It possible to split and merge documents (of the same type only).

Split Document

To split a multi-page PDF or TIFF document, select it and choose the Split Document action to be selected. This extracts all the pages in the document into single-page documents.

Merge Documents

To merge documents, select two or more documents. They can be single-page or multi-page documents, but must be the same document type. Choose the Merge Documents action. This combines the selected documents into a single multi-page document. If multiple document types are selected, the process will not complete, and an error will be displayed.
