Direct Debit Receipts - Parameters

The following new parameters determine the Direct Debit functionality.

Parameter

Description
BACSEXEC

An optional operating system command that is run after creates the file.

BACSFILE

The path and name for the generated file. The filename can contain tokens that are replaced when creating the file:

[DATE=YYYYMMDD] Replaced with the creation date in the format specified.

[TIME=HHMMSS] or [TIME=HHMM] Replaced with the creation time in the format specified.

[FDATE=YYYYMMDD] Replaced with the date in the format specified.

[BANK] Replaced with the bank code.

[KCO] Replaced with the number of the logged-in company.

[USER] Replaced with the user ID of the current user.

[BACSMED] Replaced with the name of the file definition specified in BACSMED.

For example: /home/dev/BACS/bacs[KCO][BANK][BACSMED][FDATE=YYYYMMDD].aba.

BACSMED

This has two entries separated by a comma:

  • The first is the medium for the file (for example, 6 for ANZ Direct Credits, or the name of the configurable form).

  • The second entry is the default description of the entries in the file.

BACSUSER

A comma-separated list of the default values for User Identification Number and User Preferred Specification. These can be overridden by entries on individual bank account records (see 1 - Bank Accounts).
DDMAIL The email address to use when emailing the ABA file generated as part of Direct Debit processing. This could be the user who will load the file into third-party banking software.
DDPOST Whether to post a Direct Debit receipt batch after creation. If not set, the receipt batch will remain unposted.
DDTTYPE The batch type for Direct Debit receipts.
USEDD Whether to allow customers to be configured as Direct Debit Y/N (default N). If this is Y, a new Direct Debit box is available on the Customer Statement tab; if that is ed, a new Direct Debit tab will be visible with associated fields that are linked (see 1 - Direct Debit Receipts - Customer).
