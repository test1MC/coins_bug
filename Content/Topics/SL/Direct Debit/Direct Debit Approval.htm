Direct Debit Approval

A new The Direct Debit Approval Workbench allows you to generate, approve, email or cancel Direct Debit extracts. The browse shows Direct Debit records created using Print Statements.

Fields

Field
Description

Period

The in which the Direct Debit record was created.

The date. A link on this field shows the Direct Debit details that have been created during the Statement Print process (see - Direct Debit Details).

Statement Date

The statement date that was used when the record was created.

Amount

The total amount of the payments.

Status

The status of the Direct Debit record:

  • To Email - The extract has not been generated (or emailed).
  • To Approve - The extract has been created and is awaiting approval (which also creates a cash batch).
  • Incomplete - The extract has been approved and an unposted cash batch has been created.
  • Posted - The cash batch has been posted.
  • Posting - The batch is currently being processed, or is stuck in Posting (see 1.4 - Direct Debit Processing - Resolving Problems).

Batch Number

The number of the cash batch (if it has been created).

A link on this field shows the batch details of the cash batch in , including a tab for Direct Debit Details.

Actions

Generate Extract

This generates the ABA file and produces a report.

The extract file is generated using the form configuration specified by the SL parameter BACSMED, and the file name specified by the SL parameter BACSFILE. The file is unique to the .

If the SL parameter DDMAIL has been configured, the file is emailed to the specified address, otherwise the file is just exported.

Approve Extract

This creates a Receipt Batch for the Direct Debit details using the transaction type given in SL/DDTTYPE. It also produces an approval report.

The batch is left as unposted unless the SL parameter DDPOST = Y, in which case it is posted automatically. The Status on the Direct Debit record and Direct Debit detail records is changed to Incomplete if the batch is unposted or to Posted if SL/DDPOST = Y and the batch has been posted. Creating a receipt batch also creates an allocation record linked to the invoice to the receipt to be auto-allocated. The bank is assigned from the Create Direct Debit Records generate.

Cancel Extract

This allows the Direct Debit details to reversed if there is a problem. This would normally be an administrative function and used as a last resort. This removes all Direct Debit records for the if Open. The control record is also removed so that the Statement Print can be re-processed.

Only records that have a status of To Email or To Approve can be cancelled.
