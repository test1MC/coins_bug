Direct Debit Receipts - Customer

If the new SL parameter USEDD is set to Y, a new an additional field is available on the Customer Details Statement tab.

Field

Field
Description

Direct Debit

Whether this customer is configured for Direct Debit receipts. If this is ed, a new the Direct Debit tab is displayed.

NOTE

If a customer is marked as Direct Debit, cash cannot be entered manually for that customer.

Direct Debit Tab

The Direct Debit tab allows you to configure the Direct Debit details for the customer. The fields available depend on the region (specifically, whether or not CO/COMPLOC = Aus).

Fields

Field
Description

Bank Name

Australia only. Optional. The name of the bank.

BSB

Australia only. Required. The BSB code for the branch.

Name

Not Australia. Required. The name of the bank.

Bank Details

Not Australia. Optional. Notes about the bank.

Account Number

Mandatory. The customer’s bank account number.

Not Australia. Mandatory. The of the customer’s bank.

IBAN

Not Australia. Optional. The International Bank Account Number of the customer’s bank account.

Swift BIC

Not Australia. Optional. The SWIFT Business Identifier Code for the customer’s bank.

Limit

The limit on the amount that can be taken by Direct Debit transactions in one . If the total value of the invoices exceeds this limit, an unpaid balance is carried over to the following period.

If this is zero, there is no limit.

On Hold

If this is ed, the customer is on hold and is not included in Direct Debit processing.

Reason

The resason that the customer is on hold. This is only enabled if the On Hold box is ed, and allows you to select from a list of Direct Debit hold codes (see 1 - Direct Debit Hold Codes).
