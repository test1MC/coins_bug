Statement Print - Email

Additional fields are available on the Statement Print selection screen.

 

 

Email Template

Email Template allows the user to choose which of the configured email templates to use when statements are emailed.

Customer Print Methods

If the SL parameter DEFPRN is set to Y, a new field, Customer Print Methods, is available.

Customer Print Methods allows the user to or untick the Email, Fax and/or Print

options. This then filters the statements based on the Print Method defined on the customer record, and splits the output into separate PDFs for email and fax.

If the user ticks Email, will email separate statements to:

  • All the customers whose default print method is EMAIL.
  • All the customers whose default print method is blank but who have an email address.

sends the emails from the address specified in the SENDFROM parameter.

If the user ticks Fax, will fax separate statements to:

  • All the customers whose default print method is FAX.
  • All the customers whose default print method is blank, and who have no email address but have a FAX number.

If the user ticks Print, will print separate statements for:

  • All the customers whose default print method is PRINT.
  • All the customers whose default print method is blank and who have no FAX number or email address.

In each case produces a PDF output that shows all the transactions that were faxed, emailed or printed.

Exclude Balances

In addition, the field Exclude Balances less than, is displayed. If the user enters a value in Exclude Balances less than, statements with balances less than the value entered will not be printed, faxed or emailed.
