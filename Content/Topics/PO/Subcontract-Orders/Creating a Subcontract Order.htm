Creating a Subcontract Order Header

From the Procurement Module select Subcontract Orders.

Click

On the Main Tab enter the following details:

 

Field

Description
Type

The type of the order. Order Types define the different types of purchase orders you can create.

For example:

  • Normal order
  • Call off order (see Call-Off Orders.)
  • Variation (see Variations.)

Order types differ mainly in the way orders behave if the order value is exceeded, and in the defaults that are set up on them. They can also be used as a filter on reports and enquiries.
Normal order Call off order (see Call-Off Orders.)
Variation (see Variations.) Order types differ mainly in the way orders behave if the order value is exceeded, and in the defaults that are set up on them. They can also be used as a filter on reports and enquiries.
Head/Site Order Whether the order is raised as a head office or site order. Head office orders and site orders have different order number prefixes, and can be viewed separately on enquiries.
Confirmation Order ? If this is a confirmation order (confirming an order made outside ), tick this box.
If the order is a confirmation order, text is printed on the order to indicate this.
Contract Number The contract (and phase, if appro­priate).
Use Order Form The Order Form to use
Subcontractor The name of the supplier with whom the order is raised. If you type part of the name, fills in the first supplier that matches what you type.
Unlabelled field This field may be labeled as "Last Score" or "Avg Score". It shows your company's assessment of this company. Also the date of the last performance review (if any) is displayed.
This is the average of the performance reviews for all offices of the company during the Performance Reviews History period. If there were no reviews in this period, then the last review's score will be displayed.
Status Shows the status of the Supplier company.
Office The name of the supplier's office with which the order is raised. If the supplier has only one office, defaults this in.
Unlabelled field The Purchase Ledger supplier account of this office (if present).
Unlabelled field The Trading-As name of the Supplier (if any).
VAT Registration No The subcontractor's VAT registration number.
Trade Select the commodity, trade, or plant code.
Description Enter a description of the order.
Unlabelled field The WBS code to identify where the item is needed (this is only required on contracts that use WBS codes).
Subcontract Type The type of subcontract.
Currency A code to indicate which currency the amounts are shown in.
Attention Enter the contact at the supplier's office to whom the order should be sent.
Order Date The date on which the original order was raised.
Buyer Select the user ID of the buyer. By default, fills in your user ID.
Special Instructions Enter any special instructions to be printed on the order.
Agreement Required Whether a subcontract agreement is required for this order. Checking this field will display an Agreements Tab for completion.
Special Terms Whether there are any special payment terms on this order. If so, displays additional fields on the Terms tab.
Review Required Whether performance reviews are to be carried out on this order. If this is ticked, includes this order in the list of orders that require performance reviews. See Performance Reviews.
Review Style The review style to use for this order. Review styles allow you to give different weightings to each of the performance review criteria depending on the type of order.
Labour Only Whether this order is in relation to a labour-only subcontract. This allows to determine whether certain insurances should be validated or not.
Warranties Whether warranties are required, or have been received, for this subcontract. If warranties are required and have not been received, you will be unable to authorise draft certificates.
Performance Bond Whether a performance bond is required, or has been received, for this subcontract. If a performance bond is required and has not been received, you will be unable to authorise draft certificates.

 

Select the Addresses Tab and check the Order and Site Address.

If you selected Special Terms on the main tab, select the Terms Tab and check the default Due Type is XX Days or change IF you have agreement for special payment terms.

Field

Description
Due Type This determines how interprets the Due Days field.
N means payment is due a fixed number of days after the date of the order.
A number means the number of months after the order date that payment is due; 0 means the month of the order. The Days field shows the day of the month on which payment is due.
Days This determines the date when the payment is due.
If the Due Type is N, the Days represent the number of days from the transaction date.
If the Due Type is a number, the Days represent the date in the month. 31 means the last day of the month.

Field

Description
