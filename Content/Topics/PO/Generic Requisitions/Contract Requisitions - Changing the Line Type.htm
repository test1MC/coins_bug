Requisitions - Changing the Line Type

If the new PO parameter ALWTYPCH is set to Y and the requisition has been issued, three new options appear in the Actions menu at the foot of the Requisition Lines screen:

  • Change Line Type to Material/
  • Change Line Type to
  • Change Line Type to Text

So, for example, if the person raising the requisition simply entered their requirement as a text line the buyer could change this line to a material/ line or code line.

If the system has been configured to allow the line type to be changed, a new column Line Type appears to the left of the Item column. This shows the line type as a single character, with a tooltip explaining what the single character means.

The Code field is then validated either as a material/ code or code with reference to the line’s new line type, and all other fields on the line are re-validated according to the new line type. So, for example, the unit of quantity may need to be amended. The lookup button also reacts according to the line type.

If the line type selected is “Text”, the Code field does not appear.

If a line is changed using this technique, the data originally entered is shown in a single new field on the requisition line, and the user ID of the person making the change and the date of the change are also shown in two new fields on the requisition line.

If any order has been placed against a requisition line then it is no longer possible to amend the line type of that line.
