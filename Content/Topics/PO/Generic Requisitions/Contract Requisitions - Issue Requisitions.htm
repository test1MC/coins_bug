Requisitions - Issue Requisitions

When issuing a requisition, if a line contains a blank Code and no and category were specified at the time of data entry, this is not treated as a costing error and the requisition can be issued.

Also, if the new PO parameter ALWTYPCH is set to Y (meaning that the line type on a requisition line can be changed after the requisition has been issued), it is possible to issue a requisition even though it may only have text lines on it. Previously, a requisition with only text lines could not be issued.
