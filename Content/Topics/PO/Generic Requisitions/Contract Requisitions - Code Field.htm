Requisitions - Blank Code Field

If the new PO parameter ALWRQNCD is set to Y, you can leave the Code field blank. In this instance the new PO parameter NCDDEFU is consulted to determine the default Unit of Quantity and Price Per.

If you leave the Code blank and if the and category are not configured to be entered, the derived cost analysis is not validated since this would always fail. The Status column on the Requisition Line browse screen does not show “Costing Error” in this case. Instead, the error condition will need to be corrected when the order is raised.

Various procedures that would previously have treated requisition lines with a blank Code field as an error have been modified to accept a blank Code if the requisition is a requisition:

  • Copying requisitions - the warning message about lines with cost coding problems is not shown if the only error is one that relates to a blank Code on a requisition that has no and category.
  • Price - when adding a requisition line to a price , if the line being added is on a requisition that has no and category, the line is now be added to the , and no error is shown.
