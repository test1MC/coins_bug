Copying Orders

When you create a new order by copying an existing one, most of the data is now taken from the copied order (rather than being taken from defaults). If you change certain values on the order header, they are now updated on the copied order lines. This applies if you change:

  • The number.
  • The order type.
  • The WBS code.

In each case when a new cost analysis is generated, the data is validated and if any errors are found a message is displayed and the lines are shown with an error status.

Changed

If the on the copied order is changed, the code on the order line analysis is changed and a new analysis is generated for each line.

The following sections list special cases.

Non-WBS  to WBS

If the on the original order does not use WBS, and the on the copy is changed to a that does use WBS, the following applies:

  • If the new order is a subcontract order and a header WBS code is entered, or if the new order is a material order and a default WBS code is entered, a new analysis is generated for each line, and informs the user.
  • Otherwise, the user must enter the WBS for each line on the new order.

Multi- Orders

If the original order is a multi- order, nothing is changed on the copy.

Changed Order Type

If the order type on the new order is changed, none of the analysis values are changed, unless the or category on the original order line is blank, in which case the values from the default order type settings are used to generate the new analysis.

Changed WBS Code

If the default WBS code on a new material order is changed, the analysis is not re-generated.

If the header WBS code on a new subcontract order is changed, the order line analysis is re-generated with the new WBS code.
