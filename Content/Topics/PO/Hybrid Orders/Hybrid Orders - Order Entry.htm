Hybrid Orders - Order Entry

If the PO parameter USEHYBPO is set to Y, a new Line Add Type of Material - LS is available for the following major types of order.

  • Material
  • Overhead
  • Workshop
  • Asset

This line type is not available on the following types of order, because they are already lump sum order types:

  • Subcontract

On lump sum lines, the Quantity, Un (unit), Price and Per fields are not available, and the Amount field is an input field in which you enter the lump sum amount.

Committing the Order

If the order being committed contains at least one lump sum line, does not issue a warning that there are zero value line items, even if the parameter WARNUNPR is set to Y.

Printing the Order

When printing lump sum lines, the following fields display as blank:

  • Quantity Ordered [pol_qty]
  • Unit of Quantity [pol_uoq]
  • Price [pol_price]
  • Per [pol_per]
  • Discount (1 to 5) [pol_disc, pol_disc2 etc]

GRN Entry

Just as with Lump Sum Orders, Lump Sum lines are never GRN’d even if the order itself is set for GRNs. Lump Sum lines are therefore not visible during GRN Entry.

Invoice Matching

If the order is marked as requiring GRNs, invoice matching lines are created for any lump sum lines on the order.
