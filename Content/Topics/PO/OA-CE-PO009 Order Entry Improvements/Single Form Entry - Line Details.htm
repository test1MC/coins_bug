Single Form Entry - Line Details

A section of input lines for the Major Order Type (Material, Overhead, , Workshop or Asset) is shown. The number of input lines is dictated by the relevant PO parameter MAXLINxx (see 1 , Single Form Order Entry - Parameters).

If the Major Order Type (Material, , Workshop or Asset) allows for multiple analysis types, additional sections of lines are shown. The PO parameters ALWMLAxx dictate whether multiple analysis types are allowed (see 1, Multiple Analysis Types - Parameters).

Line Type

The line type is equivalent to the Add Type on standard order entry.

The options available are:

  •  Material Lines - The Code field is validated as a commodity code and the lookup button reacts accordingly. All other fields behave as they do on the “standard” screen.
  • Stock Lines - The Code field si validated as a stock code and the lookup button reacts accordingly. If the ST parameter LUCKINS is set to Y then, just as in the “standard” screen, it is possible to enter a Shortcut code which is then replaced in the input field by the Stock Item code. All other fields behave as they do on the “standard” screen.
  • Text Lines - The Description field changes to a text input field and becomes the only input field. The lookup button, as on the “standard” screen, provides the ability to select a phrase to be inserted into the text.
  • Clause Lines - The Code field is validated as a clause code and the lookup button reacts accordingly. The text of the clause is displayed in the Description column. If the selected clause requires “plugged” data to be entered, after clicking the Save/Next button you will have to retrieve this order on the standard browse screen and make any amendments via that screen.
  • Extras Lines - The Code field is validated as an extras code and the lookup button reacts accordingly. All other fields behave as they do on the “standard” screen.
