Single Form Order Entry - Header Fields

Most of the fields behave in the same way as the fields on the standard entry screens. The comments below highlight any differences.

Top Left Pane – New Material Order

PO Number

This field only appears if the PO parameter POALLWMN is set to Y. Field validation ensures that any value entered in this field does not already exist as a purchase order number. (Currently in standard PO Entry this validation only takes place when the order header screen is saved.)

Immediate Commit?

This is a box field. If ed, when the order is saved it is committed automatically. This field does not appear if the PO parameter COMMETH is set to P.

Immediate Release?

This is a box field that appears if workflow is turned on for the Major Order Type. If ed, once the order has been created launches the relevant workflow.

Type

The drop-down list excludes the following Order Types which are not supported at the moment:

  • BoQ Orders (as dictated by the PO parameter BQPRICE)
  • eCommerce Open Orders (as dictated by the PO parameters OPENTYPE, OPBKTYPE and OPAGTYPE)
  • Memo Orders (as dictated by the PO parameter POMEMTYP)

Number

Including “Phase” if required (only shown on Material Orders). The number entered here defaults into the number fields on the Material Lines Pane (but only if the value in the target field is blank).

Default WBS Code

This appears immediately below the Number field if the uses WBS codes (only shown on Material Orders). The WBS code entered here defaults into the WBS Code fields on the Material Lines Pane (but only if the value in the target field is blank).

Department

Only shown on Overhead Orders. The department entered here defaults into the Department fields on the Overhead Lines Pane (but only if the value in the target field is blank).

Department/Location

Only shown on Orders. The department/location entered here defaults into the Location fields on the Lines Pane (but only if the value in the target field is blank).

Workshop Ref

Only shown on Workshop Orders. The workshop reference entered here defaults into the Workshop Ref fields on the Workshop Lines Pane (but only if the value in the target field is blank).

Asset Code

Only shown on Asset Orders. The asset code entered here defaults into the Asset Code fields on the Material Lines Pane (but only if the value in the target field is blank).

Buyer

This field is shown on the “standard” entry screen but is not shown on the “single form” version. However, the PO parameter POBUYRST is still consulted. If this parameter is set to Y then the logged-in user must have a buyer record.

Top Middle Pane – Deliver To

Additional Charges

If the CO parameter TAXTYPE is set to SALES then the recently-introduced Freight and Additional Charges fields, if configured, are shown. Unlike the standard screen, there is no facility to choose whether these charges are “fixed” or not. On this screen, all and any additional charges are taken to be “fixed”.

Ship Via

Only shown if any “Ship Via” records have been set up.

FOB

Only shown if any “FOB” records have been set up.

Top Right Pane – Order To

Use Order Form

This field is shown or hidden depending on the setting of the PO parameter FRMSELEC.

Bottom Panes

Single Form Order Entry - Bottom Fields

Less frequently entered data relating to the order as a whole is entered via a series of panes rendered below all of the line details panes.

The appearance of the above panes and certain fields within each pane is governed by standard parameter settings. So, for example, if the PO parameter POHCODE is set to N, the "Hold" pane will not be visible. Conversely, if the PO parameter POHCODE is set to B , the pane will appear as illustrated above.
