Terms and Conditions Sets Maintenance

Terms and Conditions Sets Maintenance is a new function that allows you to create and maintain terms and conditions sets. These consist of lines that are automatically added to a purchase order.

The initial Terms and Conditions Sets Maintenance browse allows you to give each set a code and description.

The Terms and Conditions Set Lines browse allows you to maintain the lines on the terms and conditions set.

Fields

Field
Description

Title

The default title of the terms and conditions line.

Mandatory

A box to control whether the line is mandatory. Mandatory lines cannot be removed from an order.

Normally ed

A box to control whether the line is ed when the set is added to an order. If a line is uned it will not be printed on the order.

Title Update

A box to control whether users can change the title of the terms and conditions line on an order. By default this is uned.

Additional Text

The detailed text of an item in the terms and conditions.

If you want the user to be able to enter specific values (such as dates or amounts) when they enter the order, you can insert plugs (see Plugs in the online documentation for information); click the button and select the appropriate type of plug from the lookup.

You can also use fields from the company, office, , or project by entering them in curly brackets. For example, to include the name in the terms and conditions, enter {jc_job.job_name}.

Additional Text Update

A box to control whether users can change the additional text on an order. By default this is ed.
