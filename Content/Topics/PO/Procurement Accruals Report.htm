Procurement Accruals Report

The Accruals Report allows you to produce a report that shows accrued costs (value of materials delivered but not invoiced). If you have permission, you can also generate an accruals batch.

The report takes account of all accruals, regardless of how they occur; for example: GRNs, orders accruing based on a due date, plant allocation sheets, plant maintenance.

You can only create the accruals batch if you have access to the function %WPO3010RXXXS1.

If a contract, overhead department, asset or stock location has already been included in an accrual batch for the period, will not include it in the new batch.

Configuration

Parameter : GL/ACCTTYPE

The GL transaction type used when posting monthly accruals. This must be a reversing transaction type.

Process

Navigate to Procurment > Reports > Goods Received and Plant > Accruals Report

Complete the applicable selection screens. On the first run, DO NOT the create Batch Option

Parameters Tab

Field

Description

New Page per Contract / Department / Location

Whether to create a page break at the introduction of a new contract.

Cut Off Date

This determines which ordered items to include, and the period for the accruals batch. Maximum Length: 10 characters

If the report is being run in "Create Batch" mode, this date must relate to an open GL period.

The report will ignore:

  • GRNs dated after this datefor orders not using GRNs, items whose due date is after this date
  • confirmed plant allocation sheets whose week ending date falls after this date

Create Batch

Whether to create an accruals batch.

Foreign Currency Value

Check it to show values in foreign currencies as well as in base, for orders with foreign currency suppliers.

Exclude Effect of Later Invoices

Whether to take later invoices into account.

Include Tab

Field

Description

Include

Whether to report on commodities, plant items or both.

Contract Selection Tab

Once you have made your selections, Click Next to run the Report. Run Accruals Report in Procurement (Ensure that you do not the Create Batch option at this stage).

Once the report has been checked, re-run the report, but this time the create batch option.

A Contract Status batch will be generated (but NOT posted). This batch must then be posted.

The batch will automatically reverse in the next period (thus the batch is posting in two periods) – therefore BOTH periods must be open, otherwise the batch will ‘stall’ in Posting status.
