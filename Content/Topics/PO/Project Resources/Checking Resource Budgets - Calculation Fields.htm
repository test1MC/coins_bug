Checking Resource Budgets - Calculation Fields

Several new fields can be used as decision points in the workflow process in order to decide, for example, whether a requisition or order should be routed for further approval.

Requisition Fields

RO_LinesOverQtyResourceBudget

This field returns a list of requisition item numbers whose cumulative requisitioned quantity would exceed the budgeted resource quantity if the requisition was issued (approved). This takes into account the quantity tolerances specified by the PO parameters RBQTOL% and RBQTOLAM. The field RO_iLinesOverQtyResourceBudget returns 0 if the above list is empty or 1 if the list is not empty.

RO_LinesOverAmtResourceBudget

This field returns a list of requisition item numbers whose cumulative requisitioned amount (that is, financial value) would exceed the budgeted resource amount if the requisition was issued (approved). This takes into account the amount tolerances specified by the PO parameters RBATOL% and RBATOLAM. The field RO_iLinesOverAmtResourceBudget returns 0 if the above list is empty or 1 if the list is not empty.

RO_CostHeadsOverBudget

This field returns a list of whose associated financial budget (a Status budget) would be exceeded if the requisition was issued (approved). If the list is empty then the associated budget would not be exceeded. The field RO_iCostHeadsOverBudget returns 0 if the above list is empty or 1 if the list is not empty.

RO_CostAnalsOverBudget

This field returns a list of cost codes whose associated financial budget (a Status budget) would be exceeded if the requisition was issued (approved). If the list is empty then the associated budget would not be exceeded. The field RO_iCostAnalsOverBudget returns 0 if the above list is empty or 1 if the list is not empty.

Order Fields

An important point to note in the following paragraphs is that we refer to the Maximum Allowable Quantity and Maximum Allowable Amount.

If a resource budget can be found and the Requisitioned Quantity is non-zero and is greater than the Budgeted Quantity then the Requisitioned Quantity is the Maximum Allowable Quantity; otherwise the Budgeted Quantity is the Maximum Allowable Quantity.

Similarly, if a resource budget can be found and the Requisitioned Quantity is non-zero and the Requisitioned Amount is greater than the Budgeted Amount then the Requisitioned Amount is the Maximum Allowable Amount; otherwise the Budgeted Amount is the Maximum Allowable Amount.

RO_LinesOverQtyResourceBudget

This field returns a list of order item numbers whose cumulative ordered quantity would exceed the resource’s maximum allowable quantity if the order was committed (approved). This takes into account the amount tolerances specified by the PO parameters RBATOL% and RBATOLAM. The field RO_iLinesOverQtyResourceBudget returns 0 if the above list is empty or 1 if the list is not empty.

RO_LinesOverAmtResourceBudget

This field returns a list of order item numbers whose cumulative ordered amount (that is, financial value) would exceed the resource’s maximum allowable amount if the order was committed (approved). This takes into account the amount tolerances specified by the PO parameters RBATOL% and RBATOLAM. The field RO_iLinesOverAmtResourceBudget returns 0 if the above list is empty or 1 if the list is not empty.
