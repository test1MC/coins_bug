Checking Resource Budgets

If project resources have been used for budgets, when you enter a requisition or an order can alert you if the requisition or order exceeds the budgeted quantity.

On the project resources screens:

  • The Requisitioned Quantity is shown with a red background if it exceeds the Budgeted Quantity.
  • The Ordered Quantity is shown with a red background if it exceeds the Budgeted Quantity. However, if the Requisitioned Quantity also exceeds the Budgeted Quantity assumes that any excess requisitioned quantity has been approved via the requisition approval process, so if the Ordered Quantity does not exceed the Requisitioned Quantity it is not shown with a red background.
  • The Ordered Quantity is also shown with a red background if the total Ordered Amount (that is, the financial value) exceeds the Requisitioned Amount, and the Requisitioned Amount exceeds the Budgeted Amount (this assumes that any excess requisitioned amount must have been approved via the requisition approval process).

If there are unissued requisitions for a resource line, the letters UR show next to the requisitioned quantity. Tooltips on the quantity and the UR text show details of the issued and unissued requisition lines respectively (up to five).

Links open screens that show the requisition lines and allow you to dig deeper to the requisitions themselves.

If there are uncommitted orders for a resource line, the letters UO show next to the ordered quantity. Tooltips on the quantity and the UO text show details of the committed and uncommitted order lines respectively (up to five). Links open screens that show the order lines and allow you to dig deeper to the orders themselves.

Requisitions

If requisitions are created using the Raise Requisition action on the Project Resources tab or the Tag Resources option within a requisition, there is a guaranteed link with the budgeted resource line. This allows easy comparison with the budget, and warning notifications can be shown on the Requisition Entry screen.

When you enter a requisition using one of these methods, if has been configured to check the budget and the total Requisitioned Quantity (including all other requisitions for the same project resource line) exceeds the Budgeted Quantity on the related project resource line, the requisition line shows two exclamation marks on a red background. A tooltip on this notification shows the quantity and amount budgeted and requisitioned, and highlights the difference.

Note that the value of a requisition line is calculated using the Budgeted Rate held against the resource.

Orders

If orders are created using the Raise PO action on the Requisitions Workbench, the Tag/Add Requisition Items option within an order header, or the Raise PO action on the Project Resources screen, there is a guaranteed link with the budgeted resource line and hence warning notifications can be shown on the PO Entry screen.

When you enter an order using one of these methods, if has been configured to check the budget, the order line shows two exclamation marks on a red background if the total Ordered Quantity (including all other orders for the same project resource line) exceeds either:

  • the total Requisitioned Quantity (if non-zero and the Requisitioned Quantity exceeds the Budgeted Quantity).
  • the Budgeted Quantity (if the Requisitioned Quantity is zero) on the related project resource line.

A tooltip on this notification shows the budgeted, requisitioned, and ordered quantity and amount, and highlights the difference between the ordered and approved figures (the “approved” figure is the greater of the requisitioned figure and the budgeted figure; this assumes that if the requisitioned figure exceeds the budget, approval for the increase has been given).
