Checking Resource Budgets - Parameters

The following new parameters affect the of the functionality to check resource budgets during requisition and order entry.

Parameter

Description

POBUDTXT

Warning text that is shown at the top of the tooltip if the ordered quantity exceeds the approved quantity. The approved quantity is assumed to be the greater of the budget quantity and the requisitioned quantity.

POCHKRBD

Whether checks the ordered quantity against the approved quantity when you enter an order. The approved quantity is assumed to be the greater of the budget quantity and the requisitioned quantity.

RBATOL%

The tolerance percentage to use when checking resource budget amounts. This is only consulted by calculation fields (which are intended for use in workflow).

RBATOLAM

The tolerance amount to allow when checking resource budget amounts. This is only consulted by calculation fields (which are intended for use in workflow).

RBQTOL%

The tolerance percentage to use when checking resource budget quantities. This is only consulted by calculation fields (which are intended for use in workflow).

RBQTOLAM

The tolerance amount to allow when checking resource budget quantities. This is only consulted by calculation fields (which are intended for use in workflow).

RQBUDTXT

Warning text that is shown at the top of the tooltip if the requisitioned quantity exceeds the budget amount.

RQCHKRBD

Whether checks the requisitioned quantity against the budget quantity when you enter a requisition.
RQPOQTY

Whether to allow the ordered quantity to exceed the requisitioned quantity. The options are:

  • P = Prevent the ordered quantity from exceeding the requisitioned quantity (this is the default).
  • V = Allow the ordered quantity to exceed the requisitioned quantity (but only using a ).
  • A = Allow the ordered quantity to exceed the requisitioned quantity.
