Raising a Requisition from the e

If the requisition being created is covered by the new PO parameter REQECAT, the changes documented below apply.

The words “Via e” appear in red on the requisition header.

A new box is available:

Field
Description

Disable Punchout?

The ability to raise a requisition without accessing the e may be a requirement for certain users. This box will only appear if the user has access to the function %WPO1010FRQH and if no lines have been entered against the requisition. If the requisition has been issued, even if changes are allowed to an issued requisition, the box will not appear. If the user s the Disable Punchout? box, the “Via e” text is hidden and the e is not invoked when the user clicks Save.

While the requisition remains un-issued, it can be changed at any point, so new lines can be added via the e and existing lines can be deleted or amended.

When the user clicks the Save button (and if the Disable Punchout? box has not been ed), the e is invoked. The following “holding” screen is shown:

A request to the FirstB2B e is made in order to establish a connection between the requisition header just created (or amended) and the e.

A new tab is opened in the browser, which displays the initial cross-supplier screen as determined by FirstB2B.

Once the user has filled their basket in the e, they close the e tab and click the Forward button. The basket is retrieved and the user is taken to the Requisition Lines screen.

Requisition Browse

If the requisition has been raised via the e, ‘e’ is shown in the column immediately to the right of the Requisition Number column. If the tab is relevant, the same new ‘e’ column is visible on that tab.

Requisition Summary

If the requisition has been raised via the e, the words “Via e” are shown on the Main tab.
