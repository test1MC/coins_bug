Locking the Description on PO Lines - Commodity Codes

If any of the parameters used to lock order line descriptions are set to Y, the following new fields are available on Commodity Workbench.

Field
Description

Additional Description Rule

Whether any additional text is required when using this commodity on an order.

The options are:

  • Default to Header Code (the rule on the header code applies; this is only available on detail commodity codes)
  • Not Required (the additional text box is not visible on order line entry)
  • Mandatory (the additional text box cannot be left blank during order line entry)
  • Optional (the additional text box can be left blank during order line entry)

For example, you may have a header code for bricks on which the Additional Description Rule is set to Mandatory. Typically, you may set every detail code below that head code to “Default to Header Code”. In this way, you are miniming the maintenance effort. However, you can then visit certain detail commodity codes within that same header code and set their Rule to, say, “Optional”.

Coaching Label for Additional Description

Text explaining to the user that they should enter additional information specific to the selected commodity code. So, for example, on a commodity code that represented some sort of conduit you might set this text to “Please specify size (e.g. 10m) and finish (e.g. Steel)”.

Note that you may not want to enter any coaching text on certain detail commodity codes because their related header code’s coaching text should suffice. During order entry display the coaching text from the header code if the selected commodity code’s coaching text is blank.
