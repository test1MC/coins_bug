Configuration

New PO Parameter

The PO parameter STTAXDEF has been added to the Procurement module. This parameter is only relevant for US companies (where the CO parameter TAXTYPE is set to SALES).

Parameter

Description
STTAXDEF

Orders – Default Sales Tax Code on PO line from:

• Location (Default) (Code = “L”)

• Item (Code = “I”)

Changed PO Parameters

The PO parameters in the table below have been changed to include functionality for Sales Tax Code defaulting in the Procurement module when they are set to Y.

Parameter

Description
ALWMLAAS

Allow multiple Analysis Types on Asset Orders

ALWMLAMA

Allow multiple Analysis Types on Material Orders

ALWMLAWS

Allow multiple Analysis Types on Workshop Orders

Screen and Processing Changes

The Sales Tax Code referred to throughout this document is held against the Items Tax code, as shown in Figure 1.

Sales Tax Code

PO Entry

Standard Order Entry

When a an Item is added to a an Order on any PO Order (see Figure 2 as an example), the system now checks the status of new PO parameter STTAXDEF.

  • If the parameter is set to I, and the Item selected in the Code column (#1) has a non-blank Sales Tax Code, that Item's Sales Tax Code defaults into the Sales/Use Tax Code column (#2).
  • If the parameter is not set to I, the Sales Tax Code of the Location (#3) defaults into the Sales/Use Tax Code column (#2).

Standard Order Entry

Single Form Order Entry

When a an Item is added to a Single Form Order in More Orders > Orders (Single Form) > Add Order - Single Form Entry (see Figure 1), the system now checks the status of new PO parameter STTAXDEF.

  • If the parameter is set to I, and the Item selected in the Code column (#1) has a non-blank Sales Tax Code, that Item's Sales Tax Code defaults into the Tax Code column (#2).
  • If the parameter is not set to I, the Sales Tax Code of the Location (#3) defaults into the Tax Code column (#2).

Single Form Order Entry

Multi Analysis Type Orders

When PO parameters ALWMLAAS ( Orders), ALWMLAMA (Material Orders), and ALWMLAWS (Workshop Orders) are set to Y, you can add "true" lines to other types of both Standard and Single Form purchase orders. If they are set to N, you cannot add any Items to an /Material/Workshop order. The same logic also applies for how to default the Sales Tax Code when a code is added to any of these orders.
