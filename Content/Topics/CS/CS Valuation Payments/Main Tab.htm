Due Type

The due type (whether the due date is calculated as a number of days, or as a number of months, from the date of the transaction).

The Due Type and Days fields are used to determine when payment is due.

The Due Type field controls how the Days field is interpreted. If the type is No of Days, Days represents a fixed number of days from the date of the transaction. If the Due Type is a number, it represents the number of months from the date of the transaction, and Days represents a date in the month.

  • If the due date is a fixed number of days from the date of the transaction, select No of Days in the Due Type field, and enter the number of days in the Days field. For example, "No of Days, 30" on a transaction dated 25/08/03 would give a due date of 24/09/03.
  • If the due date is a fixed date in the month, select a number in the Due Type field to represent the number of months from the transaction date and enter the date in that month in the Due Date field. For example, "2, 15" on a transaction dated 25/08/03 would give a due date of 15/10/03. A type of 0 means the month of the transaction.
  • If the due date is the last day of a month, select the number of months from the invoice date in the Due Type field, and enter 31 in the Due Date field.

Due Days

This determines the date when the payment is due.

  • If the Due Type is N, the Days represent the number of days from the transaction date.
  • If the Due Type is a number, the Days represent the date in the month. 31 means the last day of the month.

Calculate Due Date from

Use

How certificates for this series should be entered.

The options are:

  • Apps & Certs = Applications and certificates are entered separately.
  • Certs only = Only certificates are processed; applications are not used.
  • Combined App-Cert = Applications are entered and the certificates are populated automatically. This allows you to report on applications and certificates separately, but only enter a single document.

Select the default transaction type to be used when creating a certificate: Certificate, Invoice, or Receipt.

If the contract is handled on an Invoice or Receipt basis, the tax point is the certificate date. VAT is updated when the certificate is entered. Note: the Receipt option is for use when you only receive cash from a customer and wish to enter a certificate as a receipt.

If the contract is handled on a Certificate basis, the tax point is the Deposit Date. VAT is updated when the payment for the certificate is posted. You should use this option if the contract is handled on an Authenticated Receipt basis.
