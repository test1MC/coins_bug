Valuation Payment Stages Tab

This allows you to set up the valuation stages. Each stage allows you to record different amounts that relate to the valuation (for example, the amount you applied for, the amount you expect, the final certified amount). You can define up to five stages, of which the last one must be the 'certificate' stage (the 'Receipt' stage is fixed by the system).

You can name each stage and specify the time (in days) difference between each of the valuation payment stages.

On the example screen above, it expects there to be 4 stages.  (If you are using non-VPS then it starts in cert/invoiced)

 

Field

Description

Stage

The code for the valuation stage.

Description

Text to identify this stage.

Offset

The number of days after the previous stage when this stage is due.

Contact

The person who needs to be contacted when this stage is due. ( > Reports > Valuations Reports > Valuation Payment Stage Due Report shows these contacts.)
