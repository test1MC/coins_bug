Valuation Payment Stages – Rules for Cumulatives

This section explains how cumulative amounts are defaulted from preceding amounts. In the following notes, Previous Stage takes account of the CS parameter STG1BUD; if this is set to Yes, this stage does not participate in the defaulting down of the Cumulatives. In effect, it becomes an invisible stage as far as Cumulatives are concerned.

Cumulatives

For each Valuation Stage the “Cumulative” values are defaulted from preceding “Cumulative” values as follows:

  • When Valuation 1 is at Stage 1: Defaults from take-on certificate values or none (if no take-on certificate)
  • When Valuation 1 is at Stage 2 or beyond: Defaults from Previous Stage
  • When Valuation 2 (or greater) is at Stage 1: Defaults from latest Stage of previous Valuation
  • When Valuation 2 (or greater) is at Stage 2 or beyond: Defaults from Previous Stage of current Valuation

Previous Cumulatives

For each Valuation Stage the “Previous Cumulative” values are defaulted AND updated dynamically (trickle down) from preceding “Cumulative” values as follows:

  • When Valuation 1 is at any Stage: Defaults from take-on certificate values or none (if no take-on certificate)
  • When Valuation 2 is at any Stage: Defaults from latest Stage of previous Valuation

There is no “trickle down” on Cumulatives.
