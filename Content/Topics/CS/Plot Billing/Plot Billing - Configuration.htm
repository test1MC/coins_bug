Plot Billing - Configuration

Parameters

The following new VAP parameters affect plot billing:

Parameter

Description

BILLCSTA

The transaction type for applications created using Generate Plot Billing.

BILLCSTC

The transaction type for certificates created using Generate Plot Billing.

BILLPCR

Whether revenue is plot costed. If this is set to Y and the revenue head from the series is set up to be plot costed (mandatory or optional), the revenue analysis will be costed to the relevant section.

BILLPOST

Whether batches generated using Generate Plot Billing are automatically posted.

BILLVAL

Whether to generate billing from the group valuation.

BILLVAP

Whether to use Plot Billing via .

Billing Stage Types

Billing Stage Types allows you to configure the application and certificate lines that will be populated with the plot billing amounts. These also determine whether the stage is for detailed variations or a lump sum amount.

Standard Billing Stages

Standard Billing Stages is a new function that allows you to configure sets of billing stages which can be used on new series. You can create one or more sets of standard stages at level, or create separate sets for individual s.

You can identify one set of company stages, and one set for each , as the default. When a new series is created, the stages from the default set will be copied to it: if a default set for the exists, this will be copied, otherwise the default set will be copied.

The link on the Ref column takes you to a screen that allows you to set up the individual billing stages.

Application and Certificate Print

New fields Fields on the Extras tab of the configurable application and certificate documents allow you to specify the forms used to print out details of the matrix stages/plots and variations that are included in the application or certificate.
