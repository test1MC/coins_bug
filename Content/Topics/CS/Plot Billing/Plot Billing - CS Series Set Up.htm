Plot Billing - Series Set Up

If the new VAP parameter BILLVAP is set to Y, the setup of a series includes functionality to enable the billing of home builders and housing associations, including the ability to set up the stages, plots and amounts that are to be billed, and to cater for variations.

  1. To set up the series so that it uses plot-based billing, the new VAP Plot Billing Series box. (The Input Method is automatically set to Applications/Certificates.)
  2. If the new VAP parameter BILLVAL is set to Y, you can also indicate that applications and certificates can be generated using the new group valuation Generate Billing process. the Tick Group Valuation Generation box.

When you save the series, if a default set of standard billing stages has been configured (either for the specific customer account or for the ), these are copied to the series billing stages.

You can also use the Copy Standard Stages Option to copy to the series any of the standard billing stages that have been configured for the or the .

New tabs Tabs allow you to set up the billing stages, enter variations, and enter order values on a matrix.

Billing Stages

The new Billing Stages tab allows you to configure the billing stages for the series.

For each billing stage, you can enter:

  • The associated with the billing stage, and the payment stage number within the . These are optional, but allow the amount to be billed to be included in , reporting etc, and also for consolidated reporting across where different clients may have a different stage breakdown.

  • The group(s) linked to the billing stage, if billing is being generated from tick sheet valuations. These determine the stages to be billed when generating applications or certificates from the valuation matrix.

  • The billing type for the stage, which determines the input method and application or certificate line that will be populated with the value from the matrix.

Variations

The new Variations tab allows you to set up and record variations for the series. For variations that cover multiple plots, you should use a designated general plot. The combination of plot and billing stage determine which matrix element is populated with the variation amount.

  • Date the variation was agreed. When adding a new variation, this defaults to today’s date.
  • The billing stage for the variation. This is a drop down of the variation billing stages configured on the new series staging tab.
  • The plot number for the variation.
  • Description of the variation.
  • An optional external reference. This allows you to record any client reference for the variation.
  • The agreed order value for the work.
  • The cumulative amount applied for. This will be read-only.
  • The cumulative amount certified by the client. This will be read-only.
  • A text editor allows you to enter any notes for the variation.

Clicking the Detail button opens a desktop frame that lists the transactions (applications and/or certificates) entered against the variation.

Matrix

The new Matrix tab allows you to enter, for each plot and billing stage, the agreed order amount that the client is to be billed.

A tooltip on each cell shows the applied and certified amounts, and also any disputed amount.

You can use the selector to display different amounts in the cells:

  • The agreed order value for the client. This is the only cell value you can update.
  • The amount applied for.
  • The certified amount.
  • The disputed amount.
  • Unapplied - The remaining balance that is to be applied for.
  • Uncertified - The remaining balance that is to be certified.
  • Applied Not Certified - The difference between the current applied and certified amounts.
