PR Timesheets

PR Timesheet Entry

Three new fields are available in Payroll > Timesheets > [Frequency] > Employee link > Time Cards tab: PW Schedule, PW Trade, and PW Skill, as shown in PR Timesheet Entry Fields.

These same fields are also now available in these functions: Timesheets - by , Timesheets - by Employee, and Timesheets - by Multiple Employees.

PR Timesheet Entry Fields

Field Definitions

PW Schedule

The code of the Prevailing Wage Schedule. This field defaults in from the or Section level.

PW Trade

The Prevailing Wage Trade. This field defaults in with the Prevailing Wage Trade information from the Employee level.

PW Skill

The Prevailing Wage Skill level of the employee. This is used with a PW Schedule and PW Trade to determine Prevailing Wage Rates/Calculations. This field defaults in with the Prevailing Wage Skill information from the Employee level.

PR Timesheets Overrides Tab

A new PW CALC Override Type is used in Payroll > Timesheets > [Frequency] > Employee link > Overrides tab, as shown in PR Timesheets Overrides. When PW CALC is selected, the PW Schedule, PW Trade, Calc No, and Timecard fields must also be entered.

In addition, these same fields can be found in the following functions: Timesheets - by , Timesheets - by Employee, and Timesheets - by Multiple Employees.

PR Calculation

The PR Calculation function has been updated to take new takes Prevailing Wage calculations into account. For each timecard with a Costing Method, only one Calculation Type can be applied. First, the calculation process searches for calculations and rates for the timecard. If calculations are not found, it searches for PW calculations and PW rates.

Net Detail Tab

On the Timesheets > Net Detail tab, the Fringes column has been renamed as Prev Wage Fringes, and the Deductions column has been renamed as Prev Wage Deducts. These columns (shown in Net Detail Tab) include information for both Calculations and Prevailing Wage Calculations.

These changes have also been made on the Net Detail tab of the Manual Calculator and Period Maintenance functions.

Net Detail Tab

Burden Detail Tab

On the Timesheets > Burden Detail tab, the Fringes column has been renamed as Prev Wage Fringes, as shown in Burden Detail Tab. Also, the Fringes Tax Type has been changed to Prev Wage Fringes. Prev Wage Fringes now include the fringes for the Calculations and Prev Wage Calculations.

These changes have also been made on the Burden Detail tab of the Manual Calculator and Period Maintenance functions.

Note: The Burden Detail screen appears when you click the Total Hrs column hyperlink on the Burden Detail tab.

Burden Detail Tab

Prevailing Wage Calculation Detail Screen

The label of the Calculation Detail screen has been changed to Prevailing Wage Calculation Detail on the Net Detail tab for Prevailing Wage Deductions ( Prev Wage Deductions for Net Detail) and Prevailing Wage Fringes ( Prev Wage Fringes for Net Detail), and on the Burden Detail tab for Prevailing Wage Fringes ( Prev Wage Fringes for Burden Detail).

This screen includes information for Calculations and Prevailing Wage Calculations. It contains the following columns:

  • Prev Wage - This column has a box that indicates whether this is a Prevailing Wage or calculation.
  • PW Schedule - This column is used for the Prevailing Wage Schedule. It is blank for calculations.
  • Trade and PW Trade - The Trade column is for a Calculation, and the PW Trade column is for a Prevailing Wage Calculation.

These changes have also been made on the Burden Detail tab of the Manual Calculator and Period Maintenance functions.
