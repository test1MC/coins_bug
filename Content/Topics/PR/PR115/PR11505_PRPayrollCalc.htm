PR Payroll Calculation

Payroll Calculation processing (Payroll > [frequency]> Calculation) now combines time card analysis with masking. Listed below are examples of the processing that can occur.

  • If the mask is _ _ _ L, an attempt is made to substitute the first 3 characters from the time card’s . If 3 characters are not available, a warning is received during PR Calculation that the Payroll Redirection analysis for certain items (such as Gross , Gross Premium Only, Taxes, etc.) is invalid.
  • If the mask is 100_, and the is L on the time card level, the resulting is 100L.
  • If the mask is _ _ _, the resulting is retrieved completely from the time card level.
  • If the cost code of the Payroll Redirection analysis does not exist, an Invalid Analysis error is received during Validate Batch or other appropriate TS Group validations. Processing is automatically stopped, and you must address the problem by making necessary corrections and ensuring that Payroll Redirection works.
