Overview

Some COINS OA users experienced numbers being stored incorrectly against the PR record when two Print processes for the same bank were run simultaneously. A stop was added to prevent both the simultaneous Print processes and the return of the user to Print .

However, users received an error that Print was already in process for the bank and were subsequently returned to the Main tab of Print (thus re-defaulting the bank number). Once returned to the Main tab, if a user did not refresh the page before waiting to click the Next Action button after the Print function was completed, the number presented was not necessarily the next available number. This caused issues when the number assigned to the record was a duplicate and did not match the printed number or the values included in the Positive Pay files sent to the bank.

Changes have therefore been made in COINS OA Payroll to resolve this issue and are discussed in detail in this document.
