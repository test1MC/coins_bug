Timesheet Input Group Maintenance

For v11.08, PR Group Maintenance now accommodates Timesheet Input Groups, as discussed in this section.

Browse Screen

When you select the new Group Maintenance menu option in Payroll > [selected Frequency], a browse screen appears, as shown in Figure 1: Browse Screen.

Note:

The Timesheet Input Groups menu section and Group Maintenance are always shown for the US and Canadian regions regardless of whether the company is set up to use TS Input Groups. If TS Input Groups are not in use for the company, and you attempt to enter a group, the following error message appears: Timesheet Input Groups are not in use for PR Company …. You cannot add or update TS Input Groups.

Field Definitions

Input Group

The name of the TS Input Group. This cannot be updated after it has been entered.

Group Description

The description of the TS Input Group. It can be entered or updated manually for your own groups and subordinate groups.

Group Owner

This is set to User ID when a group is created. The owner of the TS Input Group is the ID of the user with permission to access the company. This can be updated manually but only for your own groups and subordinate groups. You cannot change the owner to another user who is not on the subordinate list.

Period

A period will be set once a time card has been entered for the group. The Post Batches function on the Costing Workbench can use this to retrieve all groups that need to be deleted. You cannot enter or edit this field manually.

Year

The Payroll year.

Validated

This field indicates whether the TS Input Group is validated or not. The new Validate Timesheet Input Group report sets this field for the selected groups in the current Regular/Adjusting period. The Validate Timesheet Input Group is similar to Validate Batches on the Costing Workbench, but limits the review of data to only those /timecards in the group. The PR Calculation and PR Reset functions can reset this flag. You cannot enter or edit this field manually.

Frozen

This indicates whether a Timesheet Input Group is frozen to TS Entry and Reset. A new Group Administration tool can set this field. You cannot enter or edit this field manually.

Body Detail Screen

The fields shown in Figure 2: Body Detail Screen are visible on the Body Detail section of the Timesheet Input Group Maintenance screen. These fields calculate total amounts for time cards and that are part of the Timesheet Input Group and provide Pay Salary Security, Employee Timesheet Security, and Employee Timecard Rate Security. This information is only available for the group's owners or subordinates to the group's owner.

  • You can only enter or update your groups and the groups of your subordinate users. You do not have permission to enter or update groups of another user.
  • You cannot change your own or subordinate groups' owner to another user who is not on the subordinate list.
  • You will receive an error if you attempt to enter a blank owner.
  • You will receive an error if you attempt to change the owner to another user who does not have permission to access the PR Company.
  • You can only delete your own group if the group does not have associated or time cards; otherwise, you receive an error. If you need to delete the group of a subordinate user, you must first change the owner of the group to yourself.
  • Empty groups (such as associated ) cannot be deleted, but they will be deleted when the payroll is posted.
  • Groups cannot be deleted if they are frozen.
  • A group cannot be entered if it is already entered for another pay frequency. In this case, the following error message appears: Record already exists (pr_tsgroup). You have tried to create a record which already exists.
  • An error message appears if you attempt to enter any of the following in the Input Group field: an asterisk (*), a comma (,), or a group with a blank description.
