Using the EOY Workbench in COINSplus

Create EOY for 2016

The P35L XML is now created via the EOY workbench.

The relevant users will have to be granted access to the function (%pr736 and pr736.p), using “Function Access Maintenance”.

Go to Weekly (or Monthly) Payroll > Weekly (or Monthly) Administration > Irish Routines > End of Year Internet Return.

COINSplus End of Year Internet Return Screen

If more than one tax code is set up for the relevant companies (set up against the EGOVCOS parameter), the workbench will display these. In this instance, there is a single Tax reference for both weekly and monthly payroll within the one company.

Press Return on the Registration number; takes you into a secondary screen within the workbench to create the XML file for online submission or in OA click on the company registration number.

Press the F9 key to add a new submission entry. Enter the Year as 2016 and press Return. This will bring up the detail of the workbench for the year selected.

Validate and Complete Batch

COINSplus End of Year Internet Return Workbench - Menu

The original status of the submission (XML creation batch) will be “incomplete”.

Select “complete” from the menu; will perform the necessary validation on the data contained within the submission. If there are no errors found, will bring up a further option, Create XML File (as shown below)

COINSplus EOY Return – Create XML File Screen

The availability of the Create XML file option means the file is ready for creation.

Amend P35 Summary Details

List of options and explanations of EOY Return Workbench:

Field
Description

Employee forms:

List  of  employees  within  tax  reference  of  statutory  pay  in  year  (data can be updated). Gross Pay for PRSI is now populated using the PRSI-able Pay.

Un-Complete:

Allows you to go back and make amendments to data before file creation.

Complete:

Does not mean the return is actually “complete” for file creation. You must select “complete” for the create XML option to appear. will validate the employee data for invalid PPS numbers and other field formats.

Validation errors:

This option only appears if there are validation errors. The data on the individual employee forms can be corrected via this option. You must correct any errors before the “create XML” option appears. The “complete” option must be run following any data corrections. Data corrections can be carried out using the Employee Forms option or by pressing return on the Validation Errors Option.

P35 Summary:

NOTE: details must be entered manually.

The P35 Summary now shows the following fields as updated from the fixes previously loaded: Total USC and car park levy; this will need to be checked prior to submission.

The Employee Forms record will now show the USC Pay and also the USC amount deducted

Medical Insurance - Eligible for TRS

The total amount of medical insurance paid included in the taxable benefits amount and eligible for TRS (tax relief at source). This is non-updateable when Year End Batch is at ‘Complete’ status and shows the total medical insurance amount entered on the individual Employee P35L forms.

The Total Amount figure can be amended when the Year End Batch is at ‘Incomplete’ status

Create P35 XML file

Create XML file brings up a filename prompt. The filename is based on the last four characters of the Registration number. This can be overridden to something of your choice.

COINSplus EOY Return – Create XML File

There are three quick stages the file creation will go through, displaying each message on screen:

  1. Starting Batch process
  2. Generating XML
  3. Saving XML file-Copy File (Remote to PC)

At the point of saving, the FTP (syget.p) file transfer screen will appear as shown below. You can override the Local (C:\) file name as desired - a secure network location is recommended.

COINSplus File Transfer Screen

Once you’re satisfied with the details of the file naming, press F1; the file will be saved in the requested secure local directory (in this example c:\) on your Windows PC.

This file can then be submitted to the Irish Revenue using their online service. You will need to have registered for this first – see the Revenue Online Service website at http://www.ros.ie for full details.

We recommend that you submit a test file to www.ros.ie prior to year end in order to highlight and errors. You should contact www.ros.ie in order to submit a test file.
