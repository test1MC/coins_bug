ES Start and Stop Time Processing

Timesheet Group

The first day of the week and working days are defined by Timesheet Input Group in Employee Self Service > ESS Portal > Maintenance > TImesheet Group. As of v11.09, the Default Start and End Time columns are no longer available on this screen, as shown in Figure 1: Timesheet Group Screen.

Timesheet Group Screen

Start and Stop Time Records Generation

By default, records for Start and Stop Time do not exist and must first be generated. This initial generation can be done by drilling on the Week Ending Date column on the My ESS Timesheets browse, as shown in Figure 2: Generate Start and Stop Time.

NOTE:
If First Day of Week is uned in TImesheet Group, it will be marked as No Work during generation.

Start and Stop Time Input Screen

When you drill down on the Week Ending Date in My ESS Timesheets, a Start and Stop Time screen is displayed. Here you can input the start/stop time records for the Employee.

Start and Stop Time Input Screen

(This is the Drop-down text)
  • The Start and Stop Time Input screen contains the following information:
    1. Employee information
    2. Entered Start and End Time in HH:MM format
    3. Entered Start and End Time in Timecard format
    4. information
    5. Detail information - Employees and Mgr (Manager) comments
  • When No Work is ed (below), you cannot input the time on the Start and Stop Time Input screen.

  • When Split Shift is ed (below), you can input hours on the Start and Stop Time screen, but you are required to enter Employee comments.

  • If you do not enter Employee comments, the error message below is displayed.

  • If you do enter Employee comments, it is indicated on the appropriate record in the E box (below).

  • When No Lunch is ed on the Start and Stop Time Input screen (below), Start and End Time cannot be entered and are emptied out when the record is saved.

  • Hours Worked = Shift Hours – Lunch Hours. It is displayed on the Start and Stop Time Input screen in HH:MM format and in Timecard Input format (below).

  • Paid Time Off (PTO)
  • Parameter PTOTYPE uses a comma-separated list of Pay Types that is included in Timesheets, but it is ignored when you Submit timesheets.
  • During input, the Input table on the Start and Stop Time Input screen (below) is changed.

    These changes include:

    • Calculating PTO hours separately
    • Comparing PTO to the schedule
    • Total Hours displaying the real hours
  • Named filters on this screen include All Days and Monday-Friday.

Browse Screen

Start and Stop Time changes that were made in the My ESS Timesheets browse (Figure 4: My ESS Timesheets Browse) are explained in this section.

My ESS Timesheets Browse

Column Definitions

Status

The status options in this column include:

New No Start and Stop time records have been created, and the Week Ending Date has not been drilled on.
In Process

Records are submitted or rejected.

  • When Reporting is submitted, you cannot drill on the Week Ending Date.
  • You can only update submitted reporting if it is rejected by a user.
Approved Reporting has been approved by an Approver.
Transferred Reporting has been transferred to Payroll.
Transfer in progress Transfer was stopped by error.

Start/End Completed

This column is ed when ALL records on the Start and End Time input screen are either No Work or Shift Hours.

Reporting Screen

When Start/End Completed is ed on the My ESS Timesheets screen, you can click the Work Ending Date link to go to the Reporting screen. When you click the Next button on this screen, you are taken to the Periods List screen (Figure 5: Periods List Screen), where you can open the Start/Stop Time Input screen either in the same window (Figure 6: Start/Stop Time Input Screen in Same Window) by clicking the S column (#1 below), or in a new window (Figure 7: Start/Stop Time Input Screen in New Window) by clicking the Enter Start and Stop Time button (#2 below).
