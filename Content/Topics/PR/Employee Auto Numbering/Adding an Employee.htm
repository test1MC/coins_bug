Employee Autonumbering - Adding an Employee

The Employee number field will always be updateable when adding a new record. To allow the employee number to be left blank initially, this is no longer marked as a mandatory field, though the validation requires a value for this before actually creating the new employee record.

The ‘Generate Code’ button has been removed.

  • If EMPAUTON is blank then the user is required to set the employee number.
  • If EMPAUTON is ‘IFBLANK’ then if the user enters an employee number this is used and if they do not auto-generates the number.
  • If EMPAUTON is ‘ALWAYS’ then the user is only allowed to enter a number which matches a HR ID for a person who is waiting to be added to this payroll. If they attempt to enter anything else they will get an error. If they leave the employee number field blank auto-generates the new number.
