Positive Pay Reports

New Positive Pay Report (%WCB3050RCBD)

A new Positive Pay Report replaces the old one if the CB parameter USEPOSPY = Y.

Positive Pay Report Selection Screen

The Positive Pay Report processes and reports on payments in the log, and creates an extract file which can be sent to the bank (manually or automatically). The report will examine all in the log for the specified bank that have not been previously processed. Any exceptions (such as duplicate number, negative , or a that has been previously reported with different information) are included in the report as an exception but are NOT included in the extract file.

Field

Description

Bank

The bank to produce the report or extract file for.

If this is blank, all banks are processed, and separate extract files are created in accordance with the settings on the bank account record for each bank, and if specified on the bank account record, the file will be sent to the bank by FTP.

Report Only​/ ​Process

Whether to produce a report or create an extract file.

If Process is selected, an extract file will be created for the bank and stored in the user’s your “My Files” area. Any items processed are flagged as processed; any exceptions are flagged both as processed and as being exceptions (with reason). This will prevent any processed items and exceptions from being processed again.

Detail​/ ​Summary

If Detail is selected, each will be listed on the report. If Summary is selected, only the exceptions, totals and counts for issues, voids, and total items and counts will be reported.

Effective Date

This is blank by default. Some banks include this as part of header record.

Final Cutoff

If ed, this indicates that this is the final transmission of the month. Some banks request you specify this.

Auto Transmit File

This is ed automatically if Auto Transmit File via FTP is ed on the bank account, and the Remote Host field is not blank. If this is ed, the resulting bank file will be transmitted to the bank using the specified protocol / credentials.

If this is not ed, the file in “My Files” will need to be sent to the bank.

Positive Pay Purge Report (%WCB9340RCBQ)

The Positive Pay Purge Report is a new report that allows the user you to purge log data. This deletes all log records that are older than the date entered for the bank and have been processed.

The Positive Pay Purge Report is available on the – Administration menu if the CB parameter USEPOSPY = Y.

Positive Pay Reset Utility (%WCB9350RCBQ)

Positive Pay Reset Utility is a new report that allows the user you to specify a date and have the “processed” flag cleared for items processed on that date. This allows those items to be re-processed.

The Positive Pay Reset Utility is available on the – Administration menu if the CB parameter USEPOSPY = Y.
