Positive Pay –Bank Accounts

Bank account setup has been changed to add extra fields related to positive pay. These fields are only displayed if the CB parameter USEPOSPY is set to Yes.

A new “Positive Pay” tab has been added to Bank Summary to display these new fields.

Bank Accounts – Positive Pay Fields

Field

Description

Export Checks

Whether this bank requires positive pay information. If ed, posted will be recorded in the log, and the fields below are enabled.

Export Unposted Payroll

Indicates whether unposted payroll are recorded in the log.

  • None – Unposted payroll are not recorded.

  • Printed – details are recorded when printed.

  • Manual – Manual details are recorded when they are created.

Override Account Number

An account number. This allows Positive Pay to report a different account number than is printed on , if requested by the bank.

Control Record

The Control Record that is exported as the first line of the extract, if specified by the bank.

Format

The positive pay format to use. (The formats are configured using the Positive Pay (CB - USA) form type in Document Designer.)

Default File Name

The file name to use for the positive pay extract file.

If the bank requires a particular file name format, you can specified it using the following components:

  • Text

  • %d<format>% - today’s date; for example: %dMMDDCCYY%

  • %s<format>% - sequence number; for example: %s99%

  • %b% - bank ID

Example file name: mypospay-pnc-%dMMDDYY%%s99%.txt.

If you don’t specify a file name, uses the default format “pprpt-xx-yymmdd-nn.txt”, where xx = bank id and yymmdd = today’s date and nn = sequence number for today.

Additional Format

An alternative positive pay format (some banks require a summary file).

Auto Transmit File via FTP

Whether the bank file should be sent automatically. If this is ed, the fields below are enabled.

FTP Protocol

The transfer protocol to use when sending the bank file (currently only FTP is supported).

Username

The FTP username.

Password

The password associated with the username.

Remote Host

The IP address of the remote host to which the bank file will be sent.

Port

The port to use when sending the bank file. The default value depends on the protocol selected, but the port can be changed if a custom port is required.

Remote Path

The path on the remote server to which the bank file will be sent.

Remote Filename

The filename to use for the bank file on the remote server. If this is left blank, will transmit the file using the name it was created with (see the Default File Name field).
