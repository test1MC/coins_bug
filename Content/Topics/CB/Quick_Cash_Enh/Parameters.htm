Parameters

The following new CB parameters control the functionality covered in this document.

Parameter

Description

QCRDFBNK

If this is set to Yes, the Bank Code on a Quick Cash Receipt batch defaults from the bank set up in Configuration.

QCRHIJOB

If this is set to Yes, the number cannot be entered on the Quick Cash entry screen, and the (number) and Name fields are hidden. This means that the receipt cannot be allocated to invoices and certificates by ; therefore the user will need to enter a customer code, meaning the receipt can only be allocated to invoices and certificates for that customer.

LVCCPACB

This parameter affects the Allocate Customer Cash function as follows:

Y = When Customer Cash is distributed to either or and the amount entered does not get fully allocated, then the portion that is left will post to the as Customer Cash; that is, the unallocated amount will stay in . Note that the amount of the analysis line will be updated automatically to equal the amount allocated.

       Also, if the total value of the analysis line(s) entered is less than the value of the customer cash, then the user does not need to enter a CC analysis line to the value of the difference as was standard practice before. Instead when the batch is selected to post, a warning message will be displayed to inform the user that the customer cash has not been fully allocated. If the user decides to continue then during posting a CC analysis line will be automatically created equal to the value of the customer cash that was not allocated.

N = When Customer Cash is distributed to either or and the amount entered does not get fully allocated, then the portion that is left will post to either or as unallocated cash.
