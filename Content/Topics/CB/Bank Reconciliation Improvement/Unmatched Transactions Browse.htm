Unmatched Transactions Browse (%WCB1060BCRC)

Two new fields have been added to the statement item context

Fields

Field
Description

Marked Items

The value of items that have been marked.

Difference

The value of items that still needs to be marked.

A new column has been added to the browse.

Column

Field
Description

Marked

Any item that is marked to be reconciled to the statement item will have a displayed against it.

A item can be marked either by opening it up and updating the box (and deselected in the same fashion), or using the ‘Mark Items’ action.

Browser Actions

Two new actions have been added.

Mark Items

This action allows a user to mark multiple items for reconciliation to a statement item.

There is a limitation of the maximum number of rows that can be tagged/processed in a single action. That limit is currently 100 rows, therefore if the user wants to mark for example say 200 rows then this will need to be done in at least two separate actions.

Deselect Marked Invoices

This action allows multiple marked items to be deselected at the same time.

There is a limitation of the maximum number of rows that can be tagged/processed in a single action. That limit is currently 100 rows, therefore if the user wants to deselect for example say 200 rows then this will need to be done in at least 2 separate actions.

Or if all rows are to be deselected, then this can be done in one action, by selecting the statement item on the previous page and using the ‘Deselect Marked Items’ action there.

Show/Hide Reconcile Statement Item Action

If there are any marked items against the statement item, the ‘Reconcile Statement Item’ action will be hidden.

Marked Items Filter

A new named filter with a label of ‘Marked’ shows only those open items that have been marked for reconciliation to the statement item.

Bulk Update

A bulk update button has been added to the page to allow multiple items to be marked/un-marked in one go. There is no limit to the number of rows that can be updated.

This results in the marked open items being assigned with the internal reference of the statement item and un-marked open items having their the internal reference field being assigned to zero.

Concurrent Update

A concurrent update button has been added to the page to allow all items to be opened up for update simultaneously. There is the same limit – currently 100 rows - to the number of rows that can be updated.

If the user s the box on a row then the open item will be marked.

This will result in the open item being assigned with the internal reference of the statement item.

If the user un-s the box on a row then this will result in the open item being un-marked; that is, the internal reference field will be assigned to zero.

Update Button

An Update (or Open) button has been added to the page to allow individual open items to be opened up for update. If the user s the box then the open item will be marked.

This will result in the open item being assigned with the internal reference of the statement item.

If the user un-s the box this will result in the open item being un-marked; that is, the internal reference field will be assigned to zero.
