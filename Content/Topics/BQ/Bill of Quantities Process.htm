Process

The process of in its entirety starts from configuring the module. Once the configuration of the module is done can then be used.

Before exploring , a Project//Development needs to be created, this is done outside of the module.

A Project is created in Procurement which then makes it possible to create a contract in the Status module. Once a is created a Development can be created based on this .

Developments can be created in VAP or as both are one and the same but displayed in both modules.

Though the logical next step would be to create plots on the development, it requires house types to initially be configured, this is not to say that a needs to be created but a high level house type name needs to be created, this is done within > Setup.

Once a house type is created you can then associate this with plots created against the development.

 

 

When you have a development and associated plots (with house type association) you are then able to start using to build your bill based on house types.

You can create standard house type bills within the standard house type workbench and then filter this down to a development level, making the house type bill accessible to the individual development.

At this stage you have already created the standard bill, copied it down to the development and are ready to associate it with house types of the same type at plot level, this is simply done in one easy process which makes the development items available to each plot that has the same house type.

It is worth noting that bill of quantities is formed around house types as this defines what resources are needed on each plot.

Once at Plot level you are then able to start ordering items, this can be done in either the surveyor workbench or the buyer workbench, both have the same function however the buyer workbench is reduced as the simplest form of ordering material or subcontract.
