Variation Register

Variation Register is a new workbench that shows a summary of the variations for a development. From here you can select a development, and add and maintain the list of variations for that development.

Variation Register Development Browse

Columns

Field
Description

Development

The development code. A link on this column takes you to a browse of variations for the selected development (see 1.1 , Variation Register).

Name

The name of the development.

Totals

A total of all the variations for the development. A link on this column takes you to a browse of variations for the selected development with the filter pre-set to show all variations.

Variation Statuses

A column is shown for each of the variation statuses configured (see Variation Status Codes) that are not marked as retired. Each column shows a count of the variations at that status. A link on the column takes you to the browse of variations for the selected development, with the filter pre-set to only show variations for the selected status.

Development Variations Browse

The Development Variations browse allows you to maintain the variations for a development. Column sets allow you to configure the columns to be displayed.

You can filter on all variations or just those currently in progress (that is, not live and not cancelled). Advanced filtering by QS reference, status and reason is also available.

Buttons provide shortcuts to the Plot Surveyors Workbench (and if the new BQ parameter USEHTVAR is set to Y, to the Development House Type Workbench).

Fields

Field
Description

Internal Reference

The automatically-generated unique reference for this variation within the selected development. A link on this column takes you to a browse of the items that make up the variation.

QS reference

Optional manually input reference for this variation (to help cross reference to paperwork etc.).

Variation Status

The status of the variation. The valid status codes are set up using Variation Status Codes (see 1 , Variation Status Codes).

Variation Reason

The reason for the variation. The valid reason codes are set up using Variation Reason Codes (see 1 , Variation Reason Codes).

If the variation reason has a system status of Client Extra, Client Incentive or Client Option, the Plot field is updatable.

Variation Category

The category for the variation. The valid category codes are set up using Variation Category Codes (see 1 , Variation Category Codes).

Description

The description of the variation.

Detail

The full text describing this variation in detail.

Plot

The plot number.

Estimated Cost

Manual Input of the Estimated Cost for the variation.

Quoted Cost

Manual Input of the Quoted Cost for the variation.

Approved Cost

The Approved Cost for the variation. The approved cost can only be updated if the associated system status of the variation is ‘Registered’.

Number of Linked Plot Items

The total number of plot items linked to this variation. This includes items that are to be omitted.

Plot Items NR

The non-recoverable amount from the associated plot items.

Cost Variation of Plot Items

The total cost of the plot items linked to this variation including non-recoverable .

Order Status of Linked Plot Items

The order status of the items linked to the variation.

The following three fields are only shown if the new BQ parameter USEHTVAR is set to Yes.

Number of House Type Items

The total number of house type items linked to this variation. This includes items that are to be omitted.

House Type Items NR

The non-recoverable amount from the associated house type items.

Cost Variation of House Type Items

The total cost of the house type items linked to this variation including non-recoverable .

Logged

The user who first logged the variation and the date/time it was logged.

Last Updated

The user who last updated the variation and the date/time it was updated.

Variation Items Browses

The Variation Items screen has two tabs, with browse screens that show details of the plot and development house type items that have been amended as part of the variation. Note: if the new BQ parameter USEHTVAR is not set to Y, the house type items tab will not be shown.

Variation Register Variation Items Tab

The Detail button shows details of the item.

  • For added items this shows details of the new item.
  • For deleted items this shows details of the original deleted item.
  • For updated items this shows details of the revised item, and also the changes between the original and revised item.

If the system status of the variation is Registered, Budget Approval or Budget Approved, the item can be deleted. This removes the item from the variation and reinstates the original version of the item.
