Variation Register - Changes

The variation register is available as a browse within the . The items to show can be filtered by variation status and reason.

Model

The model contains a new view type of ‘ Variation Register’. This allows you to configure a browse showing a snapshot of the variations registerYou can also predefine the variations to show on the view by their status, reason and category.

In addition when defining the BoQ variation view, a new box Only Show Variations Budget Approved since Last Period is available. If this is ed, the browse will show only variations where the budget was approved since the last period.

You can filter on all variations or just those currently in progress (that is, not live and not cancelled).

Items

New items have been introduced to allow information relating to a Variation to be captured.

Three new classes of item have been introduced:

  • Variation Manual Input
  • Variation Calculation Field
  • Variation RSP Field

These work in the same way as existing manual input, calculation and RSP fields.

There is an option to summare by . Variation selection includes the option to select by variation status and variation reason.

Update

When updating a , a new option ‘Update Variations List’ allows you update the variations list in the .

This new option is only shown if at least one ‘ Variation’ view has been set up in the model. If this option is selected then the snapshot of variations contained in the will be updated.
