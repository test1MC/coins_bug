Variation Status Codes

Variation Status Codes is a new function that allows you to set up status codes that can be allocated to variations in Variation Register.

Variation Status Codes Maintenance

Fields

Field
Description

Code

The status of the variation.

Description

The description of the variation status.

System Status

The code for the system status. This determines the type of status you are setting up.

You can create multiple variation status codes with the same system status if required so that these more detailed status codes are held against each variation for enquiry and reporting purposes. As a minimum to operating the Variation Register there should be a status code for each of the Registered, Live and Cancelled system status.

The options are:

  • Registered. Variations which have just been logged and the details are being prepared.
  • Budget Approval. This is used to track variations where approval/​authoration for the budget amount only has been requested. This would be a precursor to the items being requested. Note no restriction will be placed on the items being updated.
  • Budget Approved. This is used to track variations where the budget has been approved.
  • Detail Approval. This is used to track variations where approval/​authoration for the complete variation including all the required item changes has been requested.
  • Detail Approved. This is used to track variations where the detail has been approved.
  • Live. This is used to make the variation live. Once a variation is live, items will be able to be ordered and any further VAP budget updates will always include the revised items. The details of the original item revisions will be kept.
  • Cancelled. This is used to track cancelled variations. When a variation is cancelled, the original items are restored and any new items removed. Details of the proposed item revisions will be kept.

Workflow

The template code for the workflow to launch when a user selects this variation status.

Update

Whether the user can manually select this status for a variation.

If this is not ed, the status can only be set by a workflow.

Retired

Whether this variation status is retired.

Default Status for New Variation

Whether this status code is used as the default for a new variation.

Only one status code can be flagged as the default.

Previous Status

A list of status codes. This allows you to control whether a specific status can be selected for a variation; if the current status of the variation is in this list, this status can be selected.

Update VAP Budget with Approved Cost

Whether the Approved Cost for variations with this status should be included in the VAP budget when the Update VAP Budget process is run (see 1 , Variations and VAP Budget Update). This will be included against the stage and plot defined on the variation reason code (see 1 , Variation Reason Codes).

Update VAP Budget with Bill Items

Whether the amended items associated with the variation should be included in the VAP budget update when the Update VAP Budget process is run. This is ed automatically if the system status is Live, and can’t be uned.

Developments with No Bill Items Only

Whether this status code can only be used on developments where there are no items (either house type or plot).

This allows Variation Register to be used 'stand alone' on migrated sites where there is no . This status can then be used for 'Live' variations on these sites but have no validation of the items against the approved amount.

Approved Cost Tolerance Check

This allows the total cost of the item changes to be checked against the Approved Cost for the variation.

  • No Check. No tolerance check is to be done.
  • Percentage. The check is based on the percentage variance only.
  • Amount. The check is based on the amount only.
  • Lowest Percentage or Amount. The check is based on the lowest of either the percentage or amount tolerance.
  • Highest Percentage or Amount. The check is based on the highest of either the percentage or amount tolerance.

Tolerance Condition

When the cost tolerance check applies.

  • Above Only. The tolerance check only applies if the items exceed the approved cost.
  • Below Only. The tolerance check only applies if the items are less than the approved cost.
  • Above and Below. The tolerance check applies regardless of whether the items are greater than or less than the approved cost.

Tolerance %

The percentage tolerance allowed.

Tolerance Amount

The fixed tolerance amount.
