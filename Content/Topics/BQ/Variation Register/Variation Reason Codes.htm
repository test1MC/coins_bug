Variation Reason Codes

Variation Reasons is a new function that allows you to maintain the Variation Reason Codes which can be allocated to variations in Variation Register.

Variation Reason Codes Maintenance

Fields

Field
Description

Code

The code to identify the reason for the variation.

Description

Description of the reason.

System Reason

The code for the system reason. This determines the type of reason you are setting up.

You can create multiple variation reason codes with the same system reason if required so that these more detailed reason codes are held against each variation for enquiry and reporting purposes.

The options are:

  • Internal Variation
  • Client Extra
  • Client Incentive
  • Client Option
  • Contra Charge

Budget DrawDown

Whether this reason represents a budget draw down.

If this is ticked, you are reducing the value of a item which holds a budget and putting the value into an item which is now going to be ordered. The negative amounts (the reduction or deletion of a item) will be ignored – only the positive items (increases or additions) will be included. The net value will normally be zero.

The cost head against which a budget allowance is to be assigned in the VAP budget (see 1 , Variations and VAP Budget Update).

If this is left blank, no allowance will be allocated to the VAP budget.

Stg

The payment stage against which the allowance is to be assigned in the VAP budget.

Plot

The plot against which the allowance is to be assigned in the VAP budget

If this is left blank, no allowance will be allocated to the VAP budget.

Retired

Whether this variation reason is retired.
