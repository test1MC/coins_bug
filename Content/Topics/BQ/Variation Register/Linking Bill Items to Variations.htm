Linking Items to Variations

When you update items and create orders for subcontract and materials within the , you can optionally link these to the variation register. You can link as many plot items as required to the same variation header, but each plot item can only be linked to one ‘in progress’ variation header at once.

A new field has been added to the Plot Item Maintenance screen.

Plot Item Maintenance

Field

Field
Description

Variation

This is a drop-down list that allows you to select the variation to link the item to. The list only includes variations that have a system status of Registered, Budget Approval or Budget Approved, and if the variations are plot-specific (that is, if they have a reason code with a system status of Client Extra, Client Incentive or Client Option), they will only be included for the plot they relate to.

If the item is already associated with a variation and the system status is Detail Approval or Detail Approved, the variation cannot be changed.

If you amend an ordered item, it will be flagged as requiring a variation. The variation order will not be able to be processed until the variation has been made live.

Item Delete

A new item row action ‘Variation Delete Item’ has been added to all plot item browses. On selecting this you are prompted to select a variation to link the deleted item to. The list only includes variations that have a system status of Registered, Budget Approval or Budget Approved, and if the variations are plot-specific (that is, if they have a reason code with a system status of Client Extra, Client Incentive or Client Option), they will only be included for the plot they relate to.

If the item is already associated with a variation whose system status is Detail Approval or Detail Approved, the variation cannot be changed and the row action is prevented.

You can still delete plot items directly where required and do not have to link all deletes to a variation.

Item Indicator Column

On item browses, the last column is an indicator which has now been changed to indicate if the item is part of a variation. So the possible codes are now:

  • C - The plot item has been overwritten from the associated house type item.
  • VA - The item has been added as part of a variation and the variation is not yet live.
  • VU - The item has been updated as part of a variation and the variation is not yet live.
