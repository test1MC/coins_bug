Raise a Subcontract Draft Certificate

Navigate to the Subcontract Ledger > Draft Certificates > Input Draft Certificate (All Types)

Draft Certificate Entry (All Types) allows you to enter draft subcontractor payment certificates (and amend existing or rejected ones) for subcontracts of all billing types.

To enter draft certificates, you must have level 1 access to the relevant contract. Access to contracts for draft certificates is set up using Subcontract Ledger > SC Security > Authorisation Levels (Location).

Ensure that the Add By selector is set to Order.

Click

If the order number is already known, it can be keyed directly into the Order Number field.  Alternatively click to use the lookup page

Use the filters to locate the required order and click to select the order to be paid against.

All remaining fields are populated automatically

Click to move onto the Add Draft Certificates screen.

Contract Security will only allow named users to raise payments or to view costs on contracts.  If a Warning Message appears stating that you do not have access to that contract then contact the Finance Department who will arrange access so that you can continue with the payment, subject to authorisation

Main Tab

On the Main Tab enter the following certificate details:

 

Field

Description

Claim date:

Date the subcontractor claim was received..

Certificate Date:

The date when the work was certified. This date is used for payment selection.

Valuation Date:

The date the work was valued. This is also the cost-effective date for costing purposes. Defaults to the certificate date

Due Date:

The date when payment is due. fills in the Due Date based on the Due Date and Type for the subcontract and the valuation date.

VAT:

Defaults from the order setup, do not change

QS Ref:

The quantity surveyor's reference e.g. Enter your initial’s

S/C Ref:

The subcontractor reference, if there is one (for example, an application reference). For invoicing subcontractors, this would normally be the invoice number.

Status:

The status of this subcontract.

Active = Normal transactions can be entered.

Inactive = Only certain fields are available for input (for example, to allow for retention).

Complete = No further transactions can be entered to the subcontract.

When you enter the last 'proper' certificate on the subcontract, select Inactive.

Changes to this field also affect the Status and Complete fields on the subcontract. In certificate entry, you cannot change the status back from Complete to Inactive or from Inactive to Active. If you need to change the status back, do so using Subcontract Maintenance.

Description:

Indicates zero movement for the period.
