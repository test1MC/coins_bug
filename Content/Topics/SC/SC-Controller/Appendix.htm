Appendix

 

Applications

  • Application can be created by SC QS user when the order is shared by MC QS
  • Several applications can be created during the same valuation period, but the latest will override all previously saved. This means that the latest application can be modified and submitted.
  • Application Details:
    Cumulative Amount - the cumulative value which SC QS wants to apply for
    Previously Submitted - Cumulative Amount from previously submitted application
    This Period - the difference between Cumulative Amount and Previously Submitted amount.

Payment Notices

Payment Notice can be created:

  • in current valuation period only AND
  • payment notice has NOT been already created for this period AND
  • payless notice has not been submitted for this period

Payment Notice can NOT be created if:

  • payment notice has been already created in current period
  • payLESS notice has been submitted for this period
  • Payment Notice Due has expired for current valuation period (Payment Due = Valuation Date + Payment Notice Due Date offset)
  • no current period is available (last Valuation Date is in the past)
  • Payment Notice Total exceeds Original Order Value

Payment Notice details:

  • if submitted application exists for current period, Cumulative Amount, Previously Submitted, This Period and Anticipated Final Accounts are taken from it
  • if there is no submitted application for current period:
    Cumulative Amount is taken from the latest submitted application
    Previously Submitted = Cumulative Amount
    This Period = 0 to indicate there is no applied amount during this period

Payless Notices

Payment Notice can be created in previous and current valuation periods.

In previous if:

  • payless notice has NOT been already created for previous and current periods AND
  • payment notice has NOT been already submitted for current period AND

In current if:

  • payless notice has NOT been already created for current periods AND

Payless Notice can NOT be created if:

  • payless notice has been already created for selected period
  • Last date to send Payless Notice has expired (=Last Valuation Date + Payment Due Date offset + Last dat to send a Payless Notice offset)
  • if Payless Notice has been created in the last valuation period
  • Payment Notice Total exceeds Current Order Value (TBD in Task 5186: Total Adjusted Net Valuation > Current Order Value)
  • Total Gross Valuation < Discount + Retention
  • Total Gross Valuation < Discount
  • Total Gross Valuation < Retention

Payless Notice details:

  • If submitted Payment Notice exists in selected period, Previous Payment Notices Values = MC Cumulative Amount from this Payment Notice
  • If there is no submitted Payment Notice, Previous Payment Notices Values = 0
