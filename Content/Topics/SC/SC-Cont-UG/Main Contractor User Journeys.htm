Main Contractor User Journeys

 

Users create SC Orders in OA/Procurement/SC Orders and when committed the Order details are published to SC Controller.

 

Note: Orders can only be created from Procurement NOT Subcontract Ledger.

 

Once an Order is committed it will appear in the All SC Orders grid (subject to user Contract Security).

 

 

The All SC Orders grid allows users to assign to others or assign to themselves:

 

 

Assign To Me will assign the SC Order to the user.

Assign To allows the user to assign to another SC Controller user

 

View provides users a view of Order details currently held in the Construction Cloud.

 

Assign by Contract allows users to assign to Orders that are unassigned at that point by Contract.

 

Note: When new Orders appear for the this Contract they are NOT automatically assigned.

 

When an Order is assigned to a SC Controller user further functionality is available:

 

 

Clear Assignment will remove current Assignment.

 

Delegate To allows users to Delegate Order and Notifications to another SC Controller user.

When the dates are applied

 

 

Save & Share will share the  SC Order including the Order Rules and Valuation Dates with the Subcontractor Tenant.

Cancel will back out of the screen not saving any changes.

 

Scenario 1: When an SC Tenant has submitted an Application to the Main Contractor Tenant:

 

 

In the Applications screen users can right click on the Application and the following options are available:

 

 

View Application will allow users a read only view of the Application for Payment.

 

Review Application allows users to review and Input the values for the Payment Notice:

 

 

MC Valuation to date is used to input the cumulative values for the Payment Notice.

Gross Application = Measured Works + Materials on Site.

 

Total = Gross Application – Discount – Retention

 

Labour Liable for Tax is an input field that users need to enter how much of the total is Labour Liable for Tax.

 

Users must add an attachment of their backup to support their Payment Notice input.

 

Submit will start the process of:

 

    Checking the Notice Limit of the user (if exceeded will require approval).

    Validate with OA that data being sent is correct

    Send to OA amd create an unposted SCCCERT batch.

    Issue a Payment Notice to the Subcontract Tenant.

 

 

Scenario 2: Create Payment Notice without Application

 

In My SC Orders Grid click om SC Order line and select Create Payment Notice

 

 

Next steps are the same as Scenario 1.

 

Scenario 3: Create Payless Notice

 

In My SC Orders Grid click om SC Order line and select Create Payless Notice

 

 

The Payless Notice input screen allows users to enter Total Gross Valuation, Discount, Retentaion and Labour Liable for Tax.

 

Users are required to attach a backup before they can submit the Payless Notice.

 

After Submit the same as scenario 1:

 

    Checking the Notice Limit of the user (if exceeded will require approval).

    Validate with OA that data being sent is correct

    Send to OA amd create an unposted SCCCERT batch.

    Issue a Payment Notice to the Subcontract Tenant.
