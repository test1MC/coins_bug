Changes to Standard Certificate Definition

The CITB Deduction field on the Main tab has been re Contractual Training Contribution.

The Body Update on the Detail tab has been redesigned, and several new fields have been added.

Standard Certificate Definition – Line Update

Fields

Field
Description

Update if Invoicing

This is a box. If ed, the line can only be updated if the billing type is I (Invoicing) (overriding the setting of the Update field). This is so that where the subcontractor has submitted an invoice, the user can adjust the to agree to the invoice while preventing override of calculations where a Self-Billing Invoice or Authenticated Receipt is being produced from the certificate.

Calculate Only if Paid Net of CIS Tax

This is a box. This allows training recovery or similar deductions to only be applied where tax is deductible.

Display Only if Paid Net of CIS Tax

This is a box. This allows training recovery, materials or similar lines to only be displayed where tax is deductible.

Account Type

This is a drop-down selector with the options GL Account and Use 2nd Cost. If this is set to GL Account, it will use the account specified in the Account field. If this is set to Use 2nd Cost, it will use 2nd Cost if set up on the subcontract and valid, otherwise it will use the account specified in the Account field.

Negative This Period Error

This is a drop-down selector with the options No Message, Warning and Error.

Set this to Warning on items which would not normally be negative this payment (such as tax). This will avoid the need to have another line with a calculation to reverse the value and check that it does not exceed 0.01.

Negative Cumulative Error

This is a drop-down selector with the options No Message, Warning and Error.

Set this to Error on items which should never be negative cumulative (such as tax). This will avoid the need to have another line with a calculation to reverse the value and check that it does not exceed 0.01.

Pro-Rate Analysis

This is a drop-down selector with the options blank, TP (This Period) and CUM (Cumulative). This is only displayed if the Certificate Definition is included in the parameter VP/VAL-CERT.

This allows lines which are included in the able amount calculation to use the distribution pro-rated from the distribution on preceding certificate lines (mainly from VAP). This would be used on a Cumulative basis for retentions or on a This Payment basis for discounts and training levy deductions or similar.

Non-Recoverable Analysis

The costing analysis for proration. This is only available on a line and is only displayed if Pro-Rate Analysis is set to This Payment or Cumulative.

This consists of a selector for the type of analysis (C – or G – GL Account, and an input field for the analysis code.

This allows the costing of the negative non-recoverable on retentions taken to a fixed code. This will then be offset by the positive non-recoverable when the retention is released.

Similarly the costing of the negative non-recoverable on discounts, training contributions, etc. will be to a fixed code.

Default Code if Registered

The code to which any amounts not attributed to a code are added if the subcontractor is registered for .

This allows automated distribution for lines which are included in the able amount calculation. This would also be a fail-safe for Apportionment calculations. This must be an input code and cannot be non-recoverable.

Default Code if Not Registered

The code to which any amounts not attributed to a code are added if the subcontractor is not registered for .

This allows automated distribution for lines which are included in the able amount calculation. This would also be a fail-safe for Apportionment calculations. This must be an input code and must have a rate of 0%.

This must be populated if Default Code if Registered is populated; either both fields or neither must be populated.

Standard Certificate Definition Check

A new action – Standard Certificate Definition Check – on the Standard Certificate Definition browse checks that the selected certificate definitions have been updated consistently within themselves and with other relevant configurations. This produces a report that lists any exceptions to pre-defined rules. There may be good reasons for some of these exceptions; equally, even if no exceptions are reported, this does not guarantee that the certificate definition will work correctly.

Update of Standard Certificate Definition

If a standard certificate definition is already in use – that is, if certificates already exist for that definition – update of the definition is prevented unless the day password has been set. This is to prevent support issues caused by changes to definitions that are in use.
