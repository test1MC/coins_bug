Subcontract and Retention – Overview

Changes have been made to correctly handle the calculation of after retention on subcontractor payments where there is a split analysis from VAP. The standard certificate definition has been updated to allow for this, and manual update of the calculated values is now prevented.

An additional change allows CITB accruals and training cost recoveries to be handled based on payments net of tax rather than -only payments.

Background to the Changes

HMRC has clarified that a self-billing invoice or authenticated receipt cannot create a tax point for the retention amount until that retention is paid. This means that on these payments must be calculated after the retention, and additional processing by is needed to ensure that the correct treatment is applied to each release of retention.

Action Required

Contractors should not need to make any changes to configuration as a result of these enhancements. Home builders who pay subcontractors through self-billing or authenticated receipts need to check existing configuration and may need to configure new certificate definitions. invoices are not affected.
