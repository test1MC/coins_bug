Standard Certificate Definition

Location: Subcontract Ledger > Configuration > Standard Certificate Definition

Standard Certificate Definition allows you to define "template" definitions for subcontractor certificates. The Subcontractor Certificate defines the method of calculation used to pay subcontractors, and the layout of the input screen for subcontractor certificates. If necessary, you can modify the definition for individual subcontracts using Subcontract Certificate Definition.

The Standard Certificate Definition provides a list of all Certificate definitions set up for a company along with the row numbers for Gross, Materials, retentions, Discount, Tax, VAT, Payment and In Use? Below a list of Certificate definition in Company 10.

Clicking on a Certificate Definition type will open the Main Details Tab the Main Tab provides details of description and a summary of Gross, Materials, Retention, Discount, Tax, VAT, Payment these are linked to the appropriate row.

The Detail tab has the rows for a certificate. Each Row is divided into sections, Matrix certificates have an additional section managed by parameter VAL-CERT:

Notes (from Toggle help)
  Row/Line Number 10 100 A line numbers. This is used to order the lines on the certificate

Use numbers that are sufficiently widely spaced (for example, 10, 20, 30 ...) to allow you to add new lines in between. If you use more than one certificate, use consistent line numbers, that is, use the same number for the same line item in different certificate definitions.
Group / Section Description Contract Works Gross A description of the line - this is displayed in the Certificate Entry screen
Column        
HEADER Subtotals   +10,+20,+30 The numbers of the lines that are to be included in the calculation, if the value of this line is a subtotal.

For example, if the value of line 50 is the sum of the values in lines 10, 20 and 30, minus the value in line 40, enter:

+10,+20,+30,-40

Subtotals can also use multiplication and division. For example, +300,*60,/100 would take the figure in line 300, multiply by line 60, then divide by line 100. This allows you to do pro-rata calculations such as splitting a single retention input figure between labour and materials retention based on the labour/materials split of the gross payment. If the calculation results in a divide by zero error then a figure of zero will be given.

To use the subtotal as a default value that users can overwrite if required (for example, a VAT amount), tick the Update field.
HEADER Rate (%) 100 100 If the value of this line is a percentage of the value in another line or lines, enter the percentage.

For example, to define Retention as 5% of the Gross Cumulative value (if the Gross Cumulative line is line 50) the Subtotals field would be +50 and the Rate would be 5.00%.

To define Discount as 2.5% of the Gross Cumulative value less Retention (if Retention is line 60), the Subtotals field would be +50,-60 and the Rate would be 2.50%.

For a VAT line, use 100%, since the VAT rate is calculated independently, based on the VAT code.
HEADER Account     If the This Period amount is credited to or debited from a General Ledger account, this is the code for that account.

You must make sure that when costing is taken into account, debits equal credits. When creating a new certificate definition, test this thoroughly before use.

Do not enter an account for VAT lines. The General Ledger posting for VAT is taken from the appropriate account code in the VAT table.

Do not enter an account for Costed Amounts. The General Ledger posting for the costed amount is taken from the default General Ledger account code specified for the combination of contract, cost head, and cost category in Contract Status.
HEADER Debit CR CR Ticked = The value is to be debited from the account

Unticked = The value is to be credited to the account.
OPTIONs Calc. Cumulative Y Y Is this line calculated on a cumulative basis? For example, VAT and Income Tax lines would normally not be calculated cumulatively
OPTIONs Display Y Y Is this line displayed on the Certificate Entry screen? If the line is for calculation only, it may not need to be displayed.
OPTIONs Update N N Can the user enter or amend this field when entering the certificate? Clear the box if the value is not to be overridden
OPTIONs Print Y Y Is this line printed on the certificate? This option is not used if certificates are printed using the configurable Print Certificate procedure.
OPTIONs Text N N Is the user given the option to enter a line of text against this line? For example, on a Contra line, a contra reference or reason could be added. This text can be printed on the certificate.
OPTIONs Enter Period N N Ticked = The user can only enter a value in the This Period column.

Unticked = The user can enter a value in either the This Period or the Cumulative column
OPTIONs Detailed Breakdown? N N Is a detailed breakdown entered for this item during certificate entry? See Subcontract Certificate Breakdown: Overview.
OPTIONs Costing from Breakdown? no no This field determines whether costing distribution lines are generated automatically from the breakdown items. Only applies if the "Breakdown Items" tickbox is also ticked.
OPTIONs Show Cumulative Y Y Ticked = Show cumulative and previous values for this line, as well as the this period value.

Unticked = Only show this period value for the line (for example, if you do not want VAT or Income Tax lines to show cumulative values).
OPTIONs Debit     Ticked = The value is to be debited from the account

Unticked= The value is to be credited to the account.
OPTIONs Detail     Ticked = Transactions are posted to the General Ledger in detail, that is, one transaction for each General Ledger code affected in each certificate.

Unticked = Transactions are posted to the General Ledger in summary, that is, a single transaction for each General Ledger code used in the batch.

Note that although you do not give a General Ledger account code for VAT or costed amounts, the Detail indicator for these lines still determines how VAT and costing postings are generated.
OPTIONs Update if Invoicing N N If ticked, the line can only be updated if the billing type is I (Invoicing) (overriding the setting of the Update field). This is so that where the subcontractor has submitted an invoice, the user can adjust the VAT to agree to the invoice while preventing override of VAT calculations where a Self-Billing Invoice or Authenticated Receipt is being produced from the certificate.
OPTIONs Calculate only if Paid Net of CIS Tax N N This allows training recovery or similar deductions to only be applied where tax is deductible.
OPTIONs Display only if Paid Net of CIS Tax N N This allows training recovery, materials or similar lines to only be displayed where tax is deductible.
Settings Item ID CWK GRS A code to identify this item. This is used by configurable reports to retrieve the value associated with this line.

If you are using more than one standard certificate, use the same Item IDs for the same item in different certificates. For example, you might use GROSSAMT as the Item ID for the gross certified amount in all certificates.
Settings Maximum Cumulative 0 0 The default maximum cumulative amount allowed on this item.

Normally you should leave this blank on the Standard Certificate Definition. Maximum cumulative values can be amended on the individual certificate definition using Subcontract Ledger > Subcontracts > Subcontract Certificate Definition.
Settings Error No Message Warning This controls what happens if the Max Cumulative value for this line is reached.

Error = does not accept the value and displays an error message.

Warning = accepts the value but displays a warning message.

No Message = takes no action.

NOTE : If a different maximum cumulative value is specified for the individual subcontract, it is that value that will be checked.
Settings Account Type GL Account GL Account If this is set to GL Account, it will use the account specified in the Account field. If this is set to Use 2nd Cost, it will use 2nd Cost if set up on the subcontract and valid, otherwise it will use the account specified in the Account field.
Setting Account     f the This Period amount is credited to or debited from a General Ledger account, enter the code for that account. You can enter underscores to use defaults from the first cost General Ledger account specified on the contract record.

You must make sure that when costing is taken into account, debits equal credits. When creating a new certificate definition, test this thoroughly before use.

Do not enter an account for VAT lines. The General Ledger posting for VAT is taken from the appropriate account code in the VAT table.

Do not enter an account for Costed Amounts. The General Ledger posting for the costed amount is taken from the default General Ledger account code specified for the combination of contract, cost head, and cost category in Contract Status.
Settings Detail Y Y  
Settings Negative This Period Error No Message No Message Set this to Warning on items which would not normally be negative this payment (such as tax). This will avoid the need to have another line with a calculation to reverse the value and check that it does not exceed 0.01.
Settings Negative Cumulative Error No Message No Message This allows lines which are included in the VA Table amount calculation to use the VAT distribution pro-rated from the VAT distribution on preceding certificate lines (mainly from VAP). This would be used on a Cumulative basis for retentions or on a This Payment basis for discounts and training levy deductions or similar.
Calculation Calculation      
VAP Related VAT Pro-Rate Analysis     This is a drop-down selector with the options blank, TP(This Period) and CUM (Cumulative).This is only displayed if the Certificate Definition is included in the parameter VP/VAL-CERT.

This allows lines which are included in the VATable amount calculation to use the VAT distribution pro-rated from the VAT distribution on preceding certificate lines (mainly from VAP). This would be used on a Cumulative basis for retentions or on a This Payment basis for discounts and training levy deductions or similar.
VAP Related Non-Recoverable Analysis     The costing analysis for VAT proration. This is only available on a VAT line and is only displayed if Pro-Rate VAT Analysis is set to This Payment or Cumulative. This consists of a selector for the type of analysis (C – Contractor G – GL Account, and an input field for the analysis code. This allows the costing of the negative non-recoverable VAT on retentions taken to a fixed code. This will then be offset by the positive non-recoverable VAT when the retention is released. Similarly the costing of the negative non-recoverable VAT on discounts, training contributions, etc.will be to a fixed code.
VAP Related Default VAT code if VAT registered     The VAT code to which any amounts not attributed to a VAT code are added if the subcontractor is registered for VAT. This allows automated VAT distribution for lines which are included in the VATable amount calculation. This would also be a fail-safe for VAT Apportionment calculations. This must be an input VAT code and cannot be non-recoverable.
VAP Related Default VAT code if not VAT registered     The VAT code to which any amounts not attributed to a VAT code are added if the subcontractor is not registered for VAT. This allows automated VAT distribution for lines which are included in the VATable amount calculation. This would also be a fail-safe for VAT Apportionment calculations. This must be an input VAT code and must have a rate o f0%. This must be populated if Default VAT Code if VAT Registered is populated; either both fields or neither must be populated.
