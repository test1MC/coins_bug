Insurance Tab

The Insurance Tab allows you to record details of insurance policies and other schemes for the subcontractor. These are used in validation when entering orders and certificates, committing orders, and processing payments.

Field Description
Scheme The Type of Scheme
Category

The category of this scheme.

This can be:

  • Insurance - Policy Details, Expiry Date etc
  • Dated Items - Expiry Date but no insurance-related details such as Policy Number etc
  • Qualification - These are for compliance items that simply need to be met once and do not expire

When using the Search box, simply enter the first letter of the required category.
Expiry Date

The expiry date of the scheme.

If this field is greyed out, this means the scheme category has been changed to 'Qualification' since the date was entered.
Policy Number

The scheme number.

If this field is greyed out, this means the scheme category was previously 'Insurance' but has been changed.
Provider

The name of the insurance provider if applicable.

If this field is greyed out, this means the scheme category was previously 'Insurance' but has been changed.
Name on Policy

The actual name to be found on the policy document (if different from the supplier / subcontractor).

If this field is greyed out, this means the scheme category was previously 'Insurance' but has been changed.
Status

The status of the information held for the scheme.

The statuses are -coded:

  • Red = 'Missing Details' if any of the required details are missing; 'Expired' if the expiry date has passed; 'Not Met' if the requirements of a Qualification scheme have not been met.
  • Amber = Number of days remaining less than a set number;
  • Green = Number of days remaining before the expiry date, or 'Met' if the requirements of a Qualification scheme have been met.

These statuses are not shown if the Validate flag is set to no.

The Validate flag has no meaning on order-specific schemes.
