Contras Workbench - Unallocated Costs Tab

The Unallocated Costs tab shows all cost transactions that match the criteria for unallocated contra costs (specified by the parameters SC/CONTJSC, SC/CONTCAT, SC/CONTSEC and SC/CONTACT; see 1, Contras Workbench - Initial Configuration). These transactions represent costs that you may want to recover by a contra charge.

You can either match transactions against other transactions, or create a contra record for them. Transactions that have been linked to a contra will be marked as matched when a invoice for the contra has been created and posted.

An additional column shows the status of the transaction:

  • Blank – Not matched.
  • M – Fully Matched.
  • C – Linked to a contra and cannot be matched.
Contra Ref
Shows the contra reference once a contra has been created against the cost. A link opens the contra in a new window.

The following actions are available:

  • Create Contra – This creates a contra for each selected transaction. The contras appear on the Contras tab (see , Contra Workbench - Contras Tab).
  • Match – This allows you to match unallocated transactions with each other. As long as the balance of the transactions you select is zero, applying this action will set the status of the transactions to Matched.
  • Unmatch – This allows you to mark previously-matched transactions as unmatched. As long as the balance of the transactions you select is zero, applying this action will set the status of the transactions to Unmatched.

You can also match (or unmatch) transactions using concurrent update; only the Matched box is available for update.

Refresh Data Option

The Refresh Data option allows you to regenerate the unallocated cost transactions. This is useful if you have changed the parameters that identify the coding for unallocated costs. You also have the option to delete existing unallocated cost transactions that do not match the current parameter settings (as long as they have not been linked to contras).
