Contra Workbench - Updating the Contra Record

You can update the contra record to specify how you want to recover the costs. The update screen also shows the source details, which are taken from the source transaction.

Fields

Field
Description
Account

The account from which you want to recover the costs.

This will ultimately be a account. The lookup shows accounts from , and Subcontract Ledger, and which ledger or ledgers the account exists in. If you select a subcontractor or supplier account that does not have a matching customer account, will attempt to create the customer account automatically, using the data from the supplier or subcontractor account (except the analysis, which will default from the SC parameter CONTANAL). The new account created can be amended. If is unable to create the account you will need to create it manually.

Recovery Amount

The amount to recover from the above account. This can be different from the contra cost.

Cost Balance

The unrecovered amount; this is calculated as the Cost Amount minus the Recovery Amount.

Cost Balance To
If the Cost Balance calculated above is non-zero, you must enter a cost code to identify where this balance is costed. Normally this would be to a cost account. You should only cost this to an unallocated costs code (as specified by the SC parameters CONTACT, CONTCAT, CONTJSC and CONTSEC) if you are intending to contra charge this amount to another subcontractor or supplier.
Recovery Code
The output code to be used on the SL invoice for the recovery amount.
Approved for Recovery

A box to indicate that the contra details have been checked and approved for recovery. This allows you to control situations where there is separation of duties (for example where the contras are maintained and approved by the Commercial team but the sales invoices are created by the Finance team).

The userid and date/time of approval are recorded when this box is ed (and cleared if uned), and are displayed to the right of the box.

This can only be updated when no corresponding sales invoice has been posted.

If Approved for Recovery has been ed, the other contra details cannot be updated.

You can also apply an administration charge to the recovery invoice.

Admin Charge %
The percentage of the cost amount to charge. If this is filled in, the Admin Charge is calculated automatically.
Admin Charge
The amount of the administration charge. If this is filled in, the Admin Charge % is calculated automatically.
Post Admin Charge to
If the Admin Charge is non-zero, this is the cost code that the administration charge will be posted to. Normally this would NOT be a cost code that matches the filtering for unallocated costs, unless you intend to charge this to another subcontractor or supplier.
Admin Charge Code
The output code to be used on the SL Invoice for the administration charge amount.
