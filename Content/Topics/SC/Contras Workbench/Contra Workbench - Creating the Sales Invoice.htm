Contra Workbench - Creating the Sales Invoice

The Create Sales Invoice action on the Contra tab allows you to create a invoice for the selected contra. This creates the invoice using the details populated on the recovery section.

Before the invoice can be created, the Approved for Recovery box must be ed.

The invoice will have the transaction type specified by the SL parameter CONTTYP.

If there is an unposted batch of the type specified by SL/CONTTYP, the invoice will be added to it, otherwise a new batch will be created.

If the SL parameter CONTSLPST is set to Y, tries to post the batch immediately. If CONTSLPST is set to N, the batch remains unposted; if necessary you will be able to delete the invoice and re-create it from the Contra Workbench (though you will not be able to update the invoice, because it is linked to a contra).

When the invoice has been posted, the original source transaction is marked as matched against the invoice.
