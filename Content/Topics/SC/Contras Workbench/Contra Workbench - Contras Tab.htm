Contra Workbench - Contras Tab

The Contras tab allows you to maintain the contras for the current . Contra transactions are created using the Create Contra action on the Unallocated Costs tab.

Fields

Field
Description
Contra Reference
The reference for the contra transaction. This is generated automatically based on the settings of the SC parameters CONTFORM and CONTCODE.
Description
A description to explain the reason for the contra; this can be printed on the contra notification.
Curr
The currency of the contra.
Source Ledger
The ledger in which the source transaction was entered.
Source Account
The account code of the supplier or subcontractor from the source transaction; for example, this could be the subcontractor who carried out work to remedy another subcontractor’s unsatisfactory work.
Internal Ref
The internal reference of the source transaction.
Cost Amount
The amount of the source transaction.
Recovery Account
The account from which you want to recover the costs; for example, this could be a subcontractor whose work was unsatisfactory.
Recovery Amount
The amount you want to recover. This could be more or less than the cost amount. Note that an administration charge can also be applied.
Recovery Code
The code for the recovery costs.
Recovery Status

The status of the contra:

  • Pending (no invoice yet)
  • Approved (Ready to invoice)
  • Invoiced (Invoice posted but unpaid)
  • Complete (Invoice paid)
Recovery Invoice Ref
The reference of the recovery invoice. This has a link that opens the invoice in a new window.

Buttons allow you to attach documents and notes to a contra record.

The following actions are available:

  • Print Notification – This allows you to print notifications for the selected contras (using a configurable document - see 1, Contras Workbench - Contra Notification Form).
  • Create Sales Invoice – This creates a sales invoice for the contra using the details populated on the recovery section. The Approved for Recovery box must be ed. If the recovery details have not been completed, an error is displayed which gives details of the fields that need to be completed. If there is an unposted batch of the type specified by SL/CONTTYP, the invoice will be added to it, otherwise a new batch will be created (see 1, Contra Workbench - Creating the Sales Invoice).
  • Cancel – Cancels the contra (changes the status to Cancelled and does not allow other actions). This also clears the contra reference from the original unallocated cost transaction linked to the contra, and changes its status from “Linked to Contra” to “New”.
