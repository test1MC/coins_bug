Contras Workbench - Initial Configuration

Subcontractors Parameters

Parameter

Description

CONTFORM

The number format for contra reference numbers

CONTCODE

Characters to precede contra reference numbers.

CONTWB

Which contra functionality to use:

  • Simple enables the new Contra Workbench.
  • Complex uses existing functionality.
  • None hides all contra functionality.

CONTJSC

A mask to identify cost transactions as awaiting contra.

CONTCAT

A cost category mask to identify cost transactions as awaiting contra.

CONTSEC

A section mask to identify cost transactions as awaiting contra.

CONTACT

An activity mask to identify cost transactions as awaiting contra.

CONTBPAY

Whether to include SL contra invoices when running Select/Deselect Certificates By Batch:

NONE (Blank)

ALWAYS (A)

PROMPT (P) - This allows you to choose, for each individual payment run, whether to include contras.

CONTMPAY

Whether to include SL contra invoices when running Select/Deselect Certificates (Manual).

Parameters

Parameter

Description

CONTTYP

The transaction type to use for contra charge Invoices.

CONTANAL

The default Customer Analysis Set values for customers auto-created from suppliers or subcontractors, This is a comma-separated list of values (one for each analysis set configured) that are used when a account is automatically created.

CONTSLPST

Whether sales invoices created from the Contra Workbench are posted automatically (Y or N). The batch is posted when the Create Sales Invoice action is run (see 1, Contra Workbench - Creating the Sales Invoice).

CONCASHT

The transaction type to use for cash transactions created when a contra payment batch is posted.
