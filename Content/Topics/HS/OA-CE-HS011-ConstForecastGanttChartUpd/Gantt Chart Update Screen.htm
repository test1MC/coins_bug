Gantt Chart Update Screen

A new link on the Construction Workbench site detail screen opens the Gantt Chart Update screen in a new window.  This displays the forecast you have currently selected.

Figure 2: Link on Construction Workbench

The Gantt Chart screen consists of two scrolling areas. 

  • On the left is a set of fixed columns showing details of each plot on the site. 
  • On the right is a timeline showing the tasks for each plot.
Figure 3: Gantt Chart Update Screen

You can scroll the rows vertically using the scroll bar on the right, or using the mouse wheel.

In addition you can scroll the fixed columns on the left and timeline on the right independently, using the horizontal scrollbars. 

Note that you can continue to scroll past the end of the project duration (for example if you want to extend the duration of the project).

3.2.1 Plot List

This shows the Plot Code, House Type, Group, Sequence, and the Programme Start and End Dates for the plot, as well as the captions from the caption group specified by the HS parameter PRWBPCG.  You can click the column heading to sort the plots by the data in that column; click again to reverse the order.

Figure 4: Plot List – Column Headings

As the number of columns could exceed the visible area, you can scroll the columns sideways, independently of the tasks in the area on the right.

3.2.2 Start and End Dates

You can double-click the start or end date for a plot to activate a calendar that allows you to change the date.

Figure 5: Calendar

When you change the start or end date, this will move all the associated tasks.  

For new projects, there are no dates and therefore no tasks: the plot start and end date will be blank.  When you first enter a start or end date, this creates the tasks.

3.2.3 Drop Down Project Menu

There are a number of functions that apply to the whole project plan.  These allow you to delay the project start or finish (except on plots on which construction has already started), or to update the build progress or build sequence.

Click the icon in the top left corner to display the menu.

Figure 6: Project Menu

The delay options prompt you for the number of days to delay by, and whether to retain existing gaps and durations.

Figure 7: Delay Action Input Form

The update options prompt you to confirm the action.

Figure 8: Update Action Confirmation Prompt

3.2.4 Right-Click Menu for Multiple Rows

You can recalculate the plan for one or more plots based on the construction programme for the plot. 

Figure 9: Right-Click Menu

Right-click the plot line (in the left-hand area) and choose Apply Programme from Start or Apply Programme from End, then click Yes at the confirmation prompt.

To recalculate several plots at the same time, use Shift+click or Ctrl+click to select the rows before you display the menu.

3.2.5 Filter

You can filter the list of plots, to show selected plots or house types, or filter according to build status.  Click the button in the top left corner.  displays a Filter frame:

Figure 10: Plot Filter

In the Plot List and House Type fields, you can enter a can-do list of codes, or click the button to use a standard lookup.

To apply the filter, click Filter On, then click OK.

The filter button is highlighted to show that the filter is applied: .

3.2.6 Task Bars

The right-hand area shows the tasks for each plot.  The of the tasks is set up on the construction stage.

Figure 11: Task Bars

When you hover over a task, a tooltip shows details of the task, including the start date, end date and duration.

Figure 12: Task Tooltip

Double-clicking on a task allows you to update details of the task: the start date, end date and duration.  You can also enter notes (which are currently only visible on this update frame).

Figure 13: Task Details Update Form

3.2.7 Daily/Weekly View

Buttons at the top of the screen allow you to switch between a daily or weekly view of the project timescales.

Figure 14: Weekly View

In weekly view, the column size is reduced but you can still drag a task down to an individual week day.

Currently Saturdays and Sundays are not shown as these are non-working days.

3.2.11 Completed Tasks

Dates for tasks that have been marked as completed are shaded with horizontal bars.

Figure 18: Completed Tasks

3.2.12 Moving Tasks

You can drag tasks to change their start date, end date or duration.

If you point to a task, the pointer changes to a black arrow.  Click and hold to drag the task to the left or right; this changes its start date.  If you move a task, this pushes other tasks in the same direction, but does not pull tasks.

For example, if you move a task to the right (that is, later), any later tasks will also be moved, but earlier tasks remain as they were.

Figure 19: Moving Tasks - Before

 Figure 20: Moving Tasks - After

Similarly, if you move a task to the left (that is, earlier), any earlier tasks will also be moved, but later tasks remain as they were.  This means you can create gaps between tasks.

Note: when dragging a task left or right, any gaps in the program are closed first.

For example if you drag the first task below two days to the right:

Figure 21: Moving Tasks with Gap - Before

Then only the stages to the left of the gap would move; the gap between the stages would be closed by two days:

Figure 22: Moving Tasks with Gap - After

3.2.13 Changing the Duration of a Task

To change the duration of a task, point to the beginning or end of the task; the pointer changes to a black arrowhead.  Click and drag the task to change its duration.  As with moving a task, other tasks will be pushed earlier or later.

Figure 23: Extending Tasks - Before

 Figure 24: Extending Tasks - After

You can drag tasks over the whole timeline for a project, but you can only move them within their own row, not to another row.

3.2.14 Dragging Multiple Tasks

You can drag multiple tasks on one or more rows.  To select multiple tasks, click and release, then drag the mouse pointer over the tasks you want to move:

Figure 25: Dragging Multiple Tasks – Selecting

Then click and release again.  The tasks you selected are highlighted in yellow:

Figure 26: Dragging Multiple Tasks – Highlight

You can then drag all the selected tasks together:

Figure 27: Dragging Multiple Tasks – After

Tasks be dragged over the whole timeline for a project.

3.2.15 Parallel Tasks

For some construction methods, certain tasks run in parallel.  For example with timber frame construction, brickwork tasks can run in parallel once the frame is complete as they are independent of the other tasks.

Up to two additional rows of tasks can be set up (see 3 Screen and Processing Changes, 3 Screen and Processing Changes).

Figure 28: Parallel Tasks

You can drag or update the tasks in the second row in the same way as the main row.  Note that you can only drag tasks within the same row; you cannot move them to a different row.

3.2.16 Read Only Mode

If the forecast type you selected is locked, you can only view the plan, not edit it.
