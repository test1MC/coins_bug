Test Message (hswssys1)

 

For testing purposes the message object hswssys1 contains a testMessage method.  This allows you to pass a simple message to and receive a response back containing the message.

A sample XML message to send a query to to access development information is shown below:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<login>

<user>xxxxxx</user>

<password>xxxxxx</password>

<CID>1</CID>

</login>

<call object="hswssys1" method="testMessage" id="0987654321">

<Test>

<Message>Hello World</Message>

</Test>

</call>

</env:Body>

</env:Envelope>

 

This would return a XML message with the following format:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<call object="hswssys1" method="testMessage" id="0987654321">

<Test>

<YourMessage>Hello World</YourMessage>

</Test>

<returnvalue />

</call>

</env:Body>

</env:Envelope>

 

If you leave the message blank you can test the error response.  With a blank message the following message will be returned:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<env:Fault>

<env:Code>

<env:Value>env:Sender</env:Value>

</env:Code>

<env:Reason>

<env:Text>Test message is blank</env:Text>

</env:Reason>

</env:Fault>

</env:Body>

</env:Envelope>
