Communigator Interface – Initial Configuration

Parameters

Several new parameters have been added to the list in Communigator Parameters. Review and set the Communigator Parameters parameters as necessary.

Parameter

New in v11.09

Description

APICLID

The REST web services API client secret. You will need to obtain this from Communigator.

ü

If this is left blank, the consent category flagged as 'Do Not Email' will be used.

If this is left blank, the consent source will not be updated.

The email address of the user to whom notifications of any errors that have occurred in the Communigator response broker should be sent to.

FEEDARCH

ü

The directory into which processed messages from the Communigator data feed are archived. This is useful when trouble shooting problems with the data feed.

Note this is only applicable when using the REST web service interface.

If this is left blank, then messages will not be archived.

System Parameters

Background Process

A new process, hsucgrbw.p, needs to be set up as a background process in Unix, to start each morning. For example:

00 07 * * * /u/coins/test/bin/session_%report.sh –b –p hsucgrbw.p > /u/coins/test/var/diag/communicator.log 2>&1

The cronjob above would start at 7:00 each morning and would output a log file to /u/coins/test/var/diag/communigator.log. The process polls Communigator to check for new responses in the Communigator data feed.

All email campaign activity data is queued into a table and can be accessed; these activities make up:

  • Email sent
  • Email opened
  • Downloads
  • Web site hit
  • Landing zone hit
  • Unsubscribes
  • Bounces
  • Event bookings
  • Event attendance
  • Event cancellation
  • New Leads
  • Campaign Statistics

When receives a response, if the relevant *EV* parameter is set, an event is created in the history.

The process creates a lock file as specified by the Communigator parameter LOCKFILE; to stop the broker, remove this lock file.

The LOGFILE parameter specifies a log file for the background process. Once running this will be written to each time it polls Communigator.

If the new Communigator parameter FEEDARCH is set, the downloaded messages will be saved to the specified directory.

Role Based

As with the middleware interface, licensing is via the role %COMMGATOR. Only users assigned to this role will be able to access the ‘Communigator’ option in the prospect mailshot.

For each user who will use these features, on the Groups tab of the user record the Communigator User box.
