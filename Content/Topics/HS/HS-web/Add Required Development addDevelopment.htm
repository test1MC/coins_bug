Add Required Development (addDevelopment)

 

A sample XML message to add a new development to the prospect’s required development list is shown below:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<login>

<user>xxxxxx</user>

<password>xxxxxx</password>

<CID>1</CID>

</login>

<call object="hswshvi1" method="addDevelopment">

<Prospect reference="487">

<Development reference="">

<Contract>2006</Contract>

<Plot1>004</Plot1>

<Plot2>005</Plot2>

<Plot3>006</Plot3>

<HouseType1>A3</HouseType1>

<HouseType2>A</HouseType2>

<HouseType3>B</HouseType3>

<Comments>Another Req Dev comments</Comments>

<NoLongerInterested>no</NoLongerInterested>

</Development>

</Prospect>

</call>

</env:Body>

</env:Envelope>

 

If the add of the development to the prospect’s required development list was successful this would return a XML message with the following format:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<call object="hswshvi1" method="addDevelopment">

<ProspectCompany>001</ProspectCompany>

<ProspectReference>000487</ProspectReference>

<DevelopmentReference>00044207</DevelopmentReference>

<returnvalue></returnvalue>

</call>

</env:Body>

</env:Envelope>

 

The DevelopmentReference returned is the unique reference which identifies the event within the selected company.
