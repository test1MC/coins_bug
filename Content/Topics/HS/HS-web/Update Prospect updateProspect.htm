As per the updatePerson method you only have to supply the fields you want to update.  If the field is not contained in the XML message then the existing value held in will be retained.

 

If you want to update the personal details for the people attached to the prospect record you will need to supply the unique reference of the person in the person node, in addition to the unique prospect reference.

 

Additional node details:

 

Node

Information

Requirement/MoveStartMonth

Requirement/MoveEndMonth

Requirement/MoveStartYear

Requirement/MoveEndYear

When updating a prospect we prompt for the looking for move timescales as a start month/year and  end month/year.  In the above example this would indicate that the purchaser is looking to move in between Aug 2006 and Nov 2006.

 

If the update of the prospect was successful this would return a XML message with the following format:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<call object="hswshvi1" method="updateProspect" id="fsnnffdfds">

<ProspectCompany>001</ProspectCompany>

<ProspectReference>000487</ProspectReference>

<returnvalue></returnvalue>

</call>

</env:Body>

</env:Envelope>
