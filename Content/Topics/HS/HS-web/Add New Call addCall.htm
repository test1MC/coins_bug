Additional node details:

 

Node

Information

Type

This should be set to SALES.

CallType

This is the type of the call. This field is validated against the codes of Call Types in .

RaisedBy

This is the unique reference of the person logins in the call. (Reference from the validate login method)

ToReview

This allows you to flag the new call as requiring review in .  There is then a separate process for reviewing these new calls in .

Source

This is the source of the call, showed in the “To Review” tab in the Customer Care Workbench. It is free texted field.

Status

This is the current status of this issue. 

Link (his_ref)

Link this issue to another issue if appropriate.

Status

Priority

Category

They are validated fields correspond to status/priority/category codes in .

Subcontractor

Supplier

 

They are account codes. Those fields are optional, but they are validated against the subcontractor/supplier accounts in .

Location

SubLocation

Appliance

All fields are optional, but validated against location/sublocation/appliance codes in .

 

If the add was successful this would return a XML message with the following format:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<call object="hswshcm1" method="addCall" id="test1234">

<ComplaintCompany>001</ComplaintCompany>

<ComplaintReference>000487</ComplaintReference>

<returnvalue></returnvalue>

</call>

</env:Body>

</env:Envelope>

 

This will return the call record unique reference (ComplaintReference node) in .  Note this reference is only unique within the selected company.

 

If the add failed for any reason the following message would be returned:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<env:Fault>

<env:Code>

<env:Value>env:Sender</env:Value>

</env:Code>

<env:Reason>

<env:Text>Development does not exist [HS235]</env:Text>

</env:Reason>

</env:Fault>

</env:Body>

</env:Envelope>

 

The env:Text node will contain the reason for failure.
