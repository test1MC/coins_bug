Add Prospect Event (addEvent)

 

A sample XML message to add a new event to the prospect’s history is shown below:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<login>

<user>xxxxxx</user>

<password>xxxxxx</password>

<CID>1</CID>

</login>

<call object="hswshvi1" method="addEvent" id="callid001">

<Prospect reference="487">

<Event>

<UserCode>ENQUIRY</UserCode>

<Date>2006-07-29</Date>

<Contract>2600</Contract>

<Plot1>001</Plot1>

<Plot2>002</Plot2>

<HouseType1>B</HouseType1>

<HouseType2>A3</HouseType2>

<Comments>New event comments</Comments>

</Event>

<EnquirySources>

<Source>

<Code>LOCAL</Code>

</Source>

<Source>

<Code>NEWLOC</Code>

</Source>

</EnquirySources>

</Prospect>

</call>

</env:Body>

</env:Envelope>

 

If the add of the prospect event was successful this would return a XML message with the following format:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<call object="hswshvi1" method="addEvent" id="callid001">

<ProspectCompany>001</ProspectCompany>

<ProspectReference>000487</ProspectReference>

<EventReference>00044206</EventReference>

<returnvalue></returnvalue>

</call>

</env:Body>

</env:Envelope>

 

The EventReference returned is the unique reference which identifies the event within the selected company.
