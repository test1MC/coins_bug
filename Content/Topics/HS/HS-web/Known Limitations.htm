3 Known Limitations

The following limitations apply to the web services at the time of writing:

    The Microsoft .NET “Web References” feature cannot be used with this web service – you must manually construct the XML string to post to the service and manually parse the XML response.

    There is no means to distinguish between an empty query result set and an error running the query on the dbQuery method.

    There are several issues around character encoding. You should use the native encoding of your progress database in XML calls at the moment. If the response string is to be loaded into the Microsoft XML parser a search and replace should be used to remove the encoding=”UTF-8” from the response header and replace it with the actual database encoding in order to ensure the response is parsed correctly. Also, you cannot currently use UTF-8 through the .NET interface as the StreamWriter emits marker bytes that the web service cannot interpret (unless you manually convert the Unicode text to a UTF-8 encoded byte array and write that byte array to a binary stream).
