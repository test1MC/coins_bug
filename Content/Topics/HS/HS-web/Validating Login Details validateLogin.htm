Validating Login Details (validateLogin)

 

The method validateLogin allows you to validate the email address and login password as person provides.  A sample XML message to validate a person’s login in is shown below:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<login>

<user>xxxxxx</user>

<password>xxxxxx</password>

<CID>1</CID>

</login>

<call object="hswsppe1" method="validateLogin" id="999">

<LoginDetails>

<Email>soap.request@coins-global.com</Email>

<Reference></Reference>

<Password>letmein</Password>

<SetLastLogin>yes</SetLastLogin>

</LoginDetails>

</call>

</env:Body>

</env:Envelope>

 

You can request verification of the login by either the persons email address or their unique person reference within .  Note the person must have a password to be able to login, blank passwords are not allowed.

The SetLastLogin node allows you to specify whether records the last login date and time for the selected person.  This information can be displayed in when viewing the person’s details.

If the login details are correct this would return a XML message with the following format:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<call object="hswsppe1" method="validateLogin" id="999">

<Email>soap.request@coins-global.com</Email>

<Reference>003017</Reference>

<returnvalue>VALID</returnvalue>

</call>

</env:Body>

</env:Envelope>

 

Note the person’s unique reference and email address are returned, even if only one of them was specified in the original request.

Alternatively if the login details were incorrect this would return a XML message with the following format:

 

<?xml version="1.0" encoding="utf-8" ?>

<env:Envelope xmlns:env="http://www.w3.org/2003/05/soap-envelope">

<env:Body>

<call object="hswsppe1" method="validateLogin" id="999">

<Email>soap.request@coins-global.com</Email>

<returnvalue>INVALID</returnvalue>

</call>

</env:Body>

</env:Envelope>
