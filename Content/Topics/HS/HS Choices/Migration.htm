Migrating from and to Choices

To enable existing users to use the new Choices functionality and to migrate existing plot and plot to the new scheme, a new data migration tool is available.

  1. Set up the specification choices you want to use.
  2. If you want to map statuses to the corresponding choice statuses, you need to set up choice statuses that use the same codes as your statuses.
  3. Create a specification template to use for the migrated records; this is needed to ensure that the new plot choice records are assigned to the correct specification area and specification item.
  4. Use to Choices Mapping to map existing standard codes to specification choices.

  5. Use to Choices Mapping to map existing standard codes to specification choices.

  6. Run Choices Data Migration to create new plot choices records. This runs as a background report.
