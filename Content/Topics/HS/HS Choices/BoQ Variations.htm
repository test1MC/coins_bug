BoQ Variations

The new Choices functionality is linked to the Boq Variations Workbench if the new HS parameter CHOBQVAR is set to Y.

Variation Reasons

If the new choices functionality is in use, a new system reason of ‘Client Choice’ is available, and the ‘Client Extra’, ‘Client Incentive’ and ‘Client Option’ reasons are removed.

Variation Detail

When selecting a variation, if the variation reason is of type ‘Client Choice’ a new ‘Customer Choices’ tab is displayed. This shows a list of all the changes to customer choices that have occurred. From here you can assign the choice to the variation.

Actions allow you to assign the choice to the variation, removed it from the variation, or flag it as ‘Variation Not Required’. When the choice is assigned to a variation or flagged as not required, if a construction status code has been set up as ‘Default BoQ Variation’, the construction status for the choice is set to this status.

You need to amend the bill items to reflect the choice.

Variation Development Browse

If the new choices functionality is being used, a new column shows the count of client choices changes that need to be reviewed.
