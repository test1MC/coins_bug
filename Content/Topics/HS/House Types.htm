House Types Setup

Standard House Types allows you to set up details of the standard house types - that is, house types that you will build on more than one development.

Standard House Types allow you to record information about the house types you build, and the specifications that belong to each house type. The specifications in turn determine which cost heads are used on plots of that house type.

For developments that have their own house types, you can set up VAP > Development Maintenance > Development House Types.

 

If you are using non-sellable plots (for example, to allocate costs to roads, sewers, or the structural parts of a block of flats), you also need to set up at least one house type for non-sellable plots.

Navigate to BOQ > Setup > Site Setup > Standard House Types

Clicking will allow you to add a new house type. Once you have added the new house type you are then able to associate this to plots that you create against the new development.

When adding a new house type, you are presented with only one mandatory Field, which is the name of the house type.

If a house type exists that is similar to the new house type, you are able to select this from the house type code, change the code to a new unique code and subsequently change the name too.

Field

Description

Market Mix The market mix code for the house type/plot.
Market mix codes are set up using House Sales > Plot Sales > Setup > Market Mix Codes.
Description A sales description of this type of house.
The description is copied to plots of this type; it is displayed in the sales workbench and can be used on other websites via the HS connection.
Gross Area The default gross area for a plot of this house type.
Unlabelled field The units in which the gross area is measured.
Floor Area The default Floor Area of a house of this type.
Unlabelled field The units in which the Floor Area is measured.
Frontage The width of this house type.
Depth The depth of this house type.
Unlabelled field The units in which the width and depth are measured.
Notional Area A measure used when prorating costs across plots to calculate the cost recovery and profit for a plot.
This allows more refinement than just floor area and takes into account factors like whether the plot is a 1-, 2- or 3-storey building or an apartment, as these take up different land areas.
Construction Programme The code for the construction programme.
Density The density for this house type.
This is used in as a method of apportioning costs. The 'Density' method in LA uses the density factor and applies it to the floor area.
Bedrooms The number of bedrooms.
Receptions The number of reception rooms.
Bathrooms The number of bathrooms.
Parking The number of parking spaces available with the house type.
SAP Rating The SAP rating for this house type.
Carbon Index The carbon index of this house type.
Retired Has this house type been retired?

Associate

Navigating to the screen will show you a blank table, as the cost heads have not been associated.

Using the options menu located in the top right hand corner you are able to associate all cost heads based on the selection in the ‘associated cost head’ field

Clicking on the ‘recreate unit cost heads’ allows the system to grab all cost heads based on the associated cost head field and list them against the house type.

The cost head tab will now look like this, fully populated with the cost heads, at this stage cost and duration is irrelevant as the bill will control this.
