Types List Field

A new Types List field is available on the Item properties screen in Items, as shown in Figure 1. This field only appears for Manual Input Items when the Breakdown Item? field is ed. It allows you to set the Type values on the Breakdown lines and combine Item values by the Breakdown Types.

The Types List field may be either blank or contain a list of comma-separated entries (such as O,Open,C,Contingency,X,Closed).

  • The preexisting Type column is now labeled as Source.
  • The new currentType column:
  • Appears if the Types List field for the Item is not blank; and
  • Is sorted by breakdown keys.
  • A new Type field is available as a Browse Filter.

Update Items

Item Breakdown records can be populated when: (1) using the Update Items action process on the Main tab of the  Workbench, and (2) Source is defined on the Item and/or entered manually by the users.

If the Types List field of an Item is not blank, the breakdown process handles the new Type field as follows:

  • Assigns the first code listed in the Types List on newly created breakdown records.
  • Keeps values in existing records (even if the record value is being changed).

Field Definitions (Selected Fields)

Select Items to Print

You can select available Items from the Available field and move them to the Selected field for printing.

Group by

Select how you would like the Items grouped on the report.

If any of the Available Items have a non-blank Breakdown Types List, a new Breakdown Type option becomes available in the Group by field (not shown in the screen above). If this option is selected, the generated report is grouped by Type.
