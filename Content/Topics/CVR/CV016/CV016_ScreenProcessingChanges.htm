Subcontract View Option

Subcontract View Properties

Field Definitions

Header Summary

The Field Group to show as a Header summary in browse View.

Number of Fixed Columns

The number of columns you wish to be fixed on the screen. For example, when configuring a view, you may want the code and description to remain on the screen when scrolling left and right through the periods. This option allows you to fix these columns on the screen.

Suppress Actual Periods

Whether to suppress Amounts in the actual periods in the screen.

The Subtotal Amount (Actuals) column can be configured in the Fields tab to show subtotal values of the actual periods.

View Usage

Whether this view should be shown as Page Only, Report Only, or Report and Page.

Build Condition

This is the Build Condition used to suppress or display a View, depending on the user's Role.

Order Number

Select the range of subcontractor order numbers to report on.

Subcontract Type

Select the range of subcontract type codes to report on. The codes are defined in Subcontract Ledger > Setup Maintenance > Subcontract Types.

Trade

The subcontract trade to report on.

/Non-

Whether to report on Only, Non- Only, or Both and Non-.

Complete/Incomplete

Whether to report on Complete Only, Incomplete Only, or Both Complete and Incomplete.

Custom Query

Enter the part of a WHERE clause to append to a certain View’s body query.

Items and Generate Spread

Online Items Recalculation

Items handles Subcontract Item full values. It can now also cater for values recalculation.

Background Update of Items

If a Subcontract Item is selected for update in Update Items, values of the Item are recalculated as for other Items. The processing of Subcontract Item values is also now allowed for:

  • Update Items, Recalculate Summary Values
  • Re-Spread Values (as a part of Update)
  • Generate Spread
  • Roll Up Spread to Item
  • Process Snapshot data

Generate New and Copy

When a new is generated as a copy of a previous one, or by using the Copy previous adjustments option, previous adjustments are copied as part of the initial Update processing. When an existing is copied via the Workbench, manually entered data are copied from the original to the new .
