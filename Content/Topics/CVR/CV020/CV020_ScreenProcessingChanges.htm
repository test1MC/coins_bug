Configuration

Parameter CV/WFAUTRLS

A new CV parameter has been added to the module to control the release of a completed for Authoration workflow.

Parameter

Description

CV/WFAUTRLS

Automatically release completed for Authoration workflow (Y,N)

Whether to automatically release completed for Authoration workflow.

This parameter is only used if CV/USECVWFL = Y and CV/WTAUTCVR is not blank.

Parameter Settings

In order for the for Authoration workflow to start automatically, the Input Complete field must be ed on the Details screen (see Section 2.1), and the following parameter settings must be used, as shown in Figure 3:

  • CV/USECVWFL = Y
  • CV/WTAUTCVR is not blank
  • CV/WFAUTRLS = Y

Once these conditions are met, the Release for Authoration browse action is hidden from all relevant screens (see Section 2.1.1).

Parameter Settings for Workflow

Workbench

Details Screen

A Details screen displays when you click the Open button on the Details tab in Workbench > Main browse > click link > s browse > click Number link. The Details screen appears in Update mode and contains a Input Complete box field, as shown in Figure 1.

Details Screen in Update Mode

Input Complete Field

One of the conditions that must be met in order for the for Authoration workflow to start automatically is the ability to the Input Complete box field on the Details Update screen.

This box is enabled (can be ed) if the following conditions are met:

  • CV/USECVWFL = Y
  • CV/WTAUTCVR is not blank
  • CV/WFAUTRLS = Y

It is disabled (read only) if the is authored or an authoration workflow is currently active.

Release for Authoration Action

In previous versions of  OA, a completed could only be released for authoration workflow by using the Release for Authoration browse action, as shown in Figure 2. If you set up  OA to automatically release a for authoration (see Section 1.2), the Release for Authoration action is hidden from all relevant lists.
