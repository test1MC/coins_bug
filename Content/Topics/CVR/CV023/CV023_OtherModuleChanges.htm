Model Specific Items and Styles: Other Module Changes

This section includes information about functions found on other modules that were modified in v11.09 to allow the use of Model specific Items and Styles.

Contract Default WBS Codes

In Status > > Default WBS Codes, the Journals field (see Figure 11) lookup now shows Items. If parameter CV/ITEMMODL is set to Y, these Items come from the Model used in the ; otherwise, Company specific Items are used.

Journals Lookup

Company-wide Items Lookup

A new lookup browse is used outside the Module if parameter CV/ITEMMODL is set to Y to allow for the selection of any Item existing in a Company. This lookup looks the same as the regular Items browse, except that a Model column has been added after the Item column, as shown in Figure 12.

Forecasting Contract Items

The Contract Items screen in Forecasting > Setup now displays the model-based Item lookup if the Source for Values field is set to CVR Item, as shown in Figure 13.

House Sales Journal Maintenance

In House Sales > Financials > Setup > Journal Maintenance, the Item lookup on the Journal Line now uses a company-wide lookup type, as shown in Figure 14.

Land Appraisal Links Configuration

In Land Appraisal > Setup >  Link Configuration, the Plot Items for Build Cost field now has a company-wide lookup that uses multi-selection options, as shown in Figure 15.

Links Configuration Screen

Land Appraisal Section Input

In Land Appraisal > Setup > Template Configuration, the Cost Recovery Items field now has a company-wide lookup that uses multi-selection options, as shown in Figure 16.

House Building Forecast Matrices

Analysis, Plot , Tick Group, and Payment Stage Matrices used in the House Building module now support Model specific items if parameter CV/ITEMMODL is set to Y.
