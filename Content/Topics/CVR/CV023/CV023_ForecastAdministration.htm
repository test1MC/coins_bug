Model Specific Items and Styles: Administration

This section includes information about functions found on the >  Administration menu that were modified in v11.09 to allow the use of Model specific Items and Styles.

Items Snapshot

If parameter CV/ITEMMODL is set to Y, Items selection on the Item Snapshot screen includes all Items specific to any of the existing Models valid for the Company.

Populate Model Specific Items and Styles

If parameter CV/ITEMMODL is set to Y, and at least one Item/Style exists in the Company with a blank Model key, a new Populate Model Specific Items and Styles function is shown on the Administration menu. It allows Company specific Items and Styles to be copied to each Model, then deleted.

When you select this function from the menu, the screen displays a warning message, as shown in Figure 5.

Populate Model Specific Items and Styles

Click the Next button to proceed.

 

  • If the Item/Style exists in a Model with the same code, it is deleted and overridden by the Company specific Item/Style.
  • If copied successfully, all Company specific Items and Styles are deleted.
  • Processing generates a detailed report.

Populate Company Specific Items and Styles

If parameter CV/ITEMMODL is not set to Y, and at least one Item/Style exists in the Company with a non-blank Model key, a new Populate Company Specific Items and Styles function is shown on the Administration menu. It allows Model specific Items and Styles to be moved to the Company level.

When you select this function from the menu, the screen displays a warning message, as shown in Figure 6.

Populate Company Specific Items and Styles

Click the Next button to proceed.

  • If a Company specific Item/Style exists with the same code, it is deleted and overridden by the Model specific Item/Style.
  • If copied successfully, all Model specific Styles are deleted.
  • Processing generates a detailed report.

Items and Styles Usage Reports

Items Usage Report (see Section 4.4.1) and Styles Usage Report (see Section 4.4.2) functions are now found on the > Administration menu. These reports are always available, irrespective of the CV/ITEMMODL parameter setting.

Styles Usage Report

The new Styles Usage Report is always based on all available Styles; it shows where and how each of them is used in the Models. The selection screen is shown in Figure 7, and a sample Report is shown in Figure 8.

Styles Usage Selection Screen

List

Select whether you wish to display an All or Overlapping (the default) list on the report.

  • All - contains all Items (used, unused)
  • Overlapping - Items that are not used or only used in a single Model will not appear in the list

Field Set

Select the Field Set that you wish to display on the report. Field sets are set up in OA Reporting & BI > Field Sets > Report Field Sets.

Sample Report

Items Usage Report

The new Items Usage Report function is always based on all available Items; it shows where and how each of them is used in the Models. The selection screen is shown in Figure 9, and a sample Report is shown in Figure 10.

Items Usage Selection Screen

List

Select whether you wish to display an All or Overlapping (the default) list on the report.

  • All - contains all Items (used, unused)
  • Overlapping - Items that are not used or only used in a single Model will not appear in the list

Field Set

Select the Field Set that you wish to display on the report. Field sets are set up in OA Reporting & BI > Field Sets > Report Field Sets.

Sample Report

The Item Code may be used in three different ways on this report, as follows:

  • Used in multiple Models (there can be different setups in each Model in CV/ITEMMODL=Y configuration, or just reused with the same setup in any configuration.
  • Used in a single Model in any configuration
  • Not used for either configuration (exists but not used on fields)
