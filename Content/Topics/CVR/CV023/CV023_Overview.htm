Model Specific Items and Styles: Overview

When Items were shared between all Models defined in a company, it caused issues if the Items were named as per the common naming convention (e.g. CCFCAC). For example:

  • Importing a new Model could overwrite an Item and break existing Models.
  • Any changes in an Item shared by different Models affected all s.
  • Except Item value changes, Items dependency also could change and affect an entire company.

Similar issues occurred with Styles.

As a result of these issues, there was a need to use a different formula for different Models in the same Company (used as per specific). This has now been accomplished by allowing Items and Styles to be maintained at the Model level. By using Model specific sets of Items and Styles, any changes made in the Model do not affect other Models.

This document describes how the module has been modified to allow the use of Model specific Items and Styles.
