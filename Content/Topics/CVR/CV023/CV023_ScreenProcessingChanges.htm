Model Specific Items and Styles: Configuration

A new CV parameter lets you choose whether to use Model specific Items and Styles.

Parameter

Description

CV/ITEMMODL

Use Model specific Items and Styles

 

Model Specific Items and Styles: Setup

This section includes information about functions found on the >  Setup menu that were modified in v11.09 to allow the use of Model specific Items and Styles.

Items and Styles

If CV/ITEMMODL is set to Y, the Items and Styles workbenches on the > Setup menu are suppressed. However, you can now perform maintenance for Items and Styles in Models.

Models

Items Tab

If CV/ITEMMODL is set to Y, a new Items tab appears on the Models browse, as shown in Figure 1. This tab shows a browse screen of Items with a set of Model specific Items.

Items Tab

As with other existing workbenches, you can add/update/delete Items, copy Items (also copying from an Item of another Model) on this browse.

Named Filters

Groups of named filters are now available on the Items browse for both ITEMMODL settings (Y or N), as shown below.

1st Named Filter (by Model):

This Model Items
Company Specific Items
All Items

3rd Named Filter:

All Items
Active Items
Retired Items
Unused Items - only appears if This Model Items option has been selected in the 1st Named Filter. If selected, this filter shows Items that are not used in the current Model.

4th Named Filter (by Item attributes):

All

Manual Input Items

Breakdown Items

Snapshot Items

Items

Post Journal Items

Post Budget Items

Post Data Store Items

Column Sets

Column sets functionality is now available in the Items browse.

Transfer Items

A new Transfer Items action appears on the Model specific Items browse if parameter CV/ITEMMODL is set to Y. This action allows multiple Items to be transferred (copied or moved) to selected Model(s). If selected, the selection screen is displayed, as shown in Figure 2.

Transfer Items

Field Definitions

Items

The list of Items (Item Code and Description) you selected for transferring.

Model

The Available list shows all Models in the Company (including the current one); the Selected list shows any Models you have selected for the Transfer process. By default, no Models are selected.

Action

Indicate whether you wish to Copy or Move the Items to the selected Model(s).

If Already Exists?

If an Item being transferred already exists in a selected Model, indicate whether you want to Overwrite it or Skip it.

If Does not Exist?

If an Item being transferred does not exist in a selected Model, indicate whether you want to Copy it or Skip it.

When you click the Next button to continue, the selected options are validated, and a warning about possible data loss is displayed regarding if you selected either of the following:

  • Action - Move selected
  • If Already Exists – Overwrite selected

Styles Tab

A new Styles tab appears on the Models browse, as shown in Figure 3. This tab is a browse showing a set of Model specific Styles.

Styles Tab

As with other existing workbenches, you can add/update/delete Styles, as well as copy Styles (also copying from a Style of another Model) on this browse.

Named Filters

Groups of named filters are now available on the Styles browse, as shown below.

1st Named Filter (by Model):

This Model Styles
Company Specific Styles - only shown if there is at least one Style with a blank Model code. It allows you to easily select company specific Styles and copy/move them to any Model using the Transfer Styles action.
All Styles

2nd Named Filter:

All Styles
Active Styles
Retired Styles - only appears if there are Styles marked as Retired
Unused Styles - only appears if This Model Styles option has been selected in the 1st Named Filter. If selected, this filter shows Styles that are not used in the current Model.

Column Sets

Column sets functionality is now available in the Styles browse.

Transfer Styles

A new Transfer Styles action appears on the Model specific Styles browse if parameter CV/ITEMMODL is set to Y. This action allows multiple Styles to be transferred (copied or moved) to selected Model(s). If selected, the selection screen is displayed, as shown in Figure 4.

Transfer Styles

Field Definitions

Styles

The list of Styles (Style Code and Description) you selected for transferring.

Model

The Available list shows all Models in the Company (including the current one); the Selected list shows any Models you have selected for the Transfer process. By default, no Models are selected.

Action

Indicate whether you wish to Copy or Move the Styles to the selected Model(s).

If Already Exists?

If a Style being transferred already exists in a selected Model, indicate whether you want to Overwrite it or Skip it.

If Does not Exist?

If a Style being transferred does not exist in a selected Model, indicate whether you want to Copy it or Skip it.

When you click the Next button to continue, the selected Styles are copied/moved to another Model(s).

Fields Tab

If parameter CV/ITEMMODL is set to Y, the Item/ Style field on the Fields tab will only handle Items/Styles specific to the parent Model.

Proration Tab

If parameter CV/ITEMMODL is set to Y, only Model specific Items/Styles are maintained on the Proration tab.

Copy Model

If parameter CV/ITEMMODL is set to Y, and you copy a Model to a new one, all original Model Items and Styles are also copied to the new Model.

If parameter CV/ITEMMODL is not set to Y, Copy Model skips copying the Items and Styles, even if the original Model has its own Items/Styles.

Model Specific Items and Styles: Export/Import Setup

This section includes information about functions found on the >  Setup menu that were modified in v11.09 to allow the use of Model specific Items and Styles.

Export Setup

If parameter CV/ITEMMODL is set to Y, the Export Setup function in > Setup selects Model specific Items and Styles for export. Otherwise, it only works with Company specific Items and Styles in the Model and disregards any Model specific Items/Styles.

Import Setup

If parameter CV/ITEMMODL is set to Y, the Import Setup function in > Setup assigns a Model code to the Items and Styles.

Model Specific Items and Styles: Administration

This section includes information about functions found on the >  Administration menu that were modified in v11.09 to allow the use of Model specific Items and Styles.

Items Snapshot

If parameter CV/ITEMMODL is set to Y, Items selection on the Item Snapshot screen includes all Items specific to any of the existing Models valid for the Company.

Populate Model Specific Items and Styles

If parameter CV/ITEMMODL is set to Y, and at least one Item/Style exists in the Company with a blank Model key, a new Populate Model Specific Items and Styles function is shown on the Administration menu. It allows Company specific Items and Styles to be copied to each Model, then deleted.

When you select this function from the menu, the screen displays a warning message, as shown in Figure 5.

Populate Model Specific Items and Styles

Click the Next button to proceed.

 

  • If the Item/Style exists in a Model with the same code, it is deleted and overridden by the Company specific Item/Style.
  • If copied successfully, all Company specific Items and Styles are deleted.

Populate Company Specific Items and Styles

If parameter CV/ITEMMODL is not set to Y, and at least one Item/Style exists in the Company with a non-blank Model key, a new Populate Company Specific Items and Styles function is shown on the Administration menu. It allows Model specific Items and Styles to be moved to the Company level.

When you select this function from the menu, the screen displays a warning message, as shown in Figure 6.

Populate Company Specific Items and Styles

Click the Next button to proceed.

  • If a Company specific Item/Style exists with the same code, it is deleted and overridden by the Model specific Item/Style.
  • If copied successfully, all Company specific Styles are deleted.
  • Processing generates a detailed report.

Items and Styles Usage Reports

Items Usage Report (see Section 4.4.1) and Styles Usage Report (see Section 4.4.2) functions are now found on the > Administration menu. These reports are always available, irrespective of the CV/ITEMMODL parameter setting.

Styles Usage Report

The new Styles Usage Report function allows you to select multiple Styles to report on. The selection screen is shown in Figure 7, and a sample Report is shown in Figure 8.

Styles Usage Selection Screen

List

Select whether you wish to display an All or Overlapping (the default) list on the report.

Field Set

Select the Field Set that you wish to display on the report. Field sets are set up in OA Reporting & BI > Field Sets > Report Field Sets.

Sample Report

Items Usage Report

The new Items Usage Report function allows you to select multiple Items to report on. The selection screen is shown in Figure 9, and a sample Report is shown in Figure 10.

Items Usage Selection Screen

List

Select whether you wish to display an All or Overlapping (the default) list on the report.

  • All - contains all Items (used, unused)
  • Overlapping - Items that are not used or only used in a single Model will not appear in the list

Field Set

Select the Field Set that you wish to display on the report. Field sets are set up in OA Reporting & BI > Field Sets > Report Field Sets.

Sample Report

The Item Code may be used in three different ways on this report, as follows:

  • Used in multiple Models (there can be different setups in each Model in CV/ITEMMODL=Y configuration, or just reused with the same setup in any configuration.
  • Used in a single Model in any configuration
  • Not used for either configuration (exists but not used on fields)

Model Specific Items and Styles: Workbench

If parameter CV/ITEMMODL is set to Y, Model specific Styles are used on the browse screen.
