Filtering and Sorting

Browse Views previously could only be filtered and sorted using source table fields (e.g., Cost Code Section, WBS Code, etc.). You can now also filter and sort by Items (for example, to show Cost Codes with forecasted revenue > $100K).

Configuration

A new CV/ITEMFILT parameter lets you decide whether you will allow Items to be filtered and sorted.

Parameter

Description

CV/ITEMFILT

Allow Filtering and Sorting on Items?

Models

Details Tab

Two new fields, List of Items for Enhanced Filtering and List of Items for Enhanced Sorting, appear on the Setup > Models > Details tab if parameter CV/ITEMFILT is set to Y, as shown in Figure 1: Enhanced Filtering/Sorting Fields.

Enhanced Filtering/Sorting Fields

Field Definitions

List of Items for Enhanced Filtering

Use this field to enter up to 10 Item codes.

List of Items for Enhanced Sorting

Since the information in this field directly corresponds with the information in the List of Items for Enhanced Filtering field, it is automatically filled in when Item codes are entered in that field. It is therefore a View only field.

Fields Tab

A new Sort field is found on the Setup > Models > Fields tab, as shown in Figure 2: Sort Field.

Sort Field

Workbench

Add New

Each time a new is created, new fields cvs_filt_items, cvs_filt_map are populated from the cmh_filt_items and cmh_filt_map fields in Models. You cannot change these fields after the is saved if parameter CV/ITEMFILT is set to Y.
