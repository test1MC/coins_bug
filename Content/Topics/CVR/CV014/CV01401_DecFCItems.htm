Decimal Items

Decimal Items were stored using 2 decimal digits precision. This precision has now been was increased to 6 decimal digits. You can also configure precision on a Item. This section describes what has been done in  OA to accommodate this change.

Items

Data Type Field

The Data Type combo field, which was previously only enabled on Manual, Calculated Items, has been updated as follows:

  • Appears on the Update form for all Items.
  • Is enabled if the Item is a Manual, Calculated type.
  • Is dynamically resolved from the Item type (such as Costs, Revenue, Accruals, etc.) or from the RSP field entered.

Item Properties

A new Decimal Places field appears on the Item properties screen, as shown in Figure 1: Decimal Places Field. It is only shown when the Data Type is Decimal.

Decimal Places Field

Items Export

The Export action on the Items browse now allows you to export Items using the standard  OA Export button.

Workbench

Manual Input

When entering data for a , Item values are saved, and dependent Items are then recalculated. The recalculation of these dependent Items has been changed to accommodate accommodates the precision setting of decimal Items, as follows:

  • Item value evaluation is done using precision (no rounding).
  • Item values are rounded to the desired precision before being stored in the database.

Manual Breakdown Input

A Item configured for breakdown input uses calculation qty*rate to store the value. The result of this calculation result is now rounded as per the Item’s precision.

Update

Update has been modified to allow allows for Item precision, as follows:

  • Item value evaluation is done using precision (no rounding).
  • Item values are rounded to the desired precision before being stored in the database.

Generate Spread

Generate Spread now allows for Item decimal precision.

Items Snapshot

Changes made in Update and Generate Spread also apply to Items Snapshot data.
