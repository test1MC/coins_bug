Employees

Employee

A code to identify the employee.

If an employee is also a user and is required to enter his own contract timesheet, the Employee ID must be the same as their OA login, and JC parameter TSDEFEMP must be set to Y.

Associated Parameters

JC/TSDEFEMP

Whether should enter the current user's ID as the default employee name in Timesheet Entry (Y or N).

Name

The employee's name.

Department

The employee's department.

Employee departments allow you to group timesheet employees for reporting and security purposes. They are set up in Contract Status > Timesheets > Setup > Employee Department.

Entry Period

The dates used for timesheet entry: Weekly (7 days), Monthly (Calendar Month), or Period ().

Non-costed?

Whether this employee is a "non-costed" employee.

If this is ticked and the employee records timesheets for overhead contracts (specified by the JC parameter TSOHJOBS), then no cost transaction will be recorded when the timesheet is posted.

However, if overhead category staff are allocated to a direct contracts, or normally costed staff are allocated to an overhead contract, the costing to the contract occurs.

Associated Parameters

JC/TSOHJOBS

A can-do listcan-do list (a list in which wildcards are used to omit or include certain values. For example, !AB*,* means include all codes except those beginning "AB". See Can-Do Lists.) of contract groups that are treated as overhead contracts.

If a timesheet line is entered for a contract whose contract group is in this list AND the employee is an overhead employee (that is, the Non-costed field in Employees is ticked), no cost transaction is created for that timesheet line when the timesheets are posted.

However, if overhead category staff are allocated to a mainstream contract, or normal costed staff are allocated to an overhead contract, the costing to the contract takes place.

Unlabelled field

Whether the cost control analysis for this employee is to a contract or General Ledger analysis.

Analysis

The cost control account for this employee. This is the account to receive the credit transactions for employee timesheet costs. You can specify a contract cost analysis - the credit entry of the timesheet posting will be posted to this contract cost instead of the G/L account.

Grade

Select the employee grade. If set, the rates for this grade are used instead of the employee rates.

Employee Grades are set up in Contract Status > Timesheets > Setup > Employee Grades.

Unlabelled field

The description of the employee grade.

Department Approver

Select the Department code of persons who will approve this employee's timesheets.

Status

Select the status of the employee: Employee (regular employee), Authoriser (employee able to authorise timesheets), or Regional Inputter.

External (not Payroll)

Whether this employee is a "paid externally" employee.

Start Date

Enter the date that the employee started work.

Leave Date

Enter the date that the employee finished work.

Email Address

Enter the email address of the approver.
