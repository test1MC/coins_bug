Employee Security

Allows you to control which employees are allowed to enter and report on timesheet information for different employees and departments.

NOTE

Department ADM could be assigned as Department Security to allow global access

NOTE

security also affects which timesheets a user has access to (see 1 - Account and Security).

This function has no impact on security for Payroll employees or HR personnel.

Associated Parameters

SY/DELEGATE

SY/DELEGATE (Current Value: N)

Whether you are using delegated security.

User ID

The ID of the user.

The user ID is the code which a person uses to identify themselves in the system; for example, when logging in.

Name

The name of the user or group with this ID.

A test - James

Security List

A can-do list(a list in which wildcards are used to omit or include certain values. For example, !AB*,* means include all codes except those beginning "AB". ) of department codes that define which employees this user is allowed to access. Blank means allow access to all employees.

Code

A can-do list(a list in which wildcards are used to omit or include certain values. For example, !AB*,* means include all codes except those beginning "AB". ) of employee codes that allow you to grant or deny the selected user access on an employee-by-employee basis. Blank allows access to all employees.
