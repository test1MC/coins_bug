Changes

SV Rate Maintenance

Payroll Rates Tab

Rate Maintenance uses Trade, Union, Skill, Earnings Category, and Employee when defining Payroll Rates. The Payroll Rates tab in > Company Config > Setup > Rate Maintenance now includes a Pay Shift field, as shown in Figure 1. This key field on the timecard is used in Rate Maintenance as the basis for defining rates. It is validated against Pay Shift codes defined in COINS. A blank option is allowed.

Payroll Rates Tab

SV Cost Transactions Workbench

The Cost Transactions workbench is displayed in the menu if a uses Cost Based .

Payroll Cost Transactions

The Payroll tab is displayed on the Cost Transactions Workbench in the > > Cost Transactions browse if the uses a Rate Type with its source set to Payroll (PRCOST). A new Pay Shift column is found on this Payroll (PRCOST) tab of the > > Cost Transactions browse, as shown in Figure 2.

SV Cost Transactions Payroll Tab

The Cost Transaction Amount recalculation now considers Pay Shift Rates entered in Rate Maintenance.

Cost Transactions All () Tab

The All () tab in > > Cost Transactions now allows a Pay Shift column to display in Column Sets.

Populate Cost Transactions

A Populate Cost Transactions option is now available on the > > Cost Transactions Options menu, as shown in Figure 3. When run, it caters for the new Pay Shift field on the PR Timecard, which is used for the Amount calculation on newly created Cost Transactions.

Populate Cost Transactions Option

SV Applications and Certificates

Cost Transactions

The Cost Transactions menu is displayed in the Applications/Certificates screen in > > Cost Transactions > Applications (or Certifications) for that use Cost Based . A Payroll menu option and screen are displayed (see Figure 4) if the uses Rate Type with source set to Payroll (PRCOST). You can also see the new Pay Shift column that appears on this screen.

If you select the All () option from this same menu (see Figure 4), its browse also shows the new Pay Shift column.

Applications/Certificates Payroll Cost Transactions Screen

Standard Cost Based Report – Cost Transactions

A Standard Cost Based Report is generated for a single Application (Certificate) from the Applications/Certificates browse action (see Figure 5) or Application/Certification summary Options menu (see Figure 6) in > > Cost Transactions > Applications (or Certifications). If at least one Pay Shift Code is defined in Payroll, a new Pay Shift column appears on the Cost Transaction Report.

Standard Cost Based Report Action

Standard Cost Based Report Menu Option

Populate Measure Using Actual Costs

Populate measure using actual costs is available as an option in the Application/Certification Options menu in > > Cost Transactions > Applications (or Certifications). When run, it populates Cost Transactions automatically, linking them to current Applications/Certifications.

SV Reports

/ Reconciliation Report

If at least one Pay Shift Code is defined in Payroll, a new Pay Shift column appears on the Cost/ Reconciliation Report.
