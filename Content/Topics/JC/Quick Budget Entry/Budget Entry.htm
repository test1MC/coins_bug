Budget Entry Workbench

Input Budget Workbench is a new function that allows the user to enter budget costs against cost codes. It is available on the Status Input menu if either of the JC parameters BUDMANUPD or FORECURR are not blank.

The initial screen is a browse of . Body selectors allow you to select which Period and Budget to enter the budget for. If the new JC parameter BUDMANUPD is blank, only the Forecast At Completion budget is available; otherwise, any entry in BUDMANUPD that exists in Memo Data Types and has type “U-Unlocked” is available.

Following the link on a displays the Budget Input Workbench.

This is a browse of cost codes, which depends on the setting of JC/USECCCAT.

  • If USECCCAT is not set to All or Used, budget costs can be entered at cost code level. There is only one row per cost code, and only cost codes which have a non-blank input method are shown.
  • If USECCCAT is set to to All or Used, budget costs can be entered at cost code/category level. There are multiple rows per cost code.

NOTE

If the selected is a and the selected budget type is one of the restricted budgets specified by the following JC parameters, the Budget Entry Workbench will be read-only: BUDAPP, BUDCURR, BUDVARI, BUDRMRC, BUDPRMRC, BUDXFER, BUDPEND, BUDUNAPP.

If the budget input method is Hours, the Budget Input: Rate and Budget: Hours at Completion fields are updateable.

If the budget input method is Value, the Budget: Cost at Completion field is updateable.

If the budget input method is Quantity, the Budget: Qty to Complete field is updateable - this is available as an additional column set field.

Links on the Actuals, Remaining Commitment and Acruals columns open in a frame.
