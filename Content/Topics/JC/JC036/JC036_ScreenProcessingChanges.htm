Screen and Processing Changes

All values in the Summary now reflect the correct information of a due to changes that were made in v11.09. These changes are described in the sections below.

Summary Browse

When you select the Summary function in Status > , the browse screen appears, as shown in Figure 1. Some changes were made in v11.09 to this screen, which are described below.

Summary Browse

  • Recogned Revenue At Completion has been relabelled as Revenue At Completion. It is now calculated using Revenue At Completion rather than Forecast.
  • EMAC – OBM has been relabelled as EMAC - IBM and is now used on both the Browse and Detail screens. It is calculated using Revenue At Completion rather than Forecast.
  • Future Recogned Revenue has been relabelled as Revenue Backlog. It is now calculated using Revenue At Completion rather than Forecast.
  • Future Margin is now calculated using Revenue At Completion rather than Forecast.
  • Future Margin % is now calculated using Revenue At Completion rather than Forecast.
  • PCT Billed is now calculated using Revenue At Completion rather than Forecast.
  • PCT of Extras is now calculated using Revenue At Completion rather than Forecast.
  • To Complete Hours is now calculated using Estimated Hours at Completion instead of Forecast Hours.
  • PCTV Hours is now calculated using Estimated Hours at Completion instead of Forecast Hours.
  • The EMAC$ and EMAC% fields have been relabelled as Current MAC $ and Current MAC %. They are now calculated using Current Revenue instead of Forecast.
  • Contract Backlog is now calculated using Current Revenue instead of Forecast Revenue.

Summary Detail

The Summary Detail screen (see Figure 2) is accessed by selecting the Summary function in Status > > Summary > drill down on link > Summary tab. Some changes were made in v11.09 to this screen, which are described below.

Job Summary Detail

  • Recogned Revenue At Completion has been relabelled as Rev At Completion. It is now calculated using Revenue At Completion rather than Forecast.
  • EMAC is now calculated using Revenue At Completion rather than Forecast.
  • EMAC % is now calculated using Current Revenue instead of Forecast.
  • EMAC – OBM has been relabelled as EMAC - IBM and is now used on both the Browse and Detail screens. It is calculated using Revenue At Completion rather than Forecast.

  • Future Recogned Revenue has been relabelled as Revenue Backlog. It is now calculated using Revenue At Completion rather than Forecast.

  • Future Margin is now calculated using Revenue At Completion rather than Forecast.
  • PCT Billed is now calculated using Revenue At Completion rather than Forecast.
  • PCT of Extras is now calculated using Revenue At Completion rather than Forecast.
  • Estimated Hours At Completion is now calculated using Estimated Hours At Completion rather than Forecast Hours at Completion.
  • Contract Backlog is now calculated using Current Revenue instead of Forecast Revenue.
