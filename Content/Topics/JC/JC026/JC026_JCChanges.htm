Field Definitions

Number

Number is a mandatory field. The Company Number for the Number lookup is used from the header. Validation checks to see if a exists in the Company entered on the CDR Header. It also checks security.

If parameter JC/CDRMJOBS:

  • is set to Y, a Number row appears on the screen. This can be used to enter a Number on each column of the grid.
  • is not set to Y, a Number is still used on the grid column as a hidden field (it defaults from the CDR Header Number).

The Number defaults as follows:

  • For the 1st column of the grid, it uses the Number from the CDR Header.
  • For the next columns added to the grid, it uses the Number from the previous column; if the Number is not entered in a previous column, it uses the Number from the CDR Header.

Section

The Section field lookup and validation were modified to use the Number from the grid column, regardless of how parameter JC/CDRMJOBS is set, i.e., the Number will be on the field whether it is hidden or not.

Activity

The Activity and fields were modified to validate the Cost Code against the Number on the grid column, rather than using the Number from the CDR Header.

Daily Reports Header and Details Processing

Linking and Daily Reports

can now be linked to Daily Reports. This means that when a CDR Detail record is changed, the Number is also changed.

The linking process performs as follows:

  • A linking record is created if it does not exist with the Number.
  • The old value of the Number field is checked against CDR details (Header). If neither uses that , the link with the old Number is removed.
  • No processing occurs if the linking record already exists or any other CDR element uses the old Number value.
  • The Number on the CDR Header is used for defaulting.
  • Detail lines of the CDR are validated and used in any processing as per the linked .

Daily Report Status Change Actions

The Submit, Release, and Reject change actions use the Numbers from CDR details instead of the CDR Header for any detail-specific checks and reporting.
