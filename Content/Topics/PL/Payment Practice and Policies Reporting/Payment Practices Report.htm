Payment Practices Report

The new Payment Practices and Performance Report provides clients with the information required to calculate the three statistics to be included in their twice-yearly Payment Practices and Performance Report.

NOTE

Before running the report, we recommend that:

  • All cash has been allocated
  • reconciliation is complete
  • Payment terms on all supplier and subcontractor records are up to date

This will help to ensure that the information on the report is complete and accurate.

Payment Practices and Performance Report Selection Screen

Fields

Version

Whether to show detailed or summary information in the PDF output file.

  • Detail - lists the number of transactions for each day
  • Summary – provides summared results

Dates

The range of payment dates to report on (both From and To are required).

Number of days after Cheque Date for Payment Received

The average number of days between the date of a cheque and the date it is received.

Number of days after BACS Date for Payment Received

The average number of days between the date of a BACS payment and the date it is received.

PL Transaction Types

A range or list of transaction types to report on. This allows you to use different transaction types for any transactions outside the scope of the reporting requirement. By default, all transaction types will be included.

PL Invoice Received Calculation Method

How to calculate the Invoice Received date.

  • Days after Invoice Date - based on the assumption that the invoice was received a number of days after the invoice date.

  • Days before Posted Date - based on the assumption that the invoice was received a number of days before the posted date.

PL Invoice Received Default Days

The default number of days after the invoice date or before the posted date to be used as the Invoice Received Date. This can be varied on particular suppliers by use of a supplier analysis set. By default this will be zero.

Supplier Analysis Set for Days Override

The supplier analysis set used to override the default invoice received date. On suppliers where this analysis set value is zero or a positive integer, this number of days will override the default input above.

SC Transaction Types

A range or list of transaction types to report on. This allows you to use different transaction types for any transactions outside the scope of the reporting requirement. By default, all transaction types will be included.

SC Amount Confirmed Calculation Method

How to calculate the date when the amount was confirmed.

  • Days after Claim Date
  • Days after Valuation Date
  • Days before Valuation Date
  • Days before Certificate Date
  • Days before Posted Date

SC Amount Confirmed Default Days

The default number of days before or after the Claim, Valuation, Certificate or Posted date to be used as the Amount Confirmed Date (equivalent to the Invoice Received Date). By default this will be zero.

Subcontractor Analysis Set for Days Override

The subcontractor analysis set used to override the default amount confirmed date. On subcontractors where this analysis set value is zero or a positive integer, this number of days will override the default input above.

Generate Payment Details

this box for a list of paid transactions to be added to the Excel output file.

Generate Due Date Details

this box for a list of transactions where payment was due in the period to be added to the Excel output file.

For best performance it is recommended to run the report at summary level (do not the “Generate” boxes).

When the selection criteria have been correctly set, they can be stored and the report run whenever required with just a change of the report date range.

Report Output

Although the legislation only requires the reporting of summared statistics , the report optionally provides more detail so that:

  • Clients can see some detail of the calculation;
  • Clients and/or their auditors can check the reasonableness of the underlying figures;
  • Clients can combine the figures with others from outside of where necessary (for example where there is a division of the company that is not accounted for on ).

The report consists of two three sections. The first shows the average number of days taken to make payments, and the percentage of payments made which were paid in 30 days or fewer, between 31 and 60 days, and in 61 days or longer.

The second section shows the percentage of payments which were not made within the agreed period.

The third section shows invoices and certificates due for payment in the period but currently unpaid, analysed by hold code. This section is not required for the published statistics, but helps highlight disputed invoices and other invoices which are overdue for payment and which are likely to adversely affect the statistics reported for the period in which they are paid.

NOTE

Where invoices are prepaid, the report lists such transactions in a section for “<1” (less than one day).
