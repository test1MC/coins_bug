GRN considerations

Where a supplier is using the Open Order process and sending Order Confirmation messages these messages may contain a Delivery Ticket number that can be used in the matching process. The SUPREF option of the PO/AUTCHK parameter should be set to Y to make use of this.  

The matching process is determined by configuration. For material orders the JC/ACCMETH states what accrual methods are valid, chose from any or all of these values

O Order
G GRN
Z Neither

The contract will then have one of the valid accrual methods.

The Supplier details record can override this accrual method using one of the valid values. The contract might normally use G for all suppliers, but a particular supplier record might have O to accrue PO values not GRNs.

Where O is used and the PL/AUTOINVM parameter is set to 2, it is possible for an eInvoice to go straight to approval if all the matching criteria described above is met.
