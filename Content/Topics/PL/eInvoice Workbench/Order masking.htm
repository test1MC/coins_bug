Order masking

Which PO numbers are to be treated specially as PINN transtions is determined by setting appropriate masks for the BASDA Invoice record in the “Message Rule” program (ec2034.p).  An <UnverifiedOrderMask> tag with three nested tags, <Company>, <Order> and <PLType> is added to this configuration. Typically,

<UnverifiedOrderMask>
<Company>*</Company>
<Order>M-*,*/*/*</Order>
<PLType>PINN</PLType>
<PLCRType>PCRN</PLCRType>
</UnverifiedOrderMask>

Here any transaction with a PO number that begins M- or any transaction with a PO number that has two “/” characters, in any Company, will be treated as a PINN transaction. The PO Number is compared with each entry in the <Order> list essentially comparing PO number character strings with wildcard masks. Negated masks, masks beginning with the “!” character, may be used at the start of the <Order> list for a small increase in processing time. By definition transactions with PO numbers matched by a negated mask will not be changed from PINV to PINN. The <PLType> tag must be one of the values in the PO/NONPOTTS parameter list.
