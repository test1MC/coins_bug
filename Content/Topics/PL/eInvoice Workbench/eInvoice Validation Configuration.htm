eInvoice Validation

 

The following recipient roles are permitted: (A)ccounts, (B)uyer, (S)ite or su(P)plier.  When deciding who should receive a given notification, will cross reference the validation table role if accounts, buyer or site with the user roles contract configuration (ci901), giving priority to any contract specific entry that may exist for that role.  In each case, the first entry with an email address will be used (i.e. the same contract and role could have several entries for different users).
