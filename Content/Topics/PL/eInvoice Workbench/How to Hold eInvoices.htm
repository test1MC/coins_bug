How to Hold eInvoices

This secton describes how the eInvoice workbench processes PL transactions, invoices and credit notes that require a Purchase Order (“PINVs”) and transactions that do not require a PO (“PINNs”) with respect to setting the HOLD status and HOLD codes. The relevant PL and PO parameters listed are taken into account.

Transaction processing

There are two classes of PL transactions, invoices and credit notes that require a Purchase Order (“PINVs”) and transactions that do not require a PO (“PINNs”) each follow separate routes through the workbench programs.

Regardless of transaction type, “Loading” a transaction in the workbench will first compare the value of the invoice with the Supplier’s authorisation limit. If DEFHOLD is true an invoice value exceeding this limit will mark the transaction as being on hold. The transaction hold code will be set to the value of DEFHCODE.

PINN processing

The Transaction Type code (coj_type) of a PINN transaction will be included in the NONPOTTS parameter list.

After pressing the <Details> button in the workbench, if NONPOCST is false or the invoice refers to a PO the transaction is given distribution type “R”.

If NONPOCST is true and the transaction does not have a PO number (a PINV transaction with no PO number would be in error), costing detail my be entered. If costing has been entered the transaction is given distribution type “C” and the Supplier’s authorisation limit is checked again. If the invoice value exceeds this limit the transaction put on hold. The transaction hold code will be set to the value of DEFHCODE. If APPRCOST, possibly modified by EAPPRCST, is true further hold processing is offered as follows:

  • If HLDREGIN is false and the distribution type is not “R” the user may change the transaction’s hold status. If HLDREGIN is true and the distribution type is “R” the transaction is put on hold. For a transaction on hold the user may then change the hold code.
  • If HLDREGIN is true and the transaction’s distribution type is “R” the transaction is put on hold. The transaction hold code is set to the value of DEFHCODE possibly modified EDEFHCOD.

PINV processing

The Transaction Type code (coj_type) of a PINV transaction will not be included in the NONPOTTS parameter list.

If HLDREGIN is true and the transaction’s distribution type is “R” the transaction is put on hold. The transaction hold code is set to the value of DEFHCODE possibly modified EDEFHCOD.

If EHLDPINV is true further hold processing is offered as follows:

  • If HLDREGIN is false and the distribution type is not “R” the user may change the transaction’s hold status. If HLDREGIN is true and the distribution type is “R” the transaction is put on hold. For a transaction on hold the user may then change the hold code.

Batch creation

The batch creation process is common for all transaction types. If DEFHOLD, possibly modified by EDEFHOLD is true the user is prompted for a hold code, defaulted to DEFHCODE possibly modified EDEFHCOD, which need not be entered. If HLDREGIN is true and the distribution type is “R” the transaction is put on hold and the hold code set the value entered. For any transaction on hold and with no code the hold code is set the value entered.

The transaction is passed to the General Ledger.
