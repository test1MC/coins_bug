Transaction Processing

Setting the EC/SWWBMEMO parameter to Y disables the PINV to PINN functionality. Changing a PINV to a PINN will happen only when the Supplier has an ECOMM Address record setup; and the Company the workbench matches a mask in the <Company> tag; and the PO number matches a mask in the <Order> tag.

A Contract number will be sought from the 3rd Part Order Cross References. If a Contract is found then the transaction will be changed from PINV to PINN using this, the PO number is not considered further. If a Contract is not found in the Cross References and the PL/NONPOWBS parameter is set to Y then costing information, including the Contract number is sought from the incoming PO number.

If the PO number has just one “/” character, that is two elements, then if the first element is a valid Contract in the current Company and this Contract does not use WBS codes then the transaction will be changed from PINV to PINN. Costing is completed using the value of NONPOCH for material invoices and by the value of NONPOPCH for plant invoices.

If the PO number has just two “/” characters, that is three elements, then if the first element is a valid Contract and the second element is a valid WBS code for that Contract the transaction will be changed from a PINV to a PINN. Costing is completed using the value of NONPOCH (materials) or NONPOPCH (plant).
