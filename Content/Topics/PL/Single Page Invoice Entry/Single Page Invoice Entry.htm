Single Page Invoice Entry

Single Page Invoice Entry is a new function on the Invoices menu. The initial browse of invoice batches, and the batch header screen, are the same as for the standard invoice batch entry, but the invoice entry screen is different.

The invoice header remains fixed at the top of the page. Most of the information is now on a single page, rather than being on separate tabs, and the layout of the fields has been improved.

Single Page Invoice Entry Screen

The fields are grouped as follows:

Main Details

  • Supplier #
  • Supplier Name
  • PO #
  • Internal Ref
  • Supplier Ref
  • Description
  • Analysis
  • Invoice Date

Supplier Details

  • Address

Tooltips on icons also show the company’s registration number, email address and telephone number.

Order Details

  • PO Original Amt
  • PO Variation Amt
  • PO Current Amt
  • Previously Invoiced
  • Remaining Balance
  • GRN Value (if the order requires GRNs)

The above fields are displayed if a purchase order number is entered.

Dates and Type

The values of these fields are set automatically, and the fields are disabled. In many cases these fields do not need to be changed. If you do want to change any of these fields, the button enables them for update.
  • Due Date
  • Discount Date
  • Cost Effective Date
  • Received Date
  • Txn Type
  • Dist Type
  • Currency
  • VAT

Invoice Amounts

  • Merchandise
  • Additional Charges
  • Net Amount
  • Discount%
  • VAT Sales/Use Tax
  • Gross Amount

Commercial Hold

  • On Hold
  • Hold Code
  • Hold Reason

Financial Hold

  • On Hold
  • Hold Code
  • Hold Reason

If the order requires GRNs, but no GRNs have been entered yet, the text “GRNs do not exist” is displayed next to the order number, to inform the user that there is no need to try to perform any invoice matching.

If the order is linked to GRNs, when you save the invoice details the Matching screen is displayed.

and Costing Distribution

New dynamic input lines have been used for the distribution (if the invoice has multiple lines), and the costing distribution (if the invoice is not registered).

NOTE

This currently only supports UK-style invoices. For US invoices, costing distribution still uses the existing method of entering data on a Costing Distribution tab when saving the invoice.

Dynamic Distribution Lines

These work as follows:

  • As you tab out of the last field on the line, a new line is automatically created.
  • The add button in the header allows you to add a new line.
  • The buttons on the line allow you to copy or delete the line.
  • You can rearrange the lines by dragging the line numbers up or down.
  • If there are more lines than will show on the screen, the header and footer (which shows the running totals) remain visible while the lines scroll.

A frame to the right of the lines shows values for reference, which are updated as you update the lines.
