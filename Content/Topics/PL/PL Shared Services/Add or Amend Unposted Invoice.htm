Add (or Amend Unposted) Invoice

This screen substantially revises the standard PL Invoice Entry screen in order to gain efficiencies of use pertinent to a Shared Service operation.

The principal change is the entry of either the PO Number or the Cost Analysis before any other data.  If the PO Number is found in only one of the companies that the user has access to, then the associated company number is assigned to the invoice.  The same logic applies if no PO Number is entered but instead a Cost Analysis is entered.  The derived company’s name is displayed on the right hand side of the screen and all screen value defaults, lookups and validation are shown with reference to this derived company.

Unless otherwise stated, all validation remains the same as that implemented by the standard PL Invoice Entry routine. 

As each invoice is saved, it is inserted into (or moved to) a purchase invoice batch for the current user and PL financial period in the destination company.  If no such un-posted PL batch already exists then it is created automatically.

If there is no current PL financial period in the destination company, then the following error message displayed “No current PL period available for company ” and the save action rejected.

As just stated, an invoice may need to be moved from one PL invoice batch to a batch in another company, this could happen any number of times.  The system ensures that the batch totals of each of these batches ; corresponds correctly to the number of invoices in each batch.

 

@Utilisation of this functionality, requires sound controls around the use of ”current” periods across companies.

Add (or Amend Unposted) Invoice – With Scan

This function appears on the standard menu as illustrated below. This routine is operated in exactly the same way as its non-Shared Service equivalent.  The user selects either the Batch or the Pool  & Bulk auto indexes the images.

A PL invoice batch is created for the current user, for the current PL financial period and in the current logged-in company (i.e. NOT necessarily the user’s prime company). “Empty” PL invoices are created and put in the PL invoice batch just created and a “pointer” record created in the Shared Serve PL invoice batch’s line table as shown below.

 

A specially formatted “Temporary Internal Reference”  is generated for each “empty” invoice.  The reason for doing this is because, by definition, Shared Service invoices may well end up in a different company  and each company may have a different strategy for generating internal references.

As each invoice is saved, it is inserted into (or moved to) a purchase invoice batch for the current user and PL financial period in the destination company.  If no such un-posted PL invoice batch already exists, then it is created automatically.

If there is no current PL financial period in the destination company then the following error message displayed “No current PL period available for company ” and the save action rejected.

As stated, an invoice may need to be moved from one PL invoice batch to a batch in another company.  This, in theory, could happen any number of times.  The system ensures that the batch totals of each of these batches, corresponds correctly to the number of invoices in each batch.
