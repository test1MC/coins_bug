5.1.2 Add (or Amend Unposted) Invoice – With Scan

This function appears on the standard menu as illustrated below. This routine is operated in exactly the same way as its non-Shared Service equivalent.  The user selects either the Batch or the Pool  & Bulk auto indexes the images.

A PL invoice batch is created for the current user, for the current PL financial period and in the current logged-in company (i.e. NOT necessarily the user’s prime company). “Empty” PL invoices are created and put in the PL invoice batch just created and a “pointer” record created in the Shared Serve PL invoice batch’s line table as shown below.

Note that a specially formatted “Temporary Internal Reference”  is generated for each “empty” invoice.  The reason for doing this is because, by definition, Shared Service invoices may well end up in a different company  and each company may have a different strategy for generating internal references.

As each invoice is saved, it is inserted into (or moved to) a purchase invoice batch for the current user and PL financial period in the destination company.  If no such un-posted PL invoice batch already exists, then it is created automatically.

If there is no current PL financial period in the destination company then the following error message displayed “No current PL period available for company ” and the save action rejected.

As stated, an invoice may need to be moved from one PL invoice batch to a batch in another company.  This, in theory, could happen any number of times.  The system ensures that the batch totals of each of these batches, corresponds correctly to the number of invoices in each batch.
