Retention on Invoices - Releasing Retention

For retention to be released, a retention-only invoice (that is, an invoice with retention and no goods amount) must be raised first. This can only be done automatically. If you try to enter an invoice with no goods amount and a negative retention amount, displays an error.

A new menu option, Release Retention, allows you to create retention-only invoices. This shows a list of open retention records.

The following actions are available:

  • Create Retention Invoices. This creates a batch of retention invoices for the selected retention lines.
  • Create Retention Invoices with Discount. This creates a batch of retention invoices for the selected retention lines and marks the order as completed.

If the new PL parameter RETDISC = Y, retention invoices are created with discount. For example, if the supplier discount is 10% and the retention invoice has gross of $100 and retention of -$100, the discount will be $10.

The invoices have no cost distribution (as there are no goods) but are set to Costed.

The batch posts automatically. The batch financial date is the current period. The invoice internal reference is generated automatically and the invoice date is today’s date.

The retention posting to the GL uses the PL retention GL account set up in PL Configuration (see 1 - Retention on Invoices - PL Configuraton). However, if the supplier has a retention account set up (see 1 - Retention on Invoices - Supplier Maintenance) this overrides the PL Configuration retention account.

A retention-only invoice can be paid either manually or as part of a computer payment run just like any other invoice.
