Provisional Costing Tab

This screen allows you to distribute the cost across one or multiple lines as required.

If the transaction is not linked to a Purchase Order, you will need to build up the cost code manually by entering (for example) the Contract, , Section and Category codes by using the lookup button after each component is selected.

For example:

Click the Save button to save the line.  This will automatically open up another costing line so that you can split the costs against multiple lines if required, selecting Save on each line.  Once all lines are entered and saved, select the undo button to end line entry.

The cost able amount and Distributed fields should match once all the cost lines have been entered. If these figures do not match, this indicates a costing imbalance between the invoice value and the figures distributed across the lines that must be corrected before the batch can be posted.

Click to return to the Invoices screen where you invoice should now be displayed.

If you have more invoices to add to the batch you may click to continue invoice entry until all invoices are entered.

Once all invoices are entered and fully costed, they will appear on the Invoices screen with the Status Unposted and you are ready to post the batch.

 

 

If the invoice status is shown as Cost Imbalance you will need to go back into that invoice and correct the costing distribution on the Provisional Costing tab.  Any invoices showing Cost Imbalance will prevent the batch from posting.
