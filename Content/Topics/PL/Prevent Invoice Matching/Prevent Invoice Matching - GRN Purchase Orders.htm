GRN Purchase Orders

Similar changes have been made for GRN purchase orders.

Match Action

When the "Match" action is taken, the following will now occur:

  • If the selected invoice matching line (or lines) refer to a that uses WBS codes:
  • If the new PO parameter ALWCLSST = N, and if the section associated with the WBS code on any of the related GRN distribution lines is closed, an error is displayed and matching is prevented.
  • If the new PO parameter ALWCLSCC = N, and if the cost code on any of the related GRN distribution lines is closed, an error is displayed and matching is prevented.

In this case you will not be able to complete the invoice matching process unless the closed code is re-opened. If the new PO parameter ALWCHOV5 = Y, you will be able to override the closed code, and change it to an open code.

Overriding the WBS Code and Cost Code

If the new parameter ALWCHOV5 = Y, for each GRN distribution line the following additional fields appear on the line when updating a matching line:

WBS Code (if the uses WBS codes)

Cost Category

In addition, the Quantity field has been renamed GRN Allocation, and is not updateable.

The default values for these fields are taken from each of the GRN distribution lines.

If the WBS Code, or Category have been overridden an asterisk is shown in a new column at the right of the Matching browse.
