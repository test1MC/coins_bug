Rate Type Validations

Validations now exist to suppress changes to Rate Types used for at the Series or level. These validations are discussed in the sections below.

Maintenance

Rate Type changes at the level (with no restrictions) are only allowed under the following conditions:

  • No Rate Type is defined on any Sales Series.
  • No transactions exist.

This means that changes are only allowed to a new Rate Type with the same source types and Extra Payroll Groups.

Series Maintenance

Rate Type changes at the Series level are now allowed for a new Rate Type with the same source types and Extra Payroll Groups at the level or if blank.

Rate Types Maintenance

Main Screen

In > Company Config > Rate Maintenance, any changes to Rate Type sources on the main Rate Types Maintenance screen in the columns Payroll Transactions from, Status Timesheet Transactions from, and Transactions from (see Figure 1: Rate Types Maintenance Main Screen) are allowed only if Rate Type is not used at the or Series level.

Also, you are now prevented from deleting a Rate Type and will receive an error message if the Rate Type is used at the or Series level.

Rate Types Maintenance Main Screen

Extra Payroll Groups Tab

Changes on the Extra Payroll Groups tab on the Rate Type Summary browse screen in Rate Types Maintenance (see Figure 2: Extra Payroll Groups Screen) are only allowed if the Rate Type is not used at the or Series level.
