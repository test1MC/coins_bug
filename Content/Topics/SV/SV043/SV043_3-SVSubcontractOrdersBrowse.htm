SV Subcontract Orders Browse

There are two Subcontract Order browses that can be displayed, depending on how SV parameter USEPACK is set (Y or N). Both of these browses are discussed below.

Subcontract Orders Browse (USEPACK = Y)

If SV parameter USEPACK is set to Y, the Subcontract Orders browse is displayed as shown in Figure 1: SC Orders Browse (USEPACK = Y).

SC Orders Browse (USEPACK = Y)

This browse now includes:

  • A new Proposed Value column.
  • Suppressed columns when either Order Currency or Subcontractor Currency is selected.
  • A new Show Amounts in body selector shown at the bottom of the screen. The currency options for this field are:
  • Order Currency
  • Subcontractor Currency
  • Base Currency
  • Currency
  • Corresponding values in the selected currency shown in the Original Value, Committed Value, and Proposed Value columns.
  • A Currency column for Report Currency found after the Proposed Value column. It shows a code indicating which currency is being used for the amounts.

Subcontract Orders Browse (USEPACK = N)

If SV parameter USEPACK is set to N, the Subcontract Orders browse is displayed, as shown in Figure 2: SC Orders Browse (USEPACK = N).

SC Orders Browse (USEPACK = N)

This browse now includes:

  • A new Proposed Value column.
  • Totals shown for all value fields. Total columns are suppressed when Order Currency or Subcontractor Currency is selected.
  • Corresponding values in the selected currency shown in the Original Value, Committed Value, and Proposed Value columns.
  • A Currency column for Report Currency found after the Proposed Value column. It shows a code indicating which currency is being used for the amounts.

Field Definitions (Selected Fields)

Currency

Order Currency is displayed in this field. Its default value is derived as follows:

  • If the SC is not associated with an SC Order, the SC Currency value is Currency.
  • Otherwise, the SC Currency value is SC Order Currency.

 

After an SC Order has been entered on an SC , it cannot be updated once the SC is saved. The SC Order field can be left blank, and when it is blank, the SC is assigned with the ’s Currency. When a change is made to an SC Order on an SC , the following occurs:

  • The Currency Code of the SC is recalculated and displayed on the page.
  • All the Resources values are recalculated and sent to Items and the SC level.

Show Amounts In

The options for this field are: Order Currency, Subcontractor Currency, Base Currency, and Currency.

  • , , Quoted, and Approved values on this screen can show multi-currency values.
  • The Total columns are suppressed if either Document Currency or Subcontractor Currency is selected.

SC Value Updates

If an SC is linked to a Client , any and value updates on the SC and its relevant Items and Resources cause the recalculation of these values, Items, and Resources. Such recalculation is based on the following:

  • Exchange rates are used for an Exchange Rate Type defined by parameter SV/CURCTYPE (if not blank) or a transactional type.
  • The effective date is the SC entry date (if an SC is not approved) or the date on which the SC was first approved.

Client Created from SC

If a Client is created with Items and Resources from an SC , the and values on the Client are calculated as follows:

  • Exchange rates are used for an Exchange Rate Type defined by parameter SV/CURCTYPE (if not blank) or a transactional type.
  • The effective date is the SC Entry date (if an SC is not approved) or the date on which the SC was first approved.
