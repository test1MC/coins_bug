eCommerce Improvements - Parameter Changes

Some parameters relating to eCommerce and PCards have been changed to reflect the current state of the system:

  • SY parameters ECOMDIR, ECOMFDIR, HUBID, HUBUSER, HUBPWD and HUBIP are no longer used and have been replaced with an equivalent EC parameter.

    Old SY Parameter

    New EC Parameter

    Descrption

    ECOMDIRECDIRThe directory to store outbound and inbound messages. You need to create the directory, and the following subdirectories: /IN /LOAD /OUT and /REJECT. We recommend $BASE/var/ECDir.
    ECOMFDIRECFDIRThe directory where background processes will place routed inbound messages before they are loaded into the appropriate routine in . You must create the directory, but will automatically create a hierarchy of subdirectories for each company, inbox/processed, document type, and schema as the messages are loaded. We recommend $BASE/var/ECFDir.
    HUBIDHUBIDThe user ID of the account used to log in to the eCommerce hub; used by the FirstB2B bounceback service.
    HUBUSERHUBUSERThe user name of the account used to log in to the eCommerce hub; used by the FirstB2B bounceback service.
    HUBPWDHUBPWDThe password for the account used to log in to the eCommerce hub; used by the FirstB2B bounceback service.
    HUBIPHUBIPThe IP address of the eCommerce hub; used by the FirstB2B bounceback service.
  • The following parameters used in receiver/router processes can no longer be set as company-specific, because the current company cannot be identified in background processes:

    • EC parameters: ECDIR, ECFDIR, HUB*, EXCH*, FTP*, FERRMAIL, XSLLOCOA, MANADMIN

    • PL parameters: PCDXMLCO, CHKDUPPC, TCRDTYPS

  • PL/CHKDUPPC now allows you to specify which duplicate checks are used, using a multi-select list. These were previously set using PL/DUPCHK, which has been removed.
  • EC/ADDFLDS has also been changed from a plain list to a multi-select list.
