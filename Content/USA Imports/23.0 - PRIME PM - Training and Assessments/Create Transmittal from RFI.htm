1. Create Transmittal from RFI - You can also create a transmittal from an RFI. This is used to create a formal record recording that the RFI has been sent. To do this use the “create transmittal” action – details of the RFI will automatically populate the transmittal screen. More details on transmittals are given in the next section.

Transmittals

  1. A transmittal is used to formally record that a communication has been sent. A transmittal can be created from any of the other communication types as described above, or it can be created from scratch. The cover sheet for the transmittal can also be generated automatically.
  2. To create a transmittal from scratch use the Add button on the transmittals tab:

  1. Fill out the appropriate information:
  1. The “Delivered by” field lists all possible delivery options.
  2. The transmittal will default to being sent from the user that created it. This can be overwritten.
  3. The transmittal needs to be sent to a person. This person can be selected via the lookup or entered directly into the “To” field.
  4. The “from” and “to” addresses will be populated based on the “from” and “to” fields, although these can be overwritten if required.
  5. The reasons for the transmittal can be selected by checking the appropriate boxes.
  6. If an acknowledgement is required for the transmittal then check the “acknowledge” box and add a due date if you have one. Once the transmittal has been acknowledged you can reopen the transmittal and enter the acknowledgement date.

  1. Once you have saved the transmittal, and via the Communications Management Workbench hyperlinked into the newly created Transmittal, you will see a number of other tabs which relate.

 

  1. The “Copies To”, “Documents” and “Linked” tabs work exactly as described in the RFI section above.

 

  1. The Items Tab is unique to transmittal, and is used to record the items that were attached to the transmittal (if any). To record items that were included in a transmittal, use the Add button and enter the details. Don’t forget to save the record before you leave the screen.

 

Transmittal Actions

  1. There are some actions that you can apply to this newly created RFI, via the main communications screen, once a Transmittal has been selected.

  1. All actions work in the same way regardless of if they are run from this Actions list, or from the Options Menus on the individual tabs.
  1. Print Transmittal - You can print a transmittal in the same way that you can print an RFI
  1. The layout of the printed Transmittal is completely configurable, so the example that you see may not look like the example in this guide.
  2. When you select to print a Transmittal, you are taken to the screen below, which will look familiar to any COINS OA user, as it is based on the main printing Output Options tab.

  1. Choose the form that you wish to use at the top of the screen – there may be just one form available.
  2. If you want to email the Transmittal, then select the “email” checkbox. If not, then uncheck this box, and the document will get added to the report queue in the usual way.
  3. If you do want to email the form then you can add text into the “message” field. It is possible to build templates that pre-populate text for an emailed RFI. If such a template exists, select the HTML checkbox.
  4. Using the Email Attachments tab, you can add any of the previously attached project documents to this email.
  5. Once you are happy with your selection options use the Next button to launch the print and/or email.

 

  1. Email Transmittal Documents – performs the same emailing functions of the previous action, without printing
  1. You can also choose to email any documents that are attached to the project, by using the Attachments Tab

Submittals

 

  1. The Submittals Tab allows you to set up and maintain submittals for this project. During the construction process, many items must be submitted, approved, and purchased for a project. Project managers typically keep this information on a spreadsheet, but they can instead replace this spreadsheet through the use of submittals.
  2. To create a submittal, use the Add Button on the Submittals Tab of the Communications Management Workbench

  1. Fill out the appropriate details:
  1. The source is the person representing the company who is presenting the submittal – e.g. if the submittal is coming from a subcontractor, the source will be the contact in the subcontractors.
  2. The responsible party is the person responsible for approving the submittal.
  3. The status, importance and impact field are all used to help categorize the submittal. The options that you chose are selected from pre-defined lists.
  4. The “Open Date” field is the date that the submittal was created
  5. The “Job Required Date” field is the date that a response is required by if the job is to proceed without delay.
  6. The “lead time” fields are used if the submittal will need to be fabricated before it is sent in. These do not need to be used unless they are relevant.
  7. The “Response Due” field is used to record the date that a response is expected by
  8. The “closed Date” field is used to record the date that the submittal was closed.
  9. Click Save when all information has been added.

 

  1. Once you have saved the submittal, and via the Communications Management Workbench hyperlinked into the newly created Submittal, you will see a number of other tabs which relate.

  1. The “Copies To”, “Documents” and “Linked” tabs work exactly as described in the previous sections above.
  2. The Revisions Tab is unique to Submittals. There can be a number of versions for any given submittal. These are controlled via the Revisions tab.

  1. To create a new submittal revision, use the Add icon in the revisions tab.
  1. Select the new revision status from the selection box, and if required add the new submitted date or the new submittal received date.

  1. When the revision is saved you will see the “Current Revision” field on the details tab has been updated

  1. Submittals can also be grouped into packages. This allows you to group a number of submittals and submit them in one package. Approval of the package by the client will lead to approval of all submittals included in the package.
  2. To group submittals into a package, first you need to create the package. You can do this from the “submittal packages” tab

  1. Use the Add button to create a new Submittal Package

  1. Once you have created the submittal package, hyperlink in the record and go to the Submittals tab. You can see all submittals in the system when the filter reads “All”
  2. To allocate the submittal to the package, highlight the submittal and use the “add to package” action.

Submittal Actions

  1. The printing and emailing of the Submittals is almost identical to those steps laid out above. Please see those previous sections for reference.
  2. All actions work in the same way regardless of if they are run from this Actions list, or from the Options Menus on the individual tabs.
  1. Send Transmittal - allows you to send a transmittal created from the submittal directly. Applying the “Send Transmittal” action immediately opens an “Add new Transmittal” page, as previously described. The transmittal information is initially taken from the submittal but can be overridden if required

Issues

 

  1. The Issues Tab of the Communication Management Workbench is used to record any potential issues that may disrupt the project.
  2. To add a new issue, simply click the Add button

  1. Fill out the appropriate fields:
  1. The “Subject” field is used to record the issue
  2. The “Responsible Party” field is used to identify who is responsible for responding to the issue
  3. The “Status”, “Importance”, “Impact” and “Type” fields are all used to help categorize the issue. The options that you chose are selected from pre-defined lists.
  4. The “Open Date” field is the date that the issue was created
  5. The “Closed Date” field is the date that the issue was closed

  1. Once you have saved the Issue, and via the Communications Management Workbench hyperlinked into the newly created record, you will see a number of other tabs which relate.

  1. The “Log”, “Copies To”, “Documents” and “Linked” tabs work exactly as described in the previous sections above.

Issues Actions

  1. The emailing of Issue Documents, creating a Task, and Sending a Transmittal is almost identical to those steps laid out above. Please see those previous sections for reference.
  2. All actions work in the same way regardless of if they are run from this Actions list, or from the Options Menus on the individual tabs.
  1. Create Meeting - you can create a meeting as a response to the issue that has been raised. More details on the creation of meetings will be given in the meetings section below

 

Meetings

 

  1. The Meetings Tab allows you to track those multiple project meetings and who attended them, create follow-on meetings and to ensure that work gets done by allocating actions from those meetings to people. Meetings can be created from some of the other communication items, and can also be created from scratch.
  2. To create a new meeting, use the Add button on the Meetings Tab.

  1. Fill out the appropriate details:
  1. The type of meeting is selected from a list of options.
  2. The subject of the meeting, the date of the meeting and the start and end times must be entered.
  3. The frequency of the meeting should be selected from the drop-down list.
  4. Click save when finished.

  1. Once you have saved the Meeting, and via the Communications Management Workbench hyperlinked into the newly created record, you will see a number of other tabs which relate.

  1. The “Copies To”, “Documents” and “Linked” tabs work exactly as described in the previous sections above.
  2. Attendees Tab is used to record who was invited to the meeting
  1. To add an invitee, use the Add button at the bottom of the page.
  2. Either select the invitee using the lookup or type their name directly into the “name” field.
  3. You can record if they actually turned up by using the “attended” checkbox.
  4. Click Save when done

  1. The Meeting Items Tab is used to record to-do items from the meeting, and who was allocated those items.
  1. To create a new action item use the Add button at the bottom of the page.
  2. An action item will create a task against the person who has the action item allocated to them. This can then be tracked.
  3. A single action item will create an action line against the person who has had the action item allocated to them for this meeting only. No task will be created, but the action item will be included in the minutes of the meeting.
  4. A recurring action item will create an action line against the person who has had the action item allocated to them. No task will be created, but the action item will be included in the minutes of every meeting that is created as a follow-on to the original meeting.

 

  1. The item numbering will be done automatically.
  2. The subject that you enter will become the subject of the task that is created.
  3. The “Closed Date” field is the date that the action item was closed
  4. The action item needs to be assigned to a person. You can select this person via a lookup, or type in the person’s name directly into the field.

  1. When the action item is saved then a task is created in that person’s activity workbench. This can be used to track progress on completing the task.

Meeting Actions

  1. The emailing of Meeting Documents, and the Printing of Meeting Minutes is almost identical to those steps laid out above. Please see those previous sections for reference.
  2. All actions work in the same way regardless of if they are run from this Actions list, or from the Options Menus on the individual tabs.
  1. Create Next Meeting - Once you have created an item there is an option to create a follow-on meeting
  1. Our example meeting was setup as monthly, and so the Next Meeting was auto-created for the same date in the next month, using the same info from the first meeting.

  1. Hyperlinking into this newly saved meeting shows that the Recurring Action from the previous meeting has been replicated here.

Daily Reports

 

  1. Daily reports are used to record & document the day-to-day events that happen on site. When the entry of these reports is important to smooth business operations, and their submittal is mandatory, reports can be run to identify which jobs have not submitted Daily Reports. These reports can be scheduled to run automatically, and can be automatically emailed to senior staff highlighting the lack of a daily report.
  2. To create a new daily report use the Add button on the “Daily Reports” tab.

  1. Fill out the appropriate details:
  1. Location: A job may have more than one location and each location may be required to submit a Daily Report.
  2. The weather & temperature fields can just be approximate.
  3. The fields in the “details” tab are controlled by the way that the system has been set up, and so the screen you see may have different fields to the screen below.
  4. Click save when all data has been entered.

  1. Once you have saved the Daily Report, and via the Communications Management Workbench hyperlinked into the newly created record, you will see a number of other tabs which relate.

  1. The “Copies To”, “Documents” and “Linked” tabs work exactly as described in the previous sections above.
  2. The “Labor” tab allows you to record approximate labor usage for the day
  1. Use the Add button at the bottom of the page to create a new entry.
  2. Choose the “trade” from the pre-defined list.
  3. Enter “Qty”, the number of tradesmen, & the Hours they worked save the record.
  4. Note – this information is currently just stored as memo data and is not used to populate any other tables in COINS such as timesheets or plant timesheets.

  1. The Subcontract, Equipment, and Material information can be added to this Daily Report in the same way

Daily Report Actions

  1. The emailing of Daily Report Documents, the Printing of Daily Reports, the sending of a Transmittal and the Creation of an Issue are almost identical to those steps laid out above. Please see those previous sections for reference.

  1. There may be communications or notes not covered by any of the above communication types. You can record these by adding a journal via the Communications Tab

  1. To add a journal, first select a journal type from the selection box and then use the Add icon on the Journals screen, found under the Communications Tab of the Communication Management Workbench.
  2. The list of journal types is controlled by your company, and so if you want to use this feature to record an activity that is not listed then talk to your system administrator.

  1. Here we’re recording the receipt of a document.

  1. Once you have saved the Journal, and via the Communications Management Workbench hyperlinked into the newly created Journal, you will see a number of other tabs which relate.

 

  1. The “Documents” and “Linked” tabs work exactly as described in the previous sections above.
  2. You can use the “Recipients” tab to set up and maintain a list of recipients for the journal entry.

Task Lists

  1. Task lists are used to plan future activities, and to track progress as those activities are completed.  Each task needs to be assigned to a person – that way you know who is responsible for what and can easily check to see how they are getting on with their lists.
  2. Tasks can be auto-created from a variety of different places within Project Management, but can also be created from scratch. However before you create a task you need to set up at least one task list. You can have as many task lists as required, and they help to subdivide the tasks.
  3. On the Lists Tab, click on the Task List option on the drop down menu, and click Add.

  1. Fill out the details:
  1. List Code: Unique code for this list
  2. Prefix: If you enter a prefix in this column, it overrides the task number prefix from the PT/TSKCODE parameter.
  3. Recurring: whether this is a recurring task list
  4. Interval: If you checked the Recurring field, you are prompted to select how often the task list should recur, such as Day, Week, Two Weeks, etc.
  5. Template: if many lists will have similar tasks, Task List Templates can be used
  6. Start Date, End Date, Next Due Date
  7. Assigned: the contact person receiving the list
  8. Click Save when all details have been entered

  1. To add a task to a task list or to check on the current status of tasks go to the Tasks Workbench, under the “lists” tab:

  1. Please make note of the “Task List” selector at the bottom of the screen. Make sure that this is filtered to the correct task list or to All Task Lists. If you have created a task and cannot see it then double-check that you’re looking at the right task list as sometimes this is hidden below the bottom of the screen.
  2. Use the Add button to create a new task and fill out the appropriate details
  1. First choose the correct task list on which this task should appear.
  2. Then add details of your task. You can create tasks for other users as well as for yourself.
  3. Add a due date when the task is due – this will be used to check if the task is overdue.
  4. The “Status”, “Importance” and “Impact” fields are all used to help categorize the task that is created from the action item. The options that you chose are selected from pre-defined lists.
  5. Click Save when complete.

  1. Once you save the task, a task is also created on your/the user’s activity workbench, which can be accessed via the icon at the top of any page.

 

  1. You can complete the task here via the Activity Workbench.

Other Tabs and Reports

  1. Once you have entered all of this data into the system there are a number of different ways to track progress; the simplest and most obvious is via the Open Items Tab.
  1. This tab shows you all items that have been created against a job but have not yet been completed. This includes action items, issues, RFIs, submittals, tasks & transmittals – basically anything with an “allocated to” and “due date” field.
  2. You can filter this list by use of the filters at the bottom of the screen – you can limit views to only those items allocated to you, to only overdue items and can also filter by item type.
  3. Clicking on the underlined hyperlinks will drill down into the item.

  1. You can also use the Project Dashboard to get a graphical overview of current project status. The colors denote the age of the different items. The items displayed on this chart can be setup with Dashboard Configuration.

  1. If you click on one of the bars, you can drill down to see the information broken up into more detail
  1. Below you can see open RFI’s, by level of impact. A mouse hover shows the percentages of the different impact levels.

 

  1. Finally, there is a small number of reports that can be run for some of the various communication types. They show the details of each of the communication types, with the question and answer, and list any items linked to them.
