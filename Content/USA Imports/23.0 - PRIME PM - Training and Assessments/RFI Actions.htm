RFI Actions

  1. There are many actions that you can apply to this newly created RFI, via the main communications screen, once an RFI has been selected.

  1. All actions work in the same way regardless of if they are run from this Actions list, or from the Options Menus on the individual tabs
  1. Create Change Request from RFI – this action automatically creates a PCO change request

 

  1. Print RFI - allows you to print and/or email a formatted copy of the RFI in PDF format. The layout of the printed RFI is completely configurable, so the example that you see may not look like the example in this guide.

 

  1. When you select to print an RFI you are taken to the screen below, which will look familiar to any COINS OA user, as it is based on the main printing Output Options tab.

 

  1. Choose the form that you wish to use at the top of the screen – there may be just one form available.
  2. If you want to email the RFI then select the “email” checkbox. If not, then uncheck this box, and the document will get added to the report queue in the usual way.
  3. If you do want to email the form then you can add text into the “message” field. It is possible to build templates that pre-populate text for an emailed RFI. If such a template exists, select the HTML checkbox.
  4. Using the Email Attachments tab, you can add any of the previously attached project documents to this email.
  5. Once you are happy with your selection options use the Next button to launch the print and/or email.

  1. Email RFI documents – performs the same emailing functions of the previous action, without printing
  1. You can also choose to email any documents that are attached to the project, by using the Attachments Tab
