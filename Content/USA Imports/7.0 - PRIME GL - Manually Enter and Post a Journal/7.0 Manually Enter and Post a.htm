Manually Enter and Post a Journal

Module: General Ledger

In , all journals are grouped together in batches. So even if only making one journal, a batch needs to be made first. This document demonstrates how perform this process individually/manually, as well as through the Bulk Load function. The final section of this document will explain the steps of how to post a journal.

  1. To enter a journal batch, use the Menu Search bar to find “GL Journal Batches” or go to General Ledger > Journals > Journal Batch in the tree menu (Figure 1).
  2. Click at the bottom of the screen (Figure 1).Figure 1: GL Journal Batches

  3. Next, fill in the Batch information (Figure 2):
    • The is the period to which all the transactions in this batch will be posted. automatically fills in the current period and financial date for the module being worked in. If other periods are open, an alternative can be selected from the drop-down list.
    • uses the Transaction Date entered in the batch header as the default transaction date for all the transactions, but a different transaction date can be chosen for individual transactions.
    • If desired to base transactions in the batch on a template, select the Template.
    • The Transaction Type is a code that identifies different categories of transaction. These codes could vary from one company to another. Transaction types also determine whether a transaction will reverse in the subsequent period. JNL (non-reversing) and ACC (reversing) are the two most commonly used transaction types. All transactions in a batch will use the same transaction type, so reversing and non-reversing transaction types cannot be mixed in the same batch. See General Ledger > Configuration > Transaction Types for details.
    • Identify the name of the batch in the Description line.
  4. Then, click in the bottom, right corner of the screen (Figure 2).Figure 2: Enter in Batch Information, Click Save

    This will open the Transactions tab. The Batch number is auto-populated, and the description from step #3 becomes the title (Figure 3)

  5. Click to enter a new transaction (Figure 3).Figure 3: Click “Add” in Transactions Tab

  6. In the first drop-down selection, choose the type of account to enter the journal in. This example will use the General Ledger (Figure 4).
  7. The Analysis is the financial analysis where the value of the transaction is recorded. In this example, it’s the GL account being debited, but this could also be a number or asset number, depending on the type of transaction. (Figure 4).
  8. Enter the debit amount in Value (Figure 4).
  9. The Description will auto-populate from the batch description, but it can be modified if needed (Figure 4).
  10. Enter the Quantity for the transaction (Figure 4).Figure 4: Fill in Transaction Information

  11. Click to save the line (Figure 5).Figure 5: Click Save

  12. Using steps 5-11, add the matching opposite credit entry so the batch balances.
  13. Then, click to add a “totalling” line to the bottom of the screen. It will list the number of transactions and shows the totals of debits and credits. The number in the middle of the bottom blue line needs to be 0.00 for the journal to be posted (Figure 6).Figure 6: Sigma “Totaling” Line

Adding Journal Entries via Bulk Load

  1. These transactions can also be added through the Bulk Load function.
  2. Click , then on the bottom taskbar (Figure 7).Figure 7: Bulk Load

  3. The Bulk function will download an editable Excel spreadsheet. Open the file.

    In the spreadsheet, lines can be added easily. This example will demonstrate costing to a (Figure 8).

    Figure 8: Editable Excel Spreadsheet

  4. Copy (select all) the Excel spreadsheet and paste into the Bulk Window (Figure 9).
  5. Then, click Save (Figure 9).Figure 9: Paste into Bulk Window and Click Save

  6. The updated Journal Batch is shown in Figure 10. Continue to make sure that the total of credits and debits is 0.00 using the button.Figure 10: Updated Journal Batch

Posting the Journal

  1. When all necessary transaction lines are added, and the batch balances, click in the bottom, right corner of the screen (Figure 11).Figure 11: Click Next

    At the main Journal Batch screen, the batch will be listed as Unposted (Figure 12).

  2. Click anywhere on the row/line of the batch to select the line. It will highlight in light blue (Figure 12).Figure 12: Click to Select Batch

  3. Using the action bar at the bottom of the screen, select Post with Report from the drop-down and click (Figure 13).Figure 13: Apply Action “Post with Report”

    A pop-up notification will appear at the bottom, right of the screen, and should read “Task submitted successfully” (Figure 14).

    Figure 14: Pop-Up Notification

  4. Hover over the icon on the top bar to see Reports (Figure 15).
  5. The most recent reports will be at the top of the list. Click the Posting Report (Figure 15).Figure 15: Click the Posting Report

    The Posting Report should now be able to be viewed (Figure 16).

    Figure 16: Posting Report
