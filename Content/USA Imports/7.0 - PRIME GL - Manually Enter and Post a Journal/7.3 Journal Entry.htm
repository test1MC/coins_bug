Journal Entry

Module: General Ledger (11.06) [Journals]

  1. Go to General Ledger > Journals > Journal Batch.Figure 1: GL Journal Batches

  2. Click to create a batch.Figure 2: GL Journal Batches - Add

  3. Fill out the Batch information:
    • The is the period to which all the transactions in this batch will be posted. COINS automatically fills in the current period and financial date for the module in which you are working. If other periods are open, you can enter an alternative period.
    • COINS uses the Transaction Date you enter in the batch header as the default transaction date for all the transactions, but you can choose a different transaction date for individual transactions.
    • The Transaction Type is a code that allows you to identify different categories of transaction. These codes could vary from one company to another. Transaction types also determine whether a transaction will reverse in the subsequent period. JNL (non-reversing) and ACC (reversing) are the two commonly used transaction types. All transactions in a batch will use the same transaction type, so you cannot mix reversing and non-reversing in the same batch. See your Configuration > Transaction Type for details.

  7. The Analysis is the financial analysis where you want to record the value of the transaction; in this example, it is a GL account, but this could be a number or asset number, dependent on the type of the transaction.
  8. Enter the debit amount in Value. Debit values are positive and credit values are negative
  9. The Description will auto-populate from the batch description, but it can be modified if needed.
  10. Add the matching opposite credit entry so the batch balances. Here you can see the balanced transactions.
  11. The button adds a total line to the bottom of the screen. It will list the number of transactions and show totals of debits and credits. It will show a 0 if the batch is in balance.Figure 5: Transaction - Add
