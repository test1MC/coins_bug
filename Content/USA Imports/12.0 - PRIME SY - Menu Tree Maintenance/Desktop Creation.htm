Desktop Creation

Module: System

This document demonstrates the process of creating desktops in COINS (steps #1-42), and editing them later (steps #43-52). Desktops can be used as a homepage for when users log in and as an alternative to using the side/tree menu. There should also be pre-made, standard desktops available, seen in Figure 1, which can be found in System > System Setup > Function Maintenance.

Figure 1: Standard Desktops

 

This example will assume that the user is creating a desktop for his- or herself.

 

  1. Go to System > User Maintenance > Users (Figure 2).
  1. Select the user that is creating this desktop, and click Open (Figure 2).

Figure 2: Open User to Add Desktop For

  1. In the Preferences Tab, change Current Desktop to the name of the desktop you’d like to make, “MYDESKTOP” for this example (Figure 3).
  1. Optionally, in the Alternative Desktops field, list the same desktop name for the user to have access to (Figure 3). There can be multiple/many desktops (separated by commas with no spaces in between) listed in Alternative Desktops, so that the user can switch between desktops using a drop-down at the top of the screen. If the user will only have one desktop, this field may be left blank.
  1. Click the Save in the bottom, right corner of the page (Figure 3).

Figure 3: Set Up Desktops in Preferences Tab

 

  1. Click back into the user by clicking on the hyperlink (Figure 4).

Figure 4: Click User ID Hyperlink

  1. Then, in the Preferences tab, click Desktop Wizard (Figure 5).

Figure 5: Click Desktop Wizard in Preferences Tab

 

  1. The Desktop Wizard gives the option to copy a user’s desktop, but this feature should be ignored (Figure 6).
  1. Click Next in the bottom, right corner of the page (Figure 6).

Figure 6: Click Next

 

  1. At the next screen, choose desktop tile defaults. These fields are all adjustable for each tile, but if most the desktop tiles will be a certain size or color, they can be set on this page (Figure 7). These fields will be explained in more detail in steps #23-25.
  1. Also, add the names of the Tabs and Sections. This example will use very basic names and numbers to demonstrate where these names will end up on the finished desktop (Figure 7).
  1. Then, click Next in the bottom, right corner of the page (Figure 7).

Figure 7: Set Up Tile Defaults, Tab/Section Names, Click Next

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

  1. The next page is the Desktop Wizard History, which shows what functions the user has been on recently and how many times. This page can be helpful in determining which functions should go into a desktop, and quickly adding them. Optionally, click the Open folder to open an individual function line or Concurrent to add multiple functions to the desktop (Figure 8).

Figure 8: Desktop Wizard History

 

  1. This step will assume that Concurrent was clicked in the previous step. Use the drop-downs under Tab/Section to select where the appropriate functions will be in the desktop. Size, Open In, and the Foreground/Background colors for each tile can be specified (Figure 9).
  1. Then, click Save in the bottom, right corner of the page (Figure 9).

Figure 9: Add Functions as Tiles, Click Save

 

  1. Then, click Next in the bottom, right corner of the page (Figure 10).

Figure 10: Click Next

 

  1. The user will then be brought to the new desktop screen, which will show the tabs and sections that were set up in step #11 (Figure 11). These can be renamed later.

Figure 11: New Desktop Screen

 

  1. Tab #3 only has one section. In that case, the section will be unlabelled, even though it was specifically labelled in step #11 (Figure 12).

Figure 12: Unlabeled Section in Tab #3

  1. Next, the user needs to add tiles to this empty desktop (Figure 13).
  1. Go a function that will be added to the desktop. Click the arrow in the upper, left corner of the screen to expand the tree menu and go to a function. Alternately, click the magnifying glass (Global Search) to search for any function. Job WIP Adjustments in the Job Status module will be demonstrated for this example. (Figure 13).

Figure 13: Navigate to a Function to Add to Desktop

 

  1. Once inside the function, use the mouse to hover over History. To get to the History icon, use the dropdown arrow located next to the new user silhouette. This will show the last 10 functions that were visited (Figure 14).
  1. Click on the Add to Desktop icon on any of the functions on the list (Figure 14).

Figure 14: Click Add to Desktop Icon

  1. This brings up the Add to Desktop window. Size, Open In, and colors (Foreground and Background) will auto-populate based on the choices made in step #10, but can be changed (Figure 15). Every tile can be sized and colored as desired.

Figure 15: Add to Desktop Screen

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

  1. There are a variety of options for the Size of the desktop tile, which are compared in Figure 16:
  • Big
  • Full
  • Half
  • Normal
  • Small
  • Tall
  • Wide

Figure 16: Desktop Tile Size Options

 

 

 

 

 

 

 

 

 

 

 

 

  1. Open In determines how the function will open when clicked on in the desktop. The available options are:
  • Main Area
  • The Desktop will be exited and the user will be directed to the function, when clicked on. It will take up the entire screen.
  • Window
  • Function will open in a new tab.
  • Frame
  • Function will open in a frame, like a pop-up window.
  • Frame with Banner
  • Function will open in a frame, like a pop-up window. Will also include a banner at the top of the frame that states the company and function.
  • Inline
  • Function will be open within the desktop tile. When selecting this option, consider the Size of the tile. Figure 17 shows a Full-sized Inline desktop tile:

Figure 17: Inline Desktop Tile, Full Size

 

 

 

 

 

 

  1. Use the Add To drop down menu to choose which Tab/Section this tile belongs (Figure 18).
  1. The Subtitle field will remain greyed-out unless Override is selected. If it is, a custom Subtitle can be added to the tile (Figure 18). Otherwise, the tile will use the module name (Job WIP Adjustments, for this example) as the Subtitle.
  1. The Icon will also auto-populate based on the function, but it can be changed if desired (Figure 18).
  1. If Values are selected (Figure 18), they will appear on the desktop tile as live data.
  1. Then, click Next to save the tile (Figure 18).

Figure 18: Enter Desktop Tile Information, Click Next

 

 

 

 

 

 

 

 

 

 

 

 

 

  1. Next, either move on to the next tile, or to see the changes, use the keyboard and press F5 to refresh. If the refresh doesn’t bring up the desktop, click Show/Hide Desktop icon (Figure 19).
  1. Go to the appropriate tab and expand the appropriate section(s) to see the tiles added to the desktop. For this example, the tile was added to Tab #1 - Section #2, with the selected colors, with the selected title, overridden subtitle, selected values, and icon (Figure 19).

Figure 19: Tab #1 – Section #2 in Desktop

 

  1. The next step is to repeat steps #20-30 with the remaining appropriate/desired functions.
  1. In this example below, we’ve navigated to General Ledger > Configuration > Transaction Types.  You can use the Magnifying Glass, Menu Search, or tree menu to search for any additional functions to add to the desktop (Figure 20).
  1. Once again, look for the function in History, and click the Add to Desktop icon (Figure 20).

Figure 20: Add Transaction Types to Desktop

  1. For the Transaction Types tile, we’ll choose Tab #2 - Section #2, and for this example, we’ll let the Title, Subtitle, and Icon auto-populate (Figure 21).
  1. Click Next (Figure 21).

Figure 21: Set Up Transaction Types Desktop Tile, Click Next

 

  1. Then, press F5 on the keyboard to refresh.
  1. See the results for this tile in Figure 22 (below):

Figure 22: Tab #2 – Section #2 in Desktop

  1. Once all the tiles have been added, and the desktop is complete, go back to System > User Maintenance > Users, and open the user file of whomever else is to get the new desktop, and put the exact name of the newly created desktop in their Current Desktop field, and optionally, the Alternative Desktops field.
  1. The desktop can be added to as many users as desired.
  1. Refer to steps #1-5 of this document.

 

  1. To edit this desktop in the future, the user needs to have it as their current desktop selected. All users with general desktop editing privileges will be able to edit it, if they have the desktop assigned to them.
  1. Hover over the user in the upper, right corner of the screen, click User Preferences, and go to the Desktop tab (Figure 23).

Figure 23: Desktop Tab in User Preferences

 

  1. Click Open in the bottom, right corner or click the appropriate Tab or Section hyperlink (Figure 24).

Figure 24: Click Open or Tab/Section Hyperlink

  1. For this example, we clicked on Section #1 under Tab #4. This will bring up the list of functions that make up each tile (Figure 25).
  1. Select the appropriate rows to reorder and use the Up and Down buttons to re-arrange the order of the desktop tiles (Figure 25).
  1. Optionally, add a blank row that can be used as a space to separate rows within a section (Figure 25). Click Add, leave the line entirely blank, and click the floppy disk to Save.
  1. Then, click Next in the bottom, right corner (Figure 25).
  1. Changes made to the desktop will apply to all users.

Figure 25: Edit Desktop Tiles in a Section, Click Next

 

  1. Click F5 to refresh.
  1. In the desktop section, the space created a new row within the section (Figure 26).

Figure 26: Blank Line Created New Row Within Desktop Section
