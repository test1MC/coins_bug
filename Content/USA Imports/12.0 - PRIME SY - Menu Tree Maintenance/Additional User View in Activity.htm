Additional User View in Activity Workbench (Workflow)

Module: OA Reporting & BI/System

This document demonstrates how to give a Workflow user “View access” and/or “Update access” to another Workflow User’s Activity Workbench.

 

  1. Using the Menu Search, enter “Activity Workbench Access Maintenance” (Figure 1).

Figure 1: Menu Search for “Activity Workbench Access Maintenance”

 

  1. Alternately, use the tree menu to go to OA Reporting & BI > Workflow > Activity Workbench Access Maintenance (Figure 2).
  1. Click the Open folder in the row of the user to which the access is being given (Figure 2).

Figure 2: Open Appropriate User to Give Access To

  1. Depending on the circumstances, under the View column and/or the Update column, use the field lookup (…) to find the user whose Activities will be seen/updated by the user in step #3.
  • In this example, “marulr” will be the Viewer (Figure 3):

Figure 3: Select the Viewer

 

  • And “jaksch” will be the selected Viewee. Click the pointing-finger (Figure 4):

Figure 4: Select the Viewee

 

  1. Once the Viewee, and/or Updatee has been selected, click the floppy disk to Save the line (Figure 5).

Figure 5: Save the Line

  1. Hover over the username in the upper, right corner of the screen and click Activity Workbench (Figure 6).

Figure 6: Hover Over Username, Select Activity Workbench

 

  1. In the Another User View tab, there will appear a hyperlink of the User ID whose view has been given. Click the hyperlink to change to this user’s Activity Workbench (Figure 7).

Figure 7: Another User View Tab, Click User ID Hyperlink

 

  1. The Viewee’s name “jaksch” is now visible at the top of the Activity Workbench (Figure 8).
  1. Their Appointments, Tasks, and Actions are now visible (Figure 8).

Figure 8: Viewee’s Appointments Tab
