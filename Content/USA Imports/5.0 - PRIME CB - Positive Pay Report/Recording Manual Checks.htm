Recording Manual Checks

Module: (11.07) [Cash Transaction Entry]

  1. Click on the Module > Cash Transactions > Cash Transaction Entry.Cash Transaction Entry

  1. Click .Cash Transaction Entry

  1. Enter all the required details
    • : Period in which transaction will be posted.
    • Transaction Date: Date the transaction is being entered
    • Effective Date: Date the transaction actually occurred
    • Manual/Print:  Manual
    • Payment/Receipt: Payment
    • Allocate: or un box to Allocate batch
    • Inter-Company: box for Inter-Company Cash Transaction
    • Bank Code: Select Bank Account that the Cash will be transferred From
    • Description: Brief Description of purpose of the transaction
  2. Click .Cash Transaction Entry – Update Batch

  3. Click .Cash Transaction Entry – Update Batch

  4. Enter all the required details
    • Reference:  A reference to identify the transaction; for example, the deposit reference.
    • Number:  A Number must be entered. It is recommended to use a common default number that is not part of the actual Number sequence.
    • Type: Payment
    • Date: The date the Transfer is being made
    • Transaction Date: The transaction date of the Transfer that will be used when matching to the Bank Statement
    • T/Type: PAY-Payment
    • Ledgers:

        G Used to distribute check to Gl account (Ie. Bank Fee)

        C used to distribute to .costcode.costcategory

        used to distribute to invoice

        used to distribute to an / Service invoice

        SC used to distribute to a SC Invoice

        CS used to distribute to a invoice

        FA used to distribute to a Fixed Asset

        CC used to distribute Unallocated Customer Cash

        BK Used for Bank Transfers 

    • Account: Account that will be debited
    • Payer/Payee: Account that the Transfer is being withdrawn from
    • Narrative: Description of the Transaction/Transfer
    • Currency: USD-US Dollar
    • Amount: Amount of Transfer
  5. Click .
  6. The CB Transaction Entry – Details opens.Cash Transaction Entry – Details

  7. Click .Analysis Lines

  8. Enter all the required details
    • Ledger: The Ledger to which this analysis is recorded
    • Analysis: The analysis of the payment or receipt
    • Ref: If you have allocated this amount to a transaction in another ledger (for example, an invoice), this shows the reference for that transaction
    • Sales/Use Tax Code: The tax code for this transaction
    • Amount: The amount assigned to this analysis
    • Sales Tax: Sales tax component of the distribution value
  9. Click . Details

  10. Click .CB Transaction Entry – Update Batch

  11. Click .CB Transaction Entry – Batches

  12. Select the batch and click on the Post apply action, to post the batch.Posting the Batch

  13. The batch gets posted.Posted Batch

  14. When the batch is posted, three reports are generated and can be viewed from the Report Status.Posted Batch

  15. To open the reports click on the hyperlink of the report in Report Status.Posted Batch
