Creating a Quick with no Corresponding Invoice

Module:

  1. Start at > Cash Transactions > Cash Transaction Entry.Cash Transaction Entry

  2. Click to create a new batch.
  3. Choose Print and Payment where highlighted to bypass creating a in COINSAdding Cash Transaction Batch

  4. Within the batch, add a transaction. Add Batch Transactions

  5. Enter the transaction.
    • Use as the ledger, and the appropriate account number which will be the analysis for the transaction
    Add Transaction Details

  6. Click when complete.
  7. Select to create the analysis line.
    1. For ledger, select the code for analysis, commonly, G for GL analysis
    2. Analysis is where the cost is going, in this example a GL account
    Transaction Analysis lines

  8. Click when complete.
  9. Select the Batch hyperlink. Batch Details Hyperlink

  10. Click on the transactions tab; you should see a status of unprinted. Unprinted Status

  11. On the tree menu, select Print /Remittances. Print /Remittances

  12. Select the hyperlink on the batch. Print /Remittances Batch

  13. Select the Selection tab, verify information is correct.
  14. Make sure you select a form and select ; a report will print: Print /Remittances Selection Screen

  15. Go back to the Cash Transaction Entry, the box for your batch and run the Post with Report action.

    You will receive an error here if you forget to print /remittances.
