Creating Service Order Board in OA

Module: Service Management

Once a Service Location and Service Location customer have been created a Service Order can be made for this customer. This step is the creation of a Service Order which will then be able to be assigned and dispatched.

  1. To create a Service Order form inside the OA environment follow the tree menu to Service Management > Service Orders

Figure 1: Service Order Main Screen

 

Click the Green “Add” button at the bottom of the screen to start the Service Order process.

  1. This brings up the main Service Order screen pictured. Going section by section try to fill out as much information as possible, in the first section it is mandatory to enter in the Location Name and Customer Name.

Figure 2: Creating a New Service Order 

 

  1. In the below imagine only Location Name and Customer Name were chosen so far, but because of the setup that was done in previous steps (Creating Service Locations, Create Location Customer, and Add Location Customer Type of Work). Most mandatory sections have been filled in because of this.

Figure 3: Beginning to Create Service Order

 

 

  1. After filling in Location Name and Customer Name everything else here was auto filled in. Now just type in who called to order the SO, who is the contact person at this location and who the technician should go to see.

Figure 4: Subsection 1 of Service Order

 

  1. Agreement autofill’s if this service order is covered under an existing agreement.

Allowable work has been set up before in the “Add Location Customer Type of Work How To Document” so select the appropriate code.

Check to make sure Service Department, Service Order Group and Service Order Type have all pulled over correctly.

Enter the Response time for the Service Order examples of this will be NRML (Normal) or PM (Planned Maintenance 45 Days).

If there is a Preferred Technician for this customer enter them in.

Primary Technician fill this in if multiple Technicians are going to be on the call.

The Coverage zone is in reference to the technician zone coverage.

Figure 5: Subsection 2 of Service Order

 

  1. The called automatically fills in when the Service Order is started as does “effective”. Make sure “Show in S&D” is checked so that we can see this on the Service and Dispatch Board in the next step for creating a Service Order.

Promised date is for if the customer was told the job would be done by a certain time.

Figure 6: Subsection 3 of Service Order

 

  1. If this order requires a PO enter it in to the Customer PO section.

PO Authorization Limit helps to ensure that the job is not overspent on.

Due Type is determining whether the payment will be due after a certain amount of days or months.

Due days is filled in if the payment is due a certain amount of days after the service is done.

Check On-Site Payment if the Technician will be collecting payment when performing the service.

Customer Reference is what will be on the customers invoice.

Priority sets how quickly the service order needs to be completed.

Source of Lead how did this service order came to be.

Related Service Order is this SO related to another service order.

Figure 7: Subsection 4 of Service Order

 

  1. TRADE is mandatory in this section. This is an example of a service order analysis set that we can crate for reporting purposes. This may look different in your environment. Examples in this section are Plumbing and HVAC.

Sales Rep can be entered if applicable.

Quoted Price is if the customer was told a price on the phone.

Not to Exceed is if there is a price that cannot be charged over.

Estimated hours is how long the service call is expected to take this is highly advised to fill out.

Retention % is any amount of money being held and billed at a later date.

Figure 8: Subsection 5 of Service Order

 

  1. Everything but Asset is mandatory in this section yet that should still be filled in. Problem code should be chosen first and this choses why the service order is happening, examples are NH No Heating or NOISY. This will fill in for the mobile technician visit.

Problem Description is where any information is know we give the Technician as much information about what is wrong as possible.

Service Order Description copies what is written in Problem Description.

Figure 9: Subsection 6 of Service Order

 

  1. When finished click the SAVE button in the bottom right section of the screen.

Figure 10: Save Button
