Security Overview

Module: System

There are many different types of security in the COINS operating system. This document will look to list out these types of security and give examples on how they work.

  1. Function level security. Function Level security can be split into two types, Users and Group. Group level security is used to give multiple users access to the same functions in the COINS system. Groups can be added to users and all the access they grant will then be transferred to the user. This allows for faster maintenance to security settings when new functionality is released, or new workflows established. Users can be used to change one person’s security at a time, this allows you to be more granular. When updating group level security best practices are to set the function to either Yes (which gives access) or Group (which does not give or deny access). When the COINS system looks through security, the user starts without access to anything, for every “Yes” they gain access and for every “Group” they stay at their current access (which is no access).
  • Group level security is set up in System > User Maintenance > Groups

  1. Batch level security. Batch level security sets up which batches created in modules you can  work on. The highest level is 9, while the lowest is a 1. If someone has a security level of 9 they can access any batch in the system. If someone has a security level of 1 they can only access batches by other level 1 security users. A user at security level 2 could access all level 1’s and level 2’s but not level’s 3’s or higher.
  • Batch security can be found in System > User Maintenance > Users

  1. Record Level Security. When you have security access to modules there are certain workbenches that may need you to have additional security found inside the module. For example even if you have access to every function in Service Management, if you do not have Service Order security found in the module you will not be able to access service orders.
