Setup Payments and

Module: Payroll

Steps to setting up the Payments and .

  1. Add a Payroll Payor Bank(s) to BANK ACCOUNTS. Go to > Setup Maintenance > Bank Accounts. Click to add the Bank Account(s) that are for payroll. Pay attention that the Routing Number and the Account are included and accurate.Add a Payroll Bank

  2. After saving the line, to edit the bank details, click on the bank record. Do not drill on the hyperlink.
    • If Stock does not include MICR line, then box Print MICR Type ". 
    • If a balancing entry is required for the File, then "Create Balancing Debit EFT Entry".
    Enter Bank Details

  3. Update the Payroll Global Parameters. Go to Payroll > Global Setup > Parameters. Make all parameters Company Specific before updating. To do this, tag the line(s) and use the action bar to Create Company Specific.
    • BNKONCHK -   enter a comma-separated list of bank codes to use for Payroll.   This list will appear in Print and in Employee Maintenance.
    • DEFCKBNK – enter the bank code from the BNKONCHK list that will appear as the Default Value when adding a new employee.
    • CASHBANK – enter the bank code that will be used for all Payments for the company.
    Update Global Parameters

  4. Update FREQUENCY SPECIFIC PAYROLL PARAMETERS. Go to Payroll > Weekly Payroll > Setup > Parameters.
    • BACSCOID1 – enter the value that should appear in position 14-23 of the File-Record Type 1 (first line of file).   If you enter 9 characters instead of 10, the Processing function will add the leading space to the file (beginning Aug 2017). 
    • BACSCOID5 – enter the 10 character value that should appear in position 41-50 of the File-Record Type 5 (second line of file).  
    • BACSFILE – default value of this parameter will be set by COINS and will be something like  “BASE/var/home/[user]/[KCO]-Weekly-[DATE=YYYYMMDD]-[TIME=HHMMSS].txt”.    the ‘Weekly’ text will be replaced for each frequency. Inclusion of the Date/Time stamp will ensure every file name will be unique and not overwritten in error.
    • BACSMED – this value must be 16
    • BACSEXEC – Reserved for COINS.
    Update Frequency-Specific Parameters
