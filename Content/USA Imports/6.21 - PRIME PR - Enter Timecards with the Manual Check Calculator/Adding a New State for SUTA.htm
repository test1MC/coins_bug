Adding a New State for SUTA reporting

Module: Payroll (version 11.07)

When a company (client) begins work in a new state, it is necessary to set up these states in OA to track the tax distributions and SUTA filings for month end reporting. This How To explains the setup involved.

  1. Begin by adding the new state. Go to Payroll > Global Setup > US Specifics > States.Add a State

  2. Next set up the Tax Authority. Go to Payroll > Company Setup > Tax Authority Configuration. Click to input the new state here.  Tax Authority Configuration

  3. COINS will then prompt you to select the Tax Authority Type, in this case we are setting the state. Then set the Taxing Authority to your newly created state, in this example we have New York. Link State to Tax Configuration

  4. Add all relevant information here.Add Details

  5. Go to the Comp Codes Tab. Add the relevant Workers Compensation codes here. Save the line. *NOTE: when using new comp codes see the next step to set them up, then continue here. Add Comp Codes

    • The new State may require new Workers Comp Codes. If this is the case, these need to be created before you can add them into the above step. Go to Payroll > Global Setup > US Specifics > Comp Codes. Add Comp Codes

  6. Next the decision of how the employee is to be taxed is needed. For example – will the employee be taxed in her home state or work state? Should this default into the timecard? This will be setup on the employee file. Go to Payroll > Employee Maintenance. This is done in two places.
    • Go the State Details tab. Enter in the new state the employee will be working in. Enter all relevant details for each state here, depending on the employee’s situation. Add Comp Codes

    • Staying on the Employee File - go to the Employee Details tab > Tax Authority Details tab. Add relevant info for the employee. Decide how the Time Card Default Rules will default the costing. Verify this when running a payroll and check what defaults in.
