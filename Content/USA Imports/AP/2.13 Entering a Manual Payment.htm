Entering a Manual Payment (Non- )

Module:

This is done when a is written outside of the system and the invoice needs to be marked as paid in the system.

  1. Go to > Payments > Manual Payments. Add a new batch. Add a Batch

  2. Verify the and Transaction Date are correct and fill out the optional Description, then click .Enter and Save Batch

  3. At the Payments screen, add a new transaction to the batch. Fill in Supplier Account, Number, Bank, Pay Date, and Amount. The number can be used for any identifying information, like an electronic transfer confirmation number.Add a Transaction

  4. Then Save the line.Save the Transaction

  5. After saving the line, go to the To Allocate tabAllocate the Transaction

  6. If you want to Fully Pay the invoice: select the line and using the action bar click Pay Invoice.Fully Pay Invoice

  7. If you want to Partially Pay an invoice: open the invoice you want to partially pay. Then enter the amount you want to pay in the amount paid field. Then Save the line.Partially Pay Invoice

  8. Check the top of the page to see if you have fully allocated the payment.
    • Note: You can post even if there is an unallocated amount, but some people do not want to allow this. (Anything not allocated to an invoice will go the “unallocated” GL account defined in the module configuration page). Unallocated cash shows on the report and can be allocated to an invoice at a future time using the Apply Unallocated Cash function in .
    Check Payment is Allocated

  9. Once the Invoices have been Selected for Payment, click .Select Invoice for Payment

  10. Select the line and using the action bar, Post the batch to complete the transaction.Post the Batch
