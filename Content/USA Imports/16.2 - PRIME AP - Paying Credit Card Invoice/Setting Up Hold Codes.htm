Setting Up Hold Codes

Module: Accounts Payable

This document will demonstrate how to set up hold codes when processing AP invoices. This useful when workflow is not set up. The system can automatically put the invoice on hold, then taken off hold by using the approval workbenches.

 

    Go to Accounts Payable>Setup Maintenance>Financial Hold Codes Mait.  These are the three hold codes available to be put on invoices.

 

Figure 1: Financial Hold Code Maintenance


 

 

 

 

 

 

    To add a hold code, click on add and enter in a code and a description.

Figure 2: Adding a Hold Code


 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Then save the line.

 

    COINS can default to a hold code when an invoice is entered.   There are three parameters that must be set. DEFCODEF, USEFINH, and DEFFINH. DEFCODEF is where the value of the hold code that should default gets entered. Go to Accounts Payable> Configuration>Parameters.

 

Figure 3: Default Hold Code


 

 

 

 

 

 

 

Search DEFCODEF and open the parameter. Enter the hold code in the value line and save the line. (INVAP in this example) The code that was entered will default when an invoice is entered.

Then search for ESEFINH. Open the line switch the setting to YES and save the line. This will allow the use of financial hold codes.

 

Figure 4: Use Financial Hold Codes


 

 

 

 

 

 

 

 

 

 

Search for the third parameter DEFFINH. Open the line and switch the value to YES. Save the line.

Figure 5: Invoice on Financial Hold Code as Default


 

 

 

 

 

 

 

 

 

  1. Now enter an invoice, for instruction on how to do this refer to “Prime AP Non-PO Related Invoice Entry”

Post the invoice and go to Accounts Payable>Payments>Select/Deselect Invoices(Manual)

Figure 6: INVAP Hold Status


 

 

 

 

 

 

 

 

 

Select the filter at the bottom of the screen to ALL and click on the filter icon. This will show all the invoices. Look at the status of the invoice that was just entered. It will show the Hold Code that was set in step 3. When the line is selected and the Pay Invoices action is selected an error will pop up.

Figure 7: Error Message


 

 

 

 

 

 

 

 

 

  1. To remove the hold, go to Accounts Payable>Invoices>Approve Invoices by Supplier.


Figure 8: Approving Invoices

 

 

 

 

 

 

 

 

 

 

Find the invoice, select it and apply the Approve Invoices action.

**Note** Invoices can also be approved by Job, Entry Batch, and Costing Batch

  1. To see who approved the invoice, go to Accounts Payable>Inquiries>Invoices Inquiry.

 


Figure 9: Invoice Inquiry

 

 

 

 

 

 

 

 

 

Hyperlink on to the Internal Reference number and go to the Hold History tab. This screen shows who put the hold on and who approved the invoice (took the hold off)

Figure 10: Hold History


 

 

 

 

**Note** if nothing appears under the Hold History tab, go to Accounts Payable> Configuration>Audit Configuration (Record Level). Open the ap_invoice line and make sure the following fields are in the audit fields text box.

 

 

ain_hold,ain_hcode,ain_hreason,ain_hold2,ain_hcode2,ain_hreason2

 

 

 

 


Figure 11: Audit Configuration

 

 

 

 

 

 

 

 

 

 

Save the line.
