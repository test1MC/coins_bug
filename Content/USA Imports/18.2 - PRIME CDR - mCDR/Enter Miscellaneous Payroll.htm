Enter Miscellaneous Payroll via mCDR

Module: Job Status/Payroll/mCDR

This document demonstrates how to set up and enter user-defined miscellaneous payroll items (such as PTO/sick pay, reimbursables, per diem pay, auto-allowances, etc.) through the mCDR (mobile Crew Daily Reports) app.

 

Setup

  1. Go to Job Status > Daily Reports > Maintenance > Daily Report Misc Payroll Defaults and click Add at the bottom of the screen (Figure 1).
  1. There are three types of Daily Report Misc Payroll Defaults that can be created and used:
  • Hours/Input
  • Allows entry of a certain number of hours for a crew member
  • An example could be “Sick Pay” or “Vacation Pay”
  • Amount
  • Allows entry of a flat, dollar amount to be paid to a crew member
  • Can or cannot be considered taxable income, depending on user-defined setup
  • An example could be an “Auto-Allowance”
  • Reimb
  • Allows entry of a dollar amount to be reimbursed to a crew member
  • Never taxable as income, and not included in Gross Pay or W-2
  • An example could be “Reimbursement” to reimburse crew members for purchases of tools, supplies, parking, tolls, etc.

Figure 1: Job Status > Daily Report Misc Payroll Defaults

  1. Decide how each will be entered. In this example, Reimbursement will be entered via a “Reimb” amount and Sick Time – PTO will be entered via “Hours/Input” (Figure 2).
  1. Enter a Code and Description (Figure 2).
  1. Select an Earnings Category for the Misc Payroll Default to link to (Figure 2).
  • Earning categories must be set up in Payroll > Global Setup > US Specifics > Earning Categories before they can be used here.
  1. Select the checkbox for Hours/Input, Amount, or Reimb to determine the entry type, based on the explanations in step 2 (Figure 2).
  1. Select the checkbox for Allow Multiple Records to allow the Daily Report Misc Payroll Default to be used multiple times on a single crew daily report (Figure 2).
  1. Select the checkbox for Description Allowed to force a Description to be entered during Misc Payroll entry (Figure 2).
  1. Click the floppy disk to Save the line (Figure 2).

Figure 2: Save the Daily Report Misc Payroll Default

 

 

 

 

 

 

 

 

 

 

  1. Next, go to Job Status > Setup > Configuration > JC Parameters (Figure 3).
  1. Find the parameter CDRPRREC (List of earnings categories for validation) (Figure 3).
  1. Optionally, enter a can-do list of earnings categories (which are specified on Daily Report Misc Payroll Defaults) that will not require Cost Code input during Misc Payroll entry on Crew Daily Reports.
  • If a cost code is not specified, costing information can be derived from the Employee's Earnings Category during the Transfer to Payroll process. Cost Code distribution is not required on the daily report for these earnings categories.

Figure 3: JC Parameter CDRPRREC

 

  1. Notice that in Figure 3, SI (the earnings category for Sick Pay – PTO) is listed and RE (the earnings category for Reimbursement) is not.
  1. This means that if both are entered on a crew daily report, the Reimbursement will always be costed to the job, and the costing of Sick Pay – PTO can optionally be directed to an Analysis specified on the Earning Category of the employee.

  1. A Cost Method (a GL account or an overhead job) and Analysis can be set on the employee, along with rates, in the Earning Categories tab (Figure 4).

Figure 4: Cost Method/Analysis specified on Employee

mCDR Entry

  1. Inside of a crew daily report in the mCDR app, Misc Payroll can be entered in two different ways: from the Labor screen or the Misc Payroll screen.
  1. First, we’ll demonstrate adding Misc Payroll from the Labor screen (Figure 6).
  1. In the Labor screen, tap on the name of a crew member and select Add Misc Payroll Transaction (Figure 6).
