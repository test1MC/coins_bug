mCDR Entry - Labor

Module: mCDR

This document demonstrates how to enter labor hours via the mCDR (mobile Crew Daily Reports) app.

Pre-requisite document: “mCDR Setup”

This document only covers the Labor entry component of the mCDR app. For the other processes, reference the separate documents for:

  • “mCDR Equipment”
  • “mCDR Inventory”
  • “mCDR Misc. Charges”
  • “mCDR Misc. Payroll”
  • “mCDR Quantities Installed”

 

  1. Enter a username and password, defined from a role-based user, and tap Sign In (Figure 1).

Figure 1: Enter Username and Password, Sign in

 

 

  1. Once logged into the app, tap the + icon in the bottom, right corner of the screen (Figure 2).

Figure 2: Tap the + Icon

 

  1. Select the appropriate Crew and Job (Figures 3 and 4).

Figure 3: Select Crew

 

Figure 4: Select Job

  1. Then, select the newly created job card (Figure 5).

Figure 5: Select Job Card

 

  1. Then, tap the + icon at the bottom, right corner of the screen and add a new crew daily report (Figure 6).

Figure 6: Add New Daily Report

  1. At the Report Settings screen, the Report Date will default in as today’s date, but can be changed (Figure 7).
  1. Specify the presence or absence of Regular (R), Overtime (O), and Double Time (D) hours as entry fields. Green = present, white = absent (Figure 7).
  1. Select the appropriate Pay Shift, if applicable (Figure 7).
  • Pay Shifts allow different pay rates for individuals on different shifts.
  • For more information on setup and entry, see the how-to document “Shift Pay on CDR + mCDR"
  1. Optionally, enter a Description (Figure 7).
  1. Then, select Save (Figure 7).

Figure 7: Specify Report Settings, Save

 

  1. Tap on the newly created crew daily report (Figure 8).

Figure 8: Select Newly Created Daily Report

  1. At the Labor screen, all the members specified on the crew will pre-populate (Figure 9).
  1. Tap the + symbol (Figure 9).
  1. Optionally, additional crew members may be added to the daily report by selecting Add Crew Members (Figure 9).
  1. To add cost codes, select Add Multiple Cost Codes (Figure 9) or Add Previously Used Cost Codes.

Figure 9: Crew Populates, Tap + Symbol

 

  1. Select the appropriate cost code(s), then Save (Figure 10).

Figure 10: Select Cost Code(s), Save

  1. Enter hours by tapping on the number and entering from the pop-up keypad (Figure 11).
  1. Optionally, enter a Start and End time (Figure 11). These times do not affect processing and would be used for informational purposes only.

Figure 11: Enter Labor Hours

 

  1. To enter the same number of hours for all crew members for the daily report (one-offs can be edited afterward), tap on the cost code header and select Copy Down (Figure 12).

Figure 12: Copy Down Labor Hours

  1. Next, tap on any employee name (Figure 13).
  1. Copy to... and Copy from… can be used to copy hours from or to an employee (Figure 13).
  1. Select Remove to remove a crew member (except the foreman of the crew) from the daily report (Figure 13). However, if a crew member has 0 hours entered for the day, nothing would flow through to Payroll, and it would have the same effect.
  1. Select Duplicate to duplicate an employee line and change certain details (Figure 13). For example, an employee line could be “duplicated” and an Earning Category could be changed for sick pay/PTO if a crew member went home sick halfway through the day.
  1. Optionally, select Add Misc Payroll Transaction to add a miscellaneous payroll transaction for an employee such as sick pay, reimbursables, auto allowances, etc. (Figure 13).
  • For more information on adding miscellaneous payroll, see the how-to document: “Enter Misc Payroll via mCDR”
  1. Select Weekly Summary to see a weekly summary involving jobs, cost codes, and hours per employee by week (Figures 13 and 14).

Figure 13: Employee Menu Options

 

Figure 14: Weekly Summary

 

  1. Next, select Details (Figure 15).

Figure 15: Select Employee “Details”

  1. The Earning Category, Trade, Skill, PW Trade, and PW Skill fields will default here based on the Employee file in Payroll. They can also be changed here (Figure 16).
  • These fields will only show up for crew members that are Payroll employees. These fields will not show up for Job Status (JC) employees.
  • For more information on the PW (prevailing wage) fields, see the how-to document: “Prevailing Wage Schemes on CDRs and mCDR"
  1. Select Save (Figure 16).

Figure 16: Employee Details, Save

 

  1. Next, select Review and Sign (Figure 17).

Figure 17: Select “Review and Sign”

  1. Read-only employee details are displayed here (Figure 18).
  1. Optionally, enter Break Comments (Figure 18).
  1. Tap on the Signature field to have each employee enter a signature. It is optional for employees to enter signatures, but will give a warning upon submittal if any employees have not signed (Figure 18).

Figure 18: Break Comments, Signature

 

  1. Custom Disclaimer text can be displayed based on the setting of the JC Parameter SIGLEGTXT in OA (Figure 19).
  1. The crew member can sign in the Signature box (Figure 19).
  1. Then, select Save (Figure 19).

Figure 19: Disclaimer, Enter Signature, Save

  1. Next, tap the triangle in the upper, right corner of the screen (Figure 20).
  1. Optionally, select Attachments (Figure 20).

Figure 20: Tap Triangle, Select Attachments

 

  1. Pictures can be attached to mobile crew daily reports, which will flow through to OA. This can be used for receipts, the job site, damage, etc.
  1. Tap the + symbol and select either Take Photo or Choose from Camera Roll (Figure 21).

Figure 21: Tap the + Symbol, Select Appropriate Photo Option

  1. Pictures can be edited/annotated to include notes, a date stamp, drawings, arrows, shapes, text, etc. (Figure 22)

Figure 22: Picture with Annotations

 

  1. When finished entering data for the daily report, select Review and Submit (Figure 23).

Figure 23: Select “Review and Submit”

  1. The Review and Submit summary screen shows what labor hours have been entered by employee and by cost code (Figure 24).
  1. Tap Next in the upper, right corner of the screen (Figure 24).

Figure 24: Review and Submit Summary Screen, Tap Next

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

  1. Optionally, enter in notes for the day (Figure 25).
  1. Enter the foreman signature (Figure 25).
  1. Tap the paper airplane symbol in the bottom, right corner of the bottom, right corner to submit the crew daily report (Figure 25).

Figure 25: Enter Notes, Foreman Signature, Tap Paper Airplane Symbol

 

  1. The crew daily report will now show as “Submitted” (Figure 26).

Figure 26: Crew Daily Report is “Submitted”

 

  1. The next step would be for the crew daily report to be approved in OA. See the how-to document: “Approve Crew Daily Reports”
