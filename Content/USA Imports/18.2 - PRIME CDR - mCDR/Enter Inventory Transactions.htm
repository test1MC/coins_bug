Enter Inventory Transactions via mCDR

Module: mCDR

This document demonstrates how to enter inventory transactions through the mCDR app and explains the necessary setup required.

 

Setup

  1. Go to Job Status > Setup > Configuration > JC Parameters (Figure 1).
  1. CDRSTTT (Crew Transaction Type) must be set to CDR (Figure 1). This parameter determines the transaction type to be used when creating inventory transactions.

Figure 1: JC Parameter CDRSTTT

 

 

 

 

 

 

 

 

 

 

 

 

mCDR Entry

  1. Inside of a crew daily report in the mCDR app, tap on the hamburger bun in the upper, left corner of the screen (Figure 2).

Figure 2: Tap the Hamburger Bun

 

  1. Select Inventory from the side menu that emerges (Figure 3).

Figure 3: Select Inventory from the side menu

  1. Tap the + symbol at the bottom, right corner of the screen (Figure 4).

Figure 4: Tap the + Symbol

 

  1. Select the Inventory Location (Figure 5).
  1. Select the Inventory Code. The Description will default in (Figure 5).

Figure 5: Enter Inventory Location and Code

  1. Enter the Quantity of inventory (Figure 6).

Figure 6: Enter Inventory Quantity

 

  1. Select the Cost Type. The Cost Category should default in (Figure 7).
  1. Then, tap Save (Figure 7).

Figure 7: Enter Cost Type/Cost Category

  1. When finished adding inventory, tap the triangle in the upper, right corner of the screen (Figure 8).

Figure 8: Tap the Triangle

 

  1. Select Review and Submit (Figure 9).

Figure 9: Select Review and Submit

 

  1. Select Next at the Review and Submit summary screen (Figure 10).

Figure 10: Select Next

 

  1. Optionally, enter any notes for the day (Figure 11).
  1. Enter the foreman signature (Figure 11).
  1. Tap the paper airplane to submit the crew daily report (Figure 11).

Figure 11: Enter Notes, Signature, Tap the Paper Airplane

Review and Approval

  1. Inventory transactions entered through crew daily reports need to be approved, before they can be posted.
  1. Go to Job Status > Daily Reports > Review and Approval (Figure 12).
  1. Select the Submitted crew daily report, select Approve Current Daily Reports from the drop-down on the bottom action bar, and click the arrow to “Apply Action” (Figure 12).
  1. Alternately, click the date hyperlink of the crew daily report to review the inventory transactions in detail in the Miscellaneous tab, before approving (Figures 12 and 13).

Figure 12: Approve from Review and Approval Workbench

 

Figure 13: Approve from Daily Report Detail

 

  1. Make sure the Action is Approve and move the daily report over to the Selected section (Figure 14).
  1. Click Next in the bottom, right corner of the screen (Figure 14).

Figure 14: Approve, Selected, Click Next

 

  1. Next, hover over the printer icon and click on Approve Current Daily Reports (Figure 15).

Figure 15: Click into “Approve Current Daily Reports” Report

 

 

  1. The report will list the inventory transactions created and direct the user to the number of the stock (inventory) batch that has to be posted manually (Figure 16).

Figure 16: Report references stock (inventory) batch

 

  1. Next, go to Inventory > Input Inventory Transaction (Figure 17).
  1. Find the number of the unposted batch referenced in step 24/Figure 16, select the line, Post, and click the arrow to “Apply Action” (Figure 17).
  • Optionally, you can click into the batch to review it or run a Posting/Listing Report first, before posting.

Figure 17: Find the Batch, Select Line, Post, Apply Action

 

 

 

 

  1. Hover over the printer and click Inventory Transaction and/or GL Cost Posting Report (Figure 18).

Figure 18: Click into Posting Report

 

  1. Review the report(s) to verify that the correct GL accounts were hit (Figure 19).
  1. Now that the inventory transaction batch is posted, the amount of inventory item will decrease accordingly.

Figure 19: Review Posting Report
