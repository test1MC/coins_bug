Series Setup

Module: Status

During setup, a Series is typically copied from a template . However, under certain circumstances, the user may need to create a new series or edit an existing one. The steps are described in this document.

Note: To add or edit series on many at once, go to > /Series > Bulk Series.

Common reasons a new series is created or edited:

  1. The bills to multiple clients.
  2. The requires a “parallel billing” – the client wants to see some of the billing items on a separate invoice from the rest of the .
    • The most common example is a change order that the client wants to treat as a new contract (but will remain as a single in COINS).
  3. Errors during setup and/or changes to billing requirements are serious enough that fixing the issue requires creating a new series to replace the existing series.
  4. The was set up without using any template .
  5. A receivable series for a may be edited if:
    • Change to the default retention % for this job (10% retention will be the default for new ).
    • Change to the default sales tax code on the bills for this . (Be sure to update the series whenever a change is made to the revenue tax code on a file.)
    • The wrong template is used.

Steps to create or edit a Series:

  1. On the tree menu, go to Status > .
  2. Locate the number and click on the hyperlink to continue to the details.Click the Job Hyperlink

  3. Click Cost Codes in the drop-down under the Cost Code Maintenance tab, which may include WBS.Cost Codes

  4. Ensure that there is a valid revenue cost code on this to accept revenue transactions. Common examples would be Billings or Revenue.Ensure Valid Revenue Cost Code

  5. If a revenue type code does not exist, click in the menu bar at the bottom of the screen to add it.
    • Refer to the how-to document forManual Cost Code Entry for instructions on adding a cost code.Add a Cost Code

  6. Next, click over to the Series tab to edit a series, if applicable.
  7. If creating a new Series, click at the bottom of the screen.Add a New Series

  8. When creating a new series, the following fields will be prepopulated, but they can be changed:
    • Client
    • Sales Tax
    • Revenue Analysis – Make sure to add the revenue cost code to the number to form the full Revenue Analysis. In this example, it will appear as “10035.00” – 10035 is the number and 00 is the revenue cost code.Prepopulated Fields, Add Revenue Cost Code to Revenue Analysis

  9. Fill in additional fields:
    • Series: a unique code by , identifying this billing series.
    • Enter By: “Period” type will be the default method, but can be changed when billing.  Enter By controls the default billing amount to be entered as “this month” (this period) or “total billed to date including this month” (cumulative), depending on how the company/PM bills.
    • Discount, Retention, Material on Site Retention, Material off Site Retention: will depend on your contract terms. Fill in the necessary info.Fill in Additional Fields

  10. There are two ways to set the remaining fields. COINS recommends to not use any other combination.
    1. Style Billing: Set these parameters if you are billing with and/or T&M using the rate tables through the module.
      1. Certificate Definition: Detail Billing
      2. Input Method: Only
      3. Use: Invoices Only
      4. Billing Type: Invoice
      5. Rate Type: Leave this field blank, which will use the Rate type from the file. Style Billing

    2. Simple Billing through module: is not billed in , cannot use .
      1. Certificate Definition: SIMPLE
      2. Input Method: Applications/Certificates
      3. Use: Invoices Only
      4. Billing Type: InvoiceLump Sum Billing through Job Receivables

  11. Once all fields have been completed, click in the bottom right corner of the page.Click Save
