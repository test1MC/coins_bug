Input Quantities Installed

Module: Status

This document explains how to track completion of key tasks to compare the percent of work complete vs. percent of budget spent. The primary goal of this process is to identify or codes that are on-pace to exceed budgeted/forecasted costs before such a budget overrun occurs.

budgets can include a “quantity” on every cost code in addition to a dollar budget and/or a hour budget. “Quantity budgeted/forecasted” represents the amount of work to perform in order to complete a task; “quantity installed” represents the amount of work performed as of a given date.

Budgeting and tracking both costs and quantities on a single cost code will allow the user to report on the dollars-spent-per-unit and/or -hours-per-unit as budgeted vs. as actually incurred.

    • Parameter Notes:

      • If JC Parameter ENTZQBUD = Yes, the user can enter quantities for any cost code, regardless of budget.
      • If JC Parameter EXCQBUD = No, then the user cannot install more quantities than budgeted.
      • If JC Parameter EXCQBUD = Yes, the user can install more quantities than the budgeted amount.Enter Quantity for This Measure, Click Save
