Weekly Cost Reporting (Simple)

Module: Status

Weekly cost reporting allows a user to track costs, accruals, and commitments for a by cost code on a weekly basis. This document demonstrates how to set up weekly cost reporting, explains the available methods that can be used to capture the costs by week, and explains selected /reports that show costs by week.

Configuration

  1. First, make sure the configuration is correct. Start off by checking that the following Parameters are set correctly.

    Correct Parameters

    Parameter

    Description

    Recommended Settings

    GL/BATTDATE

    Batch Transaction Date from Source Batch

    If set to Y, the Transaction Date of the source batch is copied to a GL Batch.

    Y

    JC/WKLYMETH

    SIMPLE – All costs against the are accumulated (regardless of date). The weekly cost movement is calculated simply by subtracting the previous cumulative figures from the newly calculated figure.

     

    SIMPLE

  2. Additional parameters are recommended to be used for Weekly Cost Reporting, and their recommended settings are shown below.

    Additional Recommended Parameters

    Parameter

    Description

    Recommended Settings

    JC/WEDAY

    This allows the user to control which day of the week the weekly cost snapshot can be generated.

    It can be anything, but it must be set.

    JC/USEWKCAL

    Controls whether COINS allows you to set up a calendar or use weekly costs.

     

    Must be set to Y

    JC/WBSMOVE

    Whether to allow cost and billing transactions to be transferred between section/activity on a after posting.

    This can be set to Y or N. If set to Y, it enables the field Allow Cost/Billing Transfer on Maintenance.

    PO/COSTMOVE

    Whether to create cost movement transactions.

    Y – PO transaction records are created and Weekly Summary records are created.

Generate Weekly Costs Snapshot

The Generate Weekly Costs Snapshot functionality allows the user to save a “snapshot” of the costing data on .

  1. Go to Status > Administration > Generate Weekly Costs Snapshot.
  2. In the Main tab, enter in the Week Ending date.
  3. Select Confirmation as the Mode. For Confirmation, COINS generates the snapshot for all selected with the week ending dates that match the date entered.
  4. Select the appropriate boxes.
    • Snapshot Accruals
      • Whether to include Accrued Costs in the report.
    • Snapshot Commitments
      • Whether to include Committed Costs in the report.
    • Snapshot Costs
      • Whether to include Actual Costs in the report.
    • Snapshot Valuation Data
      • Whether to include Valuations in the report.
    Main Tab

  5. In the Selection tab, the captured in the report can be filtered. Selection Tab

  6. Next, go to the Output Options tab.
  7. Select Scheduled as the Output Type.
  8. Optionally, if Email is ed, the report will automatically be sent to a person’s email (or multiple people) when it’s generated – and that person (or persons) would not need to log into COINS to get the report.
  9. Then, click in the bottom, right corner of the page.Output Options Tab, Click Next

    Note: Refer to the “Report Scheduling” how-to document for detailed instructions on how to set up a scheduled report in COINS.

  10. At the Scheduler Update page, enter in the Time the scheduled report will be run.
  11. If the date selected for From is in the past, this report will start right away.
  12. For To, the user can select a date in which the report will stop being generated every week. To can be left blank, if desired.
  13. For Schedule, select the option “ONE TIME EVERY WEEK”.
  14. For Week Ending, select Last Week.
  15. Then, click in the bottom, right corner of the screen.Scheduler Update, Click Save
