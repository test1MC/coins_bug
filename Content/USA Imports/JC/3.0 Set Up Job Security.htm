Set Up Security

Module: Status

The purpose of this how-to document is to show a user how to setup security for a user or users, and custome specifications. security determines which a user has access to.

  1. Go to Status > Setup > Setup Maintenance > Security > Security.
  2. Click in the row of the user to update the security for.Select User to Edit Security for

  3. For the Security List field, type in the access for the user. The Security List is a list of PMs User IDs whose each user has access to.
    • For most project managers, they will just need their own User ID (“marulr” for this example) in the Security List field, so they have access only to where they are the PM.
    • If “marulr,matpot” is entered, this setup will give Marshall Ulrich (User ID: marulr) access to any where either he is the PM or Matt Potts (User ID: matpot) is the PM.
    • If “*” is entered, this setup will give the user access to all .
    Security List

  4. Enter in the Code field for the user. The Code field on the same page is a list of number(s) that this user has access to.
    • Wildcards can be left in this list, so “02*” would give a user access to every where the number starts with 02.
    • Any user who is setting up should have access to the appropriate template (s) so that they can use that as their template.
    • 02, 03, and 04 (typed in as: 02*, 03*, 04*) would also be a valid setup to give a user access to all that start with those numbers.
    Code

  5. Click to save the changes.Click Save

  6. To apply Security List or Code changes to multiple users at once, start with ing any row (it doesn’t matter which row is selected), make sure Copy Job Security is selected from the drop-down box at the bottom of the screen (it should be the only option), then click .Copy Security for Multiple Users at Once

  7. At the next screen, enter in the Security List and/or Code to be applied, and enter in the users to apply it to in the To field (with User IDs separated by commas, with no spaces in between).
  8. Click in the bottom right corner of the screen to save.Copy Security, Click Next to Save

  9. Back at the Security screen, the changes made should be updated. Changes Updated

    Note: The Status parameter COSECDEF controls whether a missing record in in Security means 'None' or 'All'. If parameter COSECDEF is set to Y, it means that blank means access to no . If parameter COSECDEF is set to N, it means that blank means access to all . The following document assumes COSECDEF is set to Y.
