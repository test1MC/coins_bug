Employee Self Service Setup

Module: Employee Self Service (ES)

Check the service packs are up to date. Also fixes.

ESS module setup

  1. Begin with finding the System Setup on the tree menu.
  • Find the User Group (comes already standard from PRIME). It should be named: PRPortal. Remember this for the next step.
  • Create a Template User. This will auto create other users as new hires join, a system admin will be able to do this quickly.
  • Name it ESSPortal. Import the “current desktop” and “initialize desktop” from the DEV environment. Refer to it as the same name. %WESPV11, and %ESSPV1.

Figure 3: Template User Setup

  • Named User and Named Role need to be checked in setup.

Figure 4: Named User

 

  • The User Role contains all the access for a user.

Figure 5: Check ESS Portal Role

 

  1. Next go to the ESS module. Set Email Templates here first. ESS > Admin > Email Templates.

Figure 5: Set Emailing Template

 

  • The following reserved words can be used:

{username} – Name of User

{userid} – Userid that will be generated or set for user

{password} – generated password

Figure 6: Example Email setup

 

 

  1. Set ESS Parameters. ESS > Admin > ESS Parameters.

Figure 7: Set Parameters

Obligatory parameters:

  • Template User (for this sample it is ESSTEMP)
  • EmailUs  (hr_inbox@company.com)
  • WEBSITE (http://ourcompany.com)
  • Welcome Message Template (for this Demo - Portal Confirm)

 

Other Parameters:

  • DESKTOP :  Desktop Function. The default is %WESPV11 for clients but a custom desktop can be entered
  • EMAILUS : Enter Email address to send inquiries to HR department. This can be a comma separated list for multiple recipients.
  • INITPRG:  Initializing  program. 
  • LOGINFMT:   Format for login ID generation. i.e. N6 will generate a 6 digit number.  For the 11.05 and 11.06 release this is numeric only.
  • WEBSITE:  Company Website IP address
  • WELCOME:  Portal Welcome Template. Configurable and is set up in System -> Email Templates

 

  1. Set up Approvers. ESS > ESS Portal > Maintenance > Approvers. These are the users who will be reviewing and approving the timecards coming from the ESS Portal users.

 

 

  1. Set Payroll Pay Types in the ESS Portal. ESS > ESS Portal > Maintenance > Payroll Pay Types. This also is where the earning categories are linked from payroll to the ESS module. Additionally, this can be weekly, 2-weekly, normal, office employees etc. – any naming convention that is desired.

 

 

  1. Set up Timesheet Group. ESS > ESS Portal > Maintenance > Timesheet Group. This needs an approver to be setup first. Add the Valid Pay types here to include each of the earnings categories/ payroll pay types to each relevant timesheet group.

 

Benefit Tiles Details Pages. ESS > Admin > Benefit Tiles Details Pages. No setup required.

 

Set Plan Types. ESS > Admin > Plan Types. Check the box for ESS Deployable to display. Group together the ESS Tile Type Group, when possible.

 

 

  1. Set Timesheet Periods. ESS > ESS Portal > Maintenance > Timesheet Periods. Selects the timesheet group, sets a date on it – which adds the ability for portal users to input timesheets to this pay frequency. Sets expected hours. This cannot be deleted once employee data has been entered to it, only marked as dormant. Similar to a pay period in payroll.

 

  1. Set ESS PR Portal Tiles. ESS > Admin > ESS PR Portal Tiles. No setup necessary, this is allows the option to set a page as “dormant” and not appear on the Portal. For example, the ability to remove the “Pay History” tile.
