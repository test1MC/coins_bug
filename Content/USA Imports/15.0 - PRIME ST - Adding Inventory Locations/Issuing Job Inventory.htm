Issuing Job Inventory

Module: Inventory

After an Inventory Location has had inventory added to it, it can be used to issue inventory out to different jobs. This document shows how to issue inventory out to jobs.

  1. Go to Inventory > Input Inventory Transaction and click on the green “ADD” button on the bottom right of the screen

Figure 1: Inventory Transaction Entry

  1. This will bring up the create a batch screen, enter correct fiscal period (current month will default), transaction date will default to today’s date, Transaction type will be I- Inventory Issue, Location should be the Inventory warehouse where the inventory will be pulled from. Then select save in the bottom right of the screen

Figure 2: New Inventory Transaction Entry

  1. This will bring up the specific transaction screen, click the green “ADD” button on the bottom of the screen

Figure 3: Adding Transaction Entry

  1. Firstly enter in the Inventory code for the Inventory that is being issue. The description will automatically fill, enter in the quantity of units and the cost per unit. Choose analysis “contract” and select the correct contract and “built” the full cost code. Choose the transaction date, typically today

Figure 4: Editing Transaction Line

  1. Once line is accurately filled out, clip the floppy disc save button to the left of the line.

Figure 5: Saving Transaction Line

  1. Note, if issuing more inventory than the system says is on-hand at the location this message will pop up. This is currently an error or a “hard stop” and will not allow the transaction to go through. Changing the Inventory Parameter ST/STOPISS this will pop up as just a warning, which the user could ignore and issue

Figure 6: Potential Error Message

  1. Once the transactions are saved, select the line and post by using the action bar

Figure 7: Posting Inventory Transaction
