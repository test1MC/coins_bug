Learning Resources Guidelines

The following is a general guide for the creation of Learning Resources:

 

  1. As much as possible, any resources created must be generic and use screen shots from the SureStart environment or the Training Environment. Customer data must NOT be used.
  2. Client specific documentation should be avoided unless the client has paid for the service.
  3. All documentation is to be supplied to clients in PDF format, or made available via a web-page.
  4. Under no circumstances should documentation bearing the copyright be supplied to clients in ANY format that would allow them to change its contents.
  5. Documentation should be written in the form of Topics rather than a single narrative. Guidance on how this works may be found here > Topic Based Writing
  6. Microsoft Word may be used to create new documentation, as long as it conforms to the topic based authoring guidelines. However, the completed document must then be sent to either Don Barwick, or Nigel Longley so that it can be incorporated into the central resource library. The document will be checked to ensure it meets our criteria for inclusion and the library will create the PDF for issuing to clients and added to the .
  7. Existing Documentation bearing the "Created in Madcap Flare" on the copyright page can only be updated via the Flare System. Consultants should request a Review Package from Don or Nigel in order to update these topics.
